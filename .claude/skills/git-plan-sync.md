# Git-Plan Sync — 提交同步计划书

## 触发条件

每次 `git add` + `git commit` 之后，或用户说"提交"、"commit"、"push" 时，自动执行本 skill。

## 执行流程

### 第 1 步：更新工作日志（最重要！）

找到 `C:\Users\Administrator\Desktop\Unity学习一\mygame\开发计划书.md` 顶部的 `## 📋 工作日志` 表格。
在表格**最上面插入新的一行**（最新在最上面），填入：

```markdown
| 日期 | 提交短哈希 | 做了什么（一句话） | 下次继续做什么 |
```

- 日期：今天的日期
- 提交短哈希：`git log -1 --format=%h` 的结果
- 做了什么：本次提交的核心内容，一句话概括
- 下次继续：下一步该做什么，让下次打开的人直接知道从哪开始

**这是最关键的一步**——工作日志位于文件最顶部，一眼可见，用于快速恢复工作上下文。

### 第 2 步：更新模块节精简记录

在 `C:\Users\Administrator\Desktop\Unity学习一\mygame\开发计划书.md` 中，找到对应模块的章节，在章节末尾追加本次改动摘要。格式：

```markdown
> **2026-06-23**：
> - 修复：ItemUsageSystem 双重扣减（删除文件）
> - 新增：DurabilitySystem 核心（ConsumeDurability/GetRatio）
> - 改动：PlacedItem +itemDurability/+repairCount
```

如果模块在计划书中有跟踪表，同步更新表中的实现/审查列状态。

### 第 3 步：更新详细文档

在 `C:\Users\Administrator\Desktop\Unity学习一\mygame\开发计划/` 中：

- **已有模块有改动**：在对应的 `.md` 文件末尾的"版本历史"表格中追加一行变更记录
- **新模块/新系统**：新建 `.md` 文件，包含：标题、日期、状态、核心文件清单、代码关键改动、版本历史
- **删除模块**：在文档中标注"已删除"并注明替代方案

### 第 4 步：提交

执行 `git add -A && git commit -m "<message>" && git push`

---

## 格式约束

### 工作日志格式

表格在计划书最顶部，最新在上，最多保留 20 行。超出 20 行时删最旧的。

### 模块节记录必须精简

- 只写「做了什么 + 影响哪个文件」，不写设计思维过程
- 一行一条改动用 `-` 开头
- 每条改动 < 30 字

### 详细文档必须具体

- 列出改动的精确代码位置（文件名、关键方法名）
- 说明改动前后行为差异
- 标注"新增/改动/删除/修复"四类动作

---

## 示例

用户说 "提交" 后：

```
1. 工作日志 → 顶行插入：| 2026-06-23 | 3d92b2c | 物品P0修复+耐久v1.0实现 | 前置H建造物耐久 |
2. 模块节 → 物品系统节末尾追记修复摘要
3. 详细文档 → 耐久系统.md 版本历史加一行
4. git add -A && git commit -m "..." && git push
```
