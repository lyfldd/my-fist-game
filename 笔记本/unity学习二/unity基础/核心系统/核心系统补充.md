## 一、场景管理补充

### 1. SceneManager 的完整功能

原笔记只讲了 LoadScene/LoadSceneAsync，还有更多重要API：

| 方法 | 作用 |
|------|------|
| `LoadScene` | 同步加载场景（替换当前） |
| `LoadSceneAsync` | 异步加载场景 |
| `LoadScene(..., LoadSceneMode.Additive)` | **叠加加载**（保留当前场景） |
| `UnloadSceneAsync` | 卸载场景 |
| `MergeScenes` | 合并两个场景 |
| `SetActiveScene` | 设置当前活动场景 |
| `GetActiveScene` | 获取当前活动场景 |
| `GetSceneByName` | 按名称获取场景引用 |
| `SceneManager.sceneLoaded` | 场景加载完成事件 |
| `SceneManager.sceneUnloaded` | 场景卸载完成事件 |

### 2. Additive 加载模式（非常重要！）

```csharp
// Additive 模式：新场景的所有物体会叠加到当前场景
// 用于：关卡无缝流式加载、UI独立场景、世界分区

// 加载
AsyncOperation op = SceneManager.LoadSceneAsync("Level_01_Part2", LoadSceneMode.Additive);
yield return op;

// 卸载（必须异步）
yield return SceneManager.UnloadSceneAsync("Level_01_Part2");
```

**实际项目中的应用模式**：

```
主场景（永久保留）:
  - 摄像机、UI、全局管理器
  - 从不卸载

关卡分块场景（按需加载/卸载）:
  - 玩家走到区域A → 加载区域A的场景
  - 玩家离开区域B → 卸载区域B的场景
  - 避免一次性加载整个大场景
```

### 3. allowSceneActivation 的深入理解

```csharp
AsyncOperation op = SceneManager.LoadSceneAsync("Level1");
op.allowSceneActivation = false;  // 场景加载到90%暂停

// 进度只会在 0 ~ 0.9 之间变化
while (op.progress < 0.9f)
{
    // 显示加载进度条
    progressBar.value = op.progress;
    yield return null;
}

// 到达90%，加载完成但未激活
progressBar.value = 1f;
yield return new WaitForSeconds(0.5f);  // 给玩家看完成提示

// 手动激活场景（触发场景切换）
op.allowSceneActivation = true;
```

### 4. DontDestroyOnLoad 的注意事项

```csharp
// 使用 DontDestroyOnLoad 后的问题：
// 1. 离开场景时不会触发 OnDestroy
// 2. 重新进入场景会再创建一个新实例
// 3. 无法通过 SceneManager 管理其生命周期

// ✅ 推荐：用单例模式管理长期存活对象
public class GameManager : MonoBehaviour
{
    private static GameManager _instance;
    void Awake()
    {
        if (_instance == null)
        {
            _instance = this;
            DontDestroyOnLoad(gameObject);
        }
        else
        {
            Destroy(gameObject);  // 销毁重复实例
        }
    }
}
```

### 5. 场景切换时的资源清理

场景切换时 Unity 自动执行的操作：
1. 除 `DontDestroyOnLoad` 外的所有对象调用 `OnDisable()` → `OnDestroy()`
2. 自动调用 `Resources.UnloadUnusedAssets()` + `GC.Collect()`
3. 加载新场景的所有对象 → 执行 `Awake()` → `OnEnable()` → `Start()`

但要注意：**协程在场景切换时不会自动终止**（如果挂在 DontDestroyOnLoad 的对象上）

## 二、LineRenderer 补充

### 1. 性能优化 —— 使用 Simple 线条

```csharp
// 对于大量静态线条，用 Mesh 替代 LineRenderer
// LineRenderer 每帧会重新生成网格，性能开销大

// 方案一：烘焙 LineRenderer 为静态网格
void BakeToMesh()
{
    LineRenderer lr = GetComponent<LineRenderer>();
    Mesh mesh = new Mesh();
    lr.BakeMesh(mesh, true);
    gameObject.AddComponent<MeshFilter>().mesh = mesh;
    gameObject.AddComponent<MeshRenderer>().material = lr.material;
    Destroy(lr);  // 去掉 LineRenderer
}

// 方案二：使用 TrailRenderer（轨迹渲染器）
// 适合：子弹轨迹、物体移动轨迹
// TrailRenderer 和 LineRenderer 类似但自动记录历史位置
```

### 2. LineRenderer 的材质和纹理动画

```csharp
// 设置材质主纹理平铺和偏移（实现流动效果）
Material mat = lineRenderer.material;
mat.mainTextureScale = new Vector2(length, 1);
mat.mainTextureOffset = new Vector2(Time.time * speed, 0);

// 使用 Shader 属性名的替代方法
mat.SetTextureScale("_MainTex", new Vector2(length, 1));
mat.SetTextureOffset("_MainTex", new Vector2(Time.time * speed, 0));
```

## 三、射线检测补充

### 1. 射线的完整参数

原笔记覆盖了大部分，但以下细节值得注意：

```csharp
// maxDistance 是 float 类型，layerMask 是 int 类型
// 重载顺序：先 distance 后 layerMask
Physics.Raycast(ray, out hit, 100f, layerMask);

// ⚠️ 常见错误：传参顺序颠倒
Physics.Raycast(ray, out hit, layerMask, 100f);  
// ❌ 这会把 int 类型的 layerMask 错误地当作 maxDistance(float)
```

### 2. 层级碰撞矩阵

Unity Project Settings → Physics → Layer Collision Matrix 提供了一个可视化矩阵：

```
         Layer0 Layer1 Layer2 Layer3 ...
Layer0   [✅]   [✅]   [❌]   [❌]
Layer1   [✅]   [✅]   [❌]   [❌]
Layer2   [❌]   [❌]   [✅]   [✅]
Layer3   [❌]   [❌]   [✅]   [✅]
```

**代码等效**：
```csharp
// 屏蔽 Layer2 和 Layer3 之间的碰撞
Physics.IgnoreLayerCollision(2, 3, true);

// 典型层级设计：
// Layer 0: Default          ← 默认所有物体
// Layer 8: Player           ← 玩家
// Layer 9: Enemy            ← 敌人
// Layer 10: Environment     ← 场景环境
// Layer 11: Bullet          ← 子弹

// 玩家射线检测只检测"敌人"层
int layerMask = 1 << LayerMask.NameToLayer("Enemy");
Physics.Raycast(ray, out hit, 100f, layerMask);
```

### 3. BoxCast / SphereCast / CapsuleCast

除了 Raycast，还有"体"投射检测：

```csharp
// 盒体投射：沿方向检测长方体接触
Physics.BoxCast(center, halfExtents, direction, out hit, rotation, maxDistance, layerMask);

// 球体投射：沿方向检测球体接触
Physics.SphereCast(origin, radius, direction, out hit, maxDistance, layerMask);

// 胶囊体投射：沿方向检测胶囊体接触
Physics.CapsuleCast(point1, point2, radius, direction, out hit, maxDistance, layerMask);

// 应用场景：角色移动时的碰撞检测（用 CapsuleCast 检测前方是否有障碍物）
// 比单纯检测中心点的 Raycast 更准确
if (Physics.CapsuleCast(headPos, footPos, 0.3f, moveDir, out hit, stepDistance))
{
    // 前方有障碍物，不能前进
}
```

### 4. Raycast 的 offset 和穿透问题

```csharp
// 常见问题：射线从物体内部发射，检测不到碰撞
// 解决：设置起始点偏移

// 从玩家位置发射，但玩家自身有 Collider
Vector3 startPos = transform.position + Vector3.up * 0.5f;  // 偏移，避免检测到自身

// 如果必须从内部发射，使用 Physics.queriesHitBackfaces 或忽略特定层
int playerLayerMask = ~(1 << gameObject.layer);  // 忽略自身层级
```

}

### 6. NonAlloc 变体的完整使用

```csharp
int maxHits = 10;
RaycastHit[] hits = new RaycastHit[maxHits];

void FixedUpdate()
{
    // 预分配数组，避免每帧创建新数组（GC压力）
    int hitCount = Physics.RaycastNonAlloc(ray, hits, 100f);
    
    for (int i = 0; i < hitCount; i++)
    {
        // hits[i] 有效
        Debug.Log(hits[i].collider.name);
    }
    // hits 数组中 hitCount 之后的元素是上一次检测的残留数据
    // 不要使用 hits[i] for i >= hitCount
}
```

## 四、范围检测补充

### 1. 三种 Overlap API 的完整对比

| API | 形状 | 参数特点 | 性能 | 典型用途 |
|-----|------|---------|------|---------|
| `OverlapSphere` | 球体 | 简单：位置 + 半径 | 最快 | 爆炸范围、拾取检测 |
| `OverlapBox` | 立方体 | 位置 + 半长宽高 + 旋转 | 中等 | 矩形判定区域 |
| `OverlapCapsule` | 胶囊体 | 两点 + 半径 | 最慢 | 角色近战攻击范围 |

### 2. 持续范围检测的性能优化

```csharp
// 不要每帧在 Update 中调用 Overlap 方法！
void Update()
{
    // ❌ 坏做法：每帧检测
    Collider[] hits = Physics.OverlapSphere(transform.position, 10f);
}

// ✅ 好做法1：降低检测频率
private float _checkInterval = 0.2f;
private float _lastCheckTime = 0f;

void Update()
{
    if (Time.time - _lastCheckTime >= _checkInterval)
    {
        Collider[] hits = Physics.OverlapSphere(transform.position, 10f);
        _lastCheckTime = Time.time;
        // 处理结果...
    }
}

// ✅ 好做法2：使用非分配版本
Collider[] _results = new Collider[20];  // 预分配

void Update()
{
    if (Time.time - _lastCheckTime >= _checkInterval)
    {
        int count = Physics.OverlapSphereNonAlloc(transform.position, 10f, _results);
        for (int i = 0; i < count; i++)
        {
            // 使用 _results[i]
        }
        _lastCheckTime = Time.time;
    }
}
```

### 3. 范围检测 + 筛选（更精确的判定）

```csharp
Collider[] hits = Physics.OverlapSphere(transform.position, 10f, enemyMask);

foreach (var hit in hits)
{
    // 筛选1：角度检测（只在扇形范围内有效）
    Vector3 dirToTarget = (hit.transform.position - transform.position).normalized;
    float angle = Vector3.Angle(transform.forward, dirToTarget);
    if (angle > fieldOfView / 2f) continue;  // 不在视野角度内
    
    // 筛选2：视线遮挡检测（是否有障碍物挡住）
    if (Physics.Raycast(transform.position, dirToTarget, out RaycastHit obstacle, 10f))
    {
        if (obstacle.transform != hit.transform) continue;  // 被遮挡
    }
    
    // 通过所有筛选 → 有效目标
    OnTargetDetected(hit.transform);
}
```

### 4. Overlap 与 Raycast 的选择

| 场景 | 使用 Overlap | 使用 Raycast |
|------|-------------|-------------|
| 检测站桩范围 | ✅ 检测指定区域内 | ❌ 需要发射多个射线 |
| 检测指定方向 | ❌ 无法判断方向 | ✅ 一条射线即可 |
| 爆炸/光环效果 | ✅ 球形最合适 | ❌ 需要大量射线 |
| 子弹命中 | ❌ 不精确 | ✅ 最适合 |
| 拾取检测 | ✅ 简单高效 | ❌ 需要玩家瞄准 |
| 视线遮挡 | ❌ 不能替代 Raycast | ✅ 检测中间物体 |
