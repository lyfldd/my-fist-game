## 一、三角函数补充

> 原 `三角函数.md` 内容缺失，以下为完整补充

### 1. 基本三角函数

| 函数 | 公式 | Unity 对应 |
|------|------|-----------|
| 正弦 sin(θ) | 对边 / 斜边 | `Mathf.Sin(弧度)` |
| 余弦 cos(θ) | 邻边 / 斜边 | `Mathf.Cos(弧度)` |
| 正切 tan(θ) | 对边 / 邻边 | `Mathf.Tan(弧度)` |

> **注意**：Unity 三角函数参数都是**弧度**，角度转弧度用 `Mathf.Deg2Rad`

### 2. Unity 中的典型应用

```csharp
// 1. 圆周运动 —— Sin/Cos 控制XZ平面圆形轨迹
float radius = 5f;
float angle = Time.time * speed;
float x = Mathf.Cos(angle) * radius;
float z = Mathf.Sin(angle) * radius;
transform.position = new Vector3(x, 0, z);

// 2. 上下浮动 —— Sin 控制Y轴往复运动
float amplitude = 2f;
float frequency = 1f;
float yOffset = Mathf.Sin(Time.time * frequency) * amplitude;
transform.position = new Vector3(transform.position.x, yOffset, transform.position.z);

// 3. 根据角度获取方向向量
float angleDeg = 45f;
float angleRad = angleDeg * Mathf.Deg2Rad;
Vector3 direction = new Vector3(Mathf.Cos(angleRad), 0, Mathf.Sin(angleRad));
// direction 就是指向45度方向的单位向量
```

### 3. 反三角函数

| 函数 | 作用 | Unity 对应 |
|------|------|-----------|
| arcsin | 已知正弦值求角度 | `Mathf.Asin()` |
| arccos | 已知余弦值求角度 | `Mathf.Acos()` |
| arctan | 已知正切值求角度 | `Mathf.Atan()` / `Mathf.Atan2(y, x)` |

**`Mathf.Atan2(y, x)` 特别重要**——它返回从 x 轴到点 (x,y) 的角度，自动处理象限：

```csharp
// 让物体朝向鼠标位置（2D）
Vector3 mousePos = Camera.main.ScreenToWorldPoint(Input.mousePosition);
Vector3 direction = mousePos - transform.position;
float angle = Mathf.Atan2(direction.y, direction.x) * Mathf.Rad2Deg;
transform.rotation = Quaternion.Euler(0, 0, angle);
```

### 4. 三角形计算公式

```
正弦定理: a / sin(A) = b / sin(B) = c / sin(C)
余弦定理: c² = a² + b² - 2ab·cos(C)
勾股定理（余弦定理特例）: a² + b² = c²（C=90°时）
```

## 二、向量运算补充

### 1. 投影向量（原笔记只讲了投影长度，没讲投影向量）

```csharp
// 计算向量 a 在方向 b 上的投影向量
Vector3 projection = Vector3.Project(a, b.normalized);
// 等价于: Vector3.Dot(a, b.normalized) * b.normalized

// 计算垂直于 b 方向的向量分量
Vector3 rejection = a - projection;  // 即 Vector3.ProjectOnPlane(a, b)
```

**应用场景**：斜坡上的滑动、力在某个方向的分量

### 2. 反射向量

```csharp
// 入射方向 inDirection 碰到法线 inNormal 后的反射方向
Vector3 reflectDir = Vector3.Reflect(inDirection, inNormal);
```

**应用**：台球反弹、光线反射、子弹弹射

### 3. 补充的 Vector3 实用方法

| 方法 | 作用 | 与已有笔记的关系 |
|------|------|-----------------|
| `Vector3.MoveTowards` | 匀速移向目标，不会超过终点 | 补充 Lerp 之外的选择 |
| `Vector3.SmoothDamp` | 平滑阻尼移动，带加速度减速效果 | 补充平滑运动 |
| `Vector3.ProjectOnPlane` | 向量投影到平面上 | 配套 Project |
| `Vector3.Reflect` | 反射向量 | 完全新内容 |
| `Vector3.RotateTowards` | 向量朝向旋转 | 补充 Lerp/Slerp 之外的选择 |

```csharp
// MoveTowards 示例：匀速移动到目标
transform.position = Vector3.MoveTowards(
    transform.position, 
    target.position, 
    speed * Time.deltaTime
);

// SmoothDamp 示例：带弹性的跟随效果
float velocity = 0f;  // 需要外部变量记录速度
float newPos = Mathf.SmoothDamp(current, target, ref velocity, smoothTime);
```

### 4. 向量运算的性能建议

- `(a - b).magnitude` 等价于 `Vector3.Distance(a, b)`，但内部有平方根运算
- 如果只比较距离远近，用 `sqrMagnitude` 代替 `magnitude`，避免根号运算
- `Vector3.Dot` 和 `Vector3.Cross` 是纯代数运算，性能开销小

```csharp
// 推荐：用 sqrMagnitude 比大小
if ((target.position - transform.position).sqrMagnitude < detectionRange * detectionRange)
{
    // 在检测范围内
}
```

### 5. 左手坐标系下的叉乘方向

Unity 使用**左手坐标系**，叉乘方向遵循左手定则：
- `Vector3.Cross(forward, right)` → up（用左手：食指指向forward，中指指向right，拇指就是up）
- 与右手坐标系（如 OpenGL）的叉乘方向相反

## 三、坐标系补充

### 1. Transform 坐标转换方法汇总

| 方法 | 作用 |
|------|------|
| `Transform.TransformPoint` | 本地坐标 → 世界坐标（平移） |
| `Transform.InverseTransformPoint` | 世界坐标 → 本地坐标 |
| `Transform.TransformDirection` | 本地方向向量 → 世界方向（忽略缩放） |
| `Transform.InverseTransformDirection` | 世界方向向量 → 本地方向（忽略缩放） |
| `Transform.TransformVector` | 本地向量 → 世界向量（考虑缩放） |
| `Transform.InverseTransformVector` | 世界向量 → 本地向量（考虑缩放） |

```csharp
// 应用：判断目标是否在物体前方（用本地坐标系转换）
Vector3 localPos = transform.InverseTransformPoint(target.position);
if (localPos.z > 0) 
{
    // 目标在物体前方
}
```

### 2. Camera 坐标转换方法

| 方法 | 作用 |
|------|------|
| `Camera.WorldToScreenPoint` | 3D世界坐标 → 2D屏幕像素坐标 |
| `Camera.ScreenToWorldPoint` | 2D屏幕像素坐标 → 3D世界坐标 |
| `Camera.WorldToViewportPoint` | 3D世界坐标 → 归一化视口坐标 |
| `Camera.ViewportToWorldPoint` | 归一化视口坐标 → 3D世界坐标 |
| `Camera.ScreenToViewportPoint` | 屏幕像素坐标 → 归一化视口坐标 |
| `Camera.ViewportToScreenPoint` | 归一化视口坐标 → 屏幕像素坐标 |

```csharp
// 实际应用：在敌人头顶显示血条（世界坐标转屏幕坐标）
Vector3 worldPos = enemy.transform.position + Vector3.up * 2f;
Vector3 screenPos = Camera.main.WorldToScreenPoint(worldPos);
healthBarUI.transform.position = screenPos;

// 实际应用：点击屏幕将物体放到点击处（屏幕坐标转世界坐标）
Ray ray = Camera.main.ScreenPointToRay(Input.mousePosition);
RaycastHit hit;
if (Physics.Raycast(ray, out hit, 100f))
{
    transform.position = hit.point;
}
```

## 四、四元数补充

### 1. 四元数的局限性

- **不能表示超过 180 度的旋转**（插值会走"短弧"）
- **不直观**：无法直接通过数值看出旋转方向
- **累计误差**：长时间连续的旋转更新会有微小漂移
- **不满足交换律**：`q1 * q2 ≠ q2 * q1`（旋转顺序重要）

### 2. 欧拉角 vs 四元数选择指南

| 场景 | 推荐 | 原因 |
|------|------|------|
| 直接设定角度 | 欧拉角 | `transform.eulerAngles = new Vector3(0, 90, 0)` |
| 连续旋转/插值 | 四元数 | 避免万向节死锁 |
| 存储旋转状态 | 四元数 | 表示唯一，不会歧义 |
| 调试查看 | 欧拉角 | 人类可读 |
| 网络同步 | 四元数 | 4个float比3个float更稳定（无万向节） |
| 仅绕单轴旋转 | 欧拉角 | Rotate(new Vector3(0, speed, 0)) 足够 |

### 3. 实用的四元数操作

```csharp
// 1. 获取两个方向之间的旋转
Quaternion rotation = Quaternion.FromToRotation(Vector3.forward, targetDirection);

// 2. 绕指定轴旋转特定角度（叠加在当前旋转上）
transform.rotation *= Quaternion.AngleAxis(90, Vector3.up);

// 3. 四元数的逆（反向旋转）
Quaternion inverse = Quaternion.Inverse(transform.rotation);

// 4. 旋转插值：Slerp 与 Lerp 的实际选择
//    - Slerp：角速度恒定，推荐使用
//    - Lerp：角速度不均匀但计算快，小角度时差异可忽略
transform.rotation = Quaternion.Slerp(transform.rotation, targetRot, Time.deltaTime * 5f);

// 5. 限制旋转角度
//    将当前旋转限制在目标方向 ±30度范围内
float maxAngle = 30f;
Quaternion limitedRot = Quaternion.RotateTowards(
    transform.rotation, 
    targetRot, 
    maxAngle * Time.deltaTime
);
```

### 4. 万向节死锁的深入理解

```
旋转顺序 Y-X-Z (Unity默认):
1. 绕Y轴旋转 → 改变航向（偏航）
2. 绕X轴旋转 → 改变俯仰
3. 绕Z轴旋转 → 改变翻滚

死锁条件：当俯仰角(X) = ±90° 时，Y轴和Z轴对齐
→ 丢失一个旋转自由度
→ 无论怎么转Y和Z，效果等价于绕同一轴旋转
```

**实际后果**：做第一人称射击游戏时，如果直接用欧拉角控制摄像机俯仰，抬头到90°时会突然画面翻滚。解决方案：
- 用四元数累积旋转
- 或限制俯仰角度在 -89°~89° 之间

## 五、Mathf 补充

### 1. 缺失的重要方法

| 方法 | 作用 |
|------|------|
| `Mathf.LerpAngle` | 角度线性插值，自动处理 360° 环绕 |
| `Mathf.MoveTowardsAngle` | 角度匀速移动，自动处理环绕 |
| `Mathf.DeltaAngle` | 计算两个角度之间的最短差值 |
| `Mathf.PerlinNoise` | 柏林噪声，生成自然随机数 |
| `Mathf.InverseLerp` | 反插值：给定值在区间内的比例 (0~1) |
| `Mathf.Remap` | 将一个区间的值映射到另一个区间 |
| `Mathf.Approximately` | 近似相等判断（处理浮点精度误差） |

```csharp
// DeltaAngle 示例：获取两个角度之间的最短方向
float currentAngle = 350f;
float targetAngle = 10f;
float delta = Mathf.DeltaAngle(currentAngle, targetAngle);
// delta = 20°（最短路径），而不是 -340°

// Approximately 示例：避免浮点比较陷阱
if (Mathf.Approximately(a, b))  // 而不是 a == b
{
    // 安全比较浮点数
}
```

### 2. PingPong 和 Repeat 的深入用法

```csharp
// PingPong：在 0~length 之间来回往复（锯齿波）
// 常用于 UI 动画、物体浮动
float value = Mathf.PingPong(Time.time * speed, maxValue);
transform.position = Vector3.up * value;  // 上下往复

// Repeat：从 0 开始循环到 length，然后归零（三角波）
// 常用于循环播放
float t = Mathf.Repeat(Time.time, 1f);  // 每秒循环一次
```

### 3. Mathf 与 Math 的选择

| 场景 | 使用 Mathf | 使用 System.Math |
|------|-----------|-----------------|
| 参数类型 | float | double |
| 精度需求 | 游戏精度足够 | 科学计算需要更高精度 |
| 性能 | 略快（float运算） | 略慢（double运算） |
| Unity集成 | 含 Deg2Rad/Rad2Deg/Lerp 等Math没有的方法 | 不含Unity专用方法 |

> **原则**：游戏开发中始终用 `Mathf`，除非有特殊的高精度需求

## 六、向量/四元数面试常考题

```csharp
// 1. 如何让物体始终面向目标？
transform.LookAt(target);
// 或
Quaternion lookRot = Quaternion.LookRotation(target.position - transform.position);
transform.rotation = Quaternion.Slerp(transform.rotation, lookRot, Time.deltaTime * 5f);

// 2. 如何判断目标是否在物体后方？
Vector3 toTarget = target.position - transform.position;
bool isBehind = Vector3.Dot(transform.forward, toTarget) < 0;

// 3. 两个物体之间的角度是多少？
float angle = Vector3.Angle(transform.forward, target.position - transform.position);

// 4. 绕Y轴旋转物体
transform.Rotate(Vector3.up * speed * Time.deltaTime);
// 或
transform.rotation *= Quaternion.Euler(0, speed * Time.deltaTime, 0);

// 5. 获取物体当前朝向的前方向量
Vector3 forward = transform.forward; // 蓝色轴
Vector3 right = transform.right;     // 红色轴
Vector3 up = transform.up;           // 绿色轴
```
