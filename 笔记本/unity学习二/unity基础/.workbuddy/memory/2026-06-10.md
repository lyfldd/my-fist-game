# 2026-06-10 工作日志

## PDF 转 Markdown 转换

将 workspace 中所有 24 个 PDF 学习笔记转换为 Markdown 格式，并提取内嵌图片。

### 技术方案
- 使用 pdfplumber 提取文字和图片位置信息
- 使用 Pillow 从 PDF 页面渲染图裁剪出图片区域
- 根据 Y 坐标排序，将图片插入到对应文字段落之间
- 图片保存到各 PDF 所在目录的 `_extracted_images/` 子目录中

### 转换结果
- 24/24 PDF 全部成功转换
- 共计提取 320 张图片
- 转换脚本保留在 `pdf2md.py`

### 修正
- 修复了图片路径中使用 Windows 反斜杠 `\` 的问题，全部改为正斜杠 `/`，确保 Markdown 渲染器能正确加载图片
- 同时在 `pdf2md.py` 脚本中加入了 `rel_path.replace("\\", "/")` 修复，后续使用不会再出现此问题

### 第二次修正 — 图片裁剪坐标缩放错误
- 问题：`page.to_image(resolution=200)` 渲染后的图片尺寸（1653x2338）与 PDF 点坐标（595x842）不匹配，直接使用 `page.images` 的坐标裁剪导致裁剪位置完全错误
- 修复：计算 `scale_x = pil_page.width / page.width` 和 `scale_y = pil_page.height / page.height`，将 PDF 点坐标正确缩放到渲染像素坐标
- 效果：图片从修复前的 493字节~10KB（空/残缺）变为 40KB~200KB（平均90KB），内容正确完整

### 第三次修正 — 字体编码乱码
- 问题：PDF 使用自定义数学字体，将拉丁字母（a-z, A-Z）映射到了韩语音节（Hangul Syllable）的 Unicode 码位上
- 表现：`푉푒푐푡표푟3`（实际是 Vector3）、`푄푢푎푡푒푟푛푖표푛`（实际是 Quaternion）、`[푤,푣]`（实际是 [w,v]）
- 修复：建立反向映射表，大写字母 U+D434~D44D → A-Z，小写字母 U+D44E~D467 → a-z，同时修复了 ``（PUA子弹点）→ `•`
- 22 个 MD 文件全部修复，同时在 `pdf2md.py` 中添加了 `fix_pdf_encoding()` 函数确保后续 PDF 转换不再出现此问题
