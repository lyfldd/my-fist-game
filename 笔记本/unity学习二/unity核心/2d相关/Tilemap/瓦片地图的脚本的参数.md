## 二、Tilemap 组件（上半部分）

Tilemap 是瓦片地图的 “画布”，决定了瓦片的位置、颜色、动画和变换。

表格

|参数|功能说明|末日项目实战用法|
|:--|:--|:--|
|**Animation Frame Rate**|动画瓦片的播放帧率|火焰、闪烁灯等动画瓦片的播放速度，一般设为 8-12|
|**Color**|瓦片整体色调 / 染色|给整个 Tilemap 加 tint 效果，比如夜晚场景调暗、废墟场景加黄调|
|**Tile Anchor**|瓦片的锚点位置，默认 `(0.5, 0.5, 0)`（中心）|一般保持默认，确保瓦片在格子中心对齐|
|**Orientation**|瓦片的朝向|`XY`：默认 2D 平面朝向，俯视角项目保持默认即可|
|**Offset**|瓦片的整体偏移（X/Y/Z）|微调整个 Tilemap 的位置，比如和其他层级对齐|
|**Rotation**|瓦片的整体旋转|一般保持 0，除非你要做特殊视角的地图|
|**Scale**|瓦片的整体缩放|一般保持 `(1,1,1)`，和你的美术资源像素尺寸匹配|

## 三、Tilemap Renderer 组件（下半部分）

Tilemap Renderer 是瓦片地图的 “渲染器”，决定了瓦片怎么被画到屏幕上，以及前后遮挡顺序。

表格

|参数|功能说明|末日项目实战用法|
|:--|:--|:--|
|**Sort Order**|瓦片的排序方向|`Top Right`：从右上到左下排序，适合等距视角；<br><br>俯视角项目建议用 `Bottom Left`，按 Y 轴从下到上排序，保证正确遮挡|
|**Mode**|瓦片的渲染模式|`Chunk`：按区块渲染，优化性能（推荐，适合大地图）|
|**Detect Chunk Culling**|区块剔除检测，优化性能|`Auto`：自动根据相机视野剔除不可见区块，大地图必开|
|**Chunk Culling Bound**|区块剔除的边界范围|一般保持默认，或根据相机大小微调|
|**Mask Interaction**|与 Sprite Mask 的交互方式|`None`：默认，不使用遮罩；<br><br>做局部显示效果时可设为 `Visible Inside Mask`|
|**Material**|渲染使用的材质|`Sprites-Default`：默认 2D 精灵材质，保持默认即可|
|**Sorting Layer**|渲染分层|划分不同层级，比如 `Ground`/`Walls`/`Decorations`，控制渲染顺序|
|**Order in Layer**|同一层内的排序，数值越大越靠前|同一层级里，靠前的墙壁 / 建筑设高数值，靠后的设低数值，模拟前后遮挡|

## 一、Grid 组件

Grid 是整个 Tilemap 系统的 “根容器”，决定了所有子 Tilemap 的网格规则。

表格

|参数|功能说明|末日项目实战用法|
|:--|:--|:--|
|**Cell Size**|单个格子的尺寸（X/Y/Z），控制瓦片和网格的对齐|你的例子里 `X=1, Y=0.5, Z=1` 是等距视角常用的 2:1 宽高比（比如 64x32 像素）；<br><br>俯视角项目建议用 `X=1, Y=1, Z=1`，和你的瓦片像素尺寸匹配（如 32x32）|
|**Cell Gap**|格子之间的间距，默认 0（无间距）|一般保持 0，避免瓦片之间出现缝隙；<br><br>做带缝隙的地砖效果时可以设置小数值|
|**Cell Layout**|网格布局类型|`Isometric`：等距视角（你的例子）；<br><br>俯视角项目用 `Rectangular`|
|**Cell Swizzle**|坐标转换规则，适配不同视角的渲染顺序|等距视角保持默认 `XYZ` 即可，不用修改|

## 核心参数详解

表格

|参数|功能说明|末日沙盒项目实战用法|
|:--|:--|:--|
|**Max Tile Change Count**|每帧处理的瓦片变化数量上限|默认为 1000，大地图可以调高（如 2000），避免频繁修改瓦片时卡顿；一般保持默认即可|
|**Extrusion Factor**|碰撞体向外扩展的距离，用来消除瓦片之间的缝隙|给墙壁层设为 `0.05` 左右，能避免角色卡在瓦片缝隙里，提升手感|
|**Use Delaunay Mesh**|使用德劳内三角网格生成碰撞体，适配不规则瓦片形状|仅当你的瓦片是不规则多边形时勾选；标准方形瓦片不用勾，避免性能开销|
|**Material**|2D 物理材质，控制摩擦力和弹性|地面瓦片可以用高摩擦力材质，冰面 / 光滑地面用低摩擦力材质；保持 `None` 为默认即可|
|**Is Trigger**|是否作为触发碰撞体（不阻挡，只触发事件）|陷阱、毒气区域、触发式机关可以勾选，用来触发玩家掉血 / 警报逻辑|
|**Used By Effector**|是否被 2D 效果器（如 Platform Effector）使用|做单向平台、移动平台时勾选，配合效果器实现特殊碰撞逻辑|
|**Composite Operation**|碰撞体的合并模式，和 `Composite Collider 2D` 配合使用|`None`：不合并（默认）<br><br>`Union`：合并为一个整体碰撞体（推荐给墙壁层用，性能更好）|
|**Offset**|碰撞体的整体偏移量|微调整个 Tilemap 碰撞体的位置，解决和美术错位的问题|

## 进阶用法：配合 Composite Collider 2D 优化性能

给你的墙壁 / 建筑 Tilemap 做碰撞时，推荐按以下方式配置：

1. 给 Tilemap 挂上 `Tilemap Collider 2D`
2. 勾选 `Used By Composite`（会自动把 `Composite Operation` 设为 `Union`）
3. 再挂上 `Composite Collider 2D` 组件

这样 Unity 会把所有瓦片的碰撞体合并成一个整体，大幅减少物理计算量，大地图性能提升明显。

## 不同场景配置建议

表格

|瓦片类型|推荐配置|
|:--|:--|
|墙壁 / 建筑（阻挡玩家）|`Extrusion Factor = 0.05` + `Composite Operation = Union` + 配合 `Composite Collider 2D`|
|地面（只提供摩擦力）|不用挂碰撞体，或者挂了之后 `Is Trigger = false`，保持默认即可|
|陷阱 / 机关（触发事件）|`Is Trigger = true`，不合并碰撞体，用来触发玩家掉血 / 警报逻辑|

## 补充：常见问题解决

- **角色卡在瓦片缝隙里**：调高 `Extrusion Factor`，给碰撞体加一点扩展量，消除缝隙。
- **大地图物理卡顿**：开启 `Composite Operation = Union`，合并碰撞体，减少物理计算量。
- **不规则瓦片碰撞不准**：勾选 `Use Delaunay Mesh`，让碰撞体贴合瓦片形状。
