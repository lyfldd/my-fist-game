## 1. 锚点进阶：吸附效果

### 什么是吸附（Snap）

吸附指在编辑器中拖拽 UI 元素时，**Anchors（锚点蓝线）和 Pivot（轴心绿线）** 自动对齐到父容器边缘或参考线的行为。吸附是 Unity Editor 辅助功能，并非运行时行为。

### Unity 中的吸附行为

在 RectTransform 编辑模式下（Anchor Presets 面板）：

| 吸附类型 | 触发方式 | 表现 |
|---------|---------|------|
| **边缘吸附** | 拖拽矩形锚点到父容器边缘 | 锚点线自动吸附到父容器四边 |
| **网格吸附** | 按住 Ctrl（Windows）或 Cmd（Mac）拖拽 | 锚点按网格增量吸附 |
| **蓝线吸附** | 调整 Anchors 面板中的 Min/Max | 锚点自动对齐到 0/0.5/1.0 等比例位置 |
| **绿线吸附** | 调整 Pivot | 轴心自动吸附到 9 个锚点预设位置 |

### 实际应用：UI 面板拖拽吸附到屏幕边缘

在游戏中常见需求：**允许玩家拖拽 HUD 面板，松开后自动吸附到屏幕边缘**，而非停留在中间位置。

### 代码实现吸附

```csharp
using UnityEngine;

/// <summary>
/// 拖拽面板到边缘自动吸附
/// </summary>
public class DragSnap : MonoBehaviour
{
    [Header("吸附参数")]
    public float snapDistance = 100f;          // 距边缘多少像素时触发吸附
    public float snapDuration = 0.2f;          // 吸附动画时长
    public bool snapToLeft   = true;
    public bool snapToRight  = true;
    public bool snapToTop    = true;
    public bool snapToBottom = true;

    private RectTransform rectTransform;
    private Canvas parentCanvas;
    private Vector2 dragOffset;
    private bool isDragging;
    private Vector2 velocity;

    void Awake()
    {
        rectTransform = GetComponent<RectTransform>();
        parentCanvas = GetComponentInParent<Canvas>();
    }

    // ---------- 拖拽逻辑 (配合 EventTrigger 或 IPointerHandler) ----------

    public void OnBeginDrag(Vector2 screenPos)
    {
        isDragging = true;
        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            parentCanvas.transform as RectTransform,
            screenPos, parentCanvas.worldCamera, out dragOffset);

        dragOffset = rectTransform.anchoredPosition - dragOffset;
    }

    public void OnDrag(Vector2 screenPos)
    {
        if (!isDragging) return;

        RectTransformUtility.ScreenPointToLocalPointInRectangle(
            parentCanvas.transform as RectTransform,
            screenPos, parentCanvas.worldCamera, out Vector2 localPos);

        rectTransform.anchoredPosition = localPos + dragOffset;
    }

    public void OnEndDrag()
    {
        isDragging = false;
        TrySnap();
    }

    // ---------- 吸附算法 ----------

    private void TrySnap()
    {
        // 获取父容器尺寸和当前 UI 尺寸
        Rect parentRect = (parentCanvas.transform as RectTransform).rect;
        float panelHalfW = rectTransform.rect.width  * rectTransform.pivot.x;
        float panelHalfH = rectTransform.rect.height * rectTransform.pivot.y;

        Vector2 pos = rectTransform.anchoredPosition;
        float leftEdge   = -parentRect.width  / 2f + panelHalfW;
        float rightEdge  =  parentRect.width  / 2f - (rectTransform.rect.width - panelHalfW);
        float bottomEdge = -parentRect.height / 2f + panelHalfH;
        float topEdge    =  parentRect.height / 2f - (rectTransform.rect.height - panelHalfH);

        // 计算到各个边缘的距离
        float distLeft   = Mathf.Abs(pos.x - leftEdge);
        float distRight  = Mathf.Abs(pos.x - rightEdge);
        float distBottom = Mathf.Abs(pos.y - bottomEdge);
        float distTop    = Mathf.Abs(pos.y - topEdge);

        // 选择最近的有效边缘
        float minDist = float.MaxValue;
        Vector2 targetPos = pos;

        if (snapToLeft && distLeft < snapDistance && distLeft < minDist)
        {
            minDist = distLeft;
            targetPos.x = leftEdge;
        }
        if (snapToRight && distRight < snapDistance && distRight < minDist)
        {
            minDist = distRight;
            targetPos.x = rightEdge;
        }
        if (snapToBottom && distBottom < snapDistance && distBottom < minDist)
        {
            minDist = distBottom;
            targetPos.y = bottomEdge;
        }
        if (snapToTop && distTop < snapDistance && distTop < minDist)
        {
            minDist = distTop;
            targetPos.y = topEdge;
        }

        // 执行吸附
        if (minDist < snapDistance)
        {
            // 使用 DOTween 或简单 Lerp（此处用直接赋值）
            rectTransform.anchoredPosition = targetPos;
        }
    }

    // ---------- 辅助：运行时切换锚点不影响吸附 ----------
    // 注意：吸附后希望面板再拉伸时保持贴边，可以同步修改锚点：
    // 吸附到左 → anchorMin.x = 0, anchorMax.x = 0
    // 吸附到右 → anchorMin.x = 1, anchorMax.x = 1
}
```

## 2. 安全区域（Safe Area）概念

### 为什么需要 Safe Area

现代移动设备屏幕存在各种非矩形区域：

| 屏占比问题 | 代表机型 | 影响 |
|-----------|---------|------|
| **刘海屏（Notch）** | iPhone X~16、华为 Mate 系列 | 顶部/底部一定区域被遮挡 |
| **挖孔屏（Hole Punch）** | 三星 Galaxy S 系列、小米 | 摄像头区域遮挡 |
| **圆角屏（Rounded Corner）** | 几乎所有全面屏手机 | 四角渲染超出可见区域 |
| **状态栏（Status Bar）** | 所有手机 | 系统状态图标区域 |
| **导航栏（Navigation Bar）** | Android 虚拟按键 | 底部操作栏区域 |

<!-- 建议截图：刘海屏/挖孔屏/圆角屏的示意图 -->
<!-- 图片路径示例：images/p2_i1.png -->

### iOS 和 Android 的 Safe Area 区别

| 方面 | iOS | Android |
|------|-----|---------|
| API | `Screen.safeArea` 原生支持 | Unity 封装 `Screen.safeArea`，但某些厂商定制 ROM 有偏差 |
| 状态栏高度 | 包含在 safeArea 顶部 inset | 部分 ROM 不包含，需额外获取 |
| 横竖屏切换 | safeArea 自动更新，需监听 `onSafeAreaChange` | 类似，但部分设备需要延迟一帧读取 |
| 底部导航栏 | 无（iPhone 使用手势） | 部分设备有虚拟导航栏，safeArea 会自动扣除 |
| 圆角范围 | 四角均有圆角 safeArea | 只有顶部/底部，取决于厂商 |

### Canvas Scaler 与 Safe Area 的协作

最佳实践：

1. **Canvas** 设为 `Screen Space - Overlay` 或 `Screen Space - Camera`
2. **Canvas Scaler** 设为 `Scale With Screen Size` + 参考分辨率（如 1080x1920）
3. **根节点**（直接挂在 Canvas 下的容器）不做 Safe Area 适配
4. **内容容器**（存放所有 UI 的 Panel）挂载 SafeAreaHelper 脚本
5. 当 safeArea 发生变化时（如横竖屏切换、来电状态栏），自动触发重新适配

> **重要原则**：SafeArea 适配在**内容 Panel 层级**，而不是 Canvas 或子 UI 元素层级。

## 3. 代码实现 Safe Area

```csharp
using UnityEngine;
using UnityEngine.UI;

/// <summary>
/// 安全区域适配组件
/// 挂在根级内容容器（通常是一个 Panel）上
/// </summary>
[RequireComponent(typeof(RectTransform))]
public class SafeAreaHelper : MonoBehaviour
{
    private RectTransform rectTransform;
    private Rect lastSafeArea = Rect.zero;

    void Awake()
    {
        rectTransform = GetComponent<RectTransform>();
        ApplySafeArea();
    }

    void Update()
    {
        // 安全区域变化时重新适配（横竖屏切换、来电等）
        if (Screen.safeArea != lastSafeArea)
        {
            ApplySafeArea();
        }
    }

    private void ApplySafeArea()
    {
        Rect safeArea = Screen.safeArea;
        lastSafeArea = safeArea;

        // 将 safeArea 像素坐标归一化到 0-1 范围
        Vector2 anchorMin = safeArea.position;
        Vector2 anchorMax = safeArea.position + safeArea.size;

        anchorMin.x /= Screen.width;
        anchorMin.y /= Screen.height;
        anchorMax.x /= Screen.width;
        anchorMax.y /= Screen.height;

        // 设置 RectTransform 锚点
        rectTransform.anchorMin = anchorMin;
        rectTransform.anchorMax = anchorMax;

        Debug.Log($"[SafeAreaHelper] 安全区域更新: ({anchorMin:F3}, {anchorMax:F3})");
    }
}
```

### 进阶版（带 Padding 和内边距）

```csharp
using UnityEngine;

public class SafeAreaHelperEx : MonoBehaviour
{
    [Header("内边距（像素）")]
    public int topPadding    = 0;
    public int bottomPadding = 0;
    public int leftPadding   = 0;
    public int rightPadding  = 0;

    [Header("横竖屏独立配置")]
    public bool useLandscapeConfig = false;
    public int landscapeTopPadding = 0;
    public int landscapeBottomPadding = 0;
    public int landscapeLeftPadding = 0;
    public int landscapeRightPadding = 0;

    private RectTransform rectTransform;
    private ScreenOrientation lastOrientation;

    void Awake()
    {
        rectTransform = GetComponent<RectTransform>();
        ApplySafeArea();
    }

    void Update()
    {
        if (Screen.safeArea != lastSafeArea || 
            (useLandscapeConfig && Screen.orientation != lastOrientation))
        {
            ApplySafeArea();
        }
    }

    private void ApplySafeArea()
    {
        Rect safeArea = Screen.safeArea;
        lastSafeArea = safeArea;
        lastOrientation = Screen.orientation;

        bool isLandscape = Screen.width > Screen.height;

        int t = isLandscape && useLandscapeConfig ? landscapeTopPadding    : topPadding;
        int b = isLandscape && useLandscapeConfig ? landscapeBottomPadding : bottomPadding;
        int l = isLandscape && useLandscapeConfig ? landscapeLeftPadding   : leftPadding;
        int r = isLandscape && useLandscapeConfig ? landscapeRightPadding  : rightPadding;

        // 带内边距的安全区域
        Rect padded = new Rect(
            safeArea.x + l,
            safeArea.y + b,
            safeArea.width  - l - r,
            safeArea.height - t - b
        );

        Vector2 anchorMin = new Vector2(padded.xMin / Screen.width,  padded.yMin / Screen.height);
        Vector2 anchorMax = new Vector2(padded.xMax / Screen.width,  padded.yMax / Screen.height);

        rectTransform.anchorMin = anchorMin;
        rectTransform.anchorMax = anchorMax;
    }
}
```

## 4. 多分辨率适配策略总结

| 设备类型 | 推荐 CanvasScaler 模式 | 推荐 Match 值 | 备注 |
|---------|----------------------|--------------|------|
| 固定横屏（PC/主机） | Scale With Screen Size | 0（宽适配） | 锚点居中，SafeArea 一般不需要 |
| 固定横屏（移动端） | Scale With Screen Size | 0.5 | 兼顾宽高，加 SafeArea |
| 移动端（竖屏） | Scale With Screen Size | 1（高适配） | 顶部留白布局，SafeArea 必加 |
| 移动端（横竖屏切换） | Scale With Screen Size | 0.5 ~ 1 动态调整 | 运行时检测宽高比 |
| 多分辨率上线 | Constant Pixel Size | 不适用 | 像素级精确但适配量大 |

### 运行时动态调整 Match 值

```csharp
using UnityEngine.UI;

public class DynamicCanvasMatch : MonoBehaviour
{
    void Start()
    {
        CanvasScaler scaler = GetComponent<CanvasScaler>();
        
        float ratio = (float)Screen.width / Screen.height;
        
        if (ratio > 1.5f)       // 横屏（如 16:9, 16:10）
            scaler.matchWidthOrHeight = 0f;
        else if (ratio < 0.75f) // 竖屏（如 9:16, 9:18.5）
            scaler.matchWidthOrHeight = 1f;
        else                    // 接近正方形
            scaler.matchWidthOrHeight = 0.5f;
    }
}
```

## 5. 常见问题

| 问题 | 现象 | 原因 | 解决方案 |
|------|------|------|---------|
| **真机偏移** | UI 在编辑器/Prefab 模式正常，真机运行时位置偏移 | 未加 SafeArea 适配，内容跑到了刘海/圆角区域 | 在根 Panel 挂载 SafeAreaHelper |
| **文字被裁切** | 部分文字或按钮超出屏幕边缘被裁掉 | CanvasScaler Match 值选择不当，UI 区域计算错误 | 调整 Match 值，用 Reference Resolution 匹配目标设备 |
| **横竖屏切换后 UI 错位** | 旋转设备后 UI 位置完全不对 | SafeArea 未在运行时重新计算 | Update 中检测 `Screen.safeArea` 变化并重新适配 |
| **图片出现黑边** | Image 边缘出现黑色条纹 | Sprite 九宫格（Sliced）设置不当 | 检查 Sprite Editor 的 Border 设置，确保覆盖所有输入区域 |
| **吸附后锚点偏移** | 拖拽吸附后再缩放父容器，UI 位置跑偏 | 吸附只改了 AnchoredPosition 没改锚点 | 吸附后同步设置 anchorMin/anchorMax |
| **部分安卓机型 SafeArea 不准** | safeArea 返回的数值与视觉不符 | 厂商 ROM 定制，`Screen.safeArea` 未覆盖全面 | 手动获取系统状态栏高度 API，加偏移量适配 |

## 6. 知识小结

| 核心内容 | 考试重点 | 难度系数 |
|---------|---------|---------|
| 吸附（Snap）概念 | 编辑器中锚点蓝线/轴心绿线的吸附行为 | ⭐ |
| 拖拽吸附代码实现 | 计算屏幕边缘距离 + 设置 anchoredPosition | ⭐⭐⭐⭐ |
| Safe Area 需求来源 | 刘海屏、挖孔屏、圆角屏、状态栏 | ⭐⭐ |
| iOS vs Android 差异 | iOS 包含状态栏，Android 需个别处理 | ⭐⭐⭐ |
| CanvasScaler + SafeArea 协作 | 内容 Panel 层级做 SafeArea，Canvas 层级做缩放 | ⭐⭐⭐ |
| SafeAreaHelper 代码实现 | `Screen.safeArea` → 归一化 → 设置 anchorMin/Max | ⭐⭐⭐ |
| 多分辨率策略表 | 不同设备类型推荐的模式和 Match 值 | ⭐⭐ |
| 动态调整 Match | 运行时检测宽高比设置 matchWidthOrHeight | ⭐⭐⭐ |
| 常见问题排查 | 真机偏移、文字裁切、图片黑边 | ⭐⭐ |

### 项目模板：SafeArea 适配检查清单

```
[ ] 所有 Canvas 已挂载 CanvasScaler（Scale With Screen Size）
[ ] 根 Panel 已挂载 SafeAreaHelper
[ ] SafeAreaHelper 在 Awake 中执行一次，Update 中检测变更
[ ] 内边距 Padding 在真机端测试通过
[ ] 横竖屏切换后 UI 位置正常
[ ] 图片背景使用 Sliced 模式，Border 设置正确
[ ] 非交互控件（纯背景、装饰图）关闭 RaycastTarget
[ ] 在 Android 模拟器和 iOS 真机上各测试一轮
```
