## 二、八步递进学习法

> 每学一个主题，按以下顺序逐步深入

| 步骤 | 内容 |
|------|------|
| **Step 1** | **理解是什么** — 概念、作用、基本结构 |
| **Step 2** | **理解怎么用** — 基础语法、简单 C# 示例 |
| **Step 3** | **理解为什么** — 底层原理、执行流程、内部机制 |
| **Step 4** | **时间与空间复杂度分析** — 最好/平均/最坏情况，Big-O 表示法，与其他方案对比 |
| **Step 5** | **注意事项与坑点** — 常见错误、禁止用法、边界条件 |
| **Step 6** | **高级用法** — 组合使用、扩展功能、变体 |
| **Step 7** | **综合练习题** — 结合算法面试题 + Unity 游戏开发场景，精选 3~5 道高质量题目 |
| **Step 8** | **总结成结构化笔记** — 形成可复用的知识体系 |

---

## 三、学习建议

- **按顺序学习，不要跳步**，从第 1 个主题开始
- 每个主题学完后，找 AI 助手按八步法帮你梳理、补充、纠错
- 练习题不在多，**精选 3~5 道**高质量的就好
- 代码示例统一使用 **C#**
- 举例场景以**算法面试题**为主，**Unity 游戏开发**为辅

## 四、笔记本整理规范

| 规范项 | 内容 |
|--------|------|
| 文件修改方式 | 直接在原笔记文件上修改，不新建"整理"副本文件夹 |
| 文件拆分 | 内容超过 5~6 个方法/分类时，概念单独一个文件，每个方法分类独立一个文件；发现符合拆分条件时，主动询问用户，不直接动手 |
| 空文件夹 | 还未学到的内容，直接跳过不处理 |
| 用户优先 | 用户主动告知的改动优先参考 |
| 查漏补缺 | 基于官方标准补充知识点 |
