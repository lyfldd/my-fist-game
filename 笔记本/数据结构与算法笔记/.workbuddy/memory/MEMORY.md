## 一、笔记整理规范

| 规范项 | 内容 |
|--------|------|
| **文件修改方式** | 直接在原来的笔记文件上修改，不新建"整理"副本文件夹 |
| **文件拆分（方案A）** | 内容超过 5~6 个方法/分类时，概念单独一个文件，每个方法分类独立一个文件 |
| **主动询问** | 发现符合拆分条件的笔记时，主动询问用户是否拆分，不自己直接动手 |
| **空文件夹** | 还未学到的内容对应的子文件夹，直接跳过不处理 |
| **用户优先** | 用户主动告知改动时，整理时优先参考用户意见 |
| **查漏补缺** | 基于最新版本和官方标准补充知识点 |

## 二、讲解偏好

- 由浅入深、循序渐进，**不跳步**
- 多用通俗类比，举例场景以 **算法面试题** 为主，辅以 **Unity 游戏开发** 中能用到的真实场景
- 关键知识点拆解细讲，最后总结梳理结构
- **分步输出**：每次只输出当前步骤内容，用户说"下一步"才继续
- **API/方法讲解**：每个关键词/方法逐个详细举例，所有重载用法全部讲完，相同功能放在一起对比差异
- **编程语言**：代码示例统一使用 **C#**

## 三、八步递进学习法（核心）

> 原七步递进法基础上，增加了时间/空间复杂度分析步骤

| 步骤 | 内容 |
|------|------|
| Step 1 | **理解是什么** — 概念、作用、基本结构 |
| Step 2 | **理解怎么用** — 基础语法、简单 C# 示例 |
| Step 3 | **理解为什么** — 底层原理、执行流程、内部机制 |
| Step 4 | **时间与空间复杂度分析** — 最好/平均/最坏情况，Big-O 表示，与其他方案对比 |
| Step 5 | **学习注意事项与坑点** — 常见错误、禁止用法、边界情况 |
| Step 6 | **学习高级用法** — 组合使用、扩展功能、变体 |
| Step 7 | **通过综合练习题把所有知识点串起来实践**（结合算法面试题 + Unity 场景） |
| Step 8 | **总结成结构化笔记，形成可复用的知识体系** |
