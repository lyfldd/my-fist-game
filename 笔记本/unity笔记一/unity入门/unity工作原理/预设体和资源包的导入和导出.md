# 预设体（Prefab）和资源包的导入导出
# 一、概念
Prefab（预制体）：
  把场景中已经配置好的 GameObject 保存到 Assets 文件夹中，
  就生成了一个 Prefab 文件（图标显示为蓝色方块）。
  可以理解为"预制菜"——做好了存起来，以后随时复用，改一处、全部更新。

资源包（Unity Package）：
  团结引擎提供的一种打包方式，可以把 Assets 文件夹下的任意资源
  打包成 .unitypackage 文件，方便在不同项目之间共享或备份。

### 断开 Prefab 连接（解包）
  - 右键场景中的 Prefab 实例 → Prefab → Unpack（解包）
  - 解包后该对象变为普通 GameObject，不再和 Prefab 文件有关联
  - 可以基于解包后的对象创建一个全新的 Prefab

### 资源包的导入导出
  导出：
  - 在 Project 窗口中右键要导出的文件/文件夹
  - 选择 Export Package...（导出资源包）...
  - 勾选要包含的内容 → 导出为 .unitypackage 文件

  导入：
  - 菜单栏 Assets（资源） → Import Package（导入资源包） → Custom Package...（自定义资源包）...
  - 或者直接双击 .unitypackage 文件
  - 勾选要导入的内容 → 导入到当前项目的 Assets 中


# 三、原理
Prefab 的本质是一个序列化的资源文件（.prefab），存储了：
  - GameObject 的名称、层级结构
  - 每个组件的类型及所有序列化字段的值

当你把 Prefab 拖入场景时，团结引擎会：
  1. 读取 .prefab 文件中的序列化数据
  2. 反序列化还原出 GameObject 及其所有组件
  3. 在场景中生成一个 Prefab 实例，并记录它与源 Prefab 的关联

Prefab 实例 = Prefab 源数据 + 实例覆盖（Override）的差异数据
当 Prefab 源文件修改后，所有实例会自动同步（没有被 Override 的部分）。


# 四、坑点
1. Prefab 嵌套时修改要小心
   → Prefab 里可以放其他 Prefab（嵌套 Prefab）
   → 修改内层 Prefab 会影响所有引用它的外层 Prefab，注意影响范围

2. Unpack 之后无法撤回
   → Unpack 之后该对象和 Prefab 断开关联，不能重新"连回去"
   → 只能重新拖入 Assets 创建新的 Prefab

3. 导入资源包可能有版本兼容问题
   → 在新版团结引擎导入旧版制作的资源包，部分 API 可能已过时或变更
   → 建议导入前先备份项目

4. Prefab Override（覆盖）堆积
   → 如果大量实例各自有 Override，之后修改 Prefab 源文件时，
```csharp
```
 被 Override 的部分不会自动同步，可能造成数据不一致
   → 尽量保持实例的覆盖（Override）最小化

5. 导出资源包时依赖项
   → 引擎默认会自动勾选依赖的资源（贴图、材质、脚本等）
   → 建议检查一遍，防止遗漏或多带不需要的文件


# 五、应用场景
- 敌人、子弹、道具等需要重复生成的对象 → 做成 Prefab，运行时动态实例化
- 关卡中重复出现的场景元素（树、路灯、栏杆）→ Prefab 复用，统一修改
- 项目间共享 UI 组件或通用工具脚本 → 导出资源包，导入到新项目
- 美术/程序分工：美术导出资源包，程序导入使用


# 六、扩展
- Prefab Variant（预制体变体）：
  在一个 Prefab 的基础上创建"变体版本"，
  变体会继承原 Prefab 的所有属性，并可以在其基础上做局部修改。
  比如：基础士兵 Prefab → 红队士兵 Variant、蓝队士兵 Variant

- 运行时动态实例化 Prefab：
  常用 Instantiate() 方法在代码中生成 Prefab 实例，
  是游戏逻辑中最常见的操作之一（生成子弹、生成敌人等）

- Package Manager（包管理器）：
  团结引擎的资源管理工具，管理引擎内置包和第三方包，
  位于菜单栏 Window（窗口） → Package Manager（包管理器）
  和 .unitypackage 的手动导入是两套不同的系统
