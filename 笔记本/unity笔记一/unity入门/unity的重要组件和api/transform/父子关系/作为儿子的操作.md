# Transform 作为儿子的操作

## 概念

当一个对象有父对象时，它会以"儿子"的身份存在，可以：判断自己是不是某个对象的子对象（IsChildOf）、获取自己在父对象中的兄弟排序（兄弟索引/SiblingIndex）、调整自己在父对象中的排序位置。这些操作在 UI 排序、角色模型层级管理中常用。

## 用法

### 1. 判断自己是不是某个对象的子对象

```csharp
// 重载：判断 this 是否是指定 Transform 的子对象（任意层级）
bool isChild = this.transform.IsChildOf(father.transform);

// 也可以反过来判断：father 是不是 this 的父对象
bool isParent = father.transform.IsChildOf(this.transform); // false
```

**实际使用**

```csharp
if (this.transform.IsChildOf(GameObject.Find("father2").transform) == true)
{
    print("我是 father2 的儿子");
}
else
{
    print("我没有爸爸");
}
```

### 2. 获取自己在父对象中的兄弟索引（Sibling Index）

```csharp
// 获取自己的兄弟索引（父对象的第几个子对象，从0开始）
int myIndex = this.transform.GetSiblingIndex();

// 总共有多少个兄弟（父对象有几个子对象）
int siblingCount = this.transform.parent.childCount;
```

> ⚠️ 没有父对象时，`GetSiblingIndex()` 返回 0，`parent` 为 null，访问 `parent.childCount` 会报空引用。

### 3. 设置自己在父对象中的兄弟位置

```csharp
// 设置为指定索引位置
this.transform.SetSiblingIndex(2);

// 设置为第一个子对象
this.transform.SetAsFirstSibling();

// 设置为最后一个子对象
this.transform.SetAsLastSibling();
```

**SetSiblingIndex 规则：**
- 传入的值超出范围（>= childCount） → 自动移到最后
- 传入负数 → 也移到最后
- 传 0 → 移到最前面

## 坑点

- `IsChildOf` 可以跨多层级判断（爷爷辈也算），只要 this 在指定对象的层级树内就返回 true。
- 兄弟索引只管**一级子对象**的排序，不影响孙对象。
- 改变 SiblingIndex 会影响渲染顺序（先渲染索引小的），在 UI 中常用来控制遮挡关系。
- `SetAsFirstSibling` 等于 `SetSiblingIndex(0)`，`SetAsLastSibling` 等于 `SetSiblingIndex(childCount - 1)`。

## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 判断对象是否属于某角色 | `boneTransform.IsChildOf(character.transform)` |
| 角色换装后调整装备层级 | `weapon.SetSiblingIndex(2);` |
| UI 点击让某按钮置顶 | `btn.transform.SetAsLastSibling();` |
| 敌人重生后恢复到原位置顺序 | `enemy.transform.SetSiblingIndex(originalIndex);` |

## 源码（部分原理）

```csharp
// IsChildOf 内部（简化）
public bool IsChildOf(Transform parent)
{
    Transform current = this;
    while (current.parent != null)
    {
        if (current.parent == parent) return true;
        current = current.parent; // 向上遍历
    }
    return false;
}
```

递归向上找 `parent`，直到找到为止或到达根对象（parent == null）。

## 扩展

- 在 UI（Canvas）中使用时，Sibling 顺序还受 Render Mode（Screen Space Overlay / Screen Space Camera / World Space）影响。
- 兄弟索引只影响 Transform 层级，不影响 Prefab 实例。如果需要动态调整 Prefab 实例的层级，先 `Instantiate`，再调 `SetSiblingIndex`。
