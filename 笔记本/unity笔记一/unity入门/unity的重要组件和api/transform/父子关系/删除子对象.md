# Transform 删除子对象

## 概念

Unity 中删除子对象有两种方式：通过 Destroy 销毁（从场景移除）或通过 DetachChildren 断绝父子关系（子对象脱离成为根对象，但不移除）。

## 用法

### 1. 一次性断绝所有子对象的父子关系

```csharp
this.transform.DetachChildren();
```

调用后，所有一级子对象脱离父子关系，成为场景根对象（层级和 Inspector 里拖出来的效果一样）。**子对象本身不会被删除**。

场景层级变化：

```text
Before: Parent ─── Child1
               └── Child2
After:  Parent（无子对象）
         Child1（根对象）
         Child2（根对象）
```

### 2. 销毁子对象（真正删除）

```csharp
// 删除单个子对象（GameObject 方式）
GameObject childObj = this.transform.GetChild(0).gameObject;
Destroy(childObj);

// 直接删 Transform 也是可以的（会自动找到父对象的 GameObject）
Destroy(this.transform.GetChild(0).gameObject);

// 删除所有子对象
for (int i = transform.childCount - 1; i >= 0; i--)
{
    Destroy(transform.GetChild(i).gameObject);
}
```

> ⚠️ 销毁是从场景中移除，对象引用变为 `null`，之后访问会报空引用错误。

### DetachChildren vs Destroy 对比

| 操作 | 子对象是否被删除 | 子对象是否脱离父子关系 | 父子层级变化 |
|------|----------------|---------------------|------------|
| `DetachChildren()` | ❌ 否（保留在场景中） | ✅ 全部脱离 | 成为根对象 |
| `Destroy(childObj)` | ✅ 是（从场景移除） | ✅ 自动脱离 | 不存在了 |

## 坑点

- `DetachChildren()` 不会删除任何对象，只是断绝关系，子对象仍在场景中变成根对象。如果你不希望子对象在场景中残留，用 `Destroy`。
- 销毁子对象时，建议从后往前删（`childCount - 1` 到 `0`），否则删除后索引变化导致漏删。
- 如果在循环中用 `for (int i = 0; i < childCount; i++)` 删除，每次 `childCount` 会变化，需要用 `while` 或从后往前。
- `Destroy` 是延迟删除，下一帧才执行，不影响当前帧的循环。

## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 重置父对象的子对象列表 | `transform.DetachChildren()` |
| 清理敌人死亡后留下的特效 | `Destroy(effect, 3f);` |
| UI 列表清空 | `foreach (Transform child in listRoot) Destroy(child.gameObject);` |
| 动态加载完成后整理层级 | `loaded.transform.SetParent(parent, false);` |

## 源码（部分原理）

```csharp
// DetachChildren 内部（简化）
public void DetachChildren()
{
    // 遍历所有子对象
    for (int i = 0; i < childCount; i++)
    {
        GetChild(i).SetParent(null); // 逐个设置为无父对象
    }
}
```

底层就是逐个 `SetParent(null)`。

## 扩展

- `DestroyImmediate(obj)` 可以立即销毁，但**禁止在运行时使用**（会破坏 Unity 的序列化系统和生命周期）。
- 如果需要更精细的父子关系管理（如缓存子对象而不是删除），可以用对象池：`pool.Get()` / `pool.Release(obj)`。
