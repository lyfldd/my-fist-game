# Transform 获取和设置父对象
## 概念

Unity 的 Transform 支持父子层级（Hierarchy）。子对象的位置、旋转、缩放默认受父对象影响（本地坐标系统）。获取和设置父对象是构建场景层级、控制角色装备系统等需求的基础。


## 用法

### 1. 获取父对象

```csharp
```
// 获取父对象的 Transform（可能为 null）
Transform parent = this.transform.parent;

// 获取父对象的 GameObject
GameObject parentGO = this.transform.parent.gameObject;

// 获取父对象的名字
print(this.transform.parent.name);


### 2. 断绝父子关系（设为 null）

```csharp
```
// 直接赋值为 null
this.transform.parent = null;

// 等价 API
this.transform.SetParent(null);

子对象成为场景根对象，世界坐标保持不变。


### 3. 设置新的父子关系

```csharp
```
// 直接赋值
this.transform.parent = GameObject.Find("father2").transform;

// API 方式
this.transform.SetParent(GameObject.Find("father2").transform);


### SetParent 的完整重载

```csharp
```
// 重载1：只传父对象（默认保留世界坐标信息）
transform.SetParent(parent);

// 重载2：传父对象 + 是否保留世界坐标信息
transform.SetParent(parent, false);  // 不保留：子对象坐标变成相对于新父对象的本地坐标
transform.SetParent(parent, true);   // 保留：子对象在世界中的实际位置不变


### 保留 vs 不保留 世界坐标（SetParent 第二参数详解）

假设：
- 子对象当前世界坐标 = (1, 1, 1)
- 新父对象世界坐标 = (3, 3, 3)

| 模式 | 效果 |
|------|------|
| `SetParent(newParent, false)` | 子对象本地坐标 = 世界坐标 - 新父世界坐标 = (-2, -2, -2)，世界坐标变为 (1,1,1) |
| `SetParent(newParent, true)` | 子对象本地坐标自动调整，世界坐标保持 (1,1,1) 不变 |

> 不保留（false）：子对象的"实际位置"（世界坐标）会跳变，通常视觉上会有闪烁。
> 保留（true）：子对象视觉上不动，但本地坐标会变化。


## 坑点

- `transform.parent = null` 和 `transform.SetParent(null)` 在大多数情况下效果相同，但 `SetParent` 支持第二个参数控制是否保留世界坐标。
- 如果父对象在运行时被 Destroy，子对象的 `parent` 会自动变成 null，不会报错。
- `transform.parent` 返回的是 **Transform**，不是 GameObject，要访问父对象本身用 `.gameObject`。


## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 装备系统（枪挂到手上） | `gun.transform.SetParent(handBone, true);` |
| 拾取物品放入背包 | `item.transform.SetParent(inventoryRoot, false);` |
| 解除所有子对象父子关系 | `transform.DetachChildren();` |
| 角色死亡时脱离父对象 | `transform.parent = null;` |

## 扩展

- `Transform.DetachChildren()`：一次性解除所有一级子对象的父子关系，比逐个设 parent=null 更高效。
- 如果需要递归找到根对象，用 `transform.root`，无论层级多深都返回最顶层父对象。
