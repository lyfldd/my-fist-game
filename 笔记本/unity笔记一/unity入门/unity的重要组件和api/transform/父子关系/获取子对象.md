# Transform 获取子对象
## 概念

Transform 提供了多种获取子对象的方式：按名字查找（支持递归）、按索引获取、获取子对象数量。这些方法在动态构建 UI、敌人AI生成、对象池管理中非常常用。


## 用法

### 1. 按名字查找子对象

```csharp
```
// 重载1：找名字匹配的子对象
Transform cube = this.transform.Find("Cube");

// 找子对象的子对象（路径写法）
Transform deep = this.transform.Find("Arm/Hand/Finger");

- **优点**：可以找到失活（SetActive(false)）的对象。
- **缺点**：只能找一级子对象的子对象（即路径写法不能跨层级中间跳），且找不到失活的孙对象。
- 返回值是 `Transform`，如果没找到返回 `null`，不会报错。

> ⚠️ 注意：`Find` 不能跨层级中间跳过——`"Arm/Finger"` 如果 Arm 下没有直接挂载的 Finger（而是在 Arm/Hand 下），则找不到。


### 2. 获取子对象数量

```csharp
```
int count = this.transform.childCount;

> ⚠️ 失活的子对象也会被计入数量。


### 3. 按索引获取子对象

```csharp
```
// 获取第0个子对象
Transform firstChild = this.transform.GetChild(0);

// 遍历所有子对象
for (int i = 0; i < transform.childCount; i++)
{
```csharp
```
Transform child = transform.GetChild(i);
print(child.name);
}

> 子对象的顺序由 `SiblingIndex` 决定，可以在 Inspector 中拖拽调整。


## 坑点

- `Find` 找不到非直接子对象（即孙对象、孙孙对象），除非写完整路径。
- `Find` 在性能上不差，但不要在 Update 中频繁调用 Find，建议缓存结果。
- `childCount` 包括失活的对象，有时候容易忘记过滤。


## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 动态创建n个怪物 | `Instantiate(monster, parentTransform);` |
| 找场景中的"出生点" | `transform.Find("SpawnPoint")` |
| 隐藏/显示所有子对象 | `for (int i=0; i<childCount; i++) GetChild(i).gameObject.SetActive(false);` |
| 动态构建UI | `content.GetChild(index)` 配合对象池 |
