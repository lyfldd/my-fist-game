# Transform 位移

## 概念

位移 = 方向 × 速度 × 时间。Unity 中有两种方式实现位移：**手动计算**（直接改 position）和 **API 调用**（Translate）。核心难点在于坐标系的选择：相对于世界坐标系运动，还是相对于对象自身的坐标系运动。

## 用法

### 方式一：手动计算

```csharp
// 相对于自身坐标系（沿自身X轴）
this.transform.position += this.transform.right * 1 * Time.deltaTime;

// 相对于世界坐标系（沿世界X轴）
this.transform.position += Vector3.right * 1 * Time.deltaTime;
```

- `this.transform.right`：对象自身坐标系 X 轴的世界方向（随对象旋转而变化）。
- `Vector3.right`：固定的世界坐标系 X 轴方向（始终为 (1, 0, 0)）。

### 方式二：Translate API

**重载版本很多，逐一说明：**

```csharp
// 重载1：只传位移向量（默认相对自身坐标系）
transform.Translate(Vector3.forward * speed * Time.deltaTime);

// 重载2：传位移向量 + 相对坐标系
transform.Translate(Vector3.forward * speed * Time.deltaTime, Space.World);
transform.Translate(Vector3.forward * speed * Time.deltaTime, Space.Self);

// 重载3：传XYZ分量（Space.World）
transform.Translate(x, y, z);

// 重载4：传XYZ分量 + 相对坐标系
transform.Translate(x, y, z, Space.Self);

// 重载5：传单轴向量 + 角度 + 相对坐标系（绕某轴旋转后平移）
transform.Translate(Vector3.up, 90f, Space.World);
```

> ⚠️ 注意：重载5容易和重载1混淆。`Translate(Vector3, float, Space)` 第三个参数是 Space，不是轴。

### Space 参数详解

```csharp
Space.World  // 相对于世界坐标系，方向固定
Space.Self   // 相对于对象自身坐标系（默认）
```

**举例说明：**

```csharp
// 沿自身 forward 方向移动（自身坐标系）
this.transform.Translate(this.transform.forward * 1 * Time.deltaTime, Space.Self);

// 沿世界 forward 方向移动（世界坐标系）
this.transform.Translate(Vector3.forward * 1 * Time.deltaTime, Space.World);
```

### Space.Self 的易错点

当对象有旋转时，`Space.Self` 下填入的向量分量是**对象自身坐标系的分量**：

```csharp
// 对象绕Y轴旋转了45度
this.transform.rotation = Quaternion.Euler(0, 45, 0);

// 以 Space.Self 传入 forward(0,0,1)，实际世界移动方向是斜的
this.transform.Translate(new Vector3(0, 0, 1) * Time.deltaTime, Space.Self);
```

> 这等价于：在自身坐标系下，X分量0、Y分量0、Z分量1 → 实际移动 = 0×right + 0×up + 1×forward → 世界中朝45度方向移动。

## 坑点

- **最常见的坑**：`Translate(this.transform.forward, Space.World)` 和 `Translate(Vector3.forward, Space.Self)` 看起来不同但**效果相同**——都是沿世界 forward 方向移动。
- 用 `Translate` 移动不会自动检测碰撞，不会受物理引擎控制。如果需要物理碰撞，用 `Rigidbody.MovePosition`。
- `Translate` 不会考虑时间缩放（`Time.timeScale`），如果想让暂停时停止移动，需要额外处理。

## 应用场景

| 需求     | 推荐方式                                                                         |
| ------ | ---------------------------------------------------------------------------- |
| 角色自动行走 | `transform.Translate(direction * speed * Time.deltaTime, Space.Self)`        |
| 相机跟随目标 | `transform.position = Vector3.Lerp(camera.position, target.position, t)`     |
| 子弹直线飞行 | `transform.Translate(Vector3.forward * speed * Time.deltaTime, Space.World)` |
| 巡逻点移动  | `transform.Translate(direction * speed * Time.deltaTime, Space.World)`       |

## 源码（部分原理）

```csharp
// Translate 简化逻辑
public void Translate(Vector3 translation, Space relativeTo)
{
    if (relativeTo == Space.World)
    {
        // 直接加到世界坐标
        position += translation;
    }
    else
    {
        // 把方向从自身坐标系转换到世界坐标系再加
        position += rotation * translation;
    }
}
```

`rotation * translation` 是一个四元数乘向量的运算，相当于把向量从本地空间旋转到世界空间。

## 扩展

- 如果需要在移动时保持平滑，最常用 `Vector3.Lerp` 或 `Vector3.MoveTowards` 而不是直接 `Translate`。
- 帧位移公式：`direction.normalized * speed * Time.deltaTime`——`Time.deltaTime` 保证无论帧率如何，速度都是一致的。
- 不用 `Time.deltaTime` 的后果：帧率高时移动快，帧率低时移动慢（俗称"帧率依赖"）。
