# Transform 位置

## 概念

Transform 组件管理对象在三维空间中的位置。Unity 提供两套位置坐标：**世界坐标**（World Position）和**本地坐标**（Local Position）。理解两者的区别是掌握 Transform 的基础。

## 用法

### 1. 世界坐标系下的位置

```csharp
// 获取
Vector3 worldPos = this.transform.position;

// 设置（必须整体赋值，不能单独改某个分量）
this.transform.position = new Vector3(1, 2, 3);
```

Inspector 上显示的 position，在有父对象时会变成相对于父对象的值，但 `transform.position` 代码读出来的**始终是世界坐标**。

### 2. 本地坐标系下的位置

```csharp
// 获取
Vector3 localPos = this.transform.localPosition;

// 设置
this.transform.localPosition = new Vector3(1, 2, 3);
```

两者的关系：

| 场景 | position | localPosition |
|------|----------|---------------|
| 没有父对象 | 等同于 localPosition | 等同于 position |
| 父对象在原点 (0,0,0) | 等同于 localPosition | 等同于 position |
| 父对象不在原点 | 世界坐标 | 相对于父对象的坐标 |

### 单独修改某一个分量

位置不能单独改分量，必须整体赋值。方法：

```csharp
// 方法1：只改X分量
this.transform.position = new Vector3(334, this.transform.position.y, this.transform.position.z);

// 方法2：先取出来，修改后再赋值
Vector3 pos = this.transform.position;
pos.x = 334;
this.transform.position = pos;
```

## 坑点

- `transform.position` 和 `transform.localPosition` 都是 **struct**（值类型），赋值时是拷贝，不会有引用问题。
- 单独改某个分量不能直接写 `transform.position.x = 10`，这会报错，因为 Vector3 的分量没有 set 访问器。
- 即使 Inspector 上 position 显示的是世界坐标（有父对象时也会实时换算），代码里用 `transform.position` 读出来的就是世界坐标值，不会因父对象变化而变化。
- 如果父对象旋转了，子对象的本地坐标不变，但世界坐标会变。

## 应用场景

| 需求 | 对应属性 |
|------|----------|
| 获取物体世界坐标（用于距离计算） | `transform.position` |
| 设置物体相对于父对象的位置 | `transform.localPosition` |
| 让子弹在当前位置生成 | `Instantiate(bullet, transform.position, Quaternion.identity)` |
| 相对父对象做偏移 | `transform.localPosition + new Vector3(0, 1, 0)` |

## 源码（部分原理）

```csharp
// Transform.position 的 get/set（简化）
public Vector3 position
{
    get { return GetPositionFromLocal(localPosition, parent); }
    set { SetPositionFromWorld(value, parent); }
}

public Vector3 localPosition
{
    get { return m_LocalPosition; }
    set { m_LocalPosition = value; }
}
```

Unity 内部用一个 4x4 矩阵（Matrix4x4）存储 Transform 的变换：世界坐标 = 父对象矩阵 × 本地坐标。`localPosition` 存储在 C++ 层，`position` 是实时计算出来的。

## 扩展

- `transform.hasChanged` 可以判断 Transform 自上次检查后是否被移动/旋转/缩放，常用于性能优化（避免重复计算）。
- 在编辑器运行时改变 `transform.position`，会在下一帧生效，不会立刻反映在 Inspector 上（除非暂停）。
