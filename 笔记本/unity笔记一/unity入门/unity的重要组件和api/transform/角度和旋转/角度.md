# Transform 角度
## 概念

Unity 用两种方式表示旋转：**欧拉角（Euler Angles）**和**四元数（Quaternion）**。欧拉角是三个角度值（X/Y/Z），直接对应 Inspector 上 Rotation 的显示；四元数是 Unity 内部的旋转表示方式，数值不直观但没有万向锁问题。Transform 同时提供两套接口。


## 用法

### 1. 世界坐标系下的角度

```csharp
```
// 获取（欧拉角，对应 Inspector 的 rotation 值）
Vector3 worldEuler = this.transform.eulerAngles;
print(worldEuler.y); // 查看绕Y轴转了多少度

// 赋值
this.transform.eulerAngles = new Vector3(0, 45, 0);  // 绕Y轴转45度


### 2. 本地坐标系下的角度

```csharp
```
// 获取（相对于父对象的旋转）
Vector3 localEuler = this.transform.localEulerAngles;

// 赋值
this.transform.localEulerAngles = new Vector3(0, 90, 0);


### 两者的区别

| 属性 | 含义 | 有父对象时 |
|------|------|-----------|
| `eulerAngles` | 世界坐标系下的旋转角度 | 可能和 Inspector 不一样（Unity 自动换算） |
| `localEulerAngles` | 相对于父对象的旋转角度 | 直接等于 Inspector 上显示的 rotation |


### 欧拉角的特性

```csharp
```
// 四元数到欧拉角的转换（Unity 内部自动处理）
this.transform.rotation = Quaternion.identity;
print(this.transform.eulerAngles);  // 输出 (0, 0, 0)

> ⚠️ 注意：同一个旋转可以对应多种欧拉角表示，Unity 会自动选取最常用的一种。


## 坑点

- **万向锁（Gimbal Lock）**：当 X 轴旋转接近 ±90° 时，Y 和 Z 轴会"合并"，丢失一个自由度。在 EulerAngles 直接赋值时可能出现意外结果。Unity 内部会自动处理，但如果需要精确控制绕特定轴旋转，优先用四元数。
- **角度赋值必须整体赋值**：`transform.eulerAngles.x = 45;` 会报错。
- `eulerAngles.y` 返回的是 0~360 的浮点数，不是 -180~180。
- 有父对象时，`eulerAngles` 和 Inspector 显示值不同，因为 Inspector 显示的始终是 `localEulerAngles`。


## 应用场景

| 需求 | 推荐方式 |
|------|----------|
| 让对象面向某个方向 | `transform.rotation = Quaternion.Euler(0, 45, 0);` |
| 获取角色当前朝向角度 | `float angle = transform.eulerAngles.y;` |
| 限制相机俯仰角 | `localEulerAngles.x = Mathf.Clamp(localEulerAngles.x, -60, 60);` |
| 让对象始终面对相机（Billboard） | `transform.LookAt(Camera.main.transform);` |

Unity 内部的旋转始终是四元数存储，eulerAngles 只是"投影"到三个轴的角度显示，读写时做双向转换。

- 如果需要获取一个旋转在 0~360 范围内的"最直观"角度（处理负数情况）：
  ```csharp
```
  float NormalizeAngle(float angle)


  angle %= 360;
  if (angle < 0) angle += 360;
  return angle;
  }
- 欧拉角的赋值顺序是 Z→X→Y（Unity 内部顺序），这影响了旋转的叠加方式。
