# Transform 坐标转换（世界坐标 ↔ 本地坐标）

## 概念

Unity 中每个对象都有两套坐标：世界坐标（绝对位置）和本地坐标（相对于父对象的偏移）。在有父子关系时，同一个位置用两套坐标表示，数值不同。Transform 提供了 API 在两套坐标之间互相转换。

## 用法

### 世界坐标 → 本地坐标

```csharp
// Transform.InverseTransformDirection 系列
Vector3 localDir = transform.InverseTransformDirection(worldDir);      // 方向
Vector3 localPoint = transform.InverseTransformPoint(worldPos);         // 点（含位移和旋转）
Vector3 localVector = transform.InverseTransformVector(worldVec);        // 向量（不含位移）
```

**用法举例：**

```csharp
// 把世界坐标 (5, 0, 0) 转换成相对于 this.transform 的本地坐标
Vector3 worldPos = new Vector3(5, 0, 0);
Vector3 localPos = transform.InverseTransformPoint(worldPos);
print(localPos); // 如果父对象在原点，结果接近 (5,0,0)
```

### 本地坐标 → 世界坐标

```csharp
// Transform.TransformDirection 系列
Vector3 worldDir = transform.TransformDirection(localDir);      // 方向
Vector3 worldPoint = transform.TransformPoint(localPos);        // 点（含位移和旋转）
Vector3 worldVector = transform.TransformVector(localVec);     // 向量（不含位移）
```

**用法举例：**

```csharp
// 把本地坐标 (0, 1, 0)（自身上方1米）转换成世界坐标
Vector3 worldPos = transform.TransformPoint(Vector3.up);
print(worldPos); // 物体世界位置 + (0, 1, 0)
```

### 四种方法的核心区别

| API | 包含位移 | 包含旋转 | 典型用途 |
|-----|---------|---------|----------|
| `TransformPoint` | ✅ | ✅ | 坐标点转换（位置） |
| `InverseTransformPoint` | ✅ | ✅ | 坐标点转换（位置） |
| `TransformVector` | ❌ | ✅ | 偏移量、缩放方向 |
| `InverseTransformVector` | ❌ | ✅ | 偏移量、缩放方向 |
| `TransformDirection` | ❌ | ✅（只旋转方向）| 方向向量 |
| `InverseTransformDirection` | ❌ | ✅（只旋转方向）| 方向向量 |

## 坑点

- **用错方法最常见的问题**：`TransformPoint` 会把向量当作带位移的"点"来处理（相加父对象位置），`TransformVector` 不会加位移，只旋转方向。如果把偏移量用 `TransformPoint` 处理，会多加上自身世界坐标，结果错误。
- `TransformDirection` / `InverseTransformDirection` 只负责旋转，方向向量的长度不变，适合处理法线、光线方向等。
- 如果没有父对象，所有四个方向方法的结果相同。

## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 计算自身上方1米的世界坐标 | `transform.TransformPoint(Vector3.up)` |
| 敌人攻击判断是否在范围内 | `transform.InverseTransformPoint(target.position)` 用X/Z距离 |
| 获取自身 right 方向的世界向量 | `transform.TransformDirection(Vector3.right)` |
| UI 跟随3D物体 | `Camera.main.WorldToScreenPoint()` 而不是 Transform API |

## 源码（部分原理）

```csharp
// TransformPoint 内部（简化）
public Vector3 TransformPoint(Vector3 point)
{
    // position 是世界坐标起点，rotation 旋转方向
    return position + rotation * point;
}

// InverseTransformPoint 内部（反向操作）
public Vector3 InverseTransformPoint(Vector3 point)
{
    // 从世界坐标减去起点，再反旋转
    return Quaternion.Inverse(rotation) * (point - position);
}
```

`rotation * point` 是四元数旋转操作，相当于把本地向量旋转到世界空间。

## 扩展

- 经常有人把偏移量（offset）和位置（position）混淆。偏移量是向量，位置是点。偏移量不包含"在哪里"，只包含"走多远"。
- 坐标系转换在相机渲染和物理计算中也常用：如果需要把屏幕像素坐标（Input.mousePosition）转换为世界坐标，需要先用 `Camera.ScreenToWorldPoint`。
