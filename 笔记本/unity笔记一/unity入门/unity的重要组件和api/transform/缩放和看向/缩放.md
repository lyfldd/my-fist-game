# Transform 缩放
## 概念

每个对象都有缩放值，表示在 X/Y/Z 三个轴上被拉伸或压缩的比例。Unity 提供两套缩放：相对于世界坐标系的最终缩放（`lossyScale`）和相对于父对象的本地缩放（`localScale`）。


## 用法

### 获取世界缩放（最终效果）

```csharp
```
// 只读，获取该对象在世界中经过所有父对象累乘后的真实缩放
Vector3 worldScale = this.transform.lossyScale;

> ⚠️ `lossyScale` 是只读的，不能赋值。Unity 没有直接提供修改世界缩放的 API。


### 获取/设置本地缩放

```csharp
```
// 获取
Vector3 localScale = this.transform.localScale;

// 设置（必须整体赋值）
this.transform.localScale = new Vector3(2, 2, 2);  // 放大2倍
this.transform.localScale = Vector3.one;            // 恢复正常大小
this.transform.localScale = new Vector3(0.5f, 1, 2); // 不均匀缩放


### lossyScale 和 localScale 的关系

世界缩放 = 父对象世界缩放 × 本地缩放

如果父对象缩放了2倍，子对象再缩放0.5倍：
子对象世界缩放 = 父缩放(2) × 子本地缩放(0.5) = 1


### Unity 没有直接提供缩放 API

和位置、旋转不同，Unity **没有提供类似 `Scale()` 的 API**。如果想让缩放变化，只能通过修改 `localScale`：

```csharp
```
// 让对象随时间慢慢放大
this.transform.localScale = Vector3.one * (1 + Time.time * 0.1f);

// 让对象X轴脉冲缩放
this.transform.localScale = new Vector3(

## 坑点

- **最重要的坑**：`lossyScale` 是**只读**的，不能赋值。如果你需要改变世界缩放，只能修改 `localScale`（或修改父对象的 `localScale`）。
- 缩放不能单独改某一个轴：`transform.localScale.x = 2;` 会报错，必须整体赋值。
- 负数缩放：可以设置负值（如 `(-1, 1, 1)` 实现镜像），但可能导致法线方向反转，影响光照和碰撞。
- 如果父对象有非均匀缩放（XYZ 不等），子对象的缩放计算会变得复杂，尽量避免这种情况。


## 应用场景

| 需求 | 推荐方式 |
|------|----------|
| 角色受伤闪烁效果 | `transform.localScale = originalScale * (1 + shake * 0.1f);` |
| 物体爆炸时向外缩放消失 | `transform.localScale = Vector3.Lerp(scale, Vector3.zero, t);` |
| UI 血条根据血量缩放X轴 | `healthBar.localScale = new Vector3(currentHP/maxHP, 1, 1);` |
| 跳蚤敌人随机大小 | `enemy.localScale = Vector3.one * Random.Range(0.5f, 2f);` |

递归向上乘以每个父对象的本地缩放，所以叫 `lossy`（有损）——因为旋转会导致缩放方向变化，结果是近似的世界缩放，不是精确的。


## 扩展

- 如果需要在编辑器中手动设置精确的世界缩放，需要先计算本地缩放：`localScale = worldScale / parent.lossyScale`。
- Scale 在父子关系下是非线性叠加的，所以设计场景层级时尽量保持父对象 Scale 为1，避免调试困难。
