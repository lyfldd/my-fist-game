# 脚本的成员变量与 Inspector 显示
# 一、概念
Inspector 检视窗口里能看到的变量，就是脚本的成员变量。
通过反射机制，团结引擎可以将脚本的字段值序列化显示在 Inspector 中，
也可以把 Inspector 中修改的值反射写回到字段里。


# 二、Inspector 显示规则
1. public 字段 → 默认在 Inspector 中显示，可直接编辑

2. private / protected 字段 → 默认不在 Inspector 中显示
   加上 [SerializeField] 特性可强制显示：
```csharp
```
 [SerializeField]
 private int myValue;

3. public 字段不想显示 → 加上 [HideInInspector] 特性隐藏：
```csharp
```
 [HideInInspector]
 public int myValue;


# 三、可显示的类型
支持直接在 Inspector 中显示的类型：
  - int、float、double、string、bool
  - Vector2、Vector3、Vector4
  - Color、Quaternion、AnimationCurve
  - GameObject、Transform、Rigidbody 等组件引用
  - 数组 Array、List<T>
  - 枚举类型 enum

不支持直接显示的类型：
  - Dictionary<K, V>
  - 自定义 struct（结构体）
  - 自定义 class（类）

想让自定义类型也能在 Inspector 中显示：
  在类/结构体定义上方加上 [System.Serializable] 特性：
```csharp
```
[System.Serializable]
public class ItemData
{
```csharp
```
public int id;
public string name;
}
  注意：Dictionary 无论加什么特性都无法直接序列化显示


# 四、辅助特性（装饰 Inspector）
1. [Header("分组标题")]
   在变量上方加标题注释，分组显示：
```csharp
```
 [Header("基础属性")]
 public int level;
 [Header("战斗属性")]
 public int attack;

2. [Tooltip("悬停提示文字")]
   在 Inspector 中鼠标悬停时显示提示：
```csharp
```
 [Tooltip("角色的初始生命值")]
 public int hp;

3. [Space()]
   在变量之间加空白间隔：
```csharp
```
 public int a;
 [Space()]
 public int b;

4. [Range(min, max)]
   把数值显示为滑块：
```csharp
```
 [Range(0, 100)]
 public float speed;

5. [Multiline(lines)]
   多行文本框（用于 string）：
```csharp
```
 [Multiline(4)]
 public string description;

6. [ContextMenuItem("菜单名", "方法名")]
   在 Inspector 右键菜单中添加重置按钮：
```csharp
```
 [ContextMenuItem("重置金币", "ResetMoney")]
 public int money = 100;
 private void ResetMoney()
 {
```csharp
```
 money = 0;
 }

7. [ContextMenu("菜单名")]
   让方法在 Inspector 右键菜单中显示并可执行：
```csharp
```
 [ContextMenu("打印日志")]
 private void PrintLog()
 {
```csharp
```
 Debug.Log("点击了");
 }


# 五、原理
Inspector 显示变量的本质是团结引擎的序列化（Serialize）系统：
  - 引擎在编辑模式下通过反射扫描脚本的所有字段
  - 将字段值序列化（转换为可存储的数据格式），显示在 Inspector 中
  - 用户在 Inspector 中修改值 → 引擎通过反射把新值写回字段

[SerializeField] 让私有字段也被序列化，所以能显示在 Inspector 中。
但字段仍然是私有的，外部代码仍然无法直接访问。


# 六、坑点
1. [SerializeField] 只作用于单个字段，每个字段都要单独加
   → 不能加在类上方让所有字段都序列化

2. 静态字段不会显示在 Inspector
   → 静态字段属于类，不属于实例，序列化系统不处理

3. 只有实例的字段才会显示
   → 脚本挂载在哪个 GameObject 上，就显示哪个实例的值

4. 属性（property）不能直接序列化
   → Inspector 显示的是字段，不是属性
   → 如果需要让 Inspector 显示属性值，可以用一个私有字段 + 公共属性


# 七、应用场景
- 在 Inspector 中配置角色属性（生命、攻击、移动速度等）
- 拖拽引用其他 GameObject、组件、资源文件
- 用 [Header] 分组整理大量配置项
- 用 [Range] 限制数值的合法范围


# 八、扩展
- 自定义编辑器窗口（EditorWindow）可以完全自定义 Inspector 的显示方式
- [ExecuteInEditMode] 特性可以让脚本在编辑模式下也执行生命周期函数
- SerializedObject / SerializedProperty API 用于编辑器扩展脚本中访问序列化数据
