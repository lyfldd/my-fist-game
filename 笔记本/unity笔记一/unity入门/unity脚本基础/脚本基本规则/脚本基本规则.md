# Unity（团结引擎）脚本基本规则
# 一、概念
团结引擎脚本是用 C# 编写、挂载在 GameObject 上的代码文件。
引擎通过反射机制来加载和执行脚本，因此脚本的创建和命名有一套固定规则。


# 二、用法（脚本的创建规则）
1. 不在 Visual Studio 2022 里直接新建 .cs 文件创建脚本
   → 应该在团结引擎编辑器的 Project 窗口（Assets 文件夹）中
```csharp
```
 右键 → Create（创建） → C# Script（C# 脚本）
   → 这样引擎才会正确地把脚本纳入编译管理，并生成默认的 MonoBehaviour 模板

2. 脚本可以放在 Assets 文件夹下的任意位置
   → 没有强制的路径要求，但建议按功能分好文件夹（如 Scripts/Player、Scripts/Enemy）

3. 类名和文件名必须一致（重要！）
   → 引擎通过反射用文件名去查找对应的类，名称不一致会导致挂载失败
   → 旧版：不一致直接报错无法挂载
   → 团结引擎（2022+）：放宽了限制，但仍会弹出警告，强烈建议保持一致

4. 不要用中文命名文件和类名
   → 部分版本或系统环境下，中文路径/类名会导致编译报错或反射查找失败

5. 一般不管命名空间（入门阶段）
   → 团结引擎脚本默认不写 namespace，不影响功能
   → 大型项目中才需要用命名空间管理模块

6. 创建的脚本一般继承 MonoBehaviour
   → 继承 MonoBehaviour 的脚本才能挂载到 GameObject 上
   → 才能使用生命周期函数（Awake、Start、Update 等）


# 三、原理
团结引擎编辑器启动时，会扫描 Assets 下所有 .cs 文件并编译成程序集（Assembly）。
挂载脚本的本质：
  - 引擎通过文件名找到对应的 Type（反射）
  - 用反射 new 出该类的实例
  - 把实例作为组件附加到 GameObject 上

所以类名和文件名的一致性是反射能正确查找类型的前提。


# 四、坑点
1. 在 VS 里直接新建文件
   → 这样创建的脚本没有默认的 MonoBehaviour 模板
   → 引擎的元数据（.meta 文件）也不会自动生成，可能导致资源管理混乱
   → 正确做法：始终在团结引擎 Project 窗口里创建脚本

2. 创建后立刻在文件系统里重命名文件
   → 如果在引擎外重命名文件，文件名改了但类名还是旧的，造成不一致
   → 正确做法：在团结引擎 Project 窗口里重命名（引擎会同步修改 .meta 文件）
   → 注意：引擎不会自动修改 .cs 文件内的类名，需要手动进去改

3. 同名脚本放在不同文件夹
   → 同一个程序集中不能有两个同名的类（没有命名空间的情况下）
   → 会导致编译报错

4. 脚本写错导致整个引擎无法编译
   → 团结引擎中只要有一个脚本报错，整个项目会停止编译
   → 在未解决报错之前，所有脚本的修改都不会生效


# 五、应用场景
- 创建玩家控制脚本、敌人 AI 脚本、UI 管理脚本等，都遵循这套规则
- 理解这些规则是避免"脚本挂载失败"、"Missing Script"等初级报错的基础


# 六、扩展
- Assembly Definition（程序集定义）：
  大型项目中可以手动划分程序集，加速编译，隔离依赖
  入门阶段不需要关心，团结引擎会把所有脚本放到默认的 Assembly-CSharp 程序集中

- 脚本模板自定义：
  可以修改引擎安装目录下的脚本模板，让新建脚本时自动生成自定义的默认内容
