# 生命周期函数有哪些
# 一、生命周期总览
脚本生命周期分为三个阶段：

  ┌──────────────────────────────────────┐
  │  第一阶段：初始化（对象出生）         │
  │  Awake → OnEnable → Start            │
  ├──────────────────────────────────────┤
  │  第二阶段：游戏循环（每帧更新）       │
  │  FixedUpdate → Update → LateUpdate   │
  │  （失活时：OnDisable → 退出循环）     │
  │  （重新激活：OnEnable → 重入循环）    │
  ├──────────────────────────────────────┤
  │  第三阶段：销毁（对象死亡）           │
  │  OnDisable → OnDestroy               │
  └──────────────────────────────────────┘


# 二、各函数详解
### Awake
  触发时机：对象出生时（加载/创建时）立即调用，整个生命周期只调用一次
  特点：
```csharp
```
- 即使脚本被禁用（enabled=false），Awake 也会执行
- 所有脚本的 Awake 都执行完后，才开始执行所有脚本的 Start
  用途：脚本自身的初始化（获取组件、赋初始值）

### OnEnable
  触发时机：依附的 GameObject 每次从「失活→激活」时调用（可多次触发）
  特点：
```csharp
```
- 对象第一次出生时，Awake 之后紧接着执行 OnEnable
- 之后每次重新激活都会触发一次
  用途：每次激活时需要重置或重新注册的逻辑（订阅事件、重置计时器）

### Start
  触发时机：对象创建后，第一次帧更新之前调用，整个生命周期只调用一次
  特点：
```csharp
```
- 执行时，所有其他脚本的 Awake 已经完成
- 适合做跨脚本的初始化（获取其他脚本的引用）
  用途：依赖其他对象/脚本初始化完成后才能执行的逻辑

### FixedUpdate
  触发时机：物理帧更新，固定时间间隔执行（默认 0.02s，即 50次/秒）
  特点：
```csharp
```
- 和游戏帧率无关，频率固定
- 可以在 Edit → Project Settings → Time 中调整固定间隔
- 一帧内可能执行多次（帧率 < 50），也可能不执行（帧率极高）
  用途：物理相关逻辑（刚体移动、施加力）

### Update
  触发时机：每帧执行一次
  特点：
```csharp
```
- 执行频率随帧率变化，不固定
- 需要用 Time.deltaTime 来保证速度与帧率无关
  用途：游戏逻辑、输入检测、非物理运动等

### LateUpdate
  触发时机：每帧执行，在所有 Update 执行完毕之后调用
  特点：
```csharp
```
- 引擎内部在 Update 和 LateUpdate 之间处理了动画更新
- 所以 LateUpdate 时，动画状态已经是这帧的最新状态
  用途：相机跟随（等目标位置在 Update 中确定后，再更新相机位置）

### OnDisable
  触发时机：依附的 GameObject 每次从「激活→失活」时调用（可多次触发）
  用途：每次失活时需要清理的逻辑（取消订阅事件、暂停计时器）

### OnDestroy
  触发时机：对象被销毁时调用，整个生命周期只调用一次
  用途：最终清理工作（释放资源、取消订阅、保存数据）


# 三、坑点
1. Awake vs Start 的区别
   → Awake：出生立刻调用，适合自身初始化
   → Start：第一帧前调用，此时所有 Awake 已完成，适合跨脚本初始化
   → 如果 A 的 Start 里需要用 B 脚本的数据，B 的数据应该在 Awake 里初始化

2. OnEnable / OnDisable 可以被多次触发
   → 不要在这里做「只需要执行一次」的初始化，那应该放 Awake 或 Start

3. 不用的生命周期函数要删掉
   → 即使是空方法，引擎也会把它纳入调用列表，产生额外开销

4. FixedUpdate 不是每帧执行一次
   → 帧率低时，一帧内 FixedUpdate 会执行多次
   → 帧率高时，某些帧内 FixedUpdate 可能不执行


# 四、应用场景
  角色移动逻辑    → Update（输入检测）+ FixedUpdate（刚体物理移动）
  相机跟随        → LateUpdate
  对象池复用      → OnEnable（初始化状态）+ OnDisable（还回对象池）
  事件订阅管理    → OnEnable（订阅）+ OnDisable（取消订阅）
  资源释放        → OnDestroy


# 五、扩展
- MonoBehaviour 还有更多事件回调（不属于主循环，但也是生命周期相关）：
  OnCollisionEnter/Stay/Exit   → 碰撞开始/持续/结束
  OnTriggerEnter/Stay/Exit     → 触发器进入/持续/退出
  OnBecameVisible/Invisible    → 进入/离开摄像机视野
