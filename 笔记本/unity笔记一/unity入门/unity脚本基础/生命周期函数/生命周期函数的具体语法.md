# 生命周期函数的具体语法
# 一、打印输出
在团结引擎里打印信息，用于调试：
  Debug.Log("普通信息");
  Debug.LogWarning("警告信息");
  Debug.LogError("错误信息");

继承 MonoBehaviour 后也可以用 print()，相当于 Debug.Log()：
  print("信息");

代码示例：
  private void Awake()
  {
```csharp
```
  Debug.Log("出生了");
  Debug.LogError("出错了");
  }


# 二、Awake
触发时机：对象出生时立即调用，类似构造函数，整个生命周期只执行一次。
  private void Awake()
  {
```csharp
```
  Debug.Log("Awake");
  }
特点：
  - 即使脚本.enabled = false，Awake 也会执行
  - 主要用于脚本自身的初始化


# 三、OnEnable
触发时机：依附的 GameObject 每次从「失活→激活」时调用，可多次触发。
  private void OnEnable()
  {
```csharp
```
  print("OnEnable");
  }
特点：
  - 对象激活时调用，可多次触发
  - 适合做每次激活时需要重置的逻辑


# 四、Start
触发时机：对象创建后，第一次帧更新之前调用，只执行一次。
  void Start()
  {
```csharp
```
  print("Start");
  }
与 Awake 的区别：
  - Awake：出生时立刻调用
  - Start：这一帧结束后的下一帧开始前才调用
  - Awake 执行时其他脚本可能还没初始化，Start 执行时所有 Awake 已完成
  - 主要作用：做跨脚本的初始化


# 五、FixedUpdate
触发时机：物理帧更新，固定时间间隔（默认 0.02 秒，即 50次/秒）。
  private void FixedUpdate()
  {
```csharp
```
  print("物理更新");
  }
用途：
  - 所有物理相关的逻辑（刚体移动、施加力）都应写在这里
  - 不受帧率波动影响，物理模拟稳定


# 六、Update
触发时机：每帧执行一次，频率随帧率变化。
  void Update()
  {
```csharp
```
  print("游戏逻辑更新");
  }
用途：
  - 处理游戏逻辑、输入检测、非物理运动
  - 需要配合 Time.deltaTime 保证速度与帧率无关


# 七、LateUpdate
触发时机：每帧执行，在 Update 全部执行完毕之后调用。
  private void LateUpdate()
  {
```csharp
```
  print("后处理更新");
  }
用途：
  - 常用于相机跟随（等目标位置更新完再更新相机）
  - 引擎内部在 Update 和 LateUpdate 之间处理了动画更新


# 八、OnDisable
触发时机：依附的 GameObject 每次从「激活→失活」时调用。
  private void OnDisable()
  {
```csharp
```
  print("失活了");
  }
用途：
  - 每次失活时需要清理的逻辑（取消事件订阅、暂停计时器等）


# 九、OnDestroy
触发时机：对象被销毁时调用，整个生命周期只执行一次。
  private void OnDestroy()
  {
```csharp
```
  print("对象被销毁了");
  }
用途：
  - 最终清理工作（释放资源、保存数据等）


# 十、注意事项
1. 不用的生命周期函数要删除，空方法也会产生性能开销
2. 生命周期函数可以是 private 或 protected，不需要 public
3. 可以用 virtual 修饰，子类可以重写
4. 不要手动调用生命周期函数，引擎会自动调用
