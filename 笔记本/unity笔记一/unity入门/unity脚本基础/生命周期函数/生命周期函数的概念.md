# 生命周期函数的概念
# 一、概念
生命周期函数（Lifecycle Functions）是团结引擎在 MonoBehaviour 脚本的
特定时机，通过「反射」自动调用的一组固定名字的方法。

它们描述了一个脚本（组件）从出生到死亡整个过程中，各个关键节点的回调。
我们只需要在脚本里声明这些方法，引擎会在合适的时候自动调用它们，
不需要我们手动触发。

  - 写 public 也能工作，但不符合规范


# 三、原理
团结引擎在底层维护了所有 GameObject 上挂载的脚本列表。
在游戏运行的各个阶段，引擎会遍历这个列表，
通过反射查找并调用对应名字的方法。

所以生命周期函数的本质 = 「约定好名字的回调函数」，
引擎在特定时机用反射帮你调用它们。

性能优化：
  如果一个生命周期函数里不写任何逻辑，引擎也不会去调用它，
  但「空方法的存在本身」会让引擎在反射扫描时把它列入调用队列，
  产生微小的性能开销。所以不用的生命周期函数要删掉。


# 四、坑点
1. 空生命周期函数会浪费性能
   → 不需要用的生命周期函数（比如空的 Update()）要删除
   → 即使方法体是空的，引擎也会把它纳入每帧调用列表

2. 不要手动调用生命周期函数
   → Awake()、Start()、Update() 等是给引擎自动调用的
   → 手动调用 this.Update() 虽然语法上没问题，但语义混乱，容易产生 bug

3. 生命周期函数不能是静态方法
   → 静态方法属于类，不属于实例，引擎的反射调用是基于实例的
   → 把生命周期函数写成 static 会导致引擎无法正常调用

4. 在生命周期函数里抛出异常会中断执行
   → 如果 Update 里抛出未捕获的异常，引擎会在控制台报错，

# 五、应用场景
- Awake：初始化该脚本自身的数据和状态
- Start：初始化需要依赖其他脚本已初始化完成的逻辑
- Update：每帧逻辑处理（输入检测、非物理运动、状态更新）
- FixedUpdate：物理逻辑（刚体移动、碰撞力）
- LateUpdate：相机跟随、动画后处理


# 六、扩展
- 除了常用的几个，MonoBehaviour 还有其他生命周期相关回调：
  OnCollisionEnter/Stay/Exit   → 碰撞事件
  OnTriggerEnter/Stay/Exit     → 触发器事件
  OnMouseDown / OnMouseEnter   → 鼠标交互事件
  OnApplicationPause           → 应用暂停/恢复
  OnApplicationQuit            → 应用退出

- 协程（Coroutine）是一种特殊的"可暂停执行"机制，
  可以配合生命周期函数使用，实现延迟、分帧执行等效果
  使用 StartCoroutine() 开启，yield return 暂停
