# MonoBehaviour 重要的方法 —— 组件获取系列

## 一、概念

组件获取方法是 MonoBehaviour 最核心的 API 之一。通过 GetComponent 系列方法，可以在代码中获取同一个 GameObject 上挂载的其他组件，从而实现脚本之间的互相访问和协作。获取到组件引用后，可以访问对方的公共字段、属性、方法。

## 二、所有重载逐一详解

### GetComponent 系列

**`GetComponent<T>()` 泛型版本（最常用）**

```csharp
Rigidbody rb = GetComponent<Rigidbody>();
rb.AddForce(Vector3.forward);
```

> 注意：如果对象上没有这个组件，返回 null。

**`GetComponent(Type type)` Type 版本**

```csharp
Rigidbody rb = GetComponent(typeof(Rigidbody)) as Rigidbody;
```

> 需要强制类型转换（as），返回的是基类 Component 类型。

**`GetComponent(string typeName)` 字符串版本**

```csharp
Rigidbody rb = GetComponent("Rigidbody") as Rigidbody;
```

> ⚠️ 字符串要区分大小写，容易写错，不推荐使用，无法获得编译时类型检查。推荐程度：泛型版本 > Type 版本 > 字符串版本。

---

### GetComponents 系列

**`GetComponents<T>()` 无参数重载**

```csharp
Rigidbody[] rbs = GetComponents<Rigidbody>();
print(rbs.Length);
```

> 只获取该对象本身的组件，不包含子对象。

**`GetComponents<T>(List<T> results)` List 参数重载**

```csharp
List<Rigidbody> rbs = new List<Rigidbody>();
GetComponents<Rigidbody>(rbs);
print(rbs.Count);
```

> 比无参数版本性能更好（避免频繁 new 数组），适合在需要频繁查询的场景使用。传入的 List 会被清空后重新填入结果。

---

### GetComponentInChildren 系列

**`GetComponentInChildren<T>()` 默认重载**

```csharp
Collider col = GetComponentInChildren<Collider>();
```

> 会递归查找所有子对象，按照子对象层级顺序查找（先找自己，再找子，再找子的子……），找到第一个就立即返回，不会继续找。

**`GetComponentInChildren<T>(bool includeInactive)` 包含失活对象重载**

```csharp
// 只在激活的子对象中查找
Collider col = GetComponentInChildren<Collider>(false);

// 在激活和失活的子对象中都查找
Collider col2 = GetComponentInChildren<Collider>(true);
```

> `includeInactive = false`（默认）：只找激活的子对象。`includeInactive = true`：激活和失活的都找。

---

### GetComponentsInChildren 系列

**`GetComponentsInChildren<T>()` 无参数重载**

```csharp
Collider[] cols = GetComponentsInChildren<Collider>();
print(cols.Length);
```

> 默认 `includeInactive = false`（只找激活的子对象）。

**`GetComponentsInChildren<T>(bool includeInactive)` 包含失活重载**

```csharp
Collider[] cols = GetComponentsInChildren<Collider>(true);
```

**`GetComponentsInChildren<T>(List<T> results)` List 参数重载**

```csharp
List<Collider> cols = new List<Collider>();
GetComponentsInChildren<Collider>(cols);
```

**`GetComponentsInChildren<T>(bool includeInactive, List<T> results)` 双参数重载**

```csharp
List<Collider> cols = new List<Collider>();
GetComponentsInChildren<Collider>(true, cols);
```

---

### GetComponentInParent 系列

**`GetComponentInParent<T>()` 无参数重载**

```csharp
Rigidbody rb = GetComponentInParent<Rigidbody>();
```

> 会查找父对象、祖父对象……直到根对象。如果当前对象本身有这个组件，也会返回（因为向上查找从自己开始）。找不到返回 null。

**`GetComponentInParent<T>(bool includeInactive)` 包含失活重载**

```csharp
Rigidbody rb = GetComponentInParent<Rigidbody>(false);
```

---

### TryGetComponent 系列（推荐使用）

**`TryGetComponent<T>(out T component)`**

```csharp
if (TryGetComponent<Rigidbody>(out Rigidbody rb))
{
    rb.AddForce(Vector3.forward);
}
```

> 尝试获取组件，如果对象上没有这个组件，不会返回 null，而是返回 false。和 `GetComponent<T>()` 的区别：`GetComponent<T>()` 没找到时返回 null，之后访问会报 NullReferenceException。`TryGetComponent<T>()` 没找到时返回 false，不会报错，更安全。推荐在不确定对象上是否有某个组件时使用。

## 三、坑点

1. `GetComponent<Type>` 返回 null 没有做判断 → `if (GetComponent<Rigidbody>() != null) { ... }` 或者直接用 `TryGetComponent` 更安全。

2. `GetComponentInChildren` 默认不查找失活子对象 → 子对象 `SetActive(false)` 后，默认找不到它身上的组件。需要传 true 参数：`GetComponentInChildren<Collider>(true)`。

3. `GetComponent(string)` 容易写错字符串 → 类名写错不会在编译时报错，只会在运行时返回 null。建议用泛型版本 `GetComponent<T>()`。

4. `GetComponent` 在 Update 里频繁调用有性能开销 → 建议在 Start 或 Awake 里获取一次，缓存到字段中复用。不要在每帧 Update 里重复调用 GetComponent。

5. `GetComponentsInChildren` 默认包含当前对象自身 → 它不仅找子对象，也找自己身上的组件。如果只想找子对象，可以用递归手动过滤。

## 四、应用场景

| 需求 | 推荐方法 |
|------|----------|
| 获取刚体做物理移动 | `GetComponent<Rigidbody>()` |
| 动态获取敌人身上的 AI 脚本 | `GetComponent<EnemyAI>()` |
| 遍历子对象的所有碰撞体做特效 | `GetComponentsInChildren<Collider>()` |
| 判断物体有没有某组件 | `TryGetComponent`（更安全的写法） |
| 父对象获取子对象的脚本实现父子通信 | `GetComponentInChildren` |

## 五、扩展

游戏对象创建后，应该在 Start() 或 Awake() 中提前缓存组件引用，而不是每次使用时都 GetComponent：

```csharp
public class Example : MonoBehaviour
{
    private Rigidbody rb;

    private void Awake()
    {
        rb = GetComponent<Rigidbody>(); // 提前缓存
    }

    private void Update()
    {
        rb.AddForce(Vector3.forward); // 直接用缓存的引用
    }
}
```

Unity 还有一个全局查找方法：`FindObjectOfType<T>()` / `FindObjectsOfType<T>()` 可以遍历整个场景查找对象，但性能很差，慎用。如果组件可能不存在但不需要报错，用 `TryGetComponent` 是最佳实践。
