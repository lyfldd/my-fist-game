# MonoBehaviour 重要的成员属性
# 一、概念
MonoBehaviour 继承自 Component，而 Component 依附于 GameObject。
因此每个继承 MonoBehaviour 的脚本都可以直接访问几个重要的成员属性：
  - gameObject：本脚本依附的 GameObject
  - transform：本脚本依附的对象的位置、旋转、缩放信息
  - enabled：控制脚本是否激活

这些是 MonoBehaviour 自带的内置成员，不需要声明，可以直接使用。


# 二、用法
### 1. gameObject
  获取本脚本依附的 GameObject 对象。
  可以访问该对象的各种属性和方法。

  语法：
```csharp
```
gameObject                    // 省略 this，推荐写法
this.gameObject               // 带 this，显式写法，效果相同

  常用属性（直接写在 gameObject 后面点出来）：
```csharp
```
gameObject.name               // 对象名称
gameObject.activeSelf         // 对象自身是否激活
gameObject.activeInHierarchy   // 对象在场景中是否真正激活
gameObject.tag                // 对象的标签
gameObject.layer              // 对象的层级

  常用方法：
```csharp
```
gameObject.SetActive(true);   // 激活/失活对象
gameObject.CompareTag("Player"); // 判断标签

  示例：
```csharp
```
print(gameObject.name);               // 打印对象名
print(gameObject.activeSelf);         // 打印自身激活状态
gameObject.SetActive(false);          // 让对象失活（连同脚本一起失活）


### 2. transform
  获取本脚本依附对象的 Transform 组件，用于操作位置、旋转、缩放。

  语法：
```csharp
```
transform                    // 省略 this，推荐写法
this.transform               // 显式写法，效果相同
this.gameObject.transform    // 从 gameObject 再拿到 transform

  常用属性：
```csharp
```
transform.position           // 世界坐标位置（Vector3）
transform.localPosition      // 相对父对象的局部坐标
transform.rotation          // 世界坐标旋转（Quaternion）
transform.localRotation      // 局部旋转
transform.localScale        // 相对父对象的缩放
transform.lossyScale        // 世界坐标缩放（只读）
transform.eulerAngles       // 欧拉角旋转（Vector3）
transform.localEulerAngles  // 局部欧拉角旋转
transform.forward           // 对象的前方方向向量
transform.right             // 对象的右方方向向量
transform.up                // 对象的上方方向向量
transform.childCount        // 子对象数量
transform.parent            // 父对象的 transform（无父对象则为 null）
transform.root              // 根对象的 transform

  常用方法：
```csharp
```
transform.Translate(Vector3);      // 移动
transform.Rotate(Vector3);         // 旋转
transform.localScale *= 2f;         // 缩放
transform.LookAt(Transform);       // 朝向目标
transform.DetachChildren();        // 断开所有子对象的父子关系
transform.Find("子对象名");        // 按名字查找子对象（递归）
transform.GetChild(0);             // 按索引获取子对象
transform.SetParent(Transform);    // 设置父对象

  示例：
```csharp
```
print(transform.position);              // 打印世界坐标
transform.position = new Vector3(0, 1, 0); // 移动到指定位置
transform.localScale = Vector3.one * 2;    // 放大两倍


### 3. enabled
  控制本脚本是否激活。
  注意：这个属性只控制脚本组件本身，GameObject 的激活状态不受影响。

  语法：
```csharp
```
enabled                      // 省略 this，推荐写法
this.enabled                 // 显式写法，效果相同

  取值：
```csharp
```
enabled = true;   // 脚本激活，Update / FixedUpdate / LateUpdate 等正常执行
enabled = false;  // 脚本失活，所有生命周期函数暂停执行

  示例：
```csharp
```
enabled = false;  // 暂停这个脚本的逻辑
enabled = true;   // 恢复这个脚本的逻辑


### 4. 访问其他脚本的成员
  通过获取其他脚本的引用，来访问那个脚本的成员（字段、方法、属性）。

  步骤：
```csharp
```
1. 在 Inspector 中拖拽目标对象，暴露一个脚本类型的公共字段
2. 在代码中通过这个引用去访问对方的成员

  示例（访问其他脚本的公共字段）：
```csharp
```
// 声明一个其他脚本类型的公共字段
public Lesson3 otherScript;

```csharp
```
// 在 Inspector 中拖入目标脚本实例后，可以：
print(otherScript.gameObject.name);        // 访问对方对象的名称
print(otherScript.transform.position);     // 访问对方对象的位置
otherScript.enabled = false;               // 禁用对方的脚本

  原理：
```csharp
```
把 "this" 换成 "其他脚本的引用变量" 即可，
本质上就是通过 gameObject / transform / enabled 这些成员去操作目标脚本所在的 GameObject。


# 三、原理
MonoBehaviour 的 gameObject、transform、enabled 不是普通字段，
而是「快捷属性」，它们内部调用了底层 API：

  gameObject  → 实际上调用的是 Component.get_gameObject()
  transform   → 实际上调用的是 Component.get_transform()
  enabled     → 实际上控制的是 MonoBehaviour 的激活状态

这三个成员在组件被创建时就和具体的 GameObject 实例绑定了，
所以可以省略 this，直接写 gameObject / transform / enabled。

enabled 控制的是组件本身的激活状态，和 GameObject.activeSelf 是两个概念：
  enabled = false    → 脚本不执行，但对象仍然"激活"
  gameObject.SetActive(false) → 对象本身失活，连带所有组件都停止执行


# 四、坑点
1. enabled 和 SetActive 混淆
   → enabled = false：脚本暂停执行，但对象本身还是激活状态
   → gameObject.SetActive(false)：整个对象失活，所有脚本都暂停
   → 如果只是想让脚本停止逻辑，用 enabled；想让对象消失，用 SetActive

2. 访问其他脚本的引用为 null
   → 如果 Inspector 中没有拖入目标脚本实例，引用就是 null
   → 访问 null 的任何成员会报 NullReferenceException
   → 建议配合 TryGetComponent 或判断 null 后再使用

3. transform.position 和 localPosition 的区别
   → position：相对于世界原点（0,0,0）的坐标
   → localPosition：相对于父对象的位置
   → 有父对象时两者数值不同，没有父对象时两者相等


# 五、应用场景
- 控制角色移动、旋转：transform.Translate()、transform.Rotate()
- 暂停/恢复技能脚本：enabled = false / true
- 查找并操作子对象：transform.Find()、transform.GetChild()
- 获取对象的标签/名称做判断：gameObject.CompareTag()、gameObject.name
- 跨脚本通信：一个脚本读取另一个脚本的公共变量


# 六、扩展
- gameObject 还有以下常用方法：
  GetComponent<T>()           → 获取同对象上的组件
  AddComponent<T>()           → 动态添加组件
  Instantiate()               → 实例化对象（静态方法）
  Destroy()                  → 销毁对象（静态方法）

- transform.Find() 是递归查找，会找所有子子孙孙
  如果子对象名不唯一，只会返回第一个匹配的结果

- 可通过 transform.root 获取场景根对象：
  if (transform.root != null) { ... }

- 如果需要监听对象激活状态变化，可以用 OnEnable / OnDisable 生命周期函数配合 enabled 逻辑
