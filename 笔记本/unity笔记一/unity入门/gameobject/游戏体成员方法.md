# GameObject 成员方法

## 概念

GameObject 的成员方法用于在运行时动态创建空物体、添加脚本、控制激活状态，以及通过消息系统向其他对象发指令。

## 用法

### 1. 创建空物体

```csharp
// 重载1：只指定名字
GameObject obj2 = new GameObject("lyf老大");

// 重载2：指定名字 + 一个或多个脚本类型
GameObject obj2 = new GameObject("lyf老大", typeof(lesson4));

// 重载3：指定名字 + 多个脚本类型（变长参数）
GameObject obj2 = new GameObject("lyf老大", typeof(lesson4), typeof(lesson5), typeof(lesson6));
```

`typeof` 使用反射传类型，`new GameObject` 第二个参数是 `params Type[]`，所以可以传任意数量。

### 2. 为对象添加脚本

继承自 `MonoBehaviour` 的脚本**不能直接 new**，必须用 `AddComponent` 动态添加。

```csharp
// 重载1：反射传类型（返回 Component）
lesson3 les3 = obj2.AddComponent(typeof(lesson3)) as lesson3;

// 重载2：泛型传类型（返回具体类型，不需要 as）
lesson2 lesson2Obj = obj2.AddComponent<lesson2>();
```

泛型版本更好用：不用 `as` 强转，IDE 自动推断类型。

### 3. 标签比较（CompareTag）

```csharp
if (obj2.gameObject.CompareTag("Player"))
{
    // 是 Player 标签的对象
}
```

等价于 `obj2.tag == "Player"`，但 `CompareTag` 更快（内部缓存了 Tag 的哈希值）。

### 4. 设置激活 / 失活

```csharp
obj2.SetActive(true);   // 激活
obj2.SetActive(false);  // 失活
```

### 5. 消息广播系统（次要成员方法）

这些方法效率低，慎用。

**SendMessage**：只发给**自己**

```csharp
this.gameObject.SendMessage("TestFun");
this.gameObject.SendMessage("TestFun2", 100); // 带参数
```

**BroadcastMessage**：发给**自己 + 所有子对象**

```csharp
this.BroadcastMessage("TestFun");
```

**SendMessageUpwards**：发给**自己 + 所有父对象**

```csharp
this.SendMessageUpwards("TestFun");
```

> ⚠️ 这三个方法都通过字符串反射查找方法名，效率极低。Unity 2017.3+ 建议用接口（Interface）或事件系统代替。

## 坑点

- `new GameObject("名字")` 创建的是**空物体**，没有 Mesh、没有 Collider，只有 Transform。
- `AddComponent` 不能添加同一个类型的脚本两次（Unity 会报错）。
- `SendMessage` / `BroadcastMessage` / `SendMessageUpwards` 会触发所有同名方法，包括 private 方法，且无法传递自定义结构体参数（只能传递内置类型）。
- 失活的对象上，`SetActive` 调用后，`SetActive(false)` 不会自动触发 OnDisable，下一帧才会。

## 应用场景

| 需求 | 推荐方法 |
|------|----------|
| 动态生成空物体做容器 | `new GameObject("名字")` |
| 敌人死亡后添加特效脚本 | `enemy.AddComponent<ExplosionEffect>()` |
| 碰撞时判断是不是玩家 | `other.CompareTag("Player")` |
| 暂停时隐藏所有敌人 | `enemyList.ForEach(e => e.SetActive(false))` |
| 通知子对象执行某方法 | `BroadcastMessage("Die")` |

## 源码（部分原理）

```csharp
// AddComponent 内部（简化）
public T AddComponent<T>() where T : Component
{
    return (T)AddComponent(typeof(T));
}

public Component AddComponent(Type type)
{
    // 调用 C++ 层 AddComponent
    // 本质是 Assembly.Load 找到脚本类
    // 然后 GameObject_AddComponent(go, type)
}
```

`AddComponent` 内部使用了反射（`Assembly.Load`），所以第一次添加某类型脚本时有额外开销，后续调用几乎没有。

## 扩展

- `new GameObject()` 第二个参数 `params Type[]` 意味着可以传数组：`new GameObject("Name", new Type[]{ typeof(A), typeof(B) })`。
- 如果需要添加带构造函数的脚本，可以先 `AddComponent`，再通过返回值访问其 public 字段初始化。
- 消息广播系统的替代方案：
  - 接口：`public interface IDamageable { void TakeDamage(float amount); }`
  - 事件：`public static event Action OnEnemyDeath;`
