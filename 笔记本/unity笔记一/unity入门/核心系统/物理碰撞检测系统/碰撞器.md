# Collider 碰撞器组件

## 一、概念

Collider（碰撞器）是 Unity 物理系统的"形状"组件。它定义了物体的物理边界形状，物理引擎根据碰撞器的几何形状判断两个物体是否相交，从而触发碰撞或触发器回调。

Collider 有两大类：
- **普通碰撞器**：参与物理碰撞（反弹、摩擦）
- **触发器碰撞器**：只做重叠检测，不产生物理效果（勾选 `isTrigger`）

> Collider 和 Rigidbody 的关系：Collision = 两个 Collider + 其中一个带 Rigidbody；Trigger = 至少一个 Collider 勾选 isTrigger + 至少一个带 Rigidbody。

---

## 二、3D 碰撞器种类详解

### 1. Box Collider（盒碰撞器）

矩形盒子形状，最常用，性能好。

```csharp
BoxCollider box = GetComponent<BoxCollider>();
box.size = new Vector3(1f, 1f, 1f);           // 尺寸（长宽高）
box.center = Vector3.zero;                      // 中心偏移
box.isTrigger = false;                          // 是否为触发器
box.material = physicsMaterial;                 // 物理材质
box.sharedMaterial = physicsMaterial;           // 与其他对象共享材质
```

**适用场景**：箱子、建筑、门、平台等方正物体。

---

### 2. Sphere Collider（球碰撞器）

球形，性能最优的碰撞器。

```csharp
SphereCollider sphere = GetComponent<SphereCollider>();
sphere.radius = 0.5f;     // 球半径
sphere.center = Vector3.zero;
```

**适用场景**：球形物体、圆形敌人、检测半径区域。

---

### 3. Capsule Collider（胶囊碰撞器）

由半球+圆柱+半球组成，可完美包裹站立的人形角色。

```csharp
CapsuleCollider capsule = GetComponent<CapsuleCollider>();
capsule.height = 2f;       // 总高度（含两端半球）
capsule.radius = 0.5f;     // 半径
capsule.center = new Vector3(0f, 1f, 0f); // 胶囊中心（通常在角色腰部）
```

**适用场景**：角色控制器（CharacterController 内置胶囊碰撞器）、柱子、管道。

---

### 4. Wheel Collider（轮子碰撞器）

专为车轮设计的碰撞器，内置悬挂系统模拟。

```csharp
WheelCollider wheel = GetComponent<WheelCollider>();
wheel.radius = 0.5f;           // 轮子半径
wheel.motorTorque = 0f;        // 电机扭矩
wheel.brakeTorque = 0f;        // 刹车扭矩
wheel.steerAngle = 0f;         // 转向角（度）
wheel.suspensionDistance = 0.3f; // 悬挂距离
```

> ⚠️ Wheel Collider 只能检测与地面的碰撞，不能与其他刚体碰撞器碰撞。常用于赛车游戏的轮胎模拟。

---

### 5. Terrain Collider（地形碰撞器）

配合 `Terrain` 地形组件使用，根据地形高度图生成碰撞形状。

```csharp
TerrainCollider terrain = GetComponent<TerrainCollider>();
terrain.terrainData = terrainData; // 关联地形数据
terrain.material = physicsMaterial;
```

---

### 6. Mesh Collider（网格碰撞器）

根据 3D 模型网格生成精确碰撞形状。

```csharp
MeshCollider mesh = GetComponent<MeshCollider>();
mesh.sharedMesh = myMesh;      // 指定网格模型
mesh.convex = false;           // 是否凸化（与刚体配合时必须为 true）
mesh.cookingOptions = CookingOptions.CookForFasterSimulation; // 烘焙优化选项
```

| 参数 | 说明 |
|---|---|
| `convex = false` | 精确但只能用于静态物体（无刚体） |
| `convex = true` | 凸化后可用于动态刚体（精度降低，但性能大幅提升） |

> ⚠️ 非凸（convex=false）的 Mesh Collider 只能挂载在**静态物体**（无 Rigidbody 或 Rigidbody.isKinematic=true）上。

---

## 三、所有碰撞器共同参数

### 1. isTrigger（是否为触发器）

```csharp
col.isTrigger = true;  // 只检测，不产生物理碰撞
col.isTrigger = false; // 正常物理碰撞
```

> 触发器仍然会与刚体碰撞器产生重叠检测，但物体之间不会有反弹或摩擦效果。

---

### 2. Material / SharedMaterial（物理材质）

```csharp
col.material = bouncyMaterial;     // 运行时修改（不影响其他对象）
col.sharedMaterial = bouncyMaterial; // 共享材质（所有使用此材质的碰撞器都变）
```

> `sharedMaterial` 是直接引用碰撞器共享的 PhysicMaterial，改动会影响所有引用此材质的碰撞器；`material` 是实例化一份副本，只影响当前对象。

---

### 3. Center（碰撞器中心偏移）

```csharp
col.center = new Vector3(0f, 0.5f, 0f); // 本地坐标系偏移
```

---

### 4. Bounds（碰撞器包围盒）

```csharp
Bounds bounds = col.bounds;
Vector3 min = bounds.min;
Vector3 max = bounds.max;
```

- `bounds` 是碰撞器在**世界坐标**下的轴对齐包围盒（AABB），用于快速判断碰撞候选对象
- 调试用：`Debug.DrawLine` 可视化 bounds 边界

---

## 四、组合使用：复合碰撞器

一个对象可以有**多个碰撞器**，组成更复杂的碰撞形状。

**典型例子：坦克**

```
Tank（根对象，有 Rigidbody）
  ├── Body (BoxCollider)       ← 身体
  ├── Turret (BoxCollider)     ← 炮塔
  ├── LeftTrack (BoxCollider)  ← 左履带
  └── RightTrack (BoxCollider) ← 右履带
```

```csharp
// 在根对象加刚体，子对象加碰撞器
rb = gameObject.AddComponent<Rigidbody>();
body.AddComponent<BoxCollider>();
turret.AddComponent<BoxCollider>();
```

> ⚠️ 刚体只需一个，加在根对象上。所有子对象的碰撞器都会自动参与物理运算。

---

## 五、Cooking Options（Mesh Collider 专属）

Mesh Collider 在生成碰撞网格前有一个"烘焙"（Cooking）步骤：

```csharp
mesh.cookingOptions = CookingOptions.CookForFasterSimulation      // 烘焙优化，性能更好
                    | CookingOptions.CollisoinIsConvex;          // 要求必须是凸网格
```

| 选项 | 效果 |
|---|---|
| `CookForFasterSimulation` | 烘焙优化，提升运行时性能 |
| `CookForFakeConvex` | 把非凸网格当作凸处理（用于 kinematic 物体） |
| `None` | 不使用烘焙选项，网格精确但慢 |

> 推荐始终加上 `CookForFasterSimulation`，可以减少内存占用和运行时间。

## 六、坑点

1. **碰撞器尺寸与视觉模型不匹配**是最常见的 Bug：模型缩小了但碰撞器还是原来大小，导致看起来没碰到但物理碰撞已触发。检查方法：编辑器中勾选 `Gizmos → Collider` 显示碰撞器线框。
2. **Mesh Collider `convex = false` 不能与刚体碰撞**：给动态刚体物体使用网格碰撞器必须勾选 `convex`（凸化），但凸化会简化形状（无法有内凹）。
3. **两个 Mesh Collider（非凸）不能互相碰撞**：两个非凸网格碰撞器之间永远检测不到碰撞，只能有一个是凸的。
4. **碰撞器碰撞面之间太薄（<0.01）会导致 Z-Fighting**：两个碰撞体重叠或相切时出现闪烁。
5. **Wheel Collider 只能与 Terrain/Plane 碰撞**：它不会与普通刚体碰撞器产生碰撞响应，常导致赛车游戏轮子穿过车身的 Bug。
6. **碰撞器缩放**：通过 `transform.localScale` 改变父对象大小时，碰撞器会自动缩放，无需手动修改 size。

## 七、应用场景

**获取碰撞器的碰撞点做特效**

```csharp
private void OnCollisionEnter(Collision collision)
{
    ContactPoint contact = collision.contacts[0];
    Quaternion rot = Quaternion.FromToRotation(Vector3.up, contact.normal);
    Vector3 pos = contact.point;
    Instantiate(hitEffect, pos, rot);
}
```

**射线检测（不用碰撞回调，直接检测碰撞）**

```csharp
void Update()
{
    if (Physics.Raycast(transform.position, transform.forward, out RaycastHit hit, 10f))
    {
        Debug.Log($"击中了：{hit.collider.name}");
    }
}
```

## 八、原理

物理引擎对碰撞器的存储和检测使用 **Bounding Volume（包围体）**：

- **Box/Sphere/Capsule**：直接用解析几何公式计算相交，性能最快
- **Mesh/Convex Hull**：先用 AABB 粗筛，再用 SAT（分离轴定理）精确检测，性能较慢

碰撞检测发生在 `Physics.Simulate` 中（FixedUpdate 期间），与渲染帧独立运行，这就是为什么物理效果不受帧率影响的原因。

---

## 九、扩展

- **`Physics.CheckSphere / CheckBox / CheckCapsule`**：不使用碰撞器，直接在空间指定位置做形状检测，返回 bool
- **`Physics.OverlapSphere / OverlapBox / OverlapCapsule`**：返回指定形状内的所有碰撞器数组（用于区域检测，无需刚体）
- **`Collider.AttachPOV`**：将碰撞器临时附加到其他刚体上（高级用法）
- **2D 对应碰撞器**：`BoxCollider2D`、`CircleCollider2D`、`CapsuleCollider2D`、`PolygonCollider2D`（精确 2D 碰撞）
