# Rigidbody 刚体组件

## 一、概念

`Rigidbody`（刚体）是 Unity 物理系统的核心组件。挂载了 Rigidbody 的 GameObject 会受到重力、碰撞力、反作用力等物理效果影响，让物体在物理引擎的控制下运动。碰撞检测的必要条件：**两个物体都有 Collider，其中一个必须挂载 Rigidbody**。

> 没有 Rigidbody 的物体称为"静态碰撞器"，引擎不会对其做物理运算。

---

## 二、Inspector 参数详解

### 1. Mass（质量）

```csharp
rb.mass = 10f;
```

- 物体的质量（单位：kg）
- 值越大，物体越难被推动，也越难加速（惯性大）
- 两个物体的质量比决定碰撞后的速度分配比
- ⚠️ Mass 不要设太大（>1000）也不要太小（<0.01），可能导致物理数值不稳定

---

### 2. Drag（线性阻力）

```csharp
rb.drag = 0.5f;
```

- 物体运动时受到的**空气阻力**，值越大减速越快
- 设为 0 表示无阻力（匀速或恒定重力）
- 设为 Infinity（`Mathf.Infinity`）会使物体**立即停止**运动

---

### 3. Angular Drag（角阻力）

```csharp
rb.angularDrag = 0.5f;
```

- 物体**旋转**时受到的阻力
- 控制旋转衰减速度，使旋转逐渐停下来
- 同理，设为 Infinity 会使旋转立即停止

---

### 4. Use Gravity（使用重力）

```csharp
rb.useGravity = true;  // 受重力影响（默认）
rb.useGravity = false; // 不受重力影响（悬浮）
```

- 关闭后物体不再下落，常用于太空场景的飞船、或手动控制高度的对象

---

### 5. Is Kinematic（是否运动学）

```csharp
rb.isKinematic = true;  // 不受物理引擎控制
rb.isKinematic = false; // 受物理引擎控制（默认）
```

- 设为 true：物体完全由 `transform.position/rotation` 手动控制，**不受任何力或碰撞影响**（但仍然参与碰撞检测）
- 设为 false：恢复正常的物理模拟
- 典型用法：动画控制的角色（动画驱动 movement，但需要触发器检测）

---

### 6. Interpolate（插值模式）

解决物理帧（FixedUpdate）和渲染帧（Update）不同步导致的抖动：

```csharp
rb.interpolation = RigidbodyInterpolation.None;    // 无插值（默认）
rb.interpolation = RigidbodyInterpolation.Interpolate; // 渲染帧在两个物理帧之间插值（平滑）
rb.interpolation = RigidbodyInterpolation.Extrapolate; // 预测下一帧位置（更平滑但可能抖动）
```

| 模式 | 说明 | 使用场景 |
|---|---|---|
| **None** | 无插值，画面可能抖动 | 静止物体、不关心的对象 |
| **Interpolate** | 在物理帧之间插值 | 摄像机跟随的物体（推荐） |
| **Extrapolate** | 预测位置 | 高速移动物体 |

---

### 7. Collision Detection（碰撞检测模式）

```csharp
rb.collisionDetectionMode = CollisionDetectionMode.Discrete;         // 离散检测（默认）
rb.collisionDetectionMode = CollisionDetectionMode.Continuous;        // 连续检测
rb.collisionDetectionMode = CollisionDetectionMode.ContinuousDynamic; // 动态连续检测
rb.collisionDetectionMode = CollisionDetectionMode.ContinuousSpeculative; // 推测式连续
```

| 模式 | 性能 | 效果 | 使用场景 |
|---|---|---|---|
| **Discrete** | 最快 | 低速碰撞可用，高速可能穿透 | 默认，速度<10m/s |
| **Continuous** | 较慢 | 保护此刚体不被高速穿透（对方是离散） | 高速子弹检测场景 |
| **ContinuousDynamic** | 最慢 | 双向保护（自己和对方都用连续） | 超高速碰撞 |
| **ContinuousSpeculative** | 中等 | Unity推测式连续，性能更优 | 替代Continuous的选项 |

> Collision Detection 越精细，物理运算越慢。移动端建议用 Discrete。

---

### 8. Constraints（约束）

限制刚体的位置/旋转自由度：

```csharp
// 冻结所有轴向的位置和旋转
rb.constraints = RigidbodyConstraints.FreezePositionX
               | RigidbodyConstraints.FreezePositionY
               | RigidbodyConstraints.FreezePositionZ
               | RigidbodyConstraints.FreezeRotationX
               | RigidbodyConstraints.FreezeRotationY
               | RigidbodyConstraints.FreezeRotationZ;

// 只冻结 Y 轴位置（地面滑行）
rb.constraints = RigidbodyConstraints.FreezePositionY;

// 冻结 X 和 Z 轴旋转（只允许绕 Y 轴转）
rb.constraints = RigidbodyConstraints.FreezeRotationX | RigidbodyConstraints.FreezeRotationZ;
```

枚举值一览：

| 枚举值 | 冻结内容 |
|---|---|
| `FreezePositionX/Y/Z` | 锁定对应轴向的位移 |
| `FreezeRotationX/Y/Z` | 锁定对应轴向的旋转 |

## 三、重要成员详解

### 1. 获取速度

```csharp
Vector3 vel = rb.velocity;       // 当前线速度（m/s）
Vector3 angVel = rb.angularVelocity; // 当前角速度（rad/s）
```

### 2. 设置速度

```csharp
rb.velocity = Vector3.forward * 5f; // 直接设置线速度
rb.angularVelocity = Vector3.up * 2f; // 直接设置角速度
```

> ⚠️ 直接设置 velocity 会**覆盖**物理引擎的计算，导致后续 AddForce 不生效（因为速度被每帧覆盖）。正确用法见 `刚体加力.md`。

### 3. 位置和旋转

```csharp
Vector3 pos = rb.position;       // 当前位置
Quaternion rot = rb.rotation;    // 当前旋转
rb.position = new Vector3(0, 5, 0); // 移动到指定位置（会触发物理）
rb.rotation = Quaternion.Euler(0, 90, 0); // 旋转
```

### 4. 质量中心

```csharp
rb.centerOfMass = new Vector3(0, -0.5f, 0); // 设置质心位置，影响旋转物理
rb.ResetInertiaTensor(); // 重置惯性张量（修改 centerOfMass 后调用）
```

### 5. 阻力控制

```csharp
rb.drag = 0f;         // 无线性阻力
rb.angularDrag = 0f; // 无角阻力
```

### 6. 休眠控制

```csharp
if (rb.IsSleeping())
{
    rb.WakeUp(); // 唤醒刚体
}
```

> 刚体休眠是 Unity 物理引擎的优化机制：当刚体速度足够小（接近静止），引擎会让它"休眠"以节省计算量。休眠后不会触发碰撞回调中的物理效果。手动 `WakeUp()` 可以强制唤醒。

## 四、坑点

1. **Is Kinematic = true 时不能 AddForce**：运动学刚体完全由 transform 控制，不受力影响。
2. **Collision Detection 设为 Discrete 时高速物体会穿透**：子弹速度太快（>10m/s）会穿过薄墙，解决方法：改 ContinuousDynamic 或在子弹路径上加 Raycast 检测。
3. **父子关系会放大 Rigidbody 的质量计算**：当子对象有刚体但父对象没有时，物理计算可能不稳定。最佳实践：刚体挂在根对象，子对象只放 Collider。
4. **Constraints 只冻结逻辑自由度，视觉上仍可能抖动**：配合 `Interpolate` 使用效果更好。
5. **Kinematic 物体仍会触发碰撞回调**：即使不受物理影响，运动学刚体参与碰撞检测，可以用来做触发区域。
6. **Mass 与 Drag 配合不当会让物体几乎不动**：Mass=100 + Drag=0 会让物体下落极慢（加速度=重力/mass，f=ma）。

## 五、应用场景

**FPS 角色控制器（刚体版）**

```csharp
public class RigidbodyFPS : MonoBehaviour
{
    private Rigidbody rb;
    public float moveSpeed = 5f;
    public float jumpForce = 8f;

    void Start()
    {
        rb = GetComponent<Rigidbody>();
    }

    void FixedUpdate()
    {
        float h = Input.GetAxis("Horizontal");
        float v = Input.GetAxis("Vertical");
        Vector3 dir = transform.forward * v + transform.right * h;
        rb.velocity = new Vector3(dir.x * moveSpeed, rb.velocity.y, dir.z * moveSpeed);

        if (Input.GetKeyDown(KeyCode.Space))
        {
            rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
        }
    }
}
```

**物体落地后停止**

```csharp
void OnCollisionEnter(Collision collision)
{
    rb.drag = 10f;  // 落地后加大阻力，使其立即停止
    rb.angularDrag = 10f;
}
```

## 六、原理

Unity 物理引擎（PhysX）在 `FixedUpdate` 中以固定时间步长（默认 0.02s，即 50Hz）更新物理状态：

1. **Force Accumulation**：收集所有 `AddForce` 指令，累加力
2. **Integration**：用欧拉积分计算加速度、速度、位移
3. **Collision Detection**：用分离轴定理（SAT）检测碰撞，生成 Contact Point
4. **Constraint Solver**：解算约束（碰撞响应、关节、固定旋转等）
5. **Sleep Check**：速度足够小的刚体进入休眠，节省后续帧计算

> `Update` 中设置 `transform.position` 会与 `FixedUpdate` 的物理运算冲突，导致不可预测的物理行为。

---

## 七、扩展

- **RigidbodyConstraints**：枚举，灵活组合 freeze 值，如 `FreezePositionY | FreezeRotationX | FreezeRotationZ`（卡住 Y 轴位置，不允许 X/Z 轴旋转）
- **Center of Mass 与 Inertia Tensor**：质心偏移会导致旋转轴改变，可通过 `rb.inertiaTensor` 和 `rb.inertiaTensorRotation` 调整
- **Rigidbody.solverIterations**：物理求解器迭代次数，越多碰撞越精确但越慢（默认 6）
- **Physic Material + Rigidbody**：物理材质决定碰撞时的摩擦和弹性，配合刚体一起影响碰撞响应
