# 碰撞与触发检测函数

## 一、概念

碰撞检测函数是 MonoBehaviour 的生命周期函数，由 Unity 物理引擎在检测到碰撞或触发重叠时自动调用。它们是游戏交互逻辑的核心：子弹击中敌人、角色踩到地板、拾取物品、进入区域等场景都依赖这些函数。

**两类检测的区别：**

| 类型 | 条件 | 物理反馈 | 参数 |
|---|---|---|---|
| **碰撞（Collision）** | 两个碰撞器都**不勾选** `isTrigger` | 有真实物理碰撞（反弹/摩擦） | `Collision collision` |
| **触发器（Trigger）** | 至少一个碰撞器**勾选** `isTrigger` | 无物理反弹，纯逻辑检测 | `Collider other` |

---

## 二、物理碰撞检测函数

### 1. OnCollisionEnter —— 碰撞开始（仅触发一次）

```csharp
private void OnCollisionEnter(Collision collision)
{
    Debug.Log("碰撞了！碰撞对象：" + collision.gameObject.name);
}
```

- **触发时机**：两个碰撞器开始接触的**那一帧**
- **是否重复**：只触发一次，后续持续接触不再触发
- 典型用法：子弹击中、落地检测、受伤判定

---

### 2. OnCollisionStay —— 持续碰撞（每帧触发）

```csharp
private void OnCollisionStay(Collision collision)
{
    // 每帧都在碰撞中时调用
    Debug.Log("还在碰撞中...");
}
```

- **触发时机**：从碰撞开始到碰撞结束之间的**每一帧**
- 典型用法：持续伤害区域（岩浆）、摩擦力处理、地板检测

---

### 3. OnCollisionExit —— 碰撞结束（仅触发一次）

```csharp
private void OnCollisionExit(Collision collision)
{
    Debug.Log("离开了！");
}
```

- **触发时机**：两个碰撞器分离的**那一帧**
- 典型用法：离开地面检测、离开伤害区域

---

## 三、触发器检测函数

> ⚠️ 触发器函数的前提：至少一个碰撞器勾选了 `isTrigger` 选项。

### 1. OnTriggerEnter —— 进入触发器

```csharp
private void OnTriggerEnter(Collider other)
{
    Debug.Log("进入了触发器区域：" + other.name);
}
```

- 典型用法：拾取金币、进入传送门、触发剧情

---

### 2. OnTriggerStay —— 停留在触发器内

```csharp
private void OnTriggerStay(Collider other)
{
    // 每帧都在触发器重叠区域时调用
}
```

- 典型用法：持续扣血区域、吸怪磁场

---

### 3. OnTriggerExit —— 离开触发器

```csharp
private void OnTriggerExit(Collider other)
{
    Debug.Log("离开了触发区域：" + other.name);
}
```

- 典型用法：离开安全区、退出拾取范围

---

## 四、Collision 参数详解

`Collision collision` 包含丰富的碰撞信息：

```csharp
private void OnCollisionEnter(Collision collision)
{
    // 1. 碰撞对象的 GameObject
    GameObject obj = collision.gameObject;

    // 2. 碰撞对象的碰撞器
    Collider col = collision.collider;

    // 3. 碰撞对象的 Transform
    Transform t = collision.transform;

    // 4. 接触点数组（通常取第一个）
    ContactPoint[] contacts = collision.contacts;
    if (contacts.Length > 0)
    {
        Vector3 point = contacts[0].point;        // 接触点世界坐标
        Vector3 normal = contacts[0].normal;      // 接触面法线
    }

    // 5. 碰撞相对速度
    Vector3 relativeVelocity = collision.relativeVelocity;

    // 6. 碰撞器是否是触发器
    bool isTrigger = collision.collider.isTrigger;

    // 7. 刚体信息（可能为空）
    Rigidbody rb = collision.rigidbody;
}
```

## 五、触发器 Collider 参数详解

`Collider other`（没有 Collision 那么多信息）：

```csharp
private void OnTriggerEnter(Collider other)
{
    // 获取碰撞对象的 GameObject
    GameObject obj = other.gameObject;

    // 获取碰撞对象的碰撞器
    Collider col = other;

    // 获取碰撞对象的刚体（可能为空）
    Rigidbody rb = other.attachedRigidbody;

    // 获取对象所在层级
    int layer = other.gameObject.layer;

    // 获取对象的 Tag
    if (other.CompareTag("Enemy"))
    {
        Debug.Log("发现敌人！");
    }

    // 通过 Tag 或 Layer 判断对象身份
    if (other.CompareTag("Coin"))
    {
        Destroy(other.gameObject); // 拾取
    }
}
```

## 六、常见应用场景

**子弹击中敌人扣血**

```csharp
public class EnemyHealth : MonoBehaviour
{
    public int maxHP = 100;

    private int currentHP;

    void Start()
    {
        currentHP = maxHP;
    }

    // 被物理碰撞击中
    private void OnCollisionEnter(Collision collision)
    {
        if (collision.gameObject.CompareTag("Bullet"))
        {
            TakeDamage(20);
            Destroy(collision.gameObject); // 销毁子弹
        }
    }

    void TakeDamage(int damage)
    {
        currentHP -= damage;
        if (currentHP <= 0)
        {
            Die();
        }
    }

    void Die()
    {
        Destroy(gameObject);
    }
}
```

**拾取金币**

```csharp
private void OnTriggerEnter(Collider other)
{
    if (other.CompareTag("Coin"))
    {
        coinCount++;
        Destroy(other.gameObject);
        Debug.Log($"金币数量：{coinCount}");
    }
}
```

**地面检测（持续）**

```csharp
private void OnCollisionStay(Collision collision)
{
    if (collision.gameObject.CompareTag("Ground"))
    {
        isGrounded = true;
    }
}

private void OnCollisionExit(Collision collision)
{
    if (collision.gameObject.CompareTag("Ground"))
    {
        isGrounded = false;
    }
}
```

## 七、坑点

1. **父子层级陷阱**：刚体挂在父对象，碰撞脚本挂在子对象时，碰撞函数**不会响应**！因为物理运算以刚体所在的对象为根。正确做法：刚体和碰撞脚本都在同一个对象上，或把刚体移到根对象。
2. **OnTriggerEnter 不受 Rigidbody 质量影响**：触发器只做碰撞检测，不触发物理反弹，所以质量再大的物体进入触发器也不会推开触发器。
3. **OnCollisionEnter 在 FixedUpdate 中触发**：物理回调在物理帧触发，而不是渲染帧。协程里等待这些函数触发要小心时序。
4. **isTrigger 和刚体缺一不可的陷阱**：
   - 纯碰撞（OnCollision）：两个物体都要有 Collider，其中一个必须有 Rigidbody
   - 触发器（OnTrigger）：至少一个是 Rigidbody + 勾选 isTrigger
5. **OnCollisionStay 在静止时可能每帧都触发**：即使物体看起来没动，只要接触面积>0，就会持续触发。
6. **Sleep 状态下的刚体不触发碰撞回调**：刚体休眠后（速度接近0且长期不动）不会触发 OnCollision/OnTrigger。调用 `WakeUp()` 强制唤醒即可。
7. **碰撞体穿过薄物体的"隧道效应"**：当物体速度太快（>10m/s）且 CollisionDetection 为 Discrete 时，高速物体会直接穿透薄墙。解决：改 `ContinuousDynamic` 或用 Raycast 补充检测。

## 八、原理

物理引擎每帧执行步骤：

1. **Broad Phase（广布阶段）**：粗筛，用 AABB 包围盒快速找出可能相交的对象对
2. **Narrow Phase（窄布阶段）**：精确检测，对候选对象对用 SAT（分离轴定理）判断是否真正相交
3. **Contact Generation（接触点生成）**：计算两个碰撞体表面的接触点位置和法线
4. **回调触发**：如果有接触点，调用 `OnCollisionEnter`；如果没有但有重叠区域（trigger），调用 `OnTriggerEnter`

`OnCollision` 在物理帧的 **Collision Resolution** 阶段触发，`OnTrigger` 在 **Trigger Resolution** 阶段触发，两者属于不同的处理管线。

## 九、扩展

- **`Collision.gameObject`**：通过它可以获取对象的 Tag/Layer/组件，实现"碰撞到敌人就扣血"这类逻辑
- **`OnControllerColliderHit`**：CharacterController 专用碰撞函数，用于第一人称角色控制器的碰撞响应
- **`Collision2D`**：2D 物理对应的碰撞参数，与 3D 版本类似但结构为 `Collision2D`
- **`IPhysicNotify` 接口**：Unity 2022+ 提供，可精确控制碰撞事件的发送时机（更高级的控制）
