# Rigidbody 添加力与力矩

## 一、概念

给刚体添加力是让物体运动最常用的方式。通过 `AddForce` 系列方法，可以让物体在方向上加速；通过 `AddTorque` 系列方法，可以让物体绕轴旋转。理解力与速度的区别（力是加速度的来源，速度是位移的直接驱动力），以及 `ForceMode` 的四种模式，是掌握物理控制的关键。

---

## 二、用法 —— 全部 API 详解

### 1. AddForce —— 添加力

获取刚体后才能使用：

```csharp
Rigidbody rb = GetComponent<Rigidbody>();
```

**重载一：`AddForce(Vector3 force)` —— 基础重载（世界坐标系）**

```csharp
rb.AddForce(Vector3.forward * 10f);
rb.AddForce(new Vector3(5f, 0f, 0f));
```

**重载二：`AddForce(Vector3 force, ForceMode mode)` —— 指定力的模式**

```csharp
rb.AddForce(Vector3.forward * 10f, ForceMode.Force);
```

**重载三：`AddForce(float x, float y, float z)` —— 分量重载（世界坐标系）**

```csharp
rb.AddForce(0f, 10f, 0f); // 等价于 AddForce(Vector3.up * 10f)
```

**重载四：`AddForce(float x, float y, float z, ForceMode mode)` —— 分量+模式重载**

```csharp
rb.AddForce(0f, 10f, 0f, ForceMode.Impulse);
```

---

### 2. AddRelativeForce —— 添加相对力（本地坐标系）

```csharp
rb.AddRelativeForce(Vector3.forward * 10f); // 沿自身的前方推动
rb.AddRelativeForce(Vector3.up * 5f, ForceMode.Impulse); // 沿自身的上方（即使物体旋转了，向上的方向也随之旋转）
```

> `AddForce` 按世界坐标系的轴向施加力；`AddRelativeForce` 按物体自身坐标系的轴向施加力。

---

### 3. AddTorque —— 添加扭矩力（旋转）

```csharp
rb.AddTorque(Vector3.up * 10f); // 绕世界 Y 轴旋转
rb.AddTorque(new Vector3(0f, 10f, 0f), ForceMode.Force);
rb.AddTorque(0f, 10f, 0f); // 分量重载（世界坐标系）
rb.AddTorque(0f, 10f, 0f, ForceMode.Impulse);
```

> 扭矩（Torque）控制的是**角加速度**，让物体绕轴旋转，而不是直接改变朝向。

---

### 4. AddRelativeTorque —— 添加相对扭矩（本地坐标系）

```csharp
rb.AddRelativeTorque(Vector3.up * 10f); // 绕自身 Y 轴旋转
```

---

### 5. 直接改变速度

```csharp
rb.velocity = Vector3.forward * 10f; // 设置瞬时速度
rb.angularVelocity = Vector3.up * 5f; // 设置瞬时角速度
```

> ⚠️ 每帧重置 velocity 会覆盖物理引擎的积分结果，造成不可预期的行为。推荐在 `FixedUpdate` 中只调用一次，或者用 AddForce。

---

### 6. AddExplosionForce —— 模拟爆炸

```csharp
rb.AddExplosionForce(500f,       // 爆炸力大小
                     transform.position, // 爆炸中心点
                     10f,          // 爆炸影响半径
                     3f,           // 向上的偏移（让爆炸可以推离地面）
                     ForceMode.Impulse);

// 带模式的版本
rb.AddExplosionForce(500f, transform.position, 10f, 0f, ForceMode.Impulse);
```

> 爆炸力会计算每个受影响的刚体到爆炸中心的距离，距离越远力越小，模拟真实物理衰减。

---

## 三、ForceMode（力的模式）—— 四个枚举值

| 模式 | 计算公式 | 特点 | 典型用途 |
|---|---|---|---|
| **Force** | `a = F/m`（持续力） | 考虑质量，每帧施加 | 推进器、引擎 |
| **Acceleration** | `a = F`（持续加速度） | 忽略质量，统一加速 | 统一加速度 |
| **Impulse** | `Δv = F/m`（瞬时力） | 考虑质量，瞬间加速 | 炮弹发射、跳跃 |
| **VelocityChange** | `Δv = F`（瞬时速度变化） | 忽略质量，瞬间改速度 | 瞬移传送 |

```csharp
rb.AddForce(Vector3.up * 5f, ForceMode.Force);      // 持续推力（逐帧累加）
rb.AddForce(Vector3.up * 5f, ForceMode.Impulse);     // 瞬时冲力（一次性，速度跳跃）
```

> ⚠️ `ForceMode.Force` 模式下，每帧施加的力会被 `FixedDeltaTime` 自动调整，所以实际效果与帧率无关。但 `Impulse` 模式下，力已经是速度变化量，不受时间步长影响。

---

## 四、Constant Force（常量力组件）

Unity 提供了 `Constant Force` 组件（`Add Component → Physics → Constant Force`），无需脚本就能让刚体持续受力：

| 参数 | 说明 |
|---|---|
| **Force** | 持续线性力（世界坐标系） |
| **Relative Force** | 持续线性力（相对坐标系） |
| **Torque** | 持续扭矩力（世界坐标系） |
| **Relative Torque** | 持续扭矩力（相对坐标系） |

适合火箭上升、持续转向等不需要手动调用 AddForce 的场景。

---

## 五、刚体休眠与唤醒

刚体休眠后不再接受物理更新（节能），但仍可触发**碰撞检测**：

```csharp
// 检测刚体是否在休眠
if (rb.IsSleeping())
{
    rb.WakeUp(); // 强制唤醒
}

// 强制让刚体立即休眠
rb.Sleep();
```

> 休眠通常由物理引擎自动判断（速度足够小时），但可以通过 `SleepThreshold`（休眠速度阈值）调整。

---

## 六、坑点

1. **AddForce 要放在 `FixedUpdate` 里**：物理运算在 FixedUpdate 中执行，AddForce 放在 Update 里会导致每帧施加的力不均匀（帧率波动时效果不稳）。
2. **持续力会不断累加**：`Force` 模式在 FixedUpdate 里每帧施加，如果不设上限，刚体会越跑越快，记得加阻力（Drag）。
3. **质量决定响应速度**：`Force` 模式下 Mass 越大加速度越小（牛顿第二定律），要让重物移动快，需要更大的力。
4. **AddForce 方向与速度方向不一致**：物体在滑动摩擦力作用下，AddForce 推着它运动但速度方向不一定与力方向一致（摩擦力会改变方向）。
5. **爆炸力有最小半径保护**：当爆炸半径内的刚体距爆炸中心过近时（<0.01），爆炸力可能异常大，建议爆炸中心放一个小的碰撞球体作为边界。
6. **`velocity = Vector3.zero` 每帧都设会导致摩擦力无效**：因为速度每帧被强制归零，摩擦力永远无法生效。

## 七、应用场景

**角色跳跃（瞬时冲量）**

```csharp
void FixedUpdate()
{
    if (Input.GetKeyDown(KeyCode.Space) && IsGrounded())
    {
        rb.AddForce(Vector3.up * jumpForce, ForceMode.Impulse);
    }
}
```

**赛车加速（持续推力）**

```csharp
void FixedUpdate()
{
    if (Input.GetKey(KeyCode.W))
    {
        rb.AddForce(transform.forward * acceleration, ForceMode.Force);
    }
    if (Input.GetKey(KeyCode.S))
    {
        rb.AddForce(transform.forward * -acceleration * 0.5f, ForceMode.Force);
    }
}
```

**飞船转向（相对力）**

```csharp
void FixedUpdate()
{
    float h = Input.GetAxis("Horizontal");
    float v = Input.GetAxis("Vertical");
    rb.AddRelativeForce(Vector3.forward * v * thrust, ForceMode.Force);
    rb.AddRelativeTorque(Vector3.up * h * turnSpeed, ForceMode.Force);
}
```

## 八、原理

`AddForce` 不是直接改变 `velocity`，而是将力值加入**力累加器**（force accumulator），在物理引擎的**积分阶段**才计算出加速度并更新速度：

```
每帧：forceAccumulator += AddForce值
积分：acceleration = forceAccumulator / mass
      velocity += acceleration * deltaTime
      position += velocity * deltaTime
      forceAccumulator = zero（清零）
```

这就是为什么 `Force` 模式下的力会随时间累加，而 `Impulse` 模式是一次性速度跳变。

---

## 九、扩展

- **`rb.maxAngularVelocity`**：限制最大角速度，防止高速旋转导致数值不稳定，默认 7（rad/s）
- **`rb.AddForceAtPosition(Vector3 force, Vector3 position)`**：在指定位置施力，产生旋转效果（杠杆原理）
- **`rb.MovePosition/Rotation`**：让刚体移动/旋转到目标位置/旋转，物理引擎会平滑插值，避免直接修改 position 导致的物理撕裂
