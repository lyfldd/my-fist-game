# Physic Material 物理材质

## 一、概念

Physic Material（物理材质，不是材质球 `Material`）是挂在 **Collider** 上的资源，用于定义两个碰撞器接触时的物理交互效果：**摩擦力**（滑动阻力）和**弹性**（弹力/回弹力度）。

> 物理材质决定"怎么碰"，刚体决定"碰不碰"。两者配合才能产生真实的物理交互——比如篮球（弹性高）和铅球（弹性低）撞墙的效果完全不同。

---

## 二、创建方法

1. `Project` 窗口右键 → `Create` → `Physic Material`
2. 给需要物理效果的 Collider 挂上去：

```csharp
// 代码方式挂载
BoxCollider box = GetComponent<BoxCollider>();
box.sharedMaterial = bouncyPhysicsMaterial;
```

或在 Inspector 中：选中 Collider → 在 `Material` 字段拖入物理材质资源。

---

## 三、参数详解

### 1. Dynamic Friction（动态摩擦力）

```csharp
physicMaterial.dynamicFriction = 0.6f; // 范围 0~1
```

- 物体**运动时**受到的摩擦阻力
- 0 = 完全光滑（无摩擦，冰面）；1 = 摩擦力最大（粗糙表面）
- 影响：当物体有速度时，摩擦力使速度逐渐衰减

---

### 2. Static Friction（静态摩擦力）

```csharp
physicMaterial.staticFriction = 0.8f; // 范围 0~1
```

- 物体**静止时**的摩擦力（防止物体意外滑动）
- 0 = 轻微推动就会滑动；1 = 几乎不可能推动
- 当施加的力超过静态摩擦力上限时，物体才会开始滑动

> ⚠️ 静态摩擦力通常要**大于等于**动态摩擦力。

---

### 3. Bounciness（弹性/回弹系数）

```csharp
physicMaterial.bounciness = 0.5f; // 范围 0~1
```

| 值 | 效果 |
|---|---|
| 0 | 完全非弹性碰撞（撞墙后停下，不会弹） |
| 0.5 | 部分弹回（篮球） |
| 1 | 完全弹性碰撞（撞墙后以相同速度反向弹回，理想情况） |

- 真实物理中反弹系数可以 >1（如弹弓蓄力），但 Unity 中限制在 0~1

---

### 4. Friction Combine（摩擦力组合模式）

定义两个物体相撞时，如何混合双方的摩擦力：

```csharp
physicMaterial.frictionCombine = PhysicMaterialCombine.Average; // 默认
```

| 组合模式 | 计算公式 | 效果 |
|---|---|---|
| **Average** | (a + b) / 2 | 取双方平均 |
| **Minimum** | min(a, b) | 取最小的（更滑） |
| **Maximum** | max(a, b) | 取最大的（更涩） |
| **Multiply** | a × b | 乘积（非常极端） |

---

### 5. Bounce Combine（弹性组合模式）

定义两个物体相撞时的弹性混合：

```csharp
physicMaterial.bounceCombine = PhysicMaterialCombine.Average;
```

| 组合模式 | 说明 |
|---|---|
| **Average** | 平均值 |
| **Minimum** | 更不弹的 |
| **Maximum** | 更弹的（常用） |
| **Multiply** | 乘积 |

---

## 四、常用材质预设参考

| 材质 | Dynamic Friction | Static Friction | Bounciness | 场景 |
|---|---|---|---|---|
| **冰面** | 0.05 | 0.05 | 0 | 滑冰场 |
| **橡胶** | 0.9 | 1.0 | 0.8 | 球类 |
| **木头** | 0.5 | 0.6 | 0.1 | 地板 |
| **金属** | 0.3 | 0.4 | 0.05 | 机械 |
| **皮肤** | 0.7 | 0.8 | 0 | 角色 |

## 五、应用场景

**角色地面摩擦力切换**

```csharp
// 滑冰时降低摩擦力
private void OnTriggerEnter(Collider other)
{
    if (other.CompareTag("Ice"))
    {
        GetComponent<Collider>().material.dynamicFriction = 0.05f;
        GetComponent<Collider>().material.staticFriction = 0.05f;
    }
}

private void OnTriggerExit(Collider other)
{
    if (other.CompareTag("Ice"))
    {
        GetComponent<Collider>().material.dynamicFriction = 0.6f;
        GetComponent<Collider>().material.staticFriction = 0.8f;
    }
}
```

**篮球游戏（高弹性）**

```csharp
physicMaterial.bounciness = 0.8f;
physicMaterial.bounceCombine = PhysicMaterialCombine.Maximum; // 遇墙也保持弹性
```

**FPS 角色足音（地面材质识别）**

```csharp
private void OnCollisionEnter(Collision collision)
{
    PhysicMaterial mat = collision.collider.sharedMaterial;
    if (mat != null)
    {
        if (mat.dynamicFriction > 0.7f)
            PlayFootstepSound("草地");
        else
            PlayFootstepSound("地板");
    }
}
```

## 六、坑点

1. **`sharedMaterial` vs `material` 的区别**：修改 `sharedMaterial` 会影响所有引用此材质的碰撞器；修改 `material` 是创建一个副本只影响自己。需要动态切换时用 `material`，需要全局改时用 `sharedMaterial`。
2. **摩擦力组合模式的效果不直观**：尤其是 `Multiply`，两个 0.5 相乘得 0.25，远比预期的滑，容易导致物体意外滑动。
3. **Bounciness = 1 的"完全弹性"在真实物理中很少见**：实际上可能因为浮点精度问题导致物体速度不减反增，建议设为 0.9 或 0.95。
4. **Friction Combine = Average 时，摩擦力为零的一方会使整体摩擦为零**：如果地板是冰面（friction=0），角色（friction=1）走过去也会变冰面。
5. **不挂物理材质时的默认值**：Dynamic Friction = 0.4，Static Friction = 0.4，Bounciness = 0，组合模式为 Average。

---

## 七、原理

物理材质定义的参数最终进入 PhysX 物理引擎的 **Contact Constraint Solver（接触约束求解器）**：

- **摩擦力**：在接触点切平面方向施加一个反向力，防止物体相对滑动，大小受 `Friction Combine` 混合后的系数控制
- **弹性**：在接触点法线方向施加一个速度分量，使碰撞后产生反弹，速度变化量 = `-e × v_normal`，e 为 Bounciness

---

## 八、扩展

- **PhysicMaterialCombine 枚举**：`Average`、`Minimum`、`Maximum`、`Multiply`，四个枚举值可以单独控制 `frictionCombine` 和 `bounceCombine`
- **2D 物理材质**：`PhysicsMaterial2D`，参数类似但针对 2D 物理系统（Friction2D、Bounciness2D）
- **关节与物理材质的关系**：关节（Joint）连接的刚体之间的摩擦也会被物理材质影响
