# Lighting Settings 光照面板

## 一、概念

Lighting Settings（光照面板）是 Unity 的**全局光照配置窗口**，通过菜单 `Window → Rendering → Lighting` 打开。这里控制整个场景的环境光照、天空盒、烘焙设置等，影响所有光源和物体的最终光照效果。

---

## 二、光照面板内容详解

### 1. Scene（场景设置）

**Environment（环境）区块**

| 参数 | 说明 |
|---|---|
| **Skybox Material** | 指定天空盒材质，控制场景背景外观 |
| **Sun** | 指定代表太阳的方向光源（通常指向场景中 `Directional Light`） |
| **Environment Reflections** | 环境反射来源，可以是 Skybox（实时反射）或 Custom（自定义 cubemap） |

> 如果没有特殊需求，`Skybox Material` 默认使用 Unity 内置的天空盒（`Procedural` 材质）。

---

**Other Settings（其他设置）区块**

| 参数 | 说明 |
|---|---|
| **Ambient Mode** | 环境光模式：`Skybox`（基于天空盒颜色）或 `Flat`（单色环境光） |
| **Ambient Intensity** | 环境光强度 |
| **Ambient Gradient** | 环境渐变色（编辑器内可视化设置上下天空颜色） |
| **Fog** | 是否开启雾效 |
| **Fog Color** | 雾的颜色 |
| **Fog Mode** | 雾的衰减模式：Linear / Exponential / Exponential Squared |
| **Fog Density** | 指数雾的密度 |
| **Fog End / Start** | 线性雾的起点和终点 |
| **Halp** | 雾效在远距离的混合比例（0=全雾，1=无雾） |

---

### 2. Realtime Lighting（实时光照）

| 参数 | 说明 |
|---|---|
| **Realtime Global Illumination** | 是否开启实时全局光照（需要 ILPP / Progressive GPU Lightmapper 支持） |
| **Realtime Update Mode** | 实时 GI 更新频率：`On Enable`（变化时更新）或 `Via Scripting`（脚本控制） |

> 实时 GI 非常消耗性能，桌面端需要 RTX 系列显卡，移动端通常关闭。

---

### 3. Mixed Lighting（混合光照）

| 参数 | 说明 |
|---|---|
| **Baked Global Illumination** | 是否开启烘焙全局光照 |
| **Lighting Mode** | 混合光照模式（当有 Mixed 光源时生效） |

**Lighting Mode 三种模式：**

| 模式 | 描述 | 适用场景 |
|---|---|---|
| **Baked Indirect** | 只有间接光照被烘焙，直接光实时计算 | 户外场景，光源方向/颜色变化不大 |
| **Subtractive** | 直接光和间接光都烘焙，但接收烘焙阴影 | 简单室内 |
| **Shadowmask** | 远距离用烘焙阴影，近距离用实时阴影 | LOD 阴影优化 |

---

### 4. Lightmapping Settings（光照贴图设置）

| 参数 | 说明 |
|---|---|
| **Lightmap Resolution** | 光照贴图分辨率（像素/单位），值越大越精细但越大 |
| **Lightmap Size** | 单张光照贴图的最大尺寸（2048 / 4096 / 8192） |
| **Compression** | 是否压缩光照贴图 |
| **Ambient Occlusion** | 是否开启环境光遮蔽（贴图裂缝处变暗） |
| **Directional Mode** | `Directional`（主光照方向信息存储在第二张贴图）或 `Non-Directional`（只存颜色） |

---

### 5. Lightmappers（光照贴图烘焙器）

| 选项 | 说明 |
|---|---|
| **Progressive CPU** | CPU 烘焙，速度适中，进度可视化好 |
| **Progressive GPU** | GPU 烘焙（需要支持 DXR 的显卡），速度最快 |
| **Enlighten（已弃用）** | 旧版烘焙器，不再推荐使用 |

---

### 6. Filtering（降噪过滤）

| 参数 | 说明 |
|---|---|
| **Indirect Samples** | 间接光照采样数，越高质量越好越慢 |
| **Indirect Steps** | 间接光照步数（光弹射次数） |
| **Direct Samples** | 直接光照采样数 |

---

## 三、坑点

1. **烘焙光照不会自动更新**：修改光源、物体位置或材质后，必须重新点击 `Lightmaps → Generate Lighting`。
2. **移动端不推荐开启烘焙 GI**：贴图内存占用大，烘焙时间长，运行时无实时更新能力。
3. **Skybox Material 未设置时场景显示纯黑**：确保在 Lighting Settings 里指定了天空盒材质或把 Ambient Mode 设为 Flat + 指定 Ambient Color。
4. **Directional Mode = Directional 会生成两张光照贴图**：内存翻倍，如果不需要主光方向信息（漫反射场景），设为 Non-Directional 节省内存。
5. **Fog 在光照贴图烘焙时会被烘进去**：开启 Fog 后，远处物体在光照贴图里就是带雾的颜色，运行时 Fog 参数变化时不会改变烘焙区域的雾。

---

## 四、常用操作

**烘焙当前场景光照**

1. 打开 `Window → Rendering → Lighting`
2. 确保所有光源 `Mode` 设为 `Baked` 或 `Mixed`
3. 点击 `Generate Lighting`（底部按钮）
4. 等待烘焙完成

**开启实时阴影**

1. 将 `Directional Light` 的 `Shadow Type` 改为 `Soft` 或 `Hard`
2. `Project Settings → Quality → Shadows` 里确认阴影质量级别

**优化烘焙时间**

```
Lightmap Resolution: 2~10（不是越高越好，2D游戏2就够了）
Indirect Samples: 500~1000（不是越高越好）
Progressive GPU（如果有 RTX 显卡）
```

---

## 五、原理

Lighting Settings 是 Unity 渲染管线的全局配置入口。烘焙时，Unity 通过 **Lightmapper** 将场景中所有 `Baked` 和 `Mixed` 光源的光照信息预先计算并写入 **Lightmap**（一张或多张贴图）。运行时，物体不再重新计算间接光照，直接采样这些贴图，从而实现光照效果但零运行时计算成本。

---

## 六、扩展

- **Light Probe Group**：在场景中放置多个光照探头，动态物体从中插值采样环境光颜色，配合烘焙使用
- **Reflection Probe**：烘焙/实时环境反射，常用于金属、水面、玻璃等高反射物体
- **ILPP（Intelligent Light Probe Placement）**：Unity 2021+ 自动优化光照探头的放置，减少人工配置
- **Environment Lighting**：除了天空盒，`Lighting Settings → Environment → Source` 也可以设为 `Gradient`，使用上下两个颜色渐变作为环境光
