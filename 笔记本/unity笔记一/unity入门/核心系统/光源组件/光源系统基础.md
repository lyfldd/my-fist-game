# Light 光源组件

## 一、概念

`Light` 是 Unity 中负责照亮场景的组件。根据光源类型不同，它的光线可以是从点向四周发散、汇聚成锥形、还是均匀照射整个场景。光源是实时光照（实时渲染）和烘焙光照的核心载体。

---

## 二、用法 —— 全部 API 详解

### 1. 获取光源组件

```csharp
Light light = GetComponent<Light>();
light.color = Color.red;
light.intensity = 2f;
```

或在 Inspector 里暴露为 public 变量拖进去：

```csharp
public Light myLight;
```

---

### 2. Type（光源类型）

```csharp
light.type = LightType.Point;      // 灯泡类型
light.type = LightType.Spot;       // 手电筒类型
light.type = LightType.Directional; // 方向光（太阳光）
light.type = LightType.Area;       // 面光源（需烘焙）
```

**四种类型详解：**

| 类型 | 描述 | 典型用途 |
|---|---|---|
| **Directional** | 方向光，无限远，无衰减，照亮整个场景 | 模拟太阳光 |
| **Point** | 点光源，向四周均匀发散，有范围衰减 | 灯泡、火焰 |
| **Spot** | 聚光灯，有锥形范围，有角度和距离衰减 | 手电筒、车灯 |
| **Area** | 面光源（矩形/长方形），只用于烘焙光照 | 室内天花板灯 |

**Spot（聚光灯）专属参数：**

```csharp
light.spotAngle = 45f; // 光锥角度（度）
light.range = 10f;      // 最远照射距离
```

**Point（点光源）专属参数：**

```csharp
light.range = 10f; // 照射范围，超出范围无光照
```

---

### 3. Color（颜色）

```csharp
light.color = Color.white;  // 默认白色
light.color = new Color(1f, 0.8f, 0.4f, 1f); // 暖黄色
light.color = Color.red;
```

---

### 4. Mode（光源模式）

决定光源是实时的还是烘焙的：

```csharp
light.mode = LightMode.Realtime;  // 实时光照
light.mode = LightMode.Baked;     // 烘焙光照
light.mode = LightMode.Mixed;      // 混合光照（部分实时部分烘焙）
```

| 模式 | 特点 | 性能 |
|---|---|---|
| **Realtime** | 每帧实时计算，阴影实时更新 | 最贵 |
| **Baked** | 在编辑时烘焙到 Lightmap 里，运行时不消耗 | 免费 |
| **Mixed** | 烘焙基础光照 + 实时动态物体叠加 | 中等 |

---

### 5. Intensity（亮度）

```csharp
light.intensity = 1f;  // 默认强度
light.intensity = 2f;  // 更亮
light.intensity = 0f;  // 熄灭
```

---

### 6. Shadow Type（阴影类型）

```csharp
light.shadows = LightShadows.None;     // 不投射阴影
light.shadows = LightShadows.Hard;      // 硬阴影（边缘清晰，性能较低）
light.shadows = LightShadows.Soft;       // 软阴影（边缘模糊，效果更好）
```

> 阴影越软渲染成本越高。移动端通常用 Hard 或关闭阴影。

---

### 7. Cookie（纹理投影遮罩）

可以把一张带透明度的纹理（通常是 PNG）投射到场景中，形成特定形状的光影：

```csharp
light.cookie = cookieTexture; // 拖入 Texture 2D
```

典型应用：理发店旋转灯柱、百叶窗阳光光斑、窗户格子投影。

> Cookie 纹理需设置为 **Clamp** 模式，否则会重复平铺。

---

### 8. Draw Halo（光晕）

勾选后在光源位置显示一个光晕球体，常配合 `Flare` 一起用。

---

### 9. Flare（耀斑）

给光源附加一个 Lens Flare（镜头耀斑）资源，模拟人眼看强光源时产生的炫光效果：

```csharp
light.flare = flareAsset; // 拖入 Lens Flare 组件
```

> ⚠️ 只有摄像机上挂载了 **Flare Layer** 组件才能看到耀斑效果。摄像机上搜索 `Flare Layer` 组件并勾选即可。

---

### 10. Culling Mask（光线遮罩）

```csharp
light.cullingMask = 1 << LayerMask.NameToLayer("Player");
```

- 与 Camera 的 Culling Mask 逻辑相同：只照亮指定层的对象
- 常用于：只照亮玩家角色，不照亮背景（性能优化）

---

### 11. Indirect Multiplier（间接光强度）

```csharp
light.bounceIntensity = 0.5f; // 低于1：反弹光变暗；高于1：反弹光变亮
```

- 控制光照反弹（间接光照 bounce）的强度
- 设为 0 可以关闭所有反弹光

---

### 12. Realtime Shadows（实时阴影参数）

```csharp
light.shadows = LightShadows.Soft;

// 阴影分辨率
light.shadowResolution = LightShadowResolution.FromQualitySettings; // 从质量设置继承

// 阴影近/远裁切距离
light.shadowNearPlane = 0.1f;
```

---

### 13. Render Mode（渲染模式）

```csharp
light.renderMode = LightRenderMode.Auto;      // 自动（根据强度决定）
light.renderMode = LightRenderMode.ForcePixel; // 强制像素光照（始终像素渲染）
light.renderMode = LightRenderMode.ForceVertex; // 顶点光照（仅影响顶点，性能低）
```

---

## 三、坑点

1. **Area 面光源不能用于实时**：只能烘焙，用 `LightType.Area` 时 `intensity` 等参数不会在运行时生效，必须配合 Lightmapper 烘焙。
2. **Directional 光源没有衰减**：没有 range 参数，光照强度在场景各处相同（除非有遮挡）。
3. **Spot 聚光灯角度太大会出现接缝**：spotAngle 最大约 180°，太大时边缘会有视觉瑕疵。
4. **Flare 需要 Flare Layer**：没有在摄像机上挂 Flare Layer，flare 资源完全不可见。
5. **烘焙时 realtime 灯光不能参与 Lightmap**：Realtime 光源在烘焙管线里会被忽略，只参与实时光照。
6. **Directional 光可以投射动态阴影**，但阴影范围是整个场景，PC 上还行，移动端需谨慎使用。

## 四、应用场景

**手电筒效果**

```csharp
public class Flashlight : MonoBehaviour
{
    public Light spotLight;
    public KeyCode toggleKey = KeyCode.F;

    void Update()
    {
        if (Input.GetKeyDown(toggleKey))
        {
            spotLight.enabled = !spotLight.enabled;
        }
    }
}
```

**按空格切换灯光明暗**

```csharp
void Update()
{
    float t = Mathf.PingPong(Time.time, 1f); // 0→1→0 循环
    light.intensity = Mathf.Lerp(0.5f, 2f, t);
}
```

**动态颜色灯**

```csharp
void Update()
{
    light.color = Color.HSVToRGB(Time.time * 0.1f % 1f, 1f, 1f);
}
```

## 五、原理

光源计算发生在**渲染管线**的**光照计算阶段**：

- **前向渲染（Forward）**：每个物体依次受每个实时光源影响，光源数量超过阈值后改用延迟光照
- **延迟渲染（Deferred）**：光照计算在独立的 G-Buffer 阶段，多光源效率更高
- **烘焙（Baked）**：编辑器里提前算好，运行时直接采样贴图（Lightmap），CPU/GPU 零开销

---

## 六、扩展

- **Point Light 衰减公式**：`intensity / (1 + distance² * attenuation)`，Unity 用此公式模拟真实光照衰减
- **URP/HDRP 额外参数**：新渲染管线中 Light 组件增加了更多体积光（Volumetrics）、光线步数（Shadow Resolution）等选项
- **光照探头（Light Probes）**：动态物体（如角色）经过烘焙区域时，用光照探头采样间接光照颜色，保证动态物体不被烘焙光照"遗漏"
- **Reflection Probe**：与光源无关，但常与光照配合，用于金属/玻璃表面的实时反射
