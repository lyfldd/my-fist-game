────────────────────────────────────────
一、递归的概念
────────────────────────────────────────

递归：在函数内部调用自身

一个正确的递归函数必须满足两个条件：
1. 必须有结束递归的条件（称为"终止条件"或"基准情形"）
2. 每次递归调用都要让"条件"朝着终止方向变化（否则会无限递归直到栈溢出）

────────────────────────────────────────
二、递归示例一：计算阶乘
────────────────────────────────────────

// 数学定义：n! = n × (n-1) × ... × 1，其中 0! = 1
// 递归思路：n! = n × (n-1)!

static int Factorial(int n)
{
    if (n <= 1)
        return 1;           // 终止条件：n=0 或 n=1 时返回 1
    return n * Factorial(n - 1);   // 递归调用，n 每次减 1，朝终止条件靠近
}

// 调用
Console.WriteLine(Factorial(5));   // 5×4×3×2×1 = 120

// 执行过程：
// Factorial(5)
//   → 5 × Factorial(4)
//     → 4 × Factorial(3)
//       → 3 × Factorial(2)
//         → 2 × Factorial(1)
//           → 返回 1
//         → 返回 2×1 = 2
//       → 返回 3×2 = 6
//     → 返回 4×6 = 24
//   → 返回 5×24 = 120

三、递归示例二：斐波那契数列
────────────────────────────────────────

// 数学定义：F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2)
// 数列：0, 1, 1, 2, 3, 5, 8, 13, 21 ...

static int Fibonacci(int n)
{
    if (n <= 0) return 0;   // 终止条件
    if (n == 1) return 1;   // 终止条件
    return Fibonacci(n - 1) + Fibonacci(n - 2);  // 递归
}

Console.WriteLine(Fibonacci(7));   // 输出 13

────────────────────────────────────────
四、递归示例三：计数（最简单）
────────────────────────────────────────

// 从 n 倒数到 1
static void CountDown(int n)
{
    if (n <= 0)
    {
        Console.WriteLine("发射！");
        return;   // 终止
    }
    Console.WriteLine(n);
    CountDown(n - 1);   // 每次 n-1，朝 0 靠近
}

CountDown(5);
// 输出：5 4 3 2 1 发射！

五、递归的风险：栈溢出（StackOverflowException）
────────────────────────────────────────

// 每次函数调用都会在"调用栈"上占用内存
// 如果递归层数太深（比如 n = 100000），调用栈会被撑满，程序报错崩溃

// 错误示例（缺少终止条件，无限递归）：
static void WrongRecursion(int n)
{
    Console.WriteLine(n);
    WrongRecursion(n + 1);   // n 不断增大，永远不会停，栈溢出！
}

// 注意：如果递归层数可能很深，应考虑改用循环（迭代）实现

────────────────────────────────────────
六、递归 vs 循环
────────────────────────────────────────

递归：
  优点：代码简洁，逻辑清晰（尤其是树形结构、分治问题）
  缺点：有栈溢出风险；函数调用有额外开销，性能略低

循环（迭代）：
  优点：性能好，不存在栈溢出风险
  缺点：某些问题（如树的遍历）用循环写起来较复杂
