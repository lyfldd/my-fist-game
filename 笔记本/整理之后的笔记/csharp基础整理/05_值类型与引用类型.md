────────────────────────────────────────
一、值类型 vs 引用类型
────────────────────────────────────────

值类型（Value Type）：
  有符号整型：sbyte, short, int, long
  无符号整型：byte, ushort, uint, ulong
  浮点数：float, double, decimal
  特殊：bool, char
  结构体：struct（包括自定义结构体）
  枚举：enum

引用类型（Reference Type）：
  string（虽然行为像值类型，但本质是引用类型）
  数组（int[]、string[] 等）
  类（class，包括自定义类）

────────────────────────────────────────
二、使用上的区别（拷贝 vs 引用）
────────────────────────────────────────

// 值类型：赋值时"复制一份"，两个变量互相独立
int a = 10;
int b = a;       // b 是 a 的一个独立副本
b = 20;
Console.WriteLine(a);   // 输出 10（a 没变）
Console.WriteLine(b);   // 输出 20

// 引用类型：赋值时"共享同一块内存"，一个变量改了另一个也变
int[] arr = { 1, 2, 3, 4 };
int[] brr = arr;         // brr 和 arr 指向同一块内存
brr[0] = 5;
Console.WriteLine(arr[0]);   // 输出 5（arr 也跟着变了！）

────────────────────────────────────────
三、内存存储方式
────────────────────────────────────────

栈（Stack）：
  - 存储值类型的"实际数据"
  - 存储引用类型的"地址（引用）"
  - 由系统自动分配和回收
  - 速度快，空间有限

堆（Heap）：
  - 存储引用类型的"实际数据"
  - 由 C# 的 GC（垃圾回收器）自动管理和回收
  - ⚠️ C# 中不需要像 C++ 那样手动 free/delete，GC 会自动处理

内存示意图：
  栈：
    int a = 10          → [10]（直接存值）
    int[] arr = {...}   → [地址 0x001]（存的是堆上的地址）
    int[] brr = arr     → [地址 0x001]（和 arr 存的地址相同！）

  堆：
    0x001 → [1, 2, 3, 4]（arr 和 brr 都指向这里）

────────────────────────────────────────
四、特殊引用类型：string
────────────────────────────────────────

// string 是引用类型，但 C# 对它做了特殊处理，行为像值类型

string str1 = "123";
string str2 = str1;   // str2 和 str1 本来指向同一块内存
str2 = "321";         // ← 关键！对 string 重新赋值，不是修改原内存
                      // 而是在堆上开辟一块新内存，str2 指向新位置
Console.WriteLine(str1);   // 输出 "123"（str1 不变）
Console.WriteLine(str2);   // 输出 "321"

// 这叫"不可变性"：string 的内容一旦创建就不可修改
// str2 = "321" 不是改了原来那块内存，而是让 str2 指向了新内存

⚠️ 注意事项：
  - 如果在循环中频繁对 string 进行拼接、修改，会大量产生内存垃圾
  - 推荐使用 StringBuilder 来做大量字符串拼接（见进阶笔记）

────────────────────────────────────────
五、函数传参中的值类型 vs 引用类型
────────────────────────────────────────

// 值类型传参：传的是"副本"，函数内修改不影响外部
static void ModifyValue(int a)
{
    a = 100;   // 只改了副本
}
int x = 1;
ModifyValue(x);
Console.WriteLine(x);   // 输出 1（没变）

// 引用类型传参：传的是"地址"，函数内修改元素会影响外部
static void ModifyArray(int[] arr)
{
    arr[0] = 999;   // 修改了堆上的实际数据
}
int[] nums = { 1, 2, 3 };
ModifyArray(nums);
Console.WriteLine(nums[0]);   // 输出 999（变了！）

// 但是，引用类型如果在函数内"重新 new"，外部不受影响
static void ReplaceArray(int[] arr)
{
    arr = new int[] { 10, 20, 30 };   // arr 指向了新内存，但外部的 nums 还指向原来的
}
int[] nums2 = { 1, 2, 3 };
ReplaceArray(nums2);
Console.WriteLine(nums2[0]);   // 输出 1（没变）
// 想让这种操作也影响到外部，需要用 ref 关键字（见 ref和out 章节）

────────────────────────────────────────
六、null（空引用）
────────────────────────────────────────

// 引用类型的变量可以赋值为 null，表示"不指向任何对象"
string s = null;
int[] arr = null;

// ⚠️ 对 null 进行操作会抛出 NullReferenceException
// s.Length;     → 报错！
// arr[0];       → 报错！

// 使用前应先判断是否为 null
if (arr != null)
{
    Console.WriteLine(arr[0]);
}

// C# 8.0+ 提供可空引用类型和空合并运算符（了解即可）
int length = arr?.Length ?? 0;   // arr 为 null 时返回 0
