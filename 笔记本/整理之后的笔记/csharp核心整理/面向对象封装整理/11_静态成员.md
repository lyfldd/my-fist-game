静态成员
=============================================

一、概念
用 static 修饰的成员变量、方法、属性等称为静态成员。
静态成员属于类本身，不属于任何实例。

二、特点
1. 通过 类名.成员名 直接访问（不需要 new 实例）
2. 程序运行时分配内存，程序结束时释放
3. 全局唯一：所有实例共享同一份静态成员
4. 静态成员与程序"同生死"

class Test
{
    public static float PI = 3.14159f;
    public int testInt = 100;

    public static float CalcCircleArea(float r)
    {
        return PI * r * r;  // 静态方法可以访问静态成员
    }

    public void TestFun()
    {
        Console.WriteLine(PI);  // 非静态方法也可以访问静态成员
    }
}

// 使用
Test.PI = 3.14f;                  // 直接用类名访问
float area = Test.CalcCircleArea(5);
Test t = new Test();
t.TestFun();                       // 非静态方法需要实例化

三、重要规则
1. 静态方法不能直接访问非静态成员（因为没有 this）
2. 非静态方法可以访问静态成员
3. 静态构造函数可以初始化静态成员

四、静态成员的作用
1. 存储全局唯一的数据（如配置信息、游戏分数）
2. 提供工具方法（如 Math 类）
3. 实现单例模式

五、常见错误理解纠正
静态成员不是说"在程序开始时就开一个内存空间"这么简单。
实际上：
- 静态字段在类型被首次使用时加载（第一次访问该类时）
- 静态构造函数在首次访问该类时执行一次
- 静态成员存储在托管堆的特殊区域（High Frequency Heap）
