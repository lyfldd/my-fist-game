构造函数和析构函数
=============================================

一、构造函数（Constructor）
用于在创建对象时初始化成员变量。构造函数名与类名相同，无返回值。

class Person
{
    public string name;
    public int age;

    // 无参构造函数
    public Person()
    {
        Console.WriteLine("构造函数被调用");
    }

    // 带参构造函数
    public Person(string name, int age)
    {
        this.name = name;
        this.age = age;
    }
}

【注意】
1. 一旦定义了带参构造函数，编译器不再自动生成无参构造函数。
   如需无参构造函数，必须显式声明。
2. 构造函数可以重载。

二、:this() 构造函数链（调用本类其他构造函数）
class Person
{
    public string name;
    public int age;

    public Person()
    {
        Console.WriteLine("无参构造");
    }

    public Person(string name) : this()  // 先调用无参构造，再执行本构造
    {
        this.name = name;
    }

    public Person(string name, int age) : this(name)  // 先调用上面那个构造
    {
        this.age = age;
    }
}

Person p = new Person("张三", 20);
// 输出：无参构造
//       张三

【纠正】原文":this()会先去调用无参构造函数再执行本函数"的描述不够准确。
正确理解：:this() 是调用本类的另一个构造函数，执行顺序是"先执行被调用的构造函数，
再执行当前构造函数"。

三、:base() 调用父类构造函数（继承部分会详细讲）

四、析构函数（Destructor）
当对象的堆内存被 GC（垃圾回收器）回收时自动调用。
用于释放非托管资源（如文件句柄、数据库连接等）。

class Person
{
    ~Person()  // 析构函数，名字与类名相同，前面加 ~
    {
        // 释放资源代码
    }
}

【注意】
1. 析构函数不常用（现代 C# 推荐使用 IDisposable 接口）
2. 不能带参数，不能加访问修饰符
3. 一个类只能有一个析构函数
4. 析构函数由 GC 调用，具体时机不确定，不要在析构函数中放确定性代码
