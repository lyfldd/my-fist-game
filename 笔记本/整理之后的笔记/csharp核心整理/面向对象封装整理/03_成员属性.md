成员属性
=============================================

一、为什么需要成员属性？
成员属性用于保护成员变量，为获取和赋值添加逻辑处理。
解决访问权限的局限问题：public 可以被任意修改，private 无法从外部访问。
属性可以让成员在外部"只读不写"或"只写不读"。

二、基本语法
访问修饰符 属性类型 属性名
{
    get { return 字段名; }
    set { 字段名 = value; }   // value 是 C# 关键字，表示外部赋的值
}

三、完整示例
class Person
{
    private int age;
    private string name;
    private int money;

    public string Name
    {
        get { return name; }      // 获取时直接返回
        set { name = value; }    // 赋值时直接设置
    }

    public int Age
    {
        get { return age; }
        set
        {
            if (value < 0 || value > 150) return; // 加规则：合法性校验
            age = value;
        }
    }

    // Money 属性演示在赋值和获取时进行数据转换（仅作演示）
    public int Money
    {
        get { return money; }        // 正确：直接返回存储的真实值
        set { money = value; }       // 正确：直接存储赋的值
    }
}

【重要纠正】
原文示例中 Money 的 get { return money-5; } set { money = value+5; } 写法
会导致每次访问都叠加偏移量，结果不可预测。正确做法是：get 直接返回，
set 直接赋值。如果需要偏移，应在业务逻辑层处理，而非属性层。

四、get/set 单独访问修饰符（细粒度控制）
1. 不写：使用属性声明时的访问权限
2. 写的访问修饰符必须低于属性的访问修饰符
3. 不能两个都低于属性修饰符

示例：Name 属性外部可读写，但 Age 只能外部读取（只能在类内部设置）
    public string Name { get; set; }          // 默认 public，可读写

    private int age;
    public int Age
    {
        get { return age; }
        private set { age = value; }           // set 是 private，外部只能读
    }

五、自动属性（C# 3.0+ 推荐写法）
当 get/set 中不需要额外逻辑时，直接用自动属性：

    public float Height { get; set; }           // 自动生成私有字段
    public string Name { get; private set; }    // 外部只读，内部可写
    public int Id { get; } = GenerateId();      // 可在声明时初始化（C# 6.0+）
