垃圾回收机制（GC）
=============================================

一、什么是 GC
垃圾回收（Garbage Collection）自动管理托管堆上的内存。
当一个对象不再被任何引用指向时，GC 会自动回收它的内存。
栈上的值类型变量由系统自动管理，不需要 GC。

二、代（Generation）
GC 将堆内存分为三代：
  Gen 0（0代）：新分配的对象，存活时间短
  Gen 1（1代）：从 Gen 0 存活下来的对象
  Gen 2（2代）：长期存活的对象

大对象（>85KB）直接进入 Gen 2，GC 不会压缩大对象所在的内存。

三、GC 工作原理（两阶段）
第1步：标记（Mark）
  从根（静态字段、方法参数）出发，遍历所有可达对象，标记为"存活"。

第2步：压缩（Compact）
  将存活对象移到一起，释放未标记对象的内存。

第3步：晋升
  存活对象可能从 Gen 0 晋升到 Gen 1/Gen 2。

触发条件：
  Gen 0 满了 → 触发 Gen 0 GC
  Gen 1 满了 → 触发 Gen 0+1 GC
  Gen 2 满了 → 触发 Full GC（全部三代）

四、GC 的性能影响
GC 执行时会导致短暂停顿（"Stop the World"）。
大对象进入 Gen 2，且不会压缩，可能导致内存碎片。

五、手动触发 GC
GC.Collect();       // 强制执行垃圾回收（一般不推荐）
GC.Collect(0);      // 只回收 Gen 0
GC.Collect(2);      // 执行 Full GC

通常在切换场景时调用（游戏开发常见做法）。

六、析构函数与 GC
如果类实现了析构函数（~ClassName），对象会进入"待终结"队列，
GC 会延迟回收，需要两次 GC 才能完全释放内存。

【现代 C# 建议】
优先使用 IDisposable 接口和 using 语句来管理非托管资源，
而不是依赖析构函数。
