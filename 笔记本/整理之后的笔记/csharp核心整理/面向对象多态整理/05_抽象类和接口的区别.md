抽象类和接口的区别（完整对比表）
=============================================

相同点：
1. 都可以被继承
2. 都不能被实例化
3. 都可以包含方法的声明
4. 子类必须实现未实现的方法
5. 都遵循里氏替换原则

不同点：

  对比项           抽象类                              接口
  关键字           abstract class                    interface
  继承             只能单一继承（一个类只能有一个父类）  多继承（一个类可实现多个接口）
  成员变量         可以有                              不能有
  构造函数         可以有（供子类 base() 调用）        不能有
  方法类型         可以有普通方法 + 抽象方法            默认只有抽象方法
                                     （C# 8.0+ 支持 default 方法）
  访问修饰符       可以任意 public/protected/private   成员默认 public，不能 private
  静态成员         可以有 static 成员                  不能有
  适用场景         表示"是什么"（模板、基类）           表示"能做什么"（行为规范）
  代码复用         子类直接复用抽象类的普通方法          必须各自实现方法

使用建议：
1. 需要多继承时 → 用接口
2. 需要包含成员变量、构造函数 → 用抽象类
3. 需要提供一个通用模板 → 用抽象类
4. 需要定义行为契约 → 用接口
5. 两者可以结合使用！
