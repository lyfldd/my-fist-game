接口
=============================================

一、概念
接口（interface）是行为的抽象规范，只定义"能做什么"，不定义"怎么做"。
关键字：interface

二、命名规范
接口名使用 PascalCase，推荐以 I 开头：
  IAttackable、IFlyable、IMovable

三、接口的声明规范
1. 不包含成员变量（只能包含方法、属性、索引器、事件）
2. 成员不能有实现（抽象）
3. 成员不能加访问修饰符，默认 public（不能 private）
4. 不能包含构造函数和析构函数
5. 接口不能继承类，但可以继承接口
6. 接口本身可以继承多个接口

四、类实现接口
1. 类可以继承一个类 + 多个接口（先写类，再写接口）
2. 类必须实现接口的所有成员，访问修饰符必须是 public
3. 如果类同时继承了抽象类和接口，先写抽象类，再写接口

五、示例
interface IAttackable
{
    void Attack();      // 方法
    int Damage { get; set; }  // 属性
}

interface IFlyable
{
    void Fly();
}

class Warrior : Character, IAttackable, IFlyable
{
    public int Damage { get; set; }

    public void Attack() { Console.WriteLine("近战攻击"); }
    public void Fly() { Console.WriteLine("不能飞"); }
}

六、显式接口实现
当类继承多个接口，且接口间有同名方法时，用显式实现来区分：

interface IAttackable
{
    void Perform();
}

interface IMovable
{
    void Perform();
}

class Player : IAttackable, IMovable
{
    void IAttackable.Perform()
    {
        Console.WriteLine("攻击行为");
    }

    void IMovable.Perform()
    {
        Console.WriteLine("移动行为");
    }
}

// 调用时需要转换为对应接口
Player p = new Player();
((IAttackable)p).Perform();  // 攻击行为
((IMovable)p).Perform();      // 移动行为

七、接口与抽象类的区别

  对比项           抽象类                接口
  继承             单一继承              多继承
  成员变量         可以有                不能有
  构造函数         可以有                不能有
  方法实现         可以有（普通方法）    默认不能有（C# 8.0+ 支持默认实现）
  访问修饰符       可以任意              不能 private
  适用场景         "是什么"（模板）      "能做什么"（行为契约）

八、接口的优势
1. 支持多继承（一个类可以实现多个接口）
2. 解耦：高层模块依赖接口而非具体实现
3. 实现插件式开发
