多态（VOB：Virtual、Override、Base）
=============================================

一、多态的概念
多态：同一父类的对象，在执行相同方法时有不同的表现。
让不同子类对象在父类引用下，展现各自独特的行为。

二、静态多态 vs 动态多态
1. 静态多态（编译时决定）：函数重载、方法重载
2. 动态多态（运行时决定）：虚方法重写（override）

三、VOB 关键字
  virtual  ：父类中声明虚方法，允许被子类重写
  override ：子类中重写父类的虚方法
  base     ：在子类中调用父类的被重写方法

四、基础示例
class GameObject
{
    public string name;
    public GameObject(string name) { this.name = name; }

    public virtual void Attack()
    {
        Console.WriteLine("基础攻击");
    }
}

class Player : GameObject
{
    public Player(string name) : base(name) { }

    public override void Attack()
    {
        base.Attack();                      // 先调用父类方法
        Console.WriteLine(name + "使用武器攻击！"); // 再添加子类行为
    }
}

// 多态调用
GameObject obj = new Player("玩家");
obj.Attack();
// 输出：
// 基础攻击
// 玩家使用武器攻击！

五、virtual 关键字
1. 声明在基类中，表示该方法可以被派生类重写
2. 基类中的虚方法有默认实现
3. 派生类可以重写（override），也可以不重写（继承默认实现）

六、override 关键字
1. 重写基类的虚方法
2. 签名（方法名、参数、返回值）必须与基类方法完全一致
3. 重写方法的访问修饰符不能比被重写方法更严格

七、base 关键字的三大作用
1. 在子类构造函数中调用父类构造函数 → base(参数)
2. 在子类方法中调用父类的被重写方法 → base.方法名()
3. 在子类中访问被隐藏的基类成员 → base.成员名

八、new 隐藏 vs override 重写（重要区别！）

  new 隐藏：子类用 new 声明同名成员，编译时绑定，不是多态
  override重写：子类用 override 声明同名方法，运行时绑定，是多态

class Base
{
    public void Show() { Console.WriteLine("Base.Show"); }
}

class Derived : Base
{
    public new void Show() { Console.WriteLine("Derived.Show"); }
}

Base d = new Derived();
d.Show();  // Base.Show（因为 Show 不是 virtual，不构成多态！）

class Base2
{
    public virtual void Show() { Console.WriteLine("Base2.Show"); }
}

class Derived2 : Base2
{
    public override void Show() { Console.WriteLine("Derived2.Show"); }
}

Base2 d2 = new Derived2();
d2.Show();  // Derived2.Show（多态！）

【结论】只有 virtual + override 才是真正的多态！
