继承中的构造函数
=============================================

一、执行顺序
实例化子类对象时，构造函数执行顺序是：
父类构造函数 → 子类构造函数（先父后子，层层向上）

class Game
{
    public Game() { Console.WriteLine("爷爷辈 Game"); }
}

class Player : Game
{
    public Player() { Console.WriteLine("父辈 Player"); }
}

class MainPlayer : Player
{
    public MainPlayer() { Console.WriteLine("子辈 MainPlayer"); }
}

new MainPlayer();
// 输出：
// 爷爷辈 Game
// 父辈 Player
// 子辈 MainPlayer

二、:base() 调用父类构造函数
子类构造函数默认调用父类的无参构造函数。
如果父类没有无参构造函数（只有带参构造函数），子类必须显式用 :base(参数) 调用。

class Father
{
    public string name;
    public Father(string name) { this.name = name; Console.WriteLine("父类构造函数"); }
}

class Son : Father
{
    public Son(string name) : base(name)  // 显式调用父类带参构造
    {
        Console.WriteLine("子类构造函数");
    }
}

new Son("张三");
// 输出：
// 父类构造函数
// 子类构造函数

三、:this() 调用本类其他构造函数
与继承无关，但常见于在继承体系中简化代码。

四、:base() 访问基类成员（与构造函数无关）
在子类的方法中，可以用 base.成员名 访问父类的被隐藏成员：

class Base
{
    public int value = 10;
}

class Derived : Base
{
    public new int value = 20;

    public void Print()
    {
        Console.WriteLine(value);         // 20（子类）
        Console.WriteLine(base.value);   // 10（父类）
    }
}
