里氏替换原则
=============================================

一、定义
里氏替换原则（Liskov Substitution Principle，LSP）：
任何父类出现的地方，子类都可以替代，且不影响程序正确性。

核心：用父类容器装载子类对象，统一管理不同子类的行为。

二、示例
class GameObject { }

class Player : GameObject
{
    public void Attack() { Console.WriteLine("玩家攻击"); }
}

class Monster : GameObject
{
    public void Attack() { Console.WriteLine("怪物攻击"); }
}

class Boss : GameObject
{
    public void Attack() { Console.WriteLine("Boss攻击"); }
}

// 使用父类容器装载所有子类对象
GameObject[] objects = { new Player(), new Monster(), new Boss() };

foreach (GameObject obj in objects)
{
    Console.WriteLine("有一个游戏对象");
}
// 所有子类对象都可以当作 GameObject 来使用

三、is 和 as 运算符
当用父类引用指向子类对象时，想要调用子类独有的方法，需要用 is/as。

1. is：判断对象是否是指定类型，返回 bool
2. as：将对象尝试转换为指定类型，失败返回 null

GameObject player = new Player();

if (player is Player)   // 判断 player 是否是 Player 类型
{
    Player p = player as Player;  // 安全转换为 Player
    p.Attack();                   // 调用子类方法
}

// 简洁写法（C# 7.0+ 支持模式匹配）
if (player is Player p)
{
    p.Attack();
}

// switch 模式匹配（C# 8.0+）
switch (player)
{
    case Player pl: pl.Attack(); break;
    case Monster m: m.Attack(); break;
    case Boss b: b.Attack(); break;
    default: break;
}

四、is/as 的作用
is  + as = 先判断再转换，避免转换失败时抛出异常。
直接用 (Player)player 强转会抛异常，is/as 组合更安全。
