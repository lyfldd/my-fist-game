密封类
=============================================

一、概念
用 sealed 关键字修饰的类，不能被其他类继承。

sealed class FinalClass
{
    public int value;
}

// 编译错误：无法从 FinalClass 派生类
// class Derived : FinalClass { }

二、为什么需要密封类？
1. 防止类被继承，保护设计意图（如 String 类）
2. 编译器可以做一些优化（密封类的方法调用更快）
3. 当类已经足够完善，不需要再扩展时

三、sealed 与 abstract 的区别
  abstract：抽象类，必须被继承
  sealed：密封类，禁止被继承

四、使用场景
1. 工具类（如 Math 类，.NET 源码中是 sealed）
2. 业务类设计时，明确某类不需要被继承
3. 配合 sealed override 封印特定方法（见"多态"章节）
