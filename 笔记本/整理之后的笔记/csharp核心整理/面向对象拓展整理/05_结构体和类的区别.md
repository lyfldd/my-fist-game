结构体和类的区别（完整对比）
=============================================

1. 类型本质
   结构体 → 值类型
   类     → 引用类型

2. 内存位置
   结构体 → 栈（作为类成员时在堆）
   类     → 堆

3. 赋值行为
   结构体 → 值拷贝（复制整个数据）
   类     → 引用复制（两个变量指向同一对象）

4. 默认构造函数
   结构体 → 隐式存在，无法显式定义无参构造
   类     → 可以显式定义无参构造

5. 字段初始化
   结构体 → 不能在声明时初始化
   类     → 可以在声明时初始化

6. 继承
   结构体 → 不能继承（但可以实现接口）
   类     → 可以继承（单继承）

7. 访问修饰符
   结构体 → 不能使用 protected
   类     → 可以使用 protected

8. 析构函数
   结构体 → 不能有析构函数
   类     → 可以有析构函数

9. 构造函数
   结构体 → 定义带参构造后，不会覆盖隐式无参构造
           → 必须在构造函数中初始化所有字段
   类     → 定义构造函数不会自动覆盖默认构造函数（需显式声明）

10. 自身引用
    结构体 → 不能在内部声明自身类型的字段
    类     → 可以声明自身类型的字段（如链表节点）

11. 静态成员
    结构体 → 可以有 static 成员
    类     → 可以有 static 成员

12. 使用场景
    结构体：数据集合（坐标、点、矩形、颜色等小型数据）
    类    ：需要继承、多态、引用共享的对象（玩家、怪物、UI）

【常见误解纠正】
- "结构体在栈，类在堆"：这只对局部变量成立。
  结构体作为类的成员时，存储在堆上，和类实例一起。
- 结构体也可以有方法、属性、索引器、运算符重载。
