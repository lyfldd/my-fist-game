命名空间
=============================================

一、概念
命名空间（namespace）用于组织代码，将相关的类、接口、结构体等归为一组。
相当于一个容器，防止命名冲突。

二、基本语法
namespace 命名空间名
{
    class 类名 { }
    struct 结构体名 { }
}

三、命名空间的特点
1. 可以重名（写在不同文件中的同名 namespace 会被视为同一个）
2. 内部类不能重名
3. 不同命名空间可以有同名的类

四、引用命名空间
using 命名空间名;   // 引用后可以直接使用该命名空间的类

五、避免歧义（别名类名冲突）
当两个命名空间有同名类时，用完全限定名：
  命名空间名.类名

六、不同命名空间的同名类
"可以理解为平行宇宙，这个宇宙只能有一个你，但是别的宇宙可以有很多个你。"

示例：
namespace UniverseA
{
    class Person { public string Name = "宇宙A的人"; }
}

namespace UniverseB
{
    class Person { public string Name = "宇宙B的人"; }
}

// 使用时用命名空间区分
var a = new UniverseA.Person();
var b = new UniverseB.Person();

七、命名空间与访问修饰符
类不加访问修饰符默认是 internal（程序集内可见）。
可以是 public（任何地方可见）。
private 和 protected 只能用于嵌套类。

namespace MySpace
{
    class InternalClass { }      // 默认 internal
    public class PublicClass { } // 公开访问
}
