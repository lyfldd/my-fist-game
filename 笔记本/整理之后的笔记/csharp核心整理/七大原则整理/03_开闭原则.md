开闭原则（Open-Closed Principle）
=============================================

一、定义
软件实体（类、模块、函数等）应该对扩展开放，对修改关闭。
即在不修改现有代码的基础上，通过扩展来添加新功能。

核心思想：用继承和多态来扩展，而不是修改已有代码。

二、反面示例（违反开闭原则）
class Shape
{
    public string Type { get; set; }
}

class AreaCalculator
{
    public double Calculate(Shape shape)
    {
        if (shape.Type == "Circle")
            return 3.14 * 5 * 5;
        else if (shape.Type == "Rectangle")
            return 4 * 6;
        // 每添加一种新图形，都要修改这里！
        return 0;
    }
}

三、遵循开闭原则的改进
abstract class Shape
{
    public abstract double CalculateArea();
}

class Circle : Shape
{
    public double Radius { get; set; }
    public override double CalculateArea() => 3.14159 * Radius * Radius;
}

class Rectangle : Shape
{
    public double Width { get; set; }
    public double Height { get; set; }
    public override double CalculateArea() => Width * Height;
}

// 使用：添加新图形不需要修改 AreaCalculator
Shape[] shapes = { new Circle { Radius = 5 }, new Rectangle { Width = 4, Height = 6 } };
foreach (Shape s in shapes)
{
    Console.WriteLine(s.CalculateArea());  // 多态调用，各图形自己负责计算
}
