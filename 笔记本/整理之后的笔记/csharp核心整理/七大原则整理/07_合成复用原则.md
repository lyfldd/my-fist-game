合成复用原则（Composite Reuse Principle）
=============================================

一、定义
尽量使用对象组合（has-a），而不是继承（is-a）来达到复用目的。
组合比继承更灵活，可以在运行时动态改变对象的行为。

核心思想：多用组合，少用继承。

二、继承 vs 组合

继承（is-a）：
  class Car : Engine { }   // Car 是一个 Engine（强耦合，不灵活）

组合（has-a）：
  class Car
  {
      private Engine _engine;  // Car 有一个 Engine（松耦合，可替换）
      public void Start() { _engine.Start(); }
  }

三、反面示例（滥用继承）
class DBConnection
{
    public void Connect() { /* 连接数据库 */ }
}

class UserDAO : DBConnection  // UserDAO 继承数据库连接类
{
    public void SaveUser() { Connect(); /* 保存用户 */ }
}

问题：UserDAO 和数据库连接类强耦合，无法切换数据源。

四、遵循合成复用原则的改进
interface IDBConnection
{
    void Connect();
}

class MySQLConnection : IDBConnection { ... }
class PostgreSQLConnection : IDBConnection { ... }

class UserDAO
{
    private IDBConnection _connection;  // 通过接口组合

    public UserDAO(IDBConnection connection)
    {
        _connection = connection;
    }

    public void SaveUser()
    {
        _connection.Connect();  // 可以轻松切换数据源
    }
}

// 使用
var dao = new UserDAO(new MySQLConnection());
dao.SaveUser();

// 切换到 PostgreSQL 不需要改 UserDAO
var dao2 = new UserDAO(new PostgreSQLConnection());

五、什么时候用继承，什么时候用组合？
  - "是...的一种" → 用继承（Cat is Animal）
  - "有...一个" → 用组合（Car has Engine）
  - 需要运行时切换行为 → 用组合
  - 需要复用父类实现，且不会有冲突 → 可以用继承
