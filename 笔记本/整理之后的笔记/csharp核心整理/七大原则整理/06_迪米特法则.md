迪米特法则（Law of Demeter）/ 最少知识原则
=============================================

一、定义
一个对象应该对其他对象有最少的了解。
即：只和直接的朋友交流，不和"朋友的朋友"交流。

核心思想：降低对象之间的耦合度，减少对象之间的依赖。

二、反面示例（违反迪米特法则）
class Customer
{
    public Address Address = new Address();
}

class Order
{
    private Customer _customer;

    public void PrintShippingAddress()
    {
        // 直接访问了 Customer 的内部对象 Address！
        Console.WriteLine(_customer.Address.Street);
    }
}

问题：Order 直接获取了 Customer 的内部 Address 对象，
如果 Address 结构改变，Order 也需要跟着改。

三、遵循迪米特法则的改进
class Customer
{
    private Address _address = new Address();

    public string GetAddress()
    {
        return _address.Street;  // 通过公共方法暴露，而不是暴露内部对象
    }
}

class Order
{
    private Customer _customer;

    public Order(Customer customer) { _customer = customer; }

    public void PrintShippingAddress()
    {
        // Order 只和 Customer 交流，不涉及 Address
        Console.WriteLine(_customer.GetAddress());
    }
}

四、迪米特法则的通俗理解
"只和你的直接朋友说话，不要让别人替你传话。"
  - 直接朋友：方法的参数、返回值、本类创建的局部对象
  - 避免：访问朋友的朋友（链式调用过深）
