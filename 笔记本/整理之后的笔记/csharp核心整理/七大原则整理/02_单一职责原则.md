单一职责原则（Single Responsibility Principle）
=============================================

一、定义
一个类应该只有一个引起它变化的原因。
即：一个类只负责一项职责。

核心思想：让每个类专注于做一件事。

二、反面示例（违反单一职责）
class User
{
    public void SaveToDatabase() { /* 保存到数据库 */ }
    public void ValidateLogin(string username, string password) { /* 验证登录 */ }
    public void SendEmail(string to) { /* 发送邮件 */ }
    public void GenerateReport() { /* 生成报告 */ }
}

问题：User 类承担了数据存储、业务逻辑、邮件发送、报表生成等多种职责。
任何一种需求变化都会导致 User 类被修改。

三、遵循单一职责的改进
class User { }  // 纯数据类

class UserValidator
{
    public bool ValidateLogin(string username, string password) { /* 验证 */ }
}

class UserRepository
{
    public void Save(User user) { /* 数据库操作 */ }
}

class EmailService
{
    public void Send(string to, string content) { /* 发邮件 */ }
}

class ReportGenerator
{
    public void Generate(User user) { /* 生成报告 */ }
}

好处：每个类职责单一，修改其中一个不会影响其他类。
