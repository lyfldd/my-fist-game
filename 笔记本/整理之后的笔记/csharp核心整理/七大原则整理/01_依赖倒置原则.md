依赖倒置原则（Dependency Inversion Principle）
=============================================

一、定义
高层模块不应该依赖低层模块，二者都应该依赖其抽象；
抽象不应该依赖细节，细节应该依赖抽象。

核心思想：面向接口编程，而不是面向实现编程。

二、问题示例（高层依赖低层）
class EmailSender
{
    public void Send(string message) { /* 发送邮件 */ }
}

class NotificationService
{
    private EmailSender sender = new EmailSender();  // 直接依赖具体类

    public void Notify(string message)
    {
        sender.Send(message);  // 如果要改成发短信，就得改这里
    }
}

问题：NotificationService 直接依赖了 EmailSender，换一种通知方式就要改代码。

三、遵循依赖倒置原则的改进
interface IMessageSender
{
    void Send(string message);
}

class EmailSender : IMessageSender
{
    public void Send(string message) { /* 发邮件 */ }
}

class SmsSender : IMessageSender
{
    public void Send(string message) { /* 发短信 */ }
}

// 高层模块依赖接口（抽象）
class NotificationService
{
    private readonly IMessageSender _sender;

    public NotificationService(IMessageSender sender)  // 通过构造函数注入
    {
        _sender = sender;
    }

    public void Notify(string message)
    {
        _sender.Send(message);  // 切换实现不需要改这里
    }
}

// 使用：可以轻松切换不同的发送者
var emailService = new NotificationService(new EmailSender());
var smsService = new NotificationService(new SmsSender());
