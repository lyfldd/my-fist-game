里氏替换原则（Liskov Substitution Principle）
=============================================

一、定义
所有引用基类的地方必须能透明地使用其子类的对象。
即：子类对象可以替换父类对象，而程序行为不会发生变化。

核心思想：子类必须保持与父类"一致"的行为，不能破坏父类契约。

二、正确示例
abstract class Animal
{
    public abstract void Eat();
}

class Dog : Animal
{
    public override void Eat() { Console.WriteLine("狗在吃狗粮"); }
}

class Cat : Animal
{
    public override void Eat() { Console.WriteLine("猫在吃猫粮"); }
}

void Feed(Animal animal)  // 任何 Animal 的子类都可以传进来
{
    animal.Eat();
}

Feed(new Dog());  // 输出：狗在吃狗粮
Feed(new Cat());  // 输出：猫在猫粮

三、违反里氏替换原则的反面示例
class Rectangle
{
    public virtual int Width { get; set; }
    public virtual int Height { get; set; }
    public int Area() => Width * Height;
}

// Square 继承 Rectangle，但行为不一致！
class Square : Rectangle
{
    public override int Width
    {
        get => base.Width;
        set { base.Width = value; base.Height = value; } // 赋值时同时修改高
    }
    public override int Height
    {
        get => base.Height;
        set { base.Width = value; base.Height = value; }
    }
}

void UseRectangle(Rectangle r)
{
    r.Width = 5;
    r.Height = 4;
    Console.WriteLine(r.Area());  // 期望 20，但 Square 返回 16
}

问题：Square 替换 Rectangle 后，Area() 的结果不符合预期。
违反里氏替换原则！

四、总结
里氏替换原则 + 开闭原则：里氏替换要求子类不破坏父类契约，
与开闭原则一起使用，构成了 OOP 中"用多态扩展"的核心。
