接口隔离原则（Interface Segregation Principle）
=============================================

一、定义
客户端不应该被迫依赖它不需要的接口。
即：接口应该保持精简，一个接口只包含客户端真正需要的方法。

核心思想：接口粒度要小，不要把不相关的方法塞进一个大接口。

二、反面示例（违反接口隔离原则）
interface IWorker
{
    void Work();
    void Eat();
    void Sleep();
}

class Robot : IWorker
{
    public void Work() { /* 机器人工作 */ }
    public void Eat() { /* 机器人不需要吃饭 */ }
    public void Sleep() { /* 机器人不需要睡觉 */ }
}

class Human : IWorker
{
    public void Work() { /* 人类工作 */ }
    public void Eat() { /* 人类吃饭 */ }
    public void Sleep() { /* 人类睡觉 */ }
}

问题：Robot 被迫实现了它不需要的 Eat() 和 Sleep() 方法。

三、遵循接口隔离原则的改进
interface IWorkable { void Work(); }
interface IEatable  { void Eat(); }
interface ISleepable{ void Sleep(); }

// 机器人只需要工作
class Robot : IWorkable
{
    public void Work() { }
}

// 人类需要工作、吃饭、睡觉
class Human : IWorkable, IEatable, ISleepable
{
    public void Work() { }
    public void Eat() { }
    public void Sleep() { }
}
