## 四、七大设计原则（独立体系）

| 原则 | 所在笔记 | 核心关联 |
|------|----------|----------|
| [[01_依赖倒置原则]] | 七大原则 | 高层模块不依赖低层模块 |
| [[02_单一职责原则]] | 七大原则 | 一个类只做一件事 |
| [[03_开闭原则]] | 七大原则 | 对扩展开放，对修改关闭 |
| [[04_接口隔离原则]] | 七大原则 | 接口要小而专 |
| [[05_里氏替换原则]] | 七大原则/继承 | ⭐ 与继承中的里氏替换同名，原理一致 |
| [[06_迪米特法则]] | 七大原则 | 只和直接朋友通信 |
| [[07_合成复用原则]] | 七大原则 | 优先组合/聚合，而非继承 |

> ⚠️ **重点**：七大原则中的「里氏替换原则」和继承中的「里氏替换原则」是同一个原理，一个讲设计思想，一个讲语法实现。

## 五、数据结构全景图

### 5.1 选择决策表

```
需要快速查找？
├── 是 → 需要按键查找？
│       ├── 是 → [[03_Dictionary]]
│       └── 否 → 数据有顺序？
│               ├── 有 → [[02_List]] 或 [[01_Array]]
│               └── 无 → [[06_LinkedList]]
└── 否 → 需后进先出？
        ├── 是 → [[04_Stack]]
        └── 否 → 需先进先出？
                ├── 是 → [[05_Queue]]
                └── 否 → [[02_List]]
```

### 5.2 泛型 vs 非泛型

| 非泛型 | → | 泛型替代 | 性能 |
|--------|---|----------|------|
| [[01_ArrayList]] | → | [[02_List]] | 泛型更快（无装箱拆箱）|
| [[02_Hashtable]] | → | [[03_Dictionary]] | 泛型更快 |

### 5.3 底层原理

所有集合的底层都逃不出两种存储方式：
- **顺序存储**：数组实现 → [[01_Array]]、[[02_List]]
- **链式存储**：节点指针实现 → [[06_LinkedList]]

→ [[03_顺序存储和链式存储]] 详解

## 六、学习顺序建议

### 新手路线（从零开始）
```
01_控制台输入输出与变量类型
    → 02_运算符
    → 03_条件分支语句
    → 04_循环语句
    → 01_数组与枚举
    → 02_函数
    → 05_值类型与引用类型

    → 继承 → 多态 → 封装
```

```
委托和事件（理解回调机制）
    → 匿名函数（简化语法）

    → 数据结构（应用实践）
    → 多线程（性能优化）
    → 反射（运行时能力）
```
