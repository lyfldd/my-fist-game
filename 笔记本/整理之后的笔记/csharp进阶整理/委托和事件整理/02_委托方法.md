# 委托的方法

---

## 一、基本绑定与调用

### 1.1 绑定单个方法

```csharp
Action a = Test; // 绑定一个方法
a();               // 调用，等价于 a.Invoke();
```

### 1.2 多播委托（+= / -=）

委托底层是方法链表，可以用 `+=` 追加方法，`-=` 移除方法。

```csharp
Action a = A;    // 绑定方法 A
a += B;          // 追加方法 B（底层调用 Delegate.Combine）
a += C;          // 追加方法 C

// 调用时会按顺序执行 A → B → C
a();

// 移除方法
a -= B;          // 从链表中移除 B（底层调用 Delegate.Remove）
```

### 1.3 安全调用（?.Invoke）

委托可能为 null，调用前要检查。

```csharp
// ❌ 不安全：委托为 null 时会抛异常
a();

// ✅ 安全写法
a?.Invoke();

// ✅ 传统写法
if (a != null) a();
```

## 二、委托的属性

### 2.1 Method — 获取方法信息

```csharp
Action a = Test;
MethodInfo method = a.Method;

Console.WriteLine(method.Name);       // Test
Console.WriteLine(method.ReturnType); // System.Void
```

### 2.2 Target — 获取所属对象

```csharp
// 静态方法：Target = null
Action staticA = ClassName.StaticMethod;
Console.WriteLine(staticA.Target); // null

// 实例方法：Target = 方法所属的对象
Player p = new Player();
Action instanceA = p.Attack;
Player target = (Player)instanceA.Target; // 返回 p 对象
```

### 2.3 GetInvocationList() — 获取方法链表

获取多播委托中**所有方法**组成的数组，常用于遍历或异常处理。

```csharp
Action a = A;
a += B;
a += C;

Delegate[] list = a.GetInvocationList();
foreach (Action item in list)
{
    item(); // 逐个执行
}
```

> 重要用途：如果链表中某个方法抛异常，可以用 `GetInvocationList()` 捕获并继续执行后续方法。

## 三、调用委托的方式

### 3.1 Invoke() — 同步调用

```csharp
a.Invoke();
// 等价于
a();
```

### 3.2 DynamicInvoke() — 弱类型调用

运行时才匹配参数类型，不检查类型安全性，**很少用**。

```csharp
Delegate d = Test;
d.DynamicInvoke(new object[] { 10 }); // 参数用 object[] 传入
```

### 3.3 BeginInvoke() / EndInvoke() — 旧式异步（已过时）

> 现在推荐使用 `async / await`，这两个方法已基本被淘汰。

## 四、精确调用策略

如果想**精确调用某一个方法**，不要用多播委托，有三种策略：

### 策略一：每个委托只绑定一个方法

```csharp
// 每个委托独立，只绑定一个方法
Action attack = Attack;
Action defend = Defend;
Action jump   = Jump;

// 想调用谁就调用谁
attack();  // 攻击
defend();  // 防御
```

> 优点：意图明确、不误触发、性能最好。

### 策略二：放进数组 / List（按索引调用）

```csharp
// 放进数组
Action[] skills = { Attack, Defend, Jump };
skills[0]();  // 攻击
skills[1]();  // 防御

// 放进 List
List<Action> skillList = new List<Action> { Attack, Defend, Jump };
// 优势：可遍历、可随机选择、可按序号/AI 随机释放
```

### 策略三：用 Dictionary（按名字/ID 调用）【推荐】

```csharp
Dictionary<string, Action> skillDict = new Dictionary<string, Action>()
{
    { "Attack", Attack },
    { "Defend", Defend },
    { "Jump",   Jump }
};

// 按名字 O(1) 查找调用，适合技能、背包、物品 ID 查找
skillDict["Attack"]();        // 攻击
if (skillDict.ContainsKey("Defend"))
    skillDict["Defend"]();
```

## 五、Delegate 静态方法

### 5.1 Delegate.Combine() — 合并委托

```csharp
Action a = A;
Action b = B;
Action c = (Action)Delegate.Combine(a, b);
// 等价于
Action c = A;
c += B;
```

### 5.2 Delegate.Remove() — 移除最后一个匹配的委托

```csharp
Action a = A + B + C;
a = (Action)Delegate.Remove(a, B); // 从链表移除最后一个 B
// 等价于
a -= B;
```

### 5.3 Delegate.RemoveAll() — 移除所有匹配的委托

```csharp
Action a = A + B + A + C;
a = (Action)Delegate.RemoveAll(a, A); // 移除所有 A
```

## 六、方法汇总表

| 方法/属性 | 作用 | 说明 |
|---|---|---|
| `Invoke()` | 同步调用所有方法 | 最常用，等价于 `a()` |
| `DynamicInvoke()` | 弱类型调用 | 运行时匹配参数，很少用 |
| `BeginInvoke()` / `EndInvoke()` | 旧式异步调用 | 已过时，推荐 `async/await` |
| `Method` | 获取方法信息 | 返回 `MethodInfo` |
| `Target` | 获取方法所属对象 | 静态方法返回 null |
| `GetInvocationList()` | 获取方法链表数组 | 用于遍历和异常处理 |
| `Delegate.Combine(a, b)` | 合并两个委托 | 等价于 `+=` |
| `Delegate.Remove(source, value)` | 移除最后一个匹配 | 等价于 `-=` |
| `Delegate.RemoveAll(source, value)` | 移除所有匹配 | 全部清除 |

## 七、实用技巧

### 7.1 多播委托 + GetInvocationList 异常处理

```csharp
Action all = MethodA;
all += MethodB;
all += MethodC;

foreach (Action handler in all.GetInvocationList())
{
    try
    {
        handler();
    }
    catch (Exception ex)
    {
        Console.WriteLine($"方法 {handler.Method.Name} 执行出错：{ex.Message}");
    }
}
```

### 7.2 用 Func 实现策略模式

```csharp
Func<int, int, int> add = (a, b) => a + b;
Func<int, int, int> sub = (a, b) => a - b;

// 运行时决定用哪个策略
Func<int, int, int> strategy = add;
int result = strategy(10, 3); // 13
```
