# 委托的概念

---

## 一、什么是委托

委托是一个**装方法的容器**，用来**存储、传递**函数。

> 简单理解：把方法当成变量来用，可以传参、赋值、放进数据结构。

- 委托本质是一个**类**（引用类型）
- 委托对象可以指向**一个或多个方法**（多播委托）
- 底层是一个**方法链表**，通过 `+=` / `-=` 管理

---

## 二、基本语法

```csharp
访问修饰符 delegate 返回值类型 委托名字(参数列表);
```

```csharp
// 示例：声明一个委托类型
delegate void MyDelegate(int a, string b);

// 约束：只有 参数为 (int, string)、返回值为 void 的方法才能绑定到此委托
```

> 注意：委托声明一般写在 `namespace` 中（与类同级），或写在类内部。

---

## 三、委托的约束规则

委托的参数列表和返回值类型**完全决定**了什么方法能被绑定。

```csharp
delegate int MyDelegate(int a, string b);
// 能绑定的方法必须是：int 类型参数、string 类型参数、返回值 int
```

## 四、C# 内置泛型委托

C# 已经封装好了两个泛型委托，**大多数场景不需要自己声明**。

### 4.1 Action（无返回值）

```csharp
Action              // 无参无返回
Action<T>           // 1 个参数
Action<T1, T2>      // 2 个参数（最多 16 个）
Action<T1, T2, T3>  // 3 个参数
...
```

```csharp
Action a1 = () => Console.WriteLine("无参无返回");
Action<int> a2 = (n) => Console.WriteLine(n);
Action<int, string> a3 = (n, s) => Console.WriteLine($"{n} {s}");
```

### 4.2 Func（有返回值）

```csharp
Func<TResult>               // 无参，返回 TResult
Func<T, TResult>            // 1 个参数，返回 TResult
Func<T1, T2, TResult>       // 2 个参数，返回 TResult
Func<T1, T2, T3, TResult>   // 3 个参数，返回 TResult
...
```

> **规律**：`Func` 最后一个泛型参数永远是**返回值**，前面的是**参数**。

```csharp
Func<int> f1 = () => 100;                        // 无参返回 int
Func<string, int> f2 = (s) => s.Length;          // string 参数，返回 int
Func<int, bool, string> f3 = (n, b) => b ? "Y" : "N"; // int+bool 参数，返回 string
```

## 五、委托的三大用途

| 用途 | 说明 | 示例 |
|---|---|---|
| **存储方法** | 把方法当成变量存起来 | `Action skill = Attack;` |
| **传递方法** | 把方法作为参数传入另一个方法 | `Sort(arr, Compare)` |
| **动态调用** | 运行时动态决定调用哪个方法 | AI 随机释放技能 |

## 六、委托与数据结构结合

委托可以装进各种数据结构，实现**行为队列**、**行为映射**等。

```csharp
// 场景：玩家打怪 → 怪物掉血 → 掉落物品 → 玩家捡起
class Player
{
    public void Attack() => Console.WriteLine("玩家攻击！");
    public void PickUp() => Console.WriteLine("玩家捡起物品！");
}
class Monster
{
    public void TakeDamage() => Console.WriteLine("怪物受伤！");
    public void DropItem() => Console.WriteLine("怪物掉落物品！");
}

static void Main()
{
    Player player = new Player();
    Monster monster = new Monster();

    // 用 Queue<Action> 存储行为队列
    Queue<Action> eventQueue = new Queue<Action>();
    eventQueue.Enqueue(player.Attack);
    eventQueue.Enqueue(monster.TakeDamage);
    eventQueue.Enqueue(monster.DropItem);
    eventQueue.Enqueue(player.PickUp);

    // 依次执行
    while (eventQueue.Count > 0)
        eventQueue.Dequeue()();
}
```

> 存放泛型委托时，用 `Action` / `Func` 作为泛型类型传入即可（如 `Queue<Action>`、`Dictionary<string, Action>`）。

## 七、委托 vs 普通方法的区别

| 对比 | 普通方法 | 委托 |
|---|---|---|
| 调用方式 | 写死 | 运行时可替换 |
| 参数传递 | 直接传值 | 可以把"行为"作为参数 |
| 多态 | 靠继承覆写 | 靠委托绑定不同方法 |
| 灵活性 | 编译时确定 | 运行时动态决定 |

> 委托最大的优势：**把"做什么"的决定推迟到运行时**。
