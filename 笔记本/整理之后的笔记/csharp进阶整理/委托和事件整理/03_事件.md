# 01_事件

## 一、问题背景：为什么需要事件？

### ❌ 不用事件：高度耦合
```csharp
class Player
{
    // 玩家必须持有所有怪物
    public List<Monster> monsters = new List<Monster>();

    public void Update()
    {
        // 每一帧都要遍历检查！
        foreach (var monster in monsters)
        {
            if (monster.IsDead && !monster.HasGivenReward)
            {
                Gold += 100;
                monster.HasGivenReward = true;
            }
        }
    }
}
```
- 玩家必须知道所有怪物
- 玩家必须主动监视怪物状态
- 怪物必须被玩家持有
- 加成就、加经验、加任务 → 玩家类越来越臃肿
- 高度耦合，后期根本没法维护

### ✅ 用事件：完全解耦
```csharp
// 怪物死了 → 玩家再调用加金币方法
// 怪物只负责说我死了
// 订阅了这个事件的就触发一下就行了
```
- 玩家不需要持有任何怪物
- 玩家不需要检查、不需要遍历
- 怪物死了自己触发，不用别人管
- 想加成就、加经验、加任务 → 直接订阅事件，不用改玩家、不用改怪物
- 结构干净、解耦、可扩展

## 二、事件的核心概念

### 事件是什么？
**事件 = "发生了一件事，大家来订阅"**

```
怪物死了 → 事件发出
玩家要加金币 → 订阅这个事件
系统要加杀怪数 → 订阅这个事件
成就系统要解锁成就 → 订阅这个事件
```

### 事件 = 中间转接器 / 广播台
| 角色 | 职责 |
|---|---|
| 怪物 | 只管发信号："我死了！" |
| 事件 | 负责转发信号："谁关心，谁来收" |
| 玩家、成就、任务、金币 | 只管听信号："收到，我执行我的逻辑" |

### 生活化比喻
- **不用事件**：你要喊人，必须一个个打电话
- **用事件**：你在群里发一句"我死了"，谁关心这件事，谁就自动收到，不用你一个个通知

## 三、事件和普通委托的核心区别

| 特性 | 普通委托（Action / 自定义 delegate） | 事件（event） |
|---|---|---|
| 外部赋值 | ✅ 可以随便赋值 | ❌ 不能直接赋值 |
| 外部覆盖 | ✅ 可以直接覆盖 = null | ❌ 不能覆盖 |
| 安全性 | 不安全，容易被误改 | 安全、规范、不会被乱改 |
| 总结 | 裸委托，容易被破坏 | 被保护起来的安全委托 |

---

## 四、声明语法

```
访问修饰符 event 委托类型 事件名字;
```

### 示例
```csharp
public delegate void MonsterDeadDelegate();  // 自定义委托

public static event MonsterDeadDelegate OnMonsterDead;  // 声明事件

// 或者直接用 Action（更简洁）
public static event Action OnMonsterDead;
```

**注意**：
- 事件不可以在类的外部赋值
- 事件不可以在类的外部调用
- 只能作为成员在类、接口和结构体里面

## 五、完整结构

### 1. 定义事件管理器（全局静态类）
```csharp
public static class GameEvent
{
    // 方式一：自定义委托
    public delegate void MonsterDeadDelegate();
    public static event MonsterDeadDelegate OnMonsterDead;

    // 方式二：直接用 Action（推荐）
    // public static event Action OnMonsterDead;
}
```

### 2. 订阅事件（谁关心谁订阅）
```csharp
class Player
{
    public int Gold { get; set; }
    public int KillCount { get; set; }

    public Player()
    {
        // 订阅事件：怪物死了 → 执行我的方法
        GameEvent.OnMonsterDead += AddGold;
        GameEvent.OnMonsterDead += AddKillCount;
    }

    void AddGold()
    {
        Gold += 100;
        Console.WriteLine("金币+100");
    }

    void AddKillCount()
    {
        KillCount++;
        Console.WriteLine("杀怪数+1");
    }
}
```

### 3. 触发事件（谁发生谁调用）
```csharp
class Monster
{
    public void Die()
    {
        Console.WriteLine("怪物死亡！");

        // 触发事件（固定写法）
        GameEvent.OnMonsterDead?.Invoke();
    }
}
```

## 六、完整使用示例

### 不用事件 vs 用事件对比

**不用事件**：
```csharp
class Player
{
    public int Gold { get; set; }
    public List<Monster> monsters = new List<Monster>();

    public void Update()
    {
        foreach (var monster in monsters)
        {
            if (monster.IsDead && !monster.HasGivenReward)
            {
                Gold += 100;
                monster.HasGivenReward = true;
            }
        }
    }
}
```

**用事件**：
```csharp
// 全局事件
public static class GameEvent
{
    public static event Action OnMonsterDead;

    public static void TriggerMonsterDead()
    {
        OnMonsterDead?.Invoke();
    }
}

// 怪物自己负责触发
class Monster
{
    public void Die()
    {
        Console.WriteLine("怪物死了，掉落金币");
        GameEvent.TriggerMonsterDead(); // 只负责喊一声：我死了！
    }
}

// 玩家只需要监听，完全不用管怪物
class Player
{
    public int Gold { get; set; }

    public Player()
    {
        GameEvent.OnMonsterDead += AddGold;
    }

    void AddGold()
    {
        Gold += 100;
        Console.WriteLine("玩家获得金币！");
    }
}

// 使用
Player player = new Player();
Monster monster = new Monster();
monster.Die();
// 全程就只有一个 monster.Die();
// 没有别的
```

## 七、修正错误汇总表

| 错误认知 | 正确理解 |
|---|---|
| 事件可以像委托一样在外部随便赋值 | ❌ 事件外部不能赋值，只能 += 和 -= |
| 事件可以直接调用 | ❌ 事件只能通过 Invoke() 触发 |
| 订阅和取消订阅用同一个匿名函数 | ❌ 匿名函数订阅后难以取消，必须用变量存着 |
