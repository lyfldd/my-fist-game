# 02_对比汇总表

## 一、协变与逆变对比表

| 特性 | 协变（Covariant） | 逆变（Contravariant） |
|---|---|---|
| 关键字 | out | in |
| 适用场景 | 返回值 / 只读接口 | 方法参数 / 输入接口 |
| 转换方向 | 子类泛型 → 父类泛型 | 父类泛型 → 子类泛型 |
| 游戏开发中的作用 | 通用刷怪、遍历、AOE 技能 | 通用受伤、事件回调、AI 行为 |
| 形象比喻 | 生产者：子类工厂可以当父类工厂用 | 消费者：父类处理器可以当子类处理器用 |
| 内存安全 | 只往外拿（只读），安全 | 只往里塞（只写），安全 |

---

## 二、口诀汇总

```
协变 out → 返回值用 → 子类泛型 → 父类泛型
逆变 in → 参数用 → 父类泛型 → 子类泛型
```

## 三、IComparable vs IComparer 对比表

| 特性 | IComparable<T> | IComparer<T> |
|---|---|---|
| 思想 | "自己和自己比" | "第三方比较器" |
| 写在哪里 | 类内部（自身） | 独立的类里 |
| 灵活性 | 低（类定义了就很难改） | 高（可以定义多种规则） |
| 调用方式 | `list.Sort()` | `list.Sort(new XxxComparer())` |
| 适用场景 | 类的固有属性（默认排序） | 复杂、可复用的排序规则 |
| 设计模式 | 模板模式 | 策略模式 |

---

## 四、匿名函数 vs 普通方法选择表

| 场景 | 推荐 | 原因 |
|---|---|---|
| 一次性回调、延迟执行 | 匿名函数 | 逻辑简单，不需要复用 |
| UI 按钮点击（单次） | 匿名函数 | 简单，不需要生命周期管理 |
| 事件订阅（长期） | 普通方法 | 需要能够取消订阅，避免内存泄漏 |
| 循环中创建委托 | 普通方法（或局部变量中转） | 避免闭包捕获循环变量 BUG |
| 逻辑复杂、需要调试 | 普通方法 | 匿名函数难调试，难复用 |
| 高频创建的对象 | 普通方法 | 避免 GC 压力 |
