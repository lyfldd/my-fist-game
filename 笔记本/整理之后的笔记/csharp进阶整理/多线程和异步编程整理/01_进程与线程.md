# 01_进程与线程

## 一、进程（Process）

### 什么是进程？
进程就是一个**正在运行的程序**。

更严谨的定义：
> **进程是操作系统分配资源的基本单位**

你双击打开一个 `.exe`（比如 QQ、浏览器、Unity 游戏、控制台程序），操作系统就会给它划一块独立内存、分配 PID、权限、句柄……这一整套"正在运行的程序实例"，就叫一个进程。

### 进程最关键的特点：独立！独立！独立！
- A 进程**不能**随便访问 B 进程的内存
- 一个程序崩了，通常不会拖垮另一个

### 每个进程有自己的
| 资源 | 说明 |
|---|---|
| 虚拟内存空间 | 独立地址范围 |
| 文件句柄 | 打开的文件 |
| 网络连接 | Socket 连接 |
| 环境变量 | 系统环境变量 |
| 安全令牌 | 权限标识 |

### 进程与程序的区别
| 概念 | 说明 |
|---|---|
| 程序 | 硬盘上的一堆代码文件（.exe、.dll） |
| 进程 | 程序被加载到内存、正在运行的实例 |

### 一个进程至少包含
- 独立的虚拟地址空间
- 至少一个线程（主线程）
- 资源句柄（文件、窗口、网络等）
- 安全上下文
- 环境变量、命令行参数
- 堆、全局数据、代码段

### 进程间通信（IPC）
两个进程默认不能直接互相访问变量，想通信必须用专门方式：
| 方式 | 说明 |
|---|---|
| 管道（Pipe） | 匿名管道 / 命名管道 |
| 共享内存（SharedMemory） | 多进程读写同一块内存 |
| Socket | 网络通信，本机也可用 |
| 文件 | 读写共享文件 |
| 消息队列 | 操作系统提供的队列 |

## 二、线程（Thread）

### 什么是线程？
> **线程是操作系统能够进行运算调度的最小单位**

- 被包含在**进程**之中
- 一条线程指的是进程中的一个单一顺序的**控制流**
- 一个进程可以并发多个线程
- 一个进程里**至少有一个线程**，这个叫**主线程**
- **进程是容器，线程是干活的**

### 线程共享 vs 线程独有

**所有线程共享这个进程的资源**：
| 共享资源 | 说明 |
|---|---|
| 堆内存（Heap） | 动态分配的对象 |
| 全局变量、静态变量 | 类级别的变量 |
| 打开的文件 | 文件句柄 |
| 网络连接 | Socket 连接 |
| 代码段 | 编译后的机器码 |

**但每个线程自己独有的东西**：
| 独有资源 | 说明 |
|---|---|
| 栈内存（Stack） | 每个线程一套栈，存局部变量 |
| 程序计数器 | 当前执行到哪一行代码 |
| 线程局部状态 | CPU 寄存器值 |

### 为什么要有多线程？

举个最常见的场景：打开一个游戏

| 线程 | 职责 |
|---|---|
| 线程 A | 渲染画面（不卡 UI） |
| 线程 B | 加载资源、网络通信 |
| 线程 C | 播放音乐 |

**如果只有一个线程**：
- 加载资源时画面就会**卡死不动**
- 音乐也会断

### 多线程的核心目的
| 目的 | 说明 |
|---|---|
| 让程序同时做多件事 | 真正并行执行 |
| 不阻塞界面 | UI 线程不卡死 |
| 利用多核 CPU | 真正并行计算 |
| I/O 等待时不浪费 CPU | 等待期间 CPU 可以干别的 |

## 三、进程 vs 线程对比

| 特性 | 进程 | 线程 |
|---|---|---|
| 单位 | 资源分配的基本单位 | 调度执行的基本单位 |
| 内存 | 独立虚拟地址空间 | 共享进程的堆和全局数据 |
| 通信 | 必须用 IPC | 可以直接读写共享变量 |
| 创建开销 | 大（需要分配独立资源） | 小（共享进程资源） |
| 稳定性 | 高（隔离，一个崩了不影响另一个） | 低（共享资源，可能相互影响） |
| 切换开销 | 大（切换整个地址空间） | 小（只切换寄存器、栈） |
| 关系 | 包含关系：进程 ≥ 1个线程 | 线程必须在进程内 |

## 四、补充知识点：线程的调度

### 操作系统如何调度线程？
- 操作系统有一个**调度器（Scheduler）**
- 把 CPU 时间切成很多小片（时间片，通常 10-20ms）
- 轮流让不同线程执行
- 切换线程时保存/恢复上下文（寄存器、栈指针等）

### 线程优先级
```csharp
thread.Priority = ThreadPriority.Highest;  // 最高
thread.Priority = ThreadPriority.AboveNormal;
thread.Priority = ThreadPriority.Normal;     // 默认
thread.Priority = ThreadPriority.BelowNormal;
thread.Priority = ThreadPriority.Lowest;    // 最低
```

### 前台线程 vs 后台线程
| 类型 | 说明 |
|---|---|
| **前台线程**（默认） | 主线程关闭，进程会等它跑完才退出 |
| **后台线程** | 主线程关闭，它直接被杀死 |

**游戏理解**：
- 前台线程 = 重要任务（必须做完）
- 后台线程 = 次要任务（游戏关了就停）

```csharp
thread.IsBackground = true;  // 设为后台线程
```

## 五、补充知识点：线程的四种状态

```
新建状态 → 运行状态 → 阻塞/睡眠状态 → 终止状态
   ↓            ↓              ↓              ↓
 new Thread()  Start()     Sleep/Join/锁     方法跑完
```

| 状态 | 说明 |
|---|---|
| **新建（Unstarted）** | `new Thread()`，还没启动 |
| **运行（Running）** | `Start()` 后，线程正在执行 |
| **阻塞/睡眠（Blocked/Waiting）** | `Sleep()`、`Join()`、等待锁 |
| **终止（Stopped）** | 方法执行完毕，线程死亡 |

## 六、修正错误汇总表

| 错误认知 | 正确理解 |
|---|---|
| 进程和线程是一回事 | ❌ 进程是容器，线程是里面干活的 |
| 线程可以直接访问其他进程的变量 | ❌ 进程间内存隔离，必须用 IPC |
| 多线程一定比单线程快 | ❌ 线程切换有开销，小任务单线程可能更快 |
| 线程越多越好 | ❌ 线程过多会导致上下文切换频繁，性能反而下降 |
| 后台线程不重要 | ❌ 但如果它是唯一的前台线程结束了，后台线程也会被强制终止 |
