# 01_匿名函数概念

## 一、什么是匿名函数？

### 定义
匿名函数 = 没有名字、临时写一段代码、用完就丢的函数

**相当于塑料袋**：只是用来临时装东西，用完就丢

> ⚠️ 脱离了委托和事件，不可以使用

### 对比：普通函数 vs 匿名函数

```csharp
// 一般的函数
void AddGold()
{
    Gold += 100;
}

// 匿名函数
() => { Gold += 100; }
```

### 语法格式
```csharp
// 无参
() => { Console.WriteLine("怪物死了"); }

// 带 int 参数
(int a) => { Console.WriteLine(a); }

// 带 string 参数
(string s) => { Console.WriteLine(s); }

// 单表达式可以省略大括号
x => x * 2;
```

### 老式匿名方法（了解即可）
```csharp
delegate()
{
    Console.WriteLine("老式写法");
};
```

---

## 二、匿名函数的特点

| 特点 | 说明 |
|---|---|
| 没有名字 | 不用专门定义方法名 |
| 临时使用 | 用完就丢，不占类的成员方法 |
| 可以直接赋值给委托 | 直接赋值给委托变量或事件 |
| 可以访问外部变量 | 比如 player、gold 等外部变量（闭包） |

---

## 三、匿名函数的主要用途

### 1. 给委托赋值
```csharp
// 原来的写法（要单独写函数）
GameEvent.OnMonsterDead += AddGold;

void AddGold()
{
    player.Gold += 100;
}

// 匿名函数写法（不用单独写函数）
GameEvent.OnMonsterDead += () =>
{
    player.Gold += 100;
    Console.WriteLine("金币+100");
};
```

### 2. 事件订阅
```csharp
button.onClick.AddListener(() => {
    打开商店();
});
```

---

## 四、为什么用匿名函数？

| 好处 | 说明 |
|---|---|
| 不用在类里写一堆零散的小方法 | 代码更紧凑 |
| 代码写在使用的地方，更直观 | 逻辑和调用处在一起 |
| 不同参数的方法可以包成统一格式 | 用委托统一接口 |
| 事件、回调、延迟执行必备 | 一次性逻辑不需要单独定义方法 |

---

## 五、匿名函数的寿命

> 匿名函数的寿命 = 引用它的委托 / 事件的寿命

**匿名函数订阅事件容易常驻内存，且难以取消**

### ❌ 匿名函数很难退订
```csharp
// 先把匿名函数交给一个变量
Action myAction = () => {
    player.Gold += 100;
};

// 订阅
GameEvent.OnMonsterDead += myAction;

// 退订
GameEvent.OnMonsterDead -= myAction;
```

### ⚠️ 使用原则
| 场景 | 用什么 |
|---|---|
| 临时执行一次 | ✅ 随便用匿名 |
| 长期事件（怪物死亡、玩家受伤、成就） | ❌ 一定用普通方法！ |

**否则**：内存泄漏、对象无法释放、越玩越卡、包体越来越大

## 六、闭包（Closure）

### 什么是闭包？
> 闭包的核心能力：即使外部函数已经执行完毕，被捕获的变量依然会被保留，供内部函数后续使用。

简单说：**函数把外部变量"抓"进自己的作用域，让变量跟着函数一起"活"下去**

### 正常的变量生命周期
```csharp
void Test()
{
    int a = 5; // 局部变量，只在Test方法内有效
}
// Test执行完，a就被销毁了，外面再也访问不到
```

### 闭包的变量生命周期
当匿名函数捕获了局部变量 a，并且把这个匿名函数返回出去时：
- 匿名函数被外部持有，不会被销毁
- 被捕获的变量 a，会被强制延长生命周期，跟着匿名函数一起存活
- 即使 Test 方法执行完，a 依然存在，匿名函数随时可以访问它

> ⚠️ 闭包捕获的值是那个父函数的最后的值

### 基础结构
```csharp
// 外层函数：接收参数，创建闭包
返回委托类型 外层函数(参数)
{
    // 返回一个匿名函数，捕获外层的参数/局部变量
    return (内部参数) => {
        // 内部逻辑：使用捕获的外部变量 + 内部参数
    };
}
```

### 具体例子
```csharp
// 外层函数：传入a，返回一个"乘以a"的函数
static Func<int, int> CreateMultiplier(int a)
{
    // 匿名函数：捕获了a，接收b，返回a*b
    return (b) => a * b;
}

// 使用
static void Main()
{
    // 1. 调用外层函数，传入a=5，得到闭包实例multiplyBy5
    // 此时a=5被multiplyBy5捕获，即使CreateMultiplier执行完，a依然存活
    Func<int, int> multiplyBy5 = CreateMultiplier(5);
    
    // 2. 执行闭包，传入b=3，计算5*3=15
    Console.WriteLine(multiplyBy5(3)); // 输出15
    Console.WriteLine(multiplyBy5(10)); // 输出50

    // 3. 再创建一个新的闭包，a=10，和之前的a=5完全独立
    Func<int, int> multiplyBy10 = CreateMultiplier(10);
    Console.WriteLine(multiplyBy10(3)); // 输出30
}
```

## 七、闭包的具体使用场景

### 场景1：给怪物生成不同的伤害计算函数
```csharp
// 通用伤害生成器：传入暴击倍率，返回对应的伤害计算函数
static Func<int, int> CreateDamageCalculator(float critRate)
{
    // 闭包：捕获了critRate
    return (baseDamage) => (int)(baseDamage * critRate);
}

// 使用
var normalDamage = CreateDamageCalculator(1.0f);    // 普通怪物：1.0倍伤害
var eliteDamage = CreateDamageCalculator(1.5f);      // 精英怪物：1.5倍伤害
var bossDamage = CreateDamageCalculator(2.0f);       // BOSS：2.0倍伤害

Console.WriteLine(normalDamage(100)); // 100
Console.WriteLine(eliteDamage(100));  // 150
Console.WriteLine(bossDamage(100));   // 200
```

### 场景2：延迟执行 + 状态保留（动画 / 计时器）
```csharp
// 怪物死亡逻辑
static void MonsterDie(int monsterId, string monsterName)
{
    // 闭包：捕获了monsterId和monsterName
    Delay(1.0f, () => {
        Console.WriteLine($"怪物{monsterId}【{monsterName}】死亡，掉落金币");
    });
}
```

### 场景3：事件订阅 + 对象状态保留
```csharp
class Player
{
    public string Name { get; set; }
    public int Gold { get; set; }

    public Player(string name)
    {
        Name = name;
        // 闭包：捕获了当前Player实例this
        GameEvent.OnMonsterDead += () => {
            Gold += 100;
            Console.WriteLine($"{Name}获得100金币，当前金币：{Gold}");
        };
    }
}
```

### 场景4：函数柯里化（把多参数函数拆成单参数）
```csharp
// 柯里化：用闭包拆成两个单参数函数
Func<int, Func<int, int>> CurriedMultiply = a => b => a * b;

// 使用
var multiplyBy5 = CurriedMultiply(5); // 先传a=5，得到闭包
Console.WriteLine(multiplyBy5(3)); // 再传b=3，得到15
```

### 闭包的优点
| 优点 | 说明 |
|---|---|
| 代码简洁 | 不用手动传参，直接捕获外部变量，减少冗余代码 |
| 状态封装 | 给函数绑定私有状态，实现"带状态的函数" |
| 灵活扩展 | 用一个通用方法生成 N 个不同逻辑的函数，复用性拉满 |
| 解耦 | 延迟执行、事件订阅时，不用传递大量参数，逻辑更清晰 |
| 函数式编程基础 | 柯里化、高阶函数的核心依赖就是闭包 |

## 八、闭包的致命坑（游戏开发必踩）

### ❌ 坑1：循环中捕获循环变量（最经典的闭包 BUG）
```csharp
// 错误写法：循环里用闭包捕获i
Action[] actions = new Action[3];
for (int i = 0; i < 3; i++)
{
    // 闭包捕获的是循环变量i的引用，不是值
    actions[i] = () => Console.WriteLine(i);
}

// 执行所有闭包
foreach (var action in actions)
{
    action();
}
// 输出：3 3 3（不是0 1 2！）
```

**解决方法**：用局部变量中转
```csharp
Action[] actions = new Action[3];
for (int i = 0; i < 3; i++)
{
    int local = i;  // 用局部变量中转
    actions[local] = () => Console.WriteLine(local);
}
// 输出：0 1 2 ✅
```

### ❌ 坑2：内存泄漏
```csharp
// 错误写法：用闭包订阅静态事件
static class GameEvent
{
    public static Action OnMonsterDead;
}

class Monster
{
    public int Id { get; set; }

    public Monster(int id)
    {
        Id = id;
        // 闭包捕获了当前Monster实例this
        GameEvent.OnMonsterDead += () => {
            Console.WriteLine($"怪物{Id}死亡");
        };
    }
}

// 问题：
// 1. 每次创建Monster，都会给静态事件加一个闭包
// 2. 静态事件永远不会被回收，闭包永远活着
// 3. 闭包捕获了Monster实例this，导致Monster永远不会被GC回收
// 4. 怪物越多，内存泄漏越严重，游戏越玩越卡
```

### ❌ 坑3：闭包捕获引用类型，导致意外修改
```csharp
// 闭包捕获引用类型变量
List<int> list = new List<int> { 1, 2, 3 };
Action action = () => {
    foreach (var num in list)
    {
        Console.WriteLine(num);
    }
};

// 修改list
list.Add(4);

// 执行闭包：输出1 2 3 4（不是1 2 3！）
action();
```

## 九、闭包的使用规范（游戏开发实战守则）

### ✅ 适合用闭包的场景
| 场景 | 说明 |
|---|---|
| 临时回调 | 延迟执行、动画结束、协程结束等一次性逻辑 |
| 简单状态封装 | 给函数绑定少量状态，比如不同倍率的伤害计算 |
| 参数包装 | 把多参数方法包装成无参 Action，放进队列 / 事件 |
| UI 按钮点击 | 单次点击，无生命周期管理需求 |

### ❌ 绝对不要用闭包的场景
| 场景 | 原因 |
|---|---|
| 订阅长期静态事件 | 会导致内存泄漏，必须用有名方法 |
| 循环中捕获循环变量 | 会出现经典 BUG，必须用局部变量中转 |
| 复杂业务逻辑 | 逻辑多行、需要复用、需要调试，必须用有名方法 |
| 高频创建的对象 | 比如怪物、子弹，频繁创建闭包会增加 GC 压力 |

---

## 十、匿名函数和普通方法的选择决策表

| 场景 | 推荐 | 原因 |
|---|---|---|
| 一次性回调、延迟执行 | 匿名函数 | 逻辑简单，不需要复用 |
| UI 按钮点击（单次） | 匿名函数 | 简单，不需要生命周期管理 |
| 事件订阅（长期） | 普通方法 | 需要能够取消订阅，避免内存泄漏 |
| 循环中创建委托 | 普通方法（或局部变量中转） | 避免闭包捕获循环变量 BUG |
| 逻辑复杂、需要调试 | 普通方法 | 匿名函数难调试，难复用 |
| 高频创建的对象 | 普通方法 | 避免 GC 压力 |

---

## 十一、修正错误汇总表

| 错误认知 | 正确理解 |
|---|---|
| 匿名函数订阅后可以随便取消 | ❌ 匿名函数难以取消，必须用变量存着 |
| 闭包捕获的是变量的值快照 | ⚠️ 捕获的是引用，闭包执行时取最新值 |
| 循环中用闭包捕获 i 会得到 0,1,2 | ❌ 闭包捕获的是 i 的引用，执行时 i 已经是 3 |
| 闭包订阅静态事件没问题 | ❌ 会导致内存泄漏，静态事件永远不会被 GC |
