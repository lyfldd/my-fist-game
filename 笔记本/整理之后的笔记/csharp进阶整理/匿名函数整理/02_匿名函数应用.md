# 02_匿名函数应用

## 一、函数当参数（委托作为参数传递）

### 核心概念
> 回调 = 我现在先不做，等一会儿做完某件事，再回头调用你给我的函数。

**"回头调用" → 所以叫回调**

### 生活例子
```
你点单（调用函数）
店员去做（执行耗时操作）
你不留下来等，去打游戏（不阻塞）
奶茶做好了，店员喊你："你的好了！"（回调）
店员喊你 = 回调
你给店员的"你的号码 / 名字" = 回调函数
```

### 为什么游戏开发必须用"函数当参数"？

| 好处 | 说明 |
|---|---|
| 解耦！ | 怪物不知道玩家是谁，技能不知道效果是什么，按钮不知道点击后干嘛 |
| 代码复用！ | 一个 Delay 函数能延迟消失、延迟音效、延迟打开宝箱 |
| 灵活扩展！ | 加新技能、新装备、新效果，不用改老代码 |
| 逻辑清晰！ | 代码写在它该出现的地方 |

## 二、函数当参数的应用场景

### 场景1：延迟执行 / 计时器
```csharp
void Delay(float time, Action onComplete)
{
    // 等待时间到
    onComplete(); // 执行你传进来的逻辑
}

// 用法
Delay(1, () => 怪物消失());
Delay(2, () => 播放音效());
Delay(3, () => 打开宝箱());
```

### 场景2：技能系统（同一个技能，不同效果）
```csharp
void UseSkill(Action onHit)
{
    播放动画();
    判定范围();
    onHit(); // 执行传进来的效果
}

// 用法
UseSkill(() => 造成燃烧());
UseSkill(() => 造成眩晕());
UseSkill(() => 召唤小怪());
```

### 场景3：怪物 / 角色死亡回调
```csharp
void TakeDamage(int dmg, Action onDead)
{
    hp -= dmg;
    if(hp <= 0) onDead();
}

// 用法
monster.TakeDamage(50, () => {
    掉落金币();
    玩家加经验();
    解锁成就();
});
```

### 场景4：UI 按钮点击事件
```csharp
button.onClick.AddListener(() => {
    打开商店();
});
```

### 场景5：任务 / 事件系统
```csharp
void CompleteQuest(Action onReward)
{
    任务完成();
    onReward();
}

// 用法
CompleteQuest(() => 获得装备());
```

### 场景6：攻击判定（不同武器不同效果）
```csharp
void Attack(Action<Enemy> effect)
{
    effect(敌人);
}

// 用法
Attack((e) => e.燃烧());
Attack((e) => e.中毒());
```

### 场景7：对话系统（不同选项不同逻辑）
```csharp
void ShowDialog(string text, Action onYes, Action onNo)
{
    if(点击YES) onYes();
    if(点击NO) onNo();
}

// 用法
ShowDialog("要接受任务吗？",
    () => { 接受任务(); },
    () => { 拒绝任务(); }
);
```

### 场景8：动画结束回调
```csharp
void PlayAnim(string name, Action onFinish)
{
    播放动画();
    动画结束时 → onFinish();
}

// 用法
PlayAnim("攻击", () => {
    造成伤害();
});
```

## 三、匿名函数包装异构方法

> 当有多个不同参数的方法时，用匿名函数全部变成无参 Action

```csharp
// int 方法
void AddGold(int amount) { }

// string 方法
void ShowMsg(string msg) { }

Queue<Action> q = new Queue<Action>();

q.Enqueue(() => AddGold(100));
q.Enqueue(() => ShowMsg("首次击杀"));
```

## 四、List<T>.Sort() 排序详解

### 基础用法

#### 1. 基础类型自动排序
```csharp
List<int> list = new List<int>();
Random random = new Random();

for (int i = 0; i < 10; i++)
{
    list.Add(random.Next(100));
}

// 默认从小到大排序
list.Sort();

// 从大到小
list.Sort();
list.Reverse();

// 或者用 Lambda
list.Sort((a, b) => b.CompareTo(a));  // 降序
```

#### 2. Lambda 自定义排序
```csharp
// 建立怪物类
class Monster
{
    public string Name { get; set; }
    public int Level { get; set; }
    public int Hp { get; set; }
}

// 添加对象
List<Monster> monsters = new List<Monster>();
monsters.Add(new Monster("lyf1", 1, 100));
monsters.Add(new Monster("lyf2", 2, 200));
monsters.Add(new Monster("lyf3", 3, 300));

// Sort 里面实现逻辑
monsters.Sort((a, b) => {
    int result = a.Level.CompareTo(b.Level);
    if (result == 0)
    {
        return a.Hp.CompareTo(b.Hp);  // 等级一样就比血量
    }
    return result;
});
```

### Lambda 排序规则
```
(a, b) => a.字段.CompareTo(b.字段)   → 升序（小→大）
(a, b) => b.字段.CompareTo(a.字段)   → 降序（大→小）
```

### CompareTo 返回值规则
| 返回值 | 含义 |
|---|---|
| 返回负数 | a 在前（小在前） |
| 返回 0 | 不动，谁也不用排谁前面 |
| 返回正数 | b 在前（大在后） |

### Sort 内置排序算法
| 元素数量 | 算法 |
|---|---|
| 元素 ≤ 16 个 | 插入排序 |
| 元素多，递归不深 | 快速排序 |
| 递归太深，怕栈溢出 | 堆排序 |

## 五、IComparable<T> 接口排序

### 实现方式
```csharp
class Item : IComparable<Item>
{
    public string Name;
    public int Price;

    public Item(string name, int price)
    {
        Name = name;
        Price = price;
    }

    public int CompareTo(Item? other)
    {
        return this.Price.CompareTo(other.Price);
    }
}

// 使用
List<Item> items = new List<Item>();
items.Add(new Item("apple", 10));
items.Add(new Item("banana", 5));
items.Add(new Item("orange", 15));

items.Sort();  // 直接调用，不用传任何东西
```

## 六、四种 Sort 重载对比

| 方法 | 核心机制 | 谁写比较逻辑 | 适用场景 |
|---|---|---|---|
| `Sort()` | IComparable<T> | 写在类内部 | 类的默认排序规则 |
| `Sort(IComparer<T>)` | IComparer<T> 接口 | 写在独立的类里 | 复杂、可复用的排序规则 |
| `Sort(Comparison<T>)` | Comparison<T> 委托 / Lambda | 写在调用处 | 临时、一次性的排序规则 |
| `Sort(index, count, ...)` | 接口 / 委托 + 范围 | 同上 | 只对列表的一部分排序 |

### Sort() 重载详解

#### 1️⃣ public void Sort()
```csharp
// 要求 T 必须实现 IComparable<T> 接口
list.Sort();
```

#### 2️⃣ public void Sort(IComparer<T>? comparer)
```csharp
// 1. 先写一个实现了 IComparer<T> 的类
public class ItemPriceComparer : IComparer<Item>
{
    public int Compare(Item a, Item b)
    {
        return a.Price.CompareTo(b.Price);
    }
}

// 2. 把它传进去
list.Sort(new ItemPriceComparer());
```

**IComparable<T> vs IComparer<T>**：
| 接口 | 思想 | 说明 |
|---|---|---|
| IComparable<T> | "自己和自己比" | 写在类里，定义默认规则 |
| IComparer<T> | "第三方比较器" | 写在外面，可以定义多种规则 |

#### 3️⃣ public void Sort(Comparison<T> comparison)
```csharp
// 直接传 Lambda 委托
list.Sort((a, b) => a.Price.CompareTo(b.Price));
```

#### 4️⃣ public void Sort(int index, int count, IComparer<T>? comparer)
```csharp
// index：从哪个索引开始排
// count：排多少个元素
list.Sort(0, 5, new ItemPriceComparer());  // 只排前5个
```

## 七、匿名函数排序 vs IComparable 排序

| 特性 | 匿名函数（Sort + Lambda） | IComparable（Sort 无参数） |
|---|---|---|
| 规则写在哪 | 调用 Sort 的地方（外部） | 类内部（自身） |
| 灵活性 | 极高（可随时换规则） | 低（类定义了就很难改） |
| 调用方式 | `list.Sort((a,b) => ...)` | `list.Sort()` |
| 适用场景 | 游戏中临时排序（按距离、按血量） | 类的固有属性（默认按 ID、按价格） |
| 设计思想 | 策略模式（用什么策略传什么） | 模板模式（自身自带模板） |

**选择原则**：
- 想让类天生就知道怎么排序自己 → 实现 IComparable<T>
- 只想在某个具体场景下排序 → 用匿名函数

## 八、修正错误汇总表

| 错误认知 | 正确理解 |
|---|---|
| Sort 默认就是从大到小 | ❌ 默认从小到大 |
| CompareTo 返回的是 bool | ❌ 返回 -1/0/1 三个值 |
| IComparable 和 IComparer 是一样的 | ❌ IComparable 是"自己比自己"，IComparer 是"第三方比较器" |
| Lambda 排序只能单字段排序 | ❌ 可以嵌套 if 实现多字段排序 |

## 九、委托当参数的游戏开发实战技巧

**技巧：统一接口 + 委托传递 = 高度解耦**

```csharp
// 不好的设计：每个效果单独写方法
void FireSkill() { 造成燃烧(); 播放动画(); }
void IceSkill() { 造成冰冻(); 播放动画(); }
void ThunderSkill() { 造成眩晕(); 播放动画(); }

// 好的设计：统一接口 + 委托传递
void UseSkill(Action effect)
{
    播放动画();
    判定范围();
    effect();
}

// 调用时传入具体效果
UseSkill(() => 造成燃烧());
UseSkill(() => 造成眩晕());
```

**好处**：技能系统只需要写一次，加新效果不用改技能代码
