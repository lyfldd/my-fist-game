# 反射（Reflection）—— Step 1：理解是什么

---

## 1.1 一句话理解

> **反射 = 程序在运行时，"照镜子"看清自己的结构，并动手操作它。**

写代码时，编译器知道 `Monster` 类有 `Name` 字段、有 `Attack()` 方法。但程序运行后，普通代码不知道这些。**反射就是让程序在运行时重新"看清"这些结构，然后动态调用它们。**

---

## 1.2 通俗类比

想象走进一个**漆黑的房间**，里面有一排上锁的柜子（类）。

| 普通方式 | 反射方式 |
|---|---|
| 你**早就知道**柜子里有什么，拿到钥匙就能开 | 你手里**没有钥匙**，但有**万能手电筒**（Type），照一照，立刻知道里面有什么（字段、属性、方法），然后临时配钥匙打开它 |

---

## 1.3 核心作用

```
反射能做什么（运行时动态）：
├── 🔍 看清一个类型有什么（字段 / 属性 / 方法 / 构造函数 / 特性）
├── 🏗️ 创建一个类型的实例（不需要提前写 new 类名()）
├── ⚙️ 调用一个方法（按名字找，运行时才决定调哪个）
├── ✏️ 读写一个字段/属性的值（甚至能读写 private 私有成员）
└── 📦 加载外部程序集（.dll），发现其中的类型并使用
```

---

## 1.4 Unity 游戏开发中的真实意义

> **Unity 本身就是一个大型反射系统，你每时每刻都在用它。**

| 你在 Unity 里做的事 | 背后反射在干什么 |
|---|---|
| Inspector 面板显示脚本字段 | Unity 扫描你的类，找到所有 `[SerializeField]` 字段并展示 |
| `GetComponent<T>()` | 泛型背后用类型信息查找组件 |
| `SendMessage("OnClick")` | 用反射按字符串找到方法并调用 |
| 保存预制体到文件 | Unity 遍历所有可序列化字段，写入文件 |
| 自定义编辑器工具窗口 | 用反射读取 GameObject 上挂的所有组件信息 |
| 热更新（ILRuntime等） | 运行时加载外部 .dll 代码并执行其中的类型 |

**简单说：** Unity 把反射封装得很好，让你感觉不到它的存在，但底层全靠它。想做自定义编辑器、写热更新系统、理解 Unity 序列化原理——反射是必须掌握的技能。

---

## 1.5 核心类一览

反射的核心全部在 `System.Reflection` 命名空间：

| 类名 | 作用 |
|---|---|
| **`Type`** | 反射的**核心入口**，代表一个类型的元数据 |
| `Assembly` | 代表一个程序集（.dll / .exe） |
| `FieldInfo` | 表示一个字段 |
| `PropertyInfo` | 表示一个属性 |
| `MethodInfo` | 表示一个方法 |
| `ConstructorInfo` | 表示一个构造函数 |

其中 **`Type`** 是重中之重——几乎所有反射操作的第一步都是拿到 `Type` 对象。
