# 反射（Reflection）—— Step 2：理解怎么用

---

## 2.1 反射核心命名空间

```csharp
using System.Reflection; // 几乎所有反射类都在这里
```

---

## 2.2 获取 Type 的三种方式

`Type` 是反射的入口，第一步永远是拿到它。

```csharp
// 方式一：已有一个实例对象
Monster m = new Monster();
Type t1 = m.GetType();

// 方式二：编译时已知类型（最常用）
Type t2 = typeof(Monster);

// 方式三：只知道字符串名字（完全运行时动态）
Type t3 = Type.GetType("MyGame.Monster"); // 必须写完整命名空间！
```

> ⚠️ **`Type.GetType("Monster")` 找不到**，必须是 `"命名空间.Monster"` 完整写法。

---

## 2.3 查询成员的 API 命名规律

**核心规律：加 s 返回数组（拿全部），不加 s 返回单个对象（按名字精准拿）。**

```csharp
// 加 s → 返回数组 → 拿全部
FieldInfo[] fields     = t.GetFields();          // 所有字段
MethodInfo[] methods   = t.GetMethods();          // 所有方法
PropertyInfo[] props   = t.GetProperties();       // 所有属性
ConstructorInfo[] ctors = t.GetConstructors();  // 所有构造函数

// 不加 s → 返回单个对象 → 按名字精准拿
FieldInfo hpField    = t.GetField("HP");             // 单个字段
MethodInfo attack    = t.GetMethod("Attack");        // 单个方法
PropertyInfo nameProp = t.GetProperty("Name");       // 单个属性
ConstructorInfo ctor  = t.GetConstructor(Type.EmptyTypes); // 单个构造函数
```

**每个 Info 对象就是一个"容器"，装着某个成员的描述信息（名字、类型、偏移量等）。**

---

## 2.4 用 Type 能查什么？

```csharp
Type t = typeof(Monster);

// === 查询基本信息 ===
t.Name;        // "Monster"（类名）
t.FullName;    // "MyGame.Monster"（含命名空间）
t.Namespace;   // "MyGame"
t.IsClass;     // True
t.BaseType;    // System.Object（父类）

// === 查询成员（加 s → 数组） ===
ConstructorInfo[] ctors = t.GetConstructors();
MethodInfo[] methods    = t.GetMethods();
PropertyInfo[] props    = t.GetProperties();
FieldInfo[] fields     = t.GetFields();

// === 查询单个成员（不加 s → 对象） ===
FieldInfo hpField    = t.GetField("HP");
MethodInfo attack    = t.GetMethod("Attack");
PropertyInfo nameProp = t.GetProperty("Name");
```

---

## 2.5 BindingFlags：访问非公开成员

**默认只返回公开成员**，要访问 private / protected / static 成员，必须加 `BindingFlags`：

```csharp
FieldInfo[] f1 = t.GetFields(); // 只返回公开字段

// 加 BindingFlags 扩大范围
FieldInfo[] f2 = t.GetFields(BindingFlags.Instance | BindingFlags.NonPublic | BindingFlags.Public); // 公开+私有实例字段
FieldInfo[] f3 = t.GetFields(BindingFlags.Static | BindingFlags.Public); // 静态公开字段
```

| 常用标志 | 含义 |
|---|---|
| `BindingFlags.Public` | 公开成员 |
| `BindingFlags.NonPublic` | 非公开成员（private / protected） |
| `BindingFlags.Instance` | 实例成员 |
| `BindingFlags.Static` | 静态成员 |
| `BindingFlags.DeclaredOnly` | 只看本类声明的，不含继承来的 |

> ⚠️ **不传 BindingFlags 默认只查公开实例成员**，其他都要显式声明。

## 2.6 拿到 Info 对象后怎么用？

> 这是最关键的一步：反射拿到 `FieldInfo`、`MethodInfo` 等对象后，每个都有自己的操作方法。

### 字段（FieldInfo）：读写值

```csharp
public class Monster
{
    public string Name = "史莱姆";
    public int HP = 100;
}
```

```csharp
Type t = typeof(Monster);
object monster = Activator.CreateInstance(t);

// 读取字段值
FieldInfo nameField = t.GetField("Name");
FieldInfo hpField = t.GetField("HP");
string name = (string)nameField.GetValue(monster); // string 是引用类型，直接返回
int hp = (int)hpField.GetValue(monster);            // int 是值类型，有装箱拆箱

// 写入字段值
nameField.SetValue(monster, "火焰史莱姆");
hpField.SetValue(monster, 200);
```

### 方法（MethodInfo）：调用

```csharp
public class Monster
{
    public void Attack(int damage)
    {
        Console.WriteLine($"造成 {damage} 点伤害！");
    }

    public void CastSkill(string name, int mpCost)
    {
        Console.WriteLine($"释放 {name}，消耗 {mpCost} MP！");
    }
}
```

```csharp
object monster = Activator.CreateInstance(typeof(Monster));

// 调用无参方法
MethodInfo die = t.GetMethod("Die");
die.Invoke(monster, null); // 无参传 null

// 调用有参方法：参数放在 object[] 数组里
MethodInfo attack = t.GetMethod("Attack");
attack.Invoke(monster, new object[] { 50 }); // int 会装箱

// 调用多参方法
MethodInfo cast = t.GetMethod("CastSkill");
cast.Invoke(monster, new object[] { "火球术", 30 });
```

### 构造函数（ConstructorInfo）：创建实例

```csharp
public class Monster
{
    public string Name;
    public int HP;

    public Monster(string name, int hp)
    {
        Name = name;
        HP = hp;
    }
}
```

```csharp
// 通过参数类型数组找到对应构造
ConstructorInfo ctor = t.GetConstructor(new Type[] { typeof(string), typeof(int) });
object monster = ctor.Invoke(new object[] { "哥布林", 200 });
```

---

### Activator：更简洁的创建实例方式

> `Activator.CreateInstance()` 是最常用的动态创建实例方式，比 `ConstructorInfo.Invoke()` 更简洁。

```csharp
// 方式1：Activator（最简洁，90%情况用这个）
object m = Activator.CreateInstance(t); // 调用无参构造

// 方式2：Activator 传参（调用有参构造）
object m = Activator.CreateInstance(t, new object[] { "哥布林", 200 });

// 方式3：泛型版（直接返回目标类型，不装箱成 object）
Monster m = Activator.CreateInstance<Monster>();
```

| 对比 | `Activator.CreateInstance()` | `ConstructorInfo.Invoke()` |
|---|---|---|
| 灵活性 | 只调无参构造（最常用） | 能指定调哪个构造 |
| 参数 | `params object[]`，自动装箱 | 手动 `object[]` |
| 适用场景 | 绝大多数情况 | 构造有多个重载时 |

**什么时候用 Activator？** 当你只有 Type 对象，不知道具体类名时，用它来 new 一个出来。

## 2.7 区分同名重载方法

**名字不同 → 直接按名字区分（最常见）。名字相同 → 用 `Type[]` 按参数类型区分。**

```csharp
// 场景：两个方法参数都是 int，但名字不同
public class Monster
{
    public void TakeHeal(int amount) { }    // 受到治疗
    public void TakeDamage(int amount) { }   // 受到攻击
}

// 直接按名字区分，不需要 Type[]
MethodInfo heal = t.GetMethod("TakeHeal");
MethodInfo damage = t.GetMethod("TakeDamage");
heal.Invoke(monster, new object[] { 50 });
damage.Invoke(monster, new object[] { 30 });

// 场景：同一个名字有多个重载（真正需要 Type[] 的情况）
public class Monster
{
    public void TakeHeal(int amount) { }
    public void TakeHeal(int amount, bool isCritical) { } // 重载
}

// 用 Type[] 按参数类型区分
MethodInfo heal1 = t.GetMethod("TakeHeal", new Type[] { typeof(int) });
MethodInfo heal2 = t.GetMethod("TakeHeal", new Type[] { typeof(int), typeof(bool) });
```

**简单说：**
- **名字不同** → `GetMethod("名字")` 直接拿
- **名字相同、参数不同** → `GetMethod("名字", new Type[]{...})` 靠参数类型区分

---

## 2.8 装箱拆箱说明

反射操作值类型（int / float / bool）时，会涉及装箱拆箱：

```csharp
// GetValue 返回 object，所以值类型要拆箱
int hp = (int)hpField.GetValue(monster); // 装箱 → 拆箱

// SetValue 参数是 object，所以值类型要装箱
hpField.SetValue(monster, 200);          // 装箱

// 引用类型（string / class）没有装箱拆箱
nameField.SetValue(monster, "火焰史莱姆"); // 直接传引用
```

## 2.9 游戏开发场景：动态调用不同怪物的技能

```csharp
public class Slime : Monster { public void Bounce() => ... }
public class Goblin : Monster { public void Steal() => ... }
public class Dragon : Monster { public void FireBreath() => ... }

// 运行时按名字找到怪物类型，动态调用专属方法
public static class MonsterSkillSystem
{
    public static void UseSkill(string monsterType, string skillName)
    {
        Type t = Type.GetType(monsterType);
        if (t == null) return;

        MethodInfo skill = t.GetMethod(skillName);
        if (skill != null)
        {
            object instance = Activator.CreateInstance(t);
            skill.Invoke(instance, null);
        }
    }
}

MonsterSkillSystem.UseSkill("Slime", "Bounce");      // 史莱姆弹跳！
MonsterSkillSystem.UseSkill("Dragon", "FireBreath"); // 巨龙喷火！
```

## 2.10 Assembly：操作程序集

> `Assembly` 用来加载和操作程序集（.dll / .exe），可以运行时加载外部 .dll 并发现其中的类型。

### 程序集是什么？

```
程序集 = .dll 或 .exe 文件，里面包含：
├── IL 代码（编译后的中间语言）
├── 元数据（类名、方法、字段等信息）
└── 清单（引用了哪些其他程序集）
```

### 核心操作

```csharp
// ① 获取当前正在运行的程序集
Assembly current = Assembly.GetExecutingAssembly();

// ② 加载外部 .dll
Assembly dll = Assembly.LoadFrom("Plugins/SkillPlugin.dll");

// ③ 获取 .dll 里所有类型
Type[] types = dll.GetTypes();

// ④ 获取 .dll 里某个具体类型
Type enemy = dll.GetType("MyPlugin.Enemy");
```

### 和 Type 的区别

| | `Type` | `Assembly` |
|---|---|---|
| 操作对象 | 一个类 | 一个 .dll / .exe |
| 用来 | 查类里的成员 | 查程序集里的类 |
| 从哪来 | `typeof()` / `GetType()` | `LoadFrom()` / `GetExecutingAssembly()` |

### 插件系统实战

```csharp
// 加载外部技能插件 .dll
Assembly pluginAsm = Assembly.LoadFrom("Plugins/SkillPlugin.dll");

// 找到所有实现了 IPlugin 接口的类并实例化
foreach (Type t in pluginAsm.GetTypes())
{
    if (typeof(IPlugin).IsAssignableFrom(t) && t.IsClass)
    {
        IPlugin plugin = (IPlugin)Activator.CreateInstance(t);
        plugin.OnLoad();
    }
}
```

### Unity 游戏开发场景

| 场景 | 怎么用 |
|---|---|
| 热更新 | 把热更新代码打包成 .dll，运行时 `Assembly.LoadFrom()` 加载执行 |
| 技能插件系统 | 玩家自己写技能脚本打包成 .dll，主游戏动态加载并实例化 |
| DLC 内容 | 把新地图/新怪物打成 .dll，运行时加载 |
| 编辑器扩展 | Unity 加载 Editor 程序集，读其中的 EditorWindow 等类 |
