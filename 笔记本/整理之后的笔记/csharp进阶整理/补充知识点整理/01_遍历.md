集合遍历
=============================================

一、foreach 循环
foreach 是遍历集合最简洁的方式。
适用于只需要读取元素、不需要修改集合结构的情况。

基本语法：
foreach (type item in collection)
{
    // 在循环体中使用 item
}

type：元素类型
  → ArrayList/List<T>：可以用具体类型，也可以用 var
  → foreach 会自动调用 GetEnumerator() 获取迭代器

示例：
// List<T> — 推荐用 var 或具体类型
List<int> list = new List<int> { 1, 2, 3 };
foreach (var item in list) { Console.WriteLine(item); }
foreach (int item in list) { Console.WriteLine(item); }

// ArrayList — 存储 object，用 var 或 object
ArrayList arr = new ArrayList { 1, "hello", 3.14 };
foreach (var item in arr) { Console.WriteLine(item); }

// Dictionary — 遍历键值对
Dictionary<string, int> dict = new Dictionary<string, int>
{
    { "数学", 90 }, { "语文", 85 }
};
foreach (var kvp in dict)
{
    Console.WriteLine($"键：{kvp.Key}, 值：{kvp.Value}");
}

foreach (var key in dict.Keys) { Console.WriteLine(key); }
foreach (var value in dict.Values) { Console.WriteLine(value); }

二、for 循环
适用于需要使用索引，或需要修改集合元素的情况。

语法：
for (int i = 0; i < collection.Count; i++)
{
    var item = collection[i];
    // 可以用索引 i，可以修改 collection[i]
}

示例：
List<int> list = new List<int> { 10, 20, 30 };
for (int i = 0; i < list.Count; i++)
{
    list[i] *= 2;  // 修改集合元素
    Console.WriteLine($"索引 {i}: {list[i]}");
}

三、foreach vs for 如何选择

  场景                          推荐方式
  只读取，不需要索引            foreach
  需要索引位置                  for
  需要修改集合元素              for
  需要反向遍历                  for 或 List<T>.FindLastIndex
  在遍历时删除元素              不用 foreach，用倒序 for

【重要】遍历时不要修改集合的大小！
for 循环正向遍历时删除元素会导致索引混乱。
正确的删除方式：
  // 倒序遍历删除（推荐）
  for (int i = list.Count - 1; i >= 0; i--)
  {
      if (list[i] > 3) list.RemoveAt(i);
  }

四、IEnumerable 和迭代器
所有可以用 foreach 遍历的集合都实现了 IEnumerable<T> 接口。
该接口只有一个方法：GetEnumerator()，返回一个迭代器。

yield return 是创建迭代器的简写方式：
IEnumerable<int> GetNumbers()
{
    yield return 1;
    yield return 2;
    yield return 3;
}
// 每次只返回一个元素，按需计算（延迟执行）
