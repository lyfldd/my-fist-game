常用集合的时间复杂度对比
=============================================

以下是 C# 中最常用集合在增、删、查、改方面的性能对比：

一、List<T>（动态数组）

  操作           时间复杂度    说明
  随机访问[i]    O(1)          直接索引访问
  尾部添加        O(1)          摊销后（扩容时O(n)）
  尾部删除        O(1)          直接删除最后元素
  头部插入        O(n)          需移动所有元素
  中间插入/删除   O(n)          需移动部分元素
  搜索（无序）    O(n)          线性查找
  搜索（有序+Sort）O(log n)    二分查找

二、LinkedList<T>（双向链表）

  操作           时间复杂度    说明
  随机访问[i]    O(n)          只能从头/尾遍历
  头部插入/删除   O(1)          只需改指针
  尾部插入/删除   O(1)          只需改指针
  中间插入/删除   O(n)          需先遍历到位置
  搜索（无序）    O(n)          线性查找

三、Dictionary<TKey, TValue>（哈希表）

  操作           时间复杂度        说明
  随机访问[Key]  O(1) 摊销        哈希计算+可能冲突
  添加           O(1) 摊销        同上
  删除           O(1) 摊销        同上
  搜索键         O(1) 摊销        哈希计算
  搜索值         O(n)            需遍历所有值

四、HashSet<T>（哈希集合）
  与 Dictionary 类似，但只存键不存值

  操作           时间复杂度
  添加           O(1) 摊销
  删除           O(1) 摊销
  搜索           O(1) 摊销
  Contains       O(1) 摊销

五、Queue<T> / Stack<T>

  操作           时间复杂度    说明
  入队/压栈      O(1)          直接添加
  出队/弹栈      O(1)          直接移除
  遍历           O(n)          线性
  搜索           O(n)          线性

六、选择建议
1. 需要快速随机访问 → List<T>
2. 需要快速任意位置插入/删除 → LinkedList<T>
3. 需要键值映射快速查找 → Dictionary<K,V>
4. 需要去重集合 → HashSet<T>
5. 需要先进先出 → Queue<T>
6. 需要先进后出 → Stack<T>
