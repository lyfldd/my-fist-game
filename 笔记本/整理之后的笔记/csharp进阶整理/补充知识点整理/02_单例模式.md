单例模式（Singleton Pattern）
=============================================

一、概念
保证一个类在整个程序中只有且仅有一个实例，并提供一个全局访问点。
属于创建型设计模式中最核心、最常用的模式之一。

生活中的类比：
  Windows 任务管理器 → 永远只有一个窗口
  打印机后台程序     → 整个电脑共用一个打印队列
  游戏 GameManager   → 整个游戏只有一个管理器

二、单例模式的三个要点
1. 私有静态变量：存储唯一实例
2. 公开静态属性：全局唯一访问入口
3. 私有构造函数：禁止外部用 new 创建实例

三、最常见写法：双重检查锁（Lazy 初始化 + 线程安全）

public class GameManager
{
    private static GameManager _instance;
    private static readonly object _lock = new object();

    public static GameManager Instance
    {
        get
        {
            if (_instance == null)  // 第一层检查：减少加锁次数，提升性能
            {
                lock (_lock)        // 加锁，防止多线程同时创建
                {
                    if (_instance == null)  // 第二层检查：确保只创建一次
                    {
                        _instance = new GameManager();
                    }
                }
            }
            return _instance;
        }
    }

    private GameManager()
    {
        Console.WriteLine("游戏管理器已创建！");
    }

    public int Score { get; set; }
    public void AddScore(int point) { Score += point; }
}

// 使用（任何地方都可以访问）
GameManager.Instance.AddScore(100);
Console.WriteLine(GameManager.Instance.Score);

四、单例模式的变体

1. 饿汉式（静态初始化，早创建）
   public class Singleton
   {
       private static readonly Singleton _instance = new Singleton();
       private Singleton() { }
       public static Singleton Instance => _instance;
   }
   优点：线程安全，由 CLR 保证
   缺点：程序启动就创建，不管用不用都占用内存

2. 懒汉式（延迟初始化，按需创建）
   即双重检查锁版本，延迟到第一次访问时创建

3. C# 简写（C# 6.0+）
   public class Singleton
   {
       public static Singleton Instance { get; } = new Singleton();
       private Singleton() { }
   }

五、线程安全说明
多线程环境下，如果不用双重检查锁（或饿汉式），
多个线程可能同时通过 null 检查，各自 new 一个实例，
破坏了单例的唯一性。

六、应用场景
1. 游戏管理器（GameManager、AudioManager）
2. 配置管理器
3. 日志管理器
4. 数据库连接池
5. 缓存管理器
