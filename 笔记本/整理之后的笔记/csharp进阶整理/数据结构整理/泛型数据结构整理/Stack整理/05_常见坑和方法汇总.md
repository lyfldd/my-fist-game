Stack<T> —— 常见坑
=============================================

| 问题 | 说明 |
|---|---|
| Pop() 空栈会崩 | 调用前先判断 stack.Count > 0 |
| Peek() 空栈会崩 | 同上 |
| ToArray() 顺序是栈顶到栈底 | 与压栈顺序相反，重建时要逆序压栈 |
| 无法按索引访问 | 不支持 stack[0]，没有索引器 |
| 无法在中间插入/删除 | 只能操作栈顶 |


二、方法汇总表

  方法              作用                   时间复杂度  空栈是否抛异常
  ─────────────────────────────────────────────────────────────────
  Push(T)           压入栈顶               O(1)       —
  Pop()              弹出栈顶（删除）        O(1)       ✅ 抛异常
  Peek()             查看栈顶（不删除）      O(1)       ✅ 抛异常
  Contains(T)        是否包含某值           O(n)       —
  Clear()            清空                  O(n)       —
  ToArray()          转为数组（栈顶→栈底）  O(n)       —
  CopyTo(T[], int)   复制到已有数组         O(n)       —
  Count              元素数量               O(1)       —
