Stack<T> —— 入栈和出栈
=============================================

一、方法汇总表

  方法        作用                          时间复杂度  空栈是否抛异常
  ───────────────────────────────────────────────────────────────────
  Push(T)      压入栈顶                      O(1)       —
  Pop()        弹出栈顶（删除并返回）          O(1)       ✅ 抛异常
  Peek()       查看栈顶（不删除）              O(1)       ✅ 抛异常


二、Push —— 压栈（入栈）

// 只能在栈顶压入元素，时间复杂度 O(1)
Stack<string> stack = new Stack<string>();
stack.Push("apple");
stack.Push("banana");
stack.Push("cherry");
// 栈顶：cherry → banana → apple：栈底


三、Pop —— 弹栈（取出并删除栈顶）

// 取出并移除栈顶元素，时间复杂度 O(1)
// 空栈调用会抛 InvalidOperationException
if (stack.Count > 0)
{
    string top = stack.Pop();
    Console.WriteLine(top); // cherry
}


四、Peek —— 查看栈顶（不删除）

// 只查看栈顶，不移除，时间复杂度 O(1)
// 空栈调用会抛 InvalidOperationException
if (stack.Count > 0)
{
    string top = stack.Peek();
    Console.WriteLine(top); // cherry（还在栈里）
}


五、安全写法总结

// ✅ 推荐：先判断再操作
if (stack.Count > 0)
{
    string top = stack.Pop();  // 安全弹出
}

// ✅ 也可以用 try-catch
try
{
    string top = stack.Pop();
}
catch (InvalidOperationException)
{
    Console.WriteLine("栈为空");
}
