List<T> —— 删除元素
=============================================

一、方法汇总表

  方法                              作用                         时间复杂度
  ─────────────────────────────────────────────────────────────────────
  Remove(T item)                   删除第一个匹配项               O(n)
  RemoveAt(int index)              删除指定索引的元素             O(n)
  RemoveRange(int index, int count) 批量删除                      O(n)
  RemoveAll(Predicate<T>)           按条件删除所有匹配项           O(n)
  Clear()                          清空所有元素                   O(n)


二、Remove —— 删除第一个匹配项

List<string> list = new List<string> { "剑", "盾", "剑", "弓" };
list.Remove("剑");   // 删除第一个 "剑"
// [盾, 剑, 弓]

// 找不到则什么都不做，不抛异常
list.Remove("药水");  // 无任何效果


三、RemoveAt —— 删除指定索引

list.RemoveAt(0);    // 删除索引 0 的元素
// [剑, 弓]

// 注意：索引超出范围会抛 ArgumentOutOfRangeException


四、RemoveRange —— 批量删除

List<int> list = new List<int> { 1, 2, 3, 4, 5 };
list.RemoveRange(1, 2);   // 从索引 1 开始删 2 个元素
// [1, 4, 5]


五、RemoveAll —— 按条件删除所有匹配项

List<int> list = new List<int> { 1, 2, 3, 4, 5, 6 };
// 删除所有偶数
list.RemoveAll(x => x % 2 == 0);
// [1, 3, 5]

// 删除所有小于 0 的元素
list.RemoveAll(x => x < 0);


六、Clear —— 清空所有元素

list.Clear();   // []  Count 变为 0，Capacity 保持不变


七、删除时的常见坑

⚠️ 不要在 foreach 循环中直接删除元素！

// ❌ 错误：foreach 中 Remove 会导致索引错乱
foreach (var item in list)
{
    if (item == 2) list.Remove(item);
}

// ✅ 正确做法 1：倒序遍历删除
for (int i = list.Count - 1; i >= 0; i--)
{
    if (list[i] == 2) list.RemoveAt(i);
}

// ✅ 正确做法 2：用 RemoveAll 条件删除
list.RemoveAll(x => x == 2);

说明：
- foreach 底层用迭代器，删除后迭代器状态被破坏，报 InvalidOperationException
- 倒序遍历删索引，前面的索引不受影响
- RemoveAll 一次性处理，效率最高
