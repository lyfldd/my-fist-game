List<T> —— 增加元素
=============================================

一、方法汇总表

  方法                              作用                   时间复杂度
  ───────────────────────────────────────────────────────────────
  Add(T item)                      末尾添加一个元素        O(1) 均摊
  AddRange(IEnumerable<T>)         末尾批量添加            O(n)
  Insert(int index, T item)       在指定索引插入          O(n)
  InsertRange(int, collection)     在指定索引批量插入      O(n)


二、Add —— 末尾添加一个元素

List<int> list = new List<int>();
list.Add(1);       // [1]
list.Add(2);       // [1, 2]
list.Add(3);       // [1, 2, 3]

// 泛型类型安全，编译时就检查类型
list.Add("abc");   // ❌ 编译报错，不允许混入其他类型


三、AddRange —— 末尾批量添加

int[] arr = { 4, 5, 6 };
list.AddRange(arr);  // [1, 2, 3, 4, 5, 6]

// 从另一个 List 添加
List<int> another = new List<int> { 7, 8 };
list.AddRange(another);  // [1, 2, 3, 4, 5, 6, 7, 8]


四、Insert —— 在指定索引插入

list.Insert(1, 99);   // 在索引 1 的位置插入 99
// [1, 99, 2, 3]      原索引 1 及之后的元素全部后移

// 注意：索引不能超过 Count，超出会抛 ArgumentOutOfRangeException
list.Insert(list.Count, 100);  // 等同于 Add，在末尾插入


五、InsertRange —— 在指定索引批量插入

List<int> toAdd = new List<int> { 10, 11, 12 };
list.InsertRange(2, toAdd);   // 从索引 2 开始批量插入


六、注意事项

- Insert / InsertRange 的索引超出范围会抛异常
- Add 偶尔会触发扩容（容量翻倍），但均摊是 O(1)
- 游戏开发中，如果知道大致数量，提前设 Capacity 可以避免扩容开销
  List<int> list = new List<int>(1000);  // 预分配 1000 容量
