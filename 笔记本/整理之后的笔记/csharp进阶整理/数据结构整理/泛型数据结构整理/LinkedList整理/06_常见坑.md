LinkedList<T> —— 常见坑
=============================================

| 问题 | 说明 |
|---|---|
| 不支持下标访问 | list[0] 会报错，没有索引器 |
| 不支持 RemoveAt | 必须先遍历到节点再用 Remove(node) 删除 |
| Remove(T) 只删第一个 | 有重复值时需要手动循环删除所有 |
| 节点不能跨链表使用 | 已有节点添加到新链表前必须先从原链表移除 |
| RemoveFirst/Last 空链表会崩 | 调用前先判断 list.Count > 0 |
| 找不到 Find 返回 null | 必须判空再访问 Value，否则 NullReferenceException |


二、对比 List<T>

| 对比 | List<T>（顺序存储） | LinkedList<T>（链式存储） |
|---|---|---|
| 按索引访问 | O(1) | 不支持 |
| 按索引插入/删除 | O(n) | 不支持（需先找节点） |
| 已知节点插入/删除 | — | O(1) |
| 头部添加 | O(n) | O(1) |
| 尾部添加 | O(1) | O(1) |
| 查找指定值 | O(n) | O(n) |
| 内存开销 | 低 | 高（每节点多 2 个指针） |
| 适用场景 | 读多写少 | 频繁在中间/头部增删 |

> 一句话总结：查多用 List<T>，频繁在中间/头部增删用 LinkedList<T>。
