Queue<T> —— 入队和出队
=============================================

一、方法汇总表

  方法                  作用                      时间复杂度  空队列是否抛异常
  ─────────────────────────────────────────────────────────────────────────
  Enqueue(T)            入队（加到队尾）            O(1)       —
  Dequeue()              出队（删除队头）            O(1)       ✅ 抛异常
  Peek()                查看队头（不删除）          O(1)       ✅ 抛异常
  TryDequeue(out T)     安全出队                   O(1)       ❌ 不抛异常
  TryPeek(out T)        安全查看队头               O(1)       ❌ 不抛异常


二、Enqueue —— 入队

queue.Enqueue("A");
queue.Enqueue("B");
queue.Enqueue("C");
// 队头：A → B → C：队尾


三、Dequeue —— 出队

// 删除并返回队头元素，时间复杂度 O(1)
// 空队列调用会抛 InvalidOperationException
if (queue.Count > 0)
{
    string head = queue.Dequeue();
    Console.WriteLine(head); // A
}
// 剩余：B → C


四、Peek —— 查看队头（不删除）

if (queue.Count > 0)
{
    string head = queue.Peek();
    Console.WriteLine(head); // B（还在队列里）
}


五、TryDequeue —— 安全出队（推荐）

// 安全出队，不抛异常
// 成功返回 true 并通过 out 拿到元素，失败返回 false
if (queue.TryDequeue(out string value))
{
    Console.WriteLine("取出：" + value);
}
else
{
    Console.WriteLine("队列为空");
}


六、TryPeek —— 安全查看队头（推荐）

// 安全查看，不删除，不抛异常
if (queue.TryPeek(out string head))
{
    Console.WriteLine("队头是：" + head);
}

> TryDequeue 和 TryPeek 不需要先判断 Count，更推荐使用。
