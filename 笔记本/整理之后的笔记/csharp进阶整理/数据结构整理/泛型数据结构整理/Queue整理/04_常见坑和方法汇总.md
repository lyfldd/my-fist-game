Queue<T> —— 常见坑
=============================================

| 问题 | 说明 |
|---|---|
| Dequeue() 空队列会崩 | 用 TryDequeue 或先判断 Count > 0 |
| Peek() 空队列会崩 | 用 TryPeek 或先判断 Count > 0 |
| 无法按索引访问 | 不支持 queue[0]，没有索引器 |
| 无法直接修改元素 | 需要 Dequeue 取出 → 修改 → Enqueue 重新入队 |
| ToArray() 是队头到队尾 | 与 Stack 的 ToArray（栈顶到栈底）方向相反 |


二、方法汇总表

  方法                作用                     时间复杂度  空队列是否抛异常
  ─────────────────────────────────────────────────────────────────────────
  Enqueue(T)          入队（加到队尾）           O(1)       —
  Dequeue()           出队（删除队头）           O(1)       ✅ 抛异常
  Peek()              查看队头（不删除）         O(1)       ✅ 抛异常
  TryDequeue(out T)   安全出队                  O(1)       ❌ 不抛异常
  TryPeek(out T)      安全查看队头              O(1)       ❌ 不抛异常
  Contains(T)         是否包含某值              O(n)       —
  Clear()             清空队列                 O(n)       —
  ToArray()           转为数组（队头→队尾）     O(n)       —
  CopyTo(T[], int)    复制到已有数组            O(n)       —
  TrimExcess()        释放多余内存              O(n)       —
  Count               元素数量                  O(1)       —


三、Stack vs Queue 对比

| 对比 | Stack<T>（栈） | Queue<T>（队列） |
|---|---|---|
| 顺序规则 | LIFO 后进先出 | FIFO 先进先出 |
| 加元素 | Push — 压栈顶 | Enqueue — 加到队尾 |
| 取元素 | Pop — 弹栈顶 | Dequeue — 取队头 |
| 查看不删 | Peek — 看栈顶 | Peek — 看队头 |
| 典型场景 | 撤销操作、括号匹配、DFS | 任务排队、消息处理、BFS |
