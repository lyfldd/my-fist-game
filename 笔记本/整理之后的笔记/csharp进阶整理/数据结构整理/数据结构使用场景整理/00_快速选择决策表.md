# 数据结构快速选择决策表

---

## 一、一句话总览

| 容器 | 核心优势 | 核心劣势 | 优先选择场景 | 绝对不选场景 |
|---|---|---|---|---|
| 数组 Array | 速度最快、内存连续、下标 O(1) | 长度固定、增删极慢 | 固定长度数据、高性能算法 | 动态数据、频繁增删 |
| List<T> | 动态扩容、下标 O(1)、遍历快 | 中间增删 O(n) | 动态列表、遍历为主、随机访问 | 频繁中间增删 |
| Dictionary<K,V> | O(1) 极速查询、键值对 | 无序、键唯一、无下标 | 高频键值查询、缓存、去重映射 | 顺序遍历、排序、键不唯一 |
| Stack<T> | 后进先出、栈顶 O(1) | 无随机访问、只能操作栈顶 | UI 回退、撤销操作、DFS | 先进先出、随机访问 |
| Queue<T> | 先进先出、头尾 O(1) | 无随机访问、只能操作头尾 | 顺序执行、排队、BFS | 后进先出、随机访问 |
| LinkedList<T> | 任意位置增删 O(1)、双向遍历 | 无下标、查找 O(n) | 频繁增删、双向遍历、环形结构 | 随机访问、高频查询 |

---

## 二、选择决策流程

  需要键值对查询？
    → 是 → Dictionary<K,V>

  需要后进先出（撤销/回退/DFS）？
    → 是 → Stack<T>

  需要先进先出（排队/BFS）？
    → 是 → Queue<T>

  需要频繁在中间增删？
    → 是 → LinkedList<T>

  数据量固定且只读/遍历？
    → 是 → Array（数组）

  其他动态列表场景？
    → List<T>（默认首选）
