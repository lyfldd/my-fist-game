泛型概念
=============================================

一、什么是泛型
泛型 = "让代码适用于任意类型的模板"
不用为 int、string、自定义类重复写一模一样的逻辑。

形象理解：
  普通代码：只能装苹果的盒子
  泛型代码：万能盒子，告诉它装什么，它就装什么

二、泛型的声明形式
泛型类    → class MyClass<T> { }
泛型接口  → interface IMyInterface<T> { }
泛型方法  → void MyMethod<T>(T param) { }
泛型委托  → delegate void MyDelegate<T>(T arg);

三、单泛型占位符 <T>
T 是占位符，可以换成任意类型名（TKey、TValue、TItem 等均可）。

// 泛型类
class Monster<T>
{
    public T Value { get; set; }
}

Monster<int> m1 = new Monster<int>();
m1.Value = 1;          // 存整数

Monster<string> m2 = new Monster<string>();
m2.Value = "lyf";     // 存字符串

四、多泛型占位符 <T1, T2, ...>
多个占位符分别代表不同类型，互不影响。

public class Bag<TPerson, TItem>
{
    public TPerson Owner { get; set; }
    public TItem Goods { get; set; }

    public void ShowInfo()
    {
        Console.WriteLine("拥有者：" + Owner);
        Console.WriteLine("物品：" + Goods);
    }
}

// 使用
var person = new Person { Name = "小明", Age = 20 };
var item = new Item { ItemName = "长剑", Price = 999 };

var bag = new Bag<Person, Item>();
bag.Owner = person;
bag.Goods = item;

【重要问题】为什么不能直接访问 T 的属性（如 Owner.Name）？
因为编译器不知道 T 到底是什么类型，泛型只是占位符，
编译器在编译时才知道具体类型。解决方法：加泛型约束。
