泛型约束
=============================================

一、为什么需要泛型约束
编译器不知道 T 是什么类型，就不知道 T 有哪些成员。
泛型约束（where）告诉编译器：T 必须满足什么条件。

语法：
  public class 类名<T> where T : 约束条件 { }

二、六种泛型约束

1. where T : 基类
   T 必须是指定类或其派生类
   public class Storage<T> where T : Entity { }

2. where T : 接口
   T 必须实现指定接口
   public class Sorter<T> where T : IComparable<T> { }

3. where T : class
   T 必须是引用类型（类、接口、数组、string、委托等）
   public class Cache<T> where T : class { }

4. where T : struct
   T 必须是值类型（int、float、struct、enum 等）
   public class Calculator<T> where T : struct { }

5. where T : new()
   T 必须有公共无参构造函数（约束允许创建 T 的实例）
   public class Factory<T> where T : new() { T item = new T(); }

6. where T : U（泛型约束泛型）
   T 和 U 之间有约束关系

三、组合约束
多个约束可以同时使用，但有顺序要求：
  引用类型：where T : class, 接口1, 接口2, new()
  值类型  ：where T : struct（值类型不能和 new() 组合）

示例（多层组合）：
public class Service<TEntity, TDto>
    where TEntity : class, new()    // TEntity：引用类型，有无参构造
    where TDto : IMapFrom<TEntity>  // TDto 必须映射自 TEntity
{ }

四、完整示例（修正原文错误）

原文写法有误，正确的约束写法：

// 约束 TPerson 必须是 Person 或其派生类
// 约束 TItem 必须是 Item 或其派生类
public class Bag<TPerson, TItem>
    where TPerson : Person
    where TItem : Item
{
    public TPerson Owner { get; set; }
    public TItem Goods { get; set; }

    public void ShowInfo()
    {
        // 加了约束后，编译器知道 Owner 有 Name/Age
        Console.WriteLine("名字：" + Owner.Name);
        Console.WriteLine("年龄：" + Owner.Age);
        Console.WriteLine("物品：" + Goods.ItemName);
        Console.WriteLine("价格：" + Goods.Price);
    }
}
