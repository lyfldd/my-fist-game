泛型方法与泛型接口
=============================================

一、泛型方法
一个方法处理多种类型的参数，不用为每种类型写一个方法。

// 普通写法：需要写三个方法
void PrintPlayer(Player p) { ... }
void PrintEnemy(Enemy e) { ... }
void PrintItem(Item i) { ... }

// 泛型方法：写一次，所有类型都能用
void Print<T>(T obj)
{
    Console.WriteLine(obj.ToString());
}

Print(player);  // 传 Player
Print(enemy);   // 传 Enemy
Print(item);    // 传 Item

二、泛型接口
不同类型实现同一接口，但参数类型不同。

public interface IDamageable<T>
{
    void TakeDamage(T damage);
}

// int 伤害
public class Player : IDamageable<int>
{
    public void TakeDamage(int damage) { Console.WriteLine($"玩家受伤：{damage}"); }
}

// float 伤害
public class Enemy : IDamageable<float>
{
    public void TakeDamage(float damage) { Console.WriteLine($"怪物受伤：{damage}"); }
}

三、泛型类 + 泛型方法的组合（最常用）

public class Backpack<T>
{
    public T Owner;  // 固定类型（由类决定）

    // 泛型方法：U 由每次调用决定
    public void PutItem<U>(U item)
    {
        Console.WriteLine($"[{Owner}] 放入了：{item}");
    }
}

public class Person { public string Name; }
public class Sword { public string Name; }

var bag = new Backpack<Person>();
bag.Owner = new Person { Name = "小明" };
bag.PutItem<Sword>(new Sword { Name = "屠龙刀" });  // 放武器
bag.PutItem<int>(100);   // 放金币
bag.PutItem<string>("药水");  // 放药水

// U 由传入的参数类型自动推断，不需要手动写
bag.PutItem("恢复药水");  // 自动推断 U 为 string
