关键字是async /await

1. 它是干什么的？
async /await = 让异步代码写得跟同步代码一样简单。
以前：
开线程 → 等线程 → 继续 → 嵌套 → 乱
现在：
一步一步写，顺序自然，逻辑清晰

2. 它和 Task 是什么关系？
Task 是 “要做的事情”
async /await 是 “优雅地做这件事的语法”
可以理解成：
Task = 车
async/await = 自动驾驶

3. 两个关键字各自的作用
async
写在方法前面
告诉编译器：
这个方法内部会使用 await
不加 async，就不能用 await
它本身不创建线程、不开启异步
只是一个 “标记”
await
写在一个 可等待对象 前面（绝大多数是 Task）
意思是：
等待这个任务完成，但不卡住当前线程
等待期间，线程可以去干别的，不会空转

4. 最关键一句话
await 不会阻塞线程，只会 “暂停方法”，等任务完了再继续。
对比：
task.Wait() → 阻塞线程（死等，啥也不干）
await task → 不阻塞线程（线程去忙别的，等好了再回来）

// async 标记方法
async Task 方法名()
{
    // 异步做事
    await Task.Run(() =>
    {
        // 子线程工作
    });

    // 上面完成后，才会走到这里
}
6. 一个超直观比喻
你（主线程）在煮开水
Wait() → 死盯着水壶，啥也不干
await → 把水壶放那，你去玩手机
水开了（Task 完成）再回来继续
