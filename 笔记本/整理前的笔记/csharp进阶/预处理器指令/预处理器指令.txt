C#游戏开发（Unity）预处理器指令笔记
核心定义：C#预处理器指令以#开头，不加分号，编译时生效、不参与程序运行，核心作用是让同一份代码适配不同场景（平台、模式），实现“一份代码，多场景适配”，是Unity游戏开发必备基础工具。
一、核心特点（游戏开发场景重点）
- 编译时执行，运行时无痕迹，不占用游戏性能、不增加包体体积；
- 核心用途是条件编译（控制哪些代码参与编译）；
- Unity中自带大量预设符号（如UNITY_EDITOR），无需手动定义；
- 无宏替换功能（区别于C/C++），符号仅分“定义/未定义”两种状态。
二、游戏开发常用指令及实战用法
1. 条件编译核心指令（最常用）
基础组合：#define（定义符号）、#undef（取消定义）、#if/#elif/#else/#endif（条件判断）
实战场景：
- 区分编辑器与打包运行：用UNITY_EDITOR预设符号，实现编辑器调试、打包后自动屏蔽无关逻辑（如作弊功能、调试按钮）；
- 多平台适配：用UNITY_ANDROID（安卓）、UNITY_IOS（iOS）、UNITY_STANDALONE_WIN（PC）等预设符号，编写不同平台专属逻辑（如触摸/键盘控制、平台SDK调用）；
- 版本功能控制：自定义DEMO_VERSION（演示版）、FULL_VERSION（完整版）符号，控制关卡解锁、功能开关；
- 调试日志控制：用DEBUG或DEVELOPMENT_BUILD，让调试日志仅在开发版显示，正式版自动剔除。
2. 辅助指令（实用工具）
- #warning：编译时输出警告（如“该功能未完善”），提醒开发者；
- #error：编译时直接报错（如“未完成代码，禁止打包”），避免误打包半成品；
- #region/#endregion：代码折叠，整理复杂逻辑（如工具方法、组件逻辑），提升代码可读性（编辑器功能，不影响编译）。
三、Unity项目常见实战示例（直接复用）
// 1. 编辑器与打包区分
using UnityEngine;
public class Player : MonoBehaviour
{
    void Start()
    {
#if UNITY_EDITOR
        Debug.Log("编辑器调试：玩家初始化"); // 打包后自动消失
        transform.position = Vector3.zero; // 编辑器快速重置位置
#else
        Debug.Log("游戏正式运行");
#endif
    }
}

// 2. 多平台控制（触摸/键盘）
void Update()
{
#if UNITY_ANDROID || UNITY_IOS
    // 手机触摸逻辑（摇杆控制）
#else
    // PC键盘鼠标逻辑（WASD控制）
#endif
}

// 3. 调试日志封装（正式版剔除）
public void GameLog(string msg)
{
#if DEBUG || DEVELOPMENT_BUILD
    Debug.Log("[游戏日志]：" + msg);
#endif
}
四、核心价值总结
1. 减少代码冗余：无需为不同平台、不同版本写多份代码，一份代码适配所有场景；
2. 提升开发效率：编辑器内方便测试（跳过开场、快速重置），打包自动清理无用逻辑；
3. 保障产品安全：调试日志、测试功能打包后自动消失，不泄露开发细节、不影响性能；
4. 灵活控制功能：快速切换Demo/完整版、开启/关闭测试功能，适配不同开发阶段需求。
备注：Unity预设符号无需手动定义，切换平台、调试/发布模式时，会自动启用对应符号（如Debug模式自动定义DEBUG）。
