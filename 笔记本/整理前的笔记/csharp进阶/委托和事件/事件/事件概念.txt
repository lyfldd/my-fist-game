就是假如说我们要做一个东西
怪物死了掉落金币
然后玩家击杀怪物获得金币
要是我想要去用委托实现的话
我要把这两个方法绑在一起
就是先是绑定一个怪物死亡，然后在·绑定一个玩家获得金币
那这个样子的话有点，玩家类要一直的关注怪物的状态
用代码实现的话就是我们要把各个怪物的实例化对象放在玩家的类里面的一个list
然后怪物死亡
玩家要不停地去遍历、检查每个怪物死没死
怪物死了 → 玩家再调用加金币方法
调用这个list里面怪物对象的每一个死亡的方法
玩家必须知道所有怪物
玩家必须主动监视怪物状态
怪物必须被玩家持有
加个成就、加个经验、加个任务 → 玩家类越来越臃肿
高度耦合，后期根本没法维护


用委托 / 事件 → 怪物自己通知，玩家不用管（解耦）
玩家不需要持有任何怪物
玩家不需要检查、不需要遍历
怪物死了自己触发，不用别人管
想加成就、加经验、加任务 → 直接订阅事件，不用改玩家、不用改怪物
结构干净、解耦、可扩展

我们先在全局定义一个游戏事件
然后我们定义一个怪物死亡的事件
然后怪物只负责说我死了
怪物触发了这个事件
然后订阅了这个事件的就触发一下就行了

接下来我们先来了解一下事件的具体概念
事件其实就是一个特殊的变量类型
声明语法
访问修饰符 event 委托类型 事件名字；
一般作为成员变量存在类中
委托怎么样，事件就怎么用
事件相对于委托的区别就是
事件不可以在类的外部赋值
不可以在类的外部调用
只能作为成员在类和接口以及结构体里面

事件就是一个 “中间转接器 / 信号分发器”
事件 = 中间转接器 / 广播台
怪物：只管发信号
“我死了！”
事件：负责转发信号
“谁关心，谁来收”
玩家、成就、任务、金币：只管听信号
“收到，我执行我的逻辑”
它们之间不直接连线，全部通过事件转接


事件 = “发生了一件事，大家来订阅”
怪物死了 → 事件发出
玩家要加金币 → 订阅这个事件
系统要加杀怪数 → 订阅这个事件
成就系统要解锁成就 → 订阅这个事件
不用事件：直接连线（很乱）
怪物 → 直接连玩家
怪物 → 直接连成就
怪物 → 直接连任务
怪物 → 直接连金币
怪物不认识任何人
玩家不认识怪物
中间只有事件在传递消息
完全解耦，随便扩展
最生活化的比喻
不用事件：
你要喊人，必须一个个打电话。
用事件：
你在群里发一句 “我死了”
谁关心这件事，谁就自动收到，不用你一个个通知。
这个群，就是事件




事件一触发，所有订阅的方法自动执行

事件和普通委托的核心区别

普通委托（Action / 自定义 delegate）
可以被外部随便赋值
可以被外部直接覆盖 = null
不安全，容易被误改

事件（event）
只能 += 订阅、-= 取消订阅
外部不能直接赋值、不能覆盖
安全、规范、不会被乱改
事件 = 被保护起来的安全委托

三、事件完整结构
1. 定义一个 “事件管理器”（全局静态）
// 全局静态类，放所有游戏事件
public static class GameEvent
{
    // 定义委托（也可以直接用 Action）
    public delegate void MonsterDeadDelegate();

    // 声明事件！前面加 event 关键字
    public static event MonsterDeadDelegate OnMonsterDead;

    // 简化写法（不用自己写委托）
    // public static event Action OnMonsterDead;
}


四、怎么订阅事件？
玩家类：关心怪物死亡
那就玩家来订阅
一般在构造函数里面来订阅一次
class Player
{
    public int Gold { get; set; }
    public int KillCount { get; set; }

    public Player()
    {
        // 订阅事件：怪物死了 → 执行我的方法
        GameEvent.OnMonsterDead += AddGold;
        GameEvent.OnMonsterDead += AddKillCount;
    }

    void AddGold()
    {
        Gold += 100;
        Console.WriteLine("金币+100");
    }

    void AddKillCount()
    {
        KillCount++;
        Console.WriteLine("杀怪数+1");
    }
}

五、怎么触发事件？（谁发生谁调用）
怪物类：死的时候触发事件
class Monster
{
    public void Die()
    {
        Console.WriteLine("怪物死亡！");

        // 触发事件（固定写法）
        GameEvent.OnMonsterDead?.Invoke();
    }
}
