## 学习风格与讲解偏好

### 核心学习特点
- 喜欢从基础概念入手，逐步深入底层原理，不满足于只记用法，追求理解本质
- 偏好结合实际场景（尤其是**游戏开发/Unity**）学习，用具体案例理解抽象知识点
- 习惯按步骤递进学习，学完一个点再进入下一个，层层递进、逐步构建完整体系

### 学习新知识的固定步骤
1. **理解是什么**（概念、作用、基本结构）
2. **理解怎么用**（基础语法、简单示例）
3. **理解为什么**（底层原理、执行流程、机制）
4. **学习注意事项与坑点**（常见错误、禁止用法、边界情况）
5. **学习高级用法**（组合使用、扩展功能）
6. **通过综合练习题把所有知识点串起来实践**
7. **总结成结构化笔记，形成可复用的知识体系**

### 讲解偏好
- 讲解由浅入深、循序渐进，**不跳步**
- 多用通俗类比 + 真实场景示例（**重点结合游戏开发/Unity 场景**）
- 关键知识点需要拆解开细讲，直到完全理解
- 最后需要总结、梳理结构，方便记忆与复习
- 讲解反射时，要重点说明在 Unity 游戏开发中的实际应用场景
- **分步输出**：每次只输出当前步骤的内容，用户主动说"下一步"或"继续"才输出下一步，不一次性全量输出
- **语法讲解偏好**：每个关键词/方法都要**逐个详细举例**，不能一笔带过；同一个关键词的所有重载用法要全部讲完；有相同功能的关键词需要**放在一起对比**，讲清楚差异和各自适用场景
- **举例原则**：每一个知识点讲解时，先用**具体可运行的完整小例子**说明用法，再对比差异，最后总结适用场景
