## 一、功能概述

玩家可驾驶车辆在游戏世界中移动。核心特性：WheelCollider 物理模拟、IInteractable 上下车交互、驾驶时摄像机切换、SHIFT 加速双档、防侧倾杆+翻滚稳定、油耗系统。

## 四、核心实现细节

### 4.1 VehicleController — 物理驱动

`Awake()` 获取 Rigidbody。`Start()` 调用 `ApplyConfig()`，从 `VehicleData` SO 加载全部参数，设置4个轮子悬挂阻尼，降低重心 `centerOfMassYOffset`。

**ApplyWheelPhysics()**（每 FixedUpdate）：
- 仅前轮设置 `steerAngle`
- 油门>0 → `motorTorque`；油门<0 → `reverseTorque`
- 刹车>0.01 → 清零 motorTorque，只设 brakeTorque（刹车优先）
- 加速时油耗×1.5
- 油尽 → `_isEngineOn = false`（引擎熄火）

**ClampSpeed()**：
- `EffectiveMaxSpeed = maxSpeed × (isBoosting ? boostSpeedMultiplier : 1)`
- `_rb.velocity.magnitude > EffectiveMaxSpeed` → 归一化速度 × 上限

**ApplyAntiRoll()**：
- 前后桥分别计算：`(travelL - travelR) × antiRollStiffness`
- 差异越大，反向力越大，抑制侧倾

**ApplyRollStabilization()**：
- 本地空间角速度阻尼：X轴(俯仰)×0.95, Z轴(侧倾)×0.92
- Y轴(偏航)保持自由，不阻碍转弯

### 4.2 VehicleInteraction — 上下车

**Enter(GameObject driver)**：
1. 保存 driver.Rigidbody → isKinematic=true, Collider → enabled=false
2. driver.SetParent(vehicle, driverSeatOffset)
3. controller.SetDriver(driver) → isEngineOn=true
4. EventBus.Publish(VehicleEnteredEvent)

**Exit()**：
1. 恢复 driver.Rigidbody → isKinematic=false, Collider → enabled=true
2. driver.SetParent(null), driver.position = exitOffset(世界空间)
3. controller.SetDriver(null) → isEngineOn=false, StopImmediate()
4. EventBus.Publish(VehicleExitedEvent)

### 4.3 VehicleInputLock — 输入接管

- 通过永久性 `Update()` 工作——每次进入/离开车辆时增加/减少自身启用状态
- 驾驶时每帧将 WASD/Space/Shift 映射到 `VehicleController` 的 API
- 通过禁用组件来控制，而非覆盖输入系统——`PlayerController` 根本不会运行，杜绝冲突

### 4.4 加速系统

SHIFT 不会增加扭矩。它会提高最高速度上限：`EffectiveMaxSpeed = maxSpeed × boostSpeedMultiplier`。加速时油耗乘以 `boostFuelMultiplier`（1.5x）。这是一种简单的"两档"方案，而不是连续动力提升。

### 4.5 编辑器工具 (CreateVehicleData)

全面的预制体搭建工具：
1. 从 Mesh 包围盒读取模型尺寸
2. 按比例缩放至目标车长（4.5m）
3. 在本地空间计算位置，在 4 个角落自动创建 `WheelCollider` 子物体
4. 添加 `Rigidbody`、`BoxCollider`、`VehicleController`、`VehicleInteraction`
5. 从计算出的尺寸填充碰撞体中心/尺寸、座位/出口偏置等 VehicleData 字段

车轮放置算法修订至第 7 版，以处理由模型导入旋转（待修复：X=-90 度）导致的 `LookRotation` 退化问题。

## 八、设计经验

- **组件禁用优于输入屏蔽**：`VehicleInputLock` 直接禁用 `PlayerController` 等组件，而非在它们内部检查"是否在驾驶"。好处是零冲突——驾驶时组件根本不在 `Update()` 中运行。
- **物理节流**：加速（SHIFT）通过提高限速来实现，而非增加扭矩。避免了速度过高后扭矩停止但玩家感受不到反馈的问题。
- **两阶段防侧翻**：`AntiRoll`（悬挂硬连接）+ `RollStabilization`（角速度阻尼）互补。仅靠防侧倾杆不足以处理急弯；仅靠阻尼会产生漂浮感。两者兼备则既有抓地力又有重量感。
- **编辑器工具价值**：从包围盒自动计算碰撞体和轮位，省去每次加新车型的手动调试。
