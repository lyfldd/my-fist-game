## 三、玩家无人机（普通）

### 3.1 使用流程

```
背包右键"无人机" → 部署 → 出现在面前地面上
背包右键"操控终端" → 打开操控界面
地面无人机 E键 → 收回背包
```

### 3.2 状态机

```
背包 ──右键部署──→ 地面待命 ──终端[起飞]──→ 飞行中
                                                │
                    电量≤1h ──→ 强制返航 ──→ 降落 ──→ E键回收
```

### 3.3 操控终端面板

```
┌─ 无人机操控终端 ───────────────────┐
│  ┌─────────────────────────────┐   │
│  │     📷 向下俯瞰画面           │   │
│  └─────────────────────────────┘   │
│  电量: [████████░░] 4.2h/6h       │
│  高度: 30m [▲][▼]  最大: 50m      │
│  自动攻击: [ON]/[OFF]  [手动开火]  │
│  [起飞] [返航] [降落] [⚠紧急回收]  │
│  ⚠ 超出范围将自动返回 / 紧急回收无视状态强制返航 │

### 3.4 参数

| 参数 | 值 |
|------|:--:|
| 总电量 | 6 游戏小时 |
| 返航预留 | 1 小时 |
| 最大高度 | 50m |
| 同时飞行 | 1 架 |
| 活动范围 | 离终端 150m |
| 武器 | 轻机枪 |
| 被击落 | 直接销毁 |
| 回收 | E键回背包 |

> **为什么同时只能飞 1 架？** 操控终端与无人机 1:1 绑定——一块芯片只能维持一条加密遥测链路。多架同时飞会导致信号串扰（玩家操控界面同时显示多个画面也会混乱）。如需多架协同，使用 AIBot 无人机仓（见 §八）。

#### 手动操控 vs 自动攻击

> **规则**: 手动操控状态下，自动攻击**暂停**。驾驶者完全控制武器触发。

| 模式 | 自动攻击 | 手动开火 | 说明 |
|------|:--:|:--:|------|
| 自动巡航 | ✅ 每 0.5s Tick | ❌ | AI 自主选目标 |
| 手动操控 | ❌ 暂停 | ✅ 左键射击 | 玩家完全接管 |
| 退出操控→回巡航 | ✅ 恢复 | ❌ | 自动恢复 Tick |

### 3.4.2 受损状态系统

| 条件 | 效果 |
|------|------|
| HP < 50% | 飞行速度降 20%（8→6.4 m/s）、orbitSpeed 降到 20°/s |
| HP < 50% | 终端 UI 显示红字警告 ⚠ "无人机受损" |
| 返航回收后 | 需修复：材料"机体碎片×2 + 电池组×1"，耗时 5s（精密装配台） |

> 修复后才能再次部署。受损不修直接再飞 → 从 50% HP 开始，更容易被击落。

### 3.5 自杀式无人机

> 炸药包改装无人机，一次性的空中炸弹。右键背包直接部署，无需终端。

#### 部署方式

```
背包右键"自杀式无人机" → 出现在面前地面上（同普通无人机）
面向无人机按 E → 锁定目标模式
  ├── 有地图标记点 → 优先锁定标记点坐标（E 键快速打击指定位置）
  ├── 锁定最近僵尸（3秒自动锁定）→ 蜂鸣升空 → 冲向目标 → 碰撞引爆
  └── 无目标 → 手动 WASD 遥控飞行（10秒后未引爆 → 自动降落回收）
```

**设计理由**：不需要操控终端（太繁琐），E 键一键锁定。无终端时也能用——末日场景下单兵携带，扔出去就炸。**按 E 时优先选地图标记点**（玩家右键地图设的标记），实现"标记→扔无人机→炸"的快速打击链。

#### 飞行模式

| 模式 | 行为 |
|------|------|
| 锁定追击 | 锁定目标后升空到 15m，直线飞向目标，碰撞引爆 |
| 标记点打击 | 锁定玩家地图标记点坐标（XZ），升空 20m 俯冲轰炸（有高度→爆炸半径+20%） |
| 遥控飞行 | 无目标时 WASD 手动遥控（最远 80m），`[空格]` 手动引爆，`[C]` 设 5 秒倒计时 |
| 倒计时引爆 | 遥控模式下按 `[C]` → 5 秒倒计时 → 自动引爆（屏幕显示倒计时，可 `[X]` 取消） |
| 超时回收 | 起飞后 10 秒未引爆 → 自动降落回地面 → E 键回收（不销毁） |

#### 爆炸参数

| 参数 | 值 |
|------|:--:|
| 爆炸伤害 | 300（中心）→ 50（边缘） |
| 伤害半径 | 8m（中心全额）→ 15m（边缘衰减） |
| 破片穿透 | 无（纯爆炸，不生成破片） |
| 对建筑伤害 | 2×（对门窗/路障有破坏力） |
| 噪音半径 | 80m（爆炸巨响，吸引全图僵尸） |
| 引爆方式 | 碰撞引爆（撞到僵尸/地面/建筑）/ 空格手动引爆 |
| 触发距离 | 距目标 < 1m 时引爆 |

#### 销毁规则

| 情况 | 结果 |
|------|------|
| 命中僵尸爆炸 | ✅ 炸了 — 销毁 |
| 命中地面/建筑 | ✅ 炸了 — 销毁 |
| 被远程火力击落（飞行中） | ⚠️ 空中爆炸 — 伤害半额（150），半径减半（7.5m） |
| 遥控手动引爆 | ✅ 立即爆炸 — 全额伤害 |
| 超时回收 | ✅ 降落后 E 键收回背包（恢复为"自杀式无人机"物品，不消耗） |
| 地面待命时被攻击 | ❌ 被毁 — 不爆炸（炸药包需要引信触发） |

## 四、AI无人机（AIBot专属）

### 4.1 核心规则

| 规则 | 说明 |
|------|------|
| 来源 | 造AIBot自带1架，不消耗物品 |
| 数量 | 1架/AIBot |
| 活动范围 | 离AIBot 100m |
| 被击落 | 自动飞回AIBot修复，冷却分级（轻度15s/重度45s） |
| AIBot死亡 | 无人机立即停用 → 帧末销毁 | （见下方销毁时序）
| 噪音 | 50m，投影到地面点（僵尸走向地面投影） |

> **AIBot 死亡销毁时序**（防"僵尸帧"）：Unity `Destroy()` 标记到帧末才执行，这一帧内无人机还能飞/攻击/Tick。
> ```csharp
> // AIBot.OnDeath()
> void OnDeath()
> {
>     // 1. 先停掉所有逻辑
>     if (_droneBay != null)
>     {
>         _droneBay.enabled = false;
>         _droneBay.aiDrone.enabled = false;
>         foreach (var d in _droneBay.deployedNormalDrones)
>             d.enabled = false;
>     }
>     // 2. 帧末销毁
>     Destroy(_droneBay?.gameObject);
> }
> ```

### 4.2 能量系统

```
独立电池 + 铀（容量各为AIBot的 1/4）
伤害/耗能/冷却 = AIBot的 1/4 等比缩放
停靠在AIBot上 → 自动从AIBot能量池补充
能量耗尽 → 强制返航 → 切断一切攻击/操控逻辑 → 只飞回AIBot
```

四能源模式照搬 AIBot：

| | 节能 | 电力 | 铀 | 爆发 |
|------|------|------|------|------|
| 速度 | 0.75x | 1x | 1.25x | 1.5x |
| 消耗倍率 | 0.5x | 1x | 1x | 1.25x |
| 激光伤害 | ❌禁用 | 基准 | 1.5x | 1.87x |
| 冷却倍率 | 0.5x | 1x | 2x | 2.5x |
| 能量源 | 电池 | 电池 | 铀 | 铀 |

耗能/伤害按 AIBot 等比例缩小。

### 4.3 双重锥形：警戒 + 攻击

```
             📷 无人机 (俯视)
             │
        ╔═════╧═════╗
        ║ 警戒锥 120° ║  ← 探测范围，检测僵尸
        ║  ┌─────┐   ║
        ║  │攻击锥│   ║  ← 60°，实际伤害范围
        ║  │ 60° │   ║
        ║  └─────┘   ║
        ╚════════════╝
             ▼ 地面
    ┌──────────────────┐  ← 警戒投影 = 大圆
    │    ┌────────┐    │
    │    │ 攻击区 │    │  ← 攻击投影 = 小圆
    │    └────────┘    │
    └──────────────────┘
    高度越高投影越大
```

| 锥形 | 角度 | 用途 | 检测方式 |
|------|:--:|------|------|
| 警戒锥 | 120° | 僵尸进入 → 触发自动攻击 | OverlapSphere + 锥角过滤 |
| 攻击锥 | 60° | 激光命中范围，造成伤害 | 锥角过滤 + 伤害结算 |

**逻辑流程**（每 0.5s Tick，与 AIBotCombat 同频）：
1. `Physics.OverlapSphereNonAlloc` 扫描警戒锥地面投影范围内的僵尸
2. 过滤：有 `DamageableZombie` 且未死亡
3. 按距离排序（优先 AIBot 当前目标 → 其次最近僵尸）
4. 锁定目标 → 发射激光 → 命中攻击锥内所有僵尸

### 4.4 仇恨系统接入

> **v5.1**: 接入 ThreatSystem。AIBot 和无人机共享威胁表，避免重复锁定同一目标。

#### 编队记忆最近攻击者

```
无人机受击 → ThreatReportEvent → ThreatSystem 记录
  → AIDrone.OnDamaged() 读取 ThreatSystem.GetTopThreats(3)
  → 优先锁定最近攻击过编队的敌人（不局限在 AIBot 当前目标）
```

| 方法 | 说明 |
|------|------|
| `ThreatSystem.GetTopThreats(int count)` | 返回最高威胁的前 N 个目标 ID |
| `ThreatSystem.GetTopThreatInCone(Vector3 origin, Vector3 dir, float angle)` | 攻击锥内最高威胁目标 |
| `AIBotCombat.CurrentPrimaryTarget` | AIBot 最高优先级武器的当前目标（协同集火用） |

#### 目标选择优先级

```
1. AIBot 主目标在攻击锥内 → 优先帮打（协同集火）
2. 编队最近攻击者（ThreatSystem 记忆）在攻击锥内 → 反击
3. 扫描附近僵尸 → 选最近未锁定（每目标最多被 3 架锁定）
4. 没有剩余 → 兜底打最近的
```

### 4.6 三种巡航模式

#### 巡航通用逻辑

```
圆心 XZ 平面做圆周运动：

angle = time × orbitSpeed (30°/s)
offset = (cos(angle) × radius,  0,  sin(angle) × radius)
targetPos = orbitCenter + offset,  y = cruiseHeight
transform → MoveTowards(targetPos, flySpeed × dt)
```

| 参数 | 说明 |
|------|------|
| `orbitCenter` | 巡航圆心（地面XZ） |
| `orbitRadius` | 巡航半径 |
| `cruiseHeight` | 巡航高度 |
| `orbitSpeed` | 30°/s（12秒一圈） |

#### 三种模式对比

| | 跟随 | 警戒 | 定点 |
|------|:--:|:--:|:--:|
| 圆心 | AIBot 位置 | AIBot 位置 | 玩家指定位置 |
| 半径 | **小圆** 5-8m | **大圆** 20-30m | 10-15m |
| 高度 | 10-15m | 15-25m | 玩家设定 |
| 跟随 AIBot | ✅ 圆心每帧更新 | ✅ 圆心每帧更新 | ❌ 圆心固定 |
| 自动攻击 | ✅ | ✅ | ✅ |
| 用途 | 贴身护航 | 广域侦察/预警 | 区域封锁 |

- **跟随**: 小半径护航，圆心绑定 AIBot，AI 移动无人机跟着移动
- **警戒**: 大半径巡逻，圆心绑定 AIBot，覆盖更广提前发现威胁
- **定点**: 圆心锁定地图某点，不跟 AIBot 走

### 4.8 寻路逻辑

无人机**空中飞行**，不需要 NavMesh（地面寻路），使用**直线插值移动**：

```
当前位置 ────→ 目标位置 (直线，无地面障碍)
```

| 参数 | 值 | 说明 |
|------|:--:|------|
| 正常飞行速度 | 8 m/s | 巡航 + 跟随 |
| 返航速度 | 12 m/s | 能量耗尽/被击落返航时加速 |
| 最小高度 | 5m | 不能低于此高度 |
| 最大高度 | 50m | |

对比 AIBot：`NavMeshAgent.SetDestination()` 地面路径 + 避障；无人机：`Vector3.MoveTowards()` / `SmoothDamp` 直线飞行。

#### 避障逻辑

无人机飞行路线可能穿过高层建筑。检测前方障碍物 → 自动爬升绕开：

```csharp
// AIDrone.Update() 每帧检测
Vector3 forward = (targetPos - transform.position).normalized;
if (Physics.Raycast(transform.position, forward, out hit, 10f, buildingLayer))
{
    // 检测到前方建筑 → 自动爬升 5m
    targetHeight = Mathf.Min(currentHeight + 5f, maxHeight);
}
// 绕过建筑后 → 恢复原巡航高度
if (!Physics.Raycast(transform.position, forward, 10f, buildingLayer))
    targetHeight = cruiseHeight;
```

### 4.9.1 AIDroneBay — 无人机停靠仓

> **职责边界**：AIDroneBay 只管停靠仓的充放电/修复冷却/释放召回/编队协调，不管飞行逻辑。

#### 职责划分

```
AIDrone（无人机本体）                 AIDroneBay（停靠仓）
├── 飞行/攻击/状态机/HP/受击          ├── 充放电管理
├── RequestDock() → 通知 Bay          ├── 修复冷却计时
├── RequestLaunch() → 通知 Bay        ├── 释放/召回逻辑
└── 不管理能量补充（由 Bay 管）       ├── 编队协调（多架调度）
                                      └── OnRepairComplete() / OnEnergyRefilled() → 通知 Drone
```

#### 接口定义

```csharp
// AIDroneBay.cs
public class AIDroneBay : MonoBehaviour
{
    [Header("停靠仓")]
    public int maxNormalDrones = 3;         // 普通无人机仓容量
    public Transform dockPoint;              // 停靠位置（AIBot 头顶）
    public float repairCooldownLight = 15f;  // 轻度受损冷却（HP>50%）
    public float repairCooldownHeavy = 45f;  // 重度受损冷却（HP≤50%）

    // 当前停靠状态
    public AIDrone aiDrone { get; private set; }          // AI无人机（专属）
    public List<Drone> dockedNormalDrones;                 // 停靠的普通无人机
    public List<Drone> deployedNormalDrones;               // 已释放的普通无人机

    // ═══ AI无人机接口 ═══
    public void LaunchAI();                                // 释放 AI 无人机
    public void RecallAI();                                // 召回 AI 无人机
    public bool IsAIDocked { get; }                        // AI 是否停靠中
    public float RepairProgress { get; }                   // 修复进度 0-1

    // ═══ 普通无人机编队接口 ═══
    public void LaunchSquad();                             // 编队全体释放
    public void RecallSquad();                             // 编队全体召回
    public void LaunchSingle(int index);                   // 释放单架
    public void RecallSingle(int index);                   // 召回单架
    public bool CanLaunch { get; }                         // 编队前提：AI 在线

    // ═══ 充电接口 ═══
    public void StartCharging(AIDrone drone);              // 开始给 AI 无人机充电
    public void StopCharging();                            // 停止充电
    public float ChargeRate { get; }                       // 当前充电速率

    // ═══ 修复接口 ═══
    public void StartRepair(AIDrone drone);                // 开始修复（盾+HP）
    public void OnRepairComplete();                        // 修复完成回调

    // ═══ 事件 ═══
    public System.Action<AIDrone> OnDroneLaunched;
    public System.Action<AIDrone> OnDroneDocked;
    public System.Action OnRepairFinished;
    public System.Action OnEnergyFull;
}
```

#### 释放流程（LaunchAI）

```
AIBot 面板点 [释放]
  → AIDroneBay.LaunchAI()
    → 检查 AI 无人机是否已停靠 (IsAIDocked == true)
    → 检查能量/HP 是否足够（HP>0, 能量>10%）
    → 实例化或激活 AIDrone: drone.Activate(dockPoint.position, dockPoint.rotation)
    → AIDrone 状态机: Docked → Flying(Following)
    → OnDroneLaunched?.Invoke(drone)
```

#### 召回流程（RecallAI）

```
AIBot 面板点 [召回]
  → AIDroneBay.RecallAI()
    → 通知 AIDrone: drone.TransitionTo(AIState.ReturningGentle)
    → AIDrone 飞回 dockPoint
    → 到达检测: Vector3.Distance(drone.position, dockPoint.position) < 2m
    → AIDrone 状态机: ReturningGentle → Docked
    → AIDroneBay 接收停靠: drone.Deactivate()
    → 开始充电: StartCharging(drone)
    → 如果需要修复: StartRepair(drone)
    → OnDroneDocked?.Invoke(drone)
```

#### 充放电管理

```
Update() 每帧 Tick:
  if (IsAIDocked && aiDrone.energy < aiDrone.maxEnergy)
    → 从 AIBot 能量池抽取
    → 速率: AIBot 当前能量 × 2%/秒（最多抽出当前能量的 2%/s）
    → AIBot 能量 < 20% : 停止给无人机充电（保护 AIBot）
    → 充电速率写入 aiDrone.energyCurrent

  充满时间: 50s（0→100%，假设 AIBot 能量充足）
```

#### 修复冷却（分级）

```
被击落返航停靠 → 检查受损程度:
  ├── HP > 50%（轻度受损）→ repairTimer = 15s
  └── HP ≤ 50%（重度受损）→ repairTimer = 45s

StartRepair()
  → Update() 每帧: _repairTimer -= dt
  → _repairTimer <= 0 → OnRepairComplete()
    → aiDrone.shield = 500 (回满)
    → aiDrone.hp = 100 (回满)
    → OnRepairFinished?.Invoke()
    → 可以再次释放
```

> **设计意图**：轻度受损 = 盾破了但本体没怎么受伤，快修（15s）。重度受损 = 本体被打残了，需要更长时间（45s）。鼓励玩家在盾破时及时召回，不要扛到本体受损。

#### 编队协调（普通无人机 ×3）

```
LaunchSquad():
  if (!CanLaunch) return;  // AI 不在线 → 不能释放
  for each dockedNormalDrones:
    drone.Activate() → 编队模式启动
    drone.SetFormation(index, total) → 分配相位+高度

RecallSquad():
  for each deployedNormalDrones:
    drone.TransitionTo(Returning)
  → 全部停靠后开始充电

普通无人机充电源: 仅 AIBot 电池（无铀），充电速率同 AI 无人机
```

// GameEvents.cs — 无人机事件

public readonly struct DroneDeployed
{
    public int DroneId { get; }          // 无人机 instanceID
    public int OwnerId { get; }          // 拥有者 instanceID（AIBot 或 Player）
    public Vector3 Position { get; }     // 释放位置
    public bool IsAIDrone { get; }       // true=AI无人机, false=普通

    public DroneDeployed(int droneId, int ownerId, Vector3 pos, bool isAI)
        => (DroneId, OwnerId, Position, IsAIDrone) = (droneId, ownerId, pos, isAI);
}

public readonly struct DroneRecalled
{
    public int DroneId { get; }
    public int OwnerId { get; }
    public bool IsForced { get; }        // true=强制返航(能量耗尽/HP=0), false=温和(玩家召回)

    public DroneRecalled(int droneId, int ownerId, bool forced)
        => (DroneId, OwnerId, IsForced) = (droneId, ownerId, forced);
}

public readonly struct DroneExploded
{
    public Vector3 Position { get; }     // 爆炸中心
    public float Radius { get; }         // 伤害半径 (15m)
    public float MaxDamage { get; }      // 中心伤害 (300)
    public float NoiseRadius { get; }    // 噪音半径 (80m)

    public DroneExploded(Vector3 pos, float radius, float maxDmg, float noise)
        => (Position, Radius, MaxDamage, NoiseRadius) = (pos, radius, maxDmg, noise);
}

public readonly struct DroneDamaged
{
    public int DroneId { get; }
    public int AttackerId { get; }
    public float Damage { get; }
    public float RemainingHP { get; }
    public bool IsShieldActive { get; }     // 能量盾是否还在

    public DroneDamaged(int droneId, int attackerId, float dmg, float hp, bool shield)
        => (DroneId, AttackerId, Damage, RemainingHP, IsShieldActive)
        = (droneId, attackerId, dmg, hp, shield);
}

public readonly struct DroneOutOfRange
{
    public int DroneId { get; }
    public int OwnerId { get; }
    public Vector3 DronePosition { get; }

    public DroneOutOfRange(int droneId, int ownerId, Vector3 pos)
        => (DroneId, OwnerId, DronePosition) = (droneId, ownerId, pos);
}

### 4.12 手动操控 vs 自动攻击

> **规则**: 手动操控状态下，自动攻击**暂停**。驾驶者完全控制武器触发。

| 模式 | 自动攻击 | 手动开火 | 说明 |
|------|:--:|:--:|------|
| 自动巡航 | ✅ 每 0.5s Tick | ❌ | AI 自主选目标 |
| 手动操控 | ❌ 暂停 | ✅ 左键射击 | 玩家完全接管 |
| 退出操控→回巡航 | ✅ 恢复 | ❌ | 自动恢复 Tick |

```
自动巡航中 ──[进入手动操控]──→ 自动攻击暂停
                                   │
                    左键射击 → 手动单发（玩家瞄准）
                    退出操控 → 自动攻击恢复
```

## 八、v5.0 新增：AIBot无人机仓 + 多无人机编队

> **日期**: 2026-06-01 · **状态**: 📋 设计定稿

### 8.1 架构

```
手持终端 (严格1架)              AIBot 无人机仓 (3普通 + 1AI)
├── 普通无人机 ×1               ├── AI无人机 ×1 (专属，精英)
│                               └── 普通无人机 ×3 (消耗品，编队)
│
├─ 右键终端→操控界面             AI无人机 = 编队"大脑"
├─ 手动操控：WASD+高度滑块       没有AI无人机在线 → 仓内普通无人机不能起飞
└─ 被击落：直接销毁              AI无人机离线 → 仓内普通无人机停靠等待
```

### 8.2 无人机仓机制

| 参数 | 值 |
|------|:--:|
| 仓容量 | **3 架** 普通无人机 |
| 装填方式 | 从背包拖"无人机"物品到仓格 |
| AI无人机 | 专属槽位，不占仓 |
| 编队前提 | AI无人机必须在线 |
| AIBot死亡 | 仓内+空中所有立即销毁 |

### 8.3 编队指令

| 指令 | 行为 |
|------|------|
| 跟随 | 小半径(5-8m)环绕 AIBot，高度分层错开 |
| 警戒 | 大半径(20-30m)分散巡逻，每人覆盖一个扇形 |
| 定点 | 编队锁定地图某点上空巡航 |
| 进攻 | 优先AIBot主目标→各自就近选未被锁定的僵尸 |
| 停火 | 只飞不输出 |
| 全员返航 | 全部返回AIbot仓 |

### 8.4 空间分离

多架自动错开相位 + 不同高度层：
`phaseOffset[i] = i × 360° / N + random(-10°, 10°)`
`height[i] = 10 + i × 6 + random(-1m, 1m)`

#### 高度分层 vs 攻击锥地面覆盖率

60° 攻击锥在不同高度下的地面投影半径（`R = H × tan(30°)`）：

| 高度 | 攻击锥地面半径 | 覆盖面积 | 特性 |
|:----:|:------------:|:------:|------|
| 10m | ≈ 17m | ~900m² | 低空精确打击，适合贴身护航 |
| 16m | ≈ 28m | ~2,400m² | 中等覆盖，均衡型 |
| 22m | ≈ 38m | ~4,500m² | 高空广域覆盖，几乎全覆盖 |

> **设计意图**：这是**有意为之的特性，不是 bug**。三个高度层形成互补：
> - **低空(10m)**：攻击半径小 → 精准点杀 → 适合跟随护航模式（打 AIBot 身边的僵尸）
> - **中空(16m)**：中等覆盖 → 均衡 → 警戒模式的默认层
> - **高空(22m)**：广域覆盖 → 压制大群 → 适合定点防守/大范围清场
>
> "越低越强"的直觉在巡航半径下不成立——低空的优势是**精确**（不打到友军），高空的优势是**覆盖**（不漏掉敌人）。编队三层互补，而非简单的高低强弱对比。

### 8.5 轻量目标分配

每架独立Tick，`Dictionary<Zombie, int>` 记录被锁定次数：
1. AIBot主目标在攻击锥内→优先锁定
2. 扫描附近僵尸，过滤已被友军锁定的
3. 选最近未锁定僵尸
4. 没有剩余→兜底打最近的

| 参数 | 值 | 说明 |
|------|:--:|------|
| 精英锁定上限 | 3 | 同一僵尸最多被 3 架无人机同时锁定（AI+2普通） |
| 普通锁定上限 | 2 | 同一僵尸最多被 2 架普通无人机同时锁定 |

> 上限防止 4 架全打同一个僵尸浪费火力。超过上限 → 自动找下一个目标。

### 8.6 编队无人机 vs AI无人机

| | AI无人机 | 普通无人机(编队) |
|------|:--:|:--:|
| 数量 | 1 | 最多3 |
| 能源 | 电池+铀，4模式 | 仅电池 |
| 能量盾 | 有(500HP，先吸收伤害) | 无 |
| 被击落 | 飞回修复30s | 直接销毁 |
| 噪音 | 50m | 30m |
| 武器 | 激光 | 轻机枪 |
| 行为 | 独立AI | 编队共享指令 |
| 定位 | 精英单位 | 消耗品 |
