## 一、架构总览

双层架构：**FactionSystem（静态阵营关系）** + **ThreatSystem（动态仇恨表）**。

```
┌─────────────────────────────────────────────┐
│              FactionSystem (静态层)           │
│  阵营关系矩阵: 7阵营×7阵营                     │
│  IsAlly() / IsHostile() / IsNeutral()        │
│  FactionData SO → 默认关系 → 运行时 SetRelation│
│  支持存档 delta 导出/导入                      │
└──────────────┬──────────────────────────────┘
               │ 查询阵营关系
               ▼
┌─────────────────────────────────────────────┐
│             ThreatSystem (动态层)             │
│  Dict<int, Dict<int, ThreatEntry>>          │
│  仇恨表 + 权重排序 + 衰减 + 距离加速            │
│  友军误伤×0.1 / Neutral临时敌对30s            │
│  每实体最多5目标 / 分帧GC清理死引用            │
└──────────────┬──────────────────────────────┘
               │ OnEnable 自注册
               ▼
┌─────────────────────────────────────────────┐
│   FactionComponent  +  InstanceRegistry      │
│  每个实体挂 FactionComponent                 │
│  InstanceRegistry: InstanceID→Transform 映射  │
│  静态轻量查找表，供位置查询和GC存活判定          │
└─────────────────────────────────────────────┘
```

## 三、FactionSystem（静态阵营层）

**文件**: `Assets/_Game/Systems/Threat/FactionSystem.cs`

MonoBehaviour 单例，挂载在场景持久 GameObject 上。注册到 `ServiceLocator`。

### FactionData (ScriptableObject)

**文件**: `Assets/_Game/Config/FactionData.cs`

```csharp
[CreateAssetMenu(menuName = "Game/Faction Data", fileName = "Faction_")]
public class FactionData : ScriptableObject
{
    public FactionType factionType;
    public string displayName;
    public Color factionColor = Color.white;
    public FactionType[] allies;      // 盟友列表
    public FactionType[] hostiles;    // 敌对列表
    // 未在两者中的 = 中立
}
```

### 初始化流程

1. `Awake()` → `LoadDefaults()`
2. 尝试从 `Resources/Factions/` 加载所有 `FactionData` SO
3. 解析 SO 中的 `allies[]` / `hostiles[]` 填入 `_relations` 字典
4. 若无 SO（运行时），调用 `EnsureDefaultRelations()` 硬编码默认关系
5. 遍历所有 7×7 阵营组合，未显式声明的补为 `Neutral`
6. 关系是**双向对称**存储的：`_relations[(a,b)]` 和 `_relations[(b,a)]` 同时写入

### 运行时查询

```csharp
bool IsHostile(FactionType a, FactionType b)
bool IsAlly(FactionType a, FactionType b)
bool IsNeutral(FactionType a, FactionType b)
```

### 运行时变更与事件通知

```csharp
public void SetRelation(FactionType a, FactionType b, FactionRelation relation)
```

- 双向写入 `_relations[(a,b)]` 和 `_relations[(b,a)]`
- 发布 `FactionRelationChangedEvent`，ThreatSystem 订阅：
  - 变为 **Ally** → 清除涉及该对阵营的所有现有仇恨条目
  - 变为 **Neutral** → 现有仇恨值衰减 50%
  - 变为 **Hostile** → 保持不变

### 存档系统接口

```csharp
// 导出运行时修改过的阵营关系（只存与默认值不同的）
List<FactionDeltaSaveData> GetFactionDeltas()

// 从存档恢复阵营关系变更
void ApplyDeltas(List<FactionDeltaSaveData> deltas)
```

`FactionDeltaSaveData` 包含三个字符串字段：`factionA`, `factionB`, `relation`。

## 四、ThreatSystem（动态仇恨层）

**文件**: `Assets/_Game/Systems/Threat/ThreatSystem.cs`

MonoBehaviour 单例。每帧 `Update()` 执行衰减、临时敌对计时器扣减、分帧 GC。`LateUpdate()` 批量清理死亡实体。

### 核心数据结构

```csharp
// 外层 key = sourceId（谁持有仇恨）
// 内层 key = targetId 或 0（0=sentinel，表示只知道位置不知道具体目标）
Dictionary<int, Dictionary<int, ThreatEntry>> _table;

// 实体 → 阵营映射（RegisterEntity 时写入）
Dictionary<int, FactionType> _entityFactions;

// Neutral 临时敌对计时器：(实体A, 实体B) → 剩余秒数
Dictionary<(int, int), float> _tempHostileTimers;
```

### 可配置参数

| 参数 | 默认值 | 说明 |
|---|---|---|
| `_maxThreatPerTarget` | 500 | 单目标仇恨值硬上限 |
| `_maxTrackedTargets` | 5 | 每个实体同时追踪的最大目标数 |
| `_friendlyFireMultiplier` | 0.1 | 友军伤害仇恨倍率（误伤几乎不记仇） |
| `_perceptionRange` | 30m | 感知距离，用于距离衰减加速判定 |

### 核心 API

```csharp
// 注册/注销
void RegisterEntity(int instanceId, FactionType faction)
void UnregisterEntity(int instanceId)     // 清"它恨谁" + "谁恨它" + 阵营 + 临时敌对

// 核心：添加仇恨
// targetId=null → sentinel条目（如 AllyAlert），AIAgent走入位置观察而非追击
void AddThreat(int sourceId, int? targetId, float value, ThreatType type, Vector3? position = null)

// 查询
int?  GetTopTarget(int sourceId)          // 按WeightedValue排序（含sentinel）
int?  GetTopRealTarget(int sourceId)      // 仅返回有targetInstanceId的目标（AIAgent Combat用）
List<ThreatEntry> GetAllThreats(int sourceId) // WeightedValue降序
float GetThreatValue(int sourceId, int targetId)   // 裸值
float GetWeightedValue(int sourceId, int targetId) // 加权值
bool  HasThreat(int sourceId)
Vector3? GetLastKnownPosition(int sourceId, int targetId)

// 清除
void ClearThreat(int sourceId, int targetId)
void ClearAllThreats(int sourceId)
```

### AddThreat 完整逻辑

1. **自伤保护**: `sourceId == targetId` 且非 sentinel → 直接返回
2. **阵营系数计算**:
   - 查询 `FactionSystem.IsAlly(srcFac, tgtFac)`
   - 如果是盟友且没有临时敌对标记 → `multiplier = 0.1`
   - 否则 → `multiplier = 1.0`
3. **Neutral 临时敌对**（双向）:
   - 非Neutral攻击Neutral → 双方互相标记30s临时敌对
   - Neutral攻击非Neutral → 同上
   - 临时敌对期间友军伤害倍率不生效（满额仇恨）
4. **写入仇恨表**:
   - 已存在目标 → 累加仇恨值（上限500），更新时间戳和位置
   - 新目标 → 若已满5个，淘汰 WeightedValue 最小的，然后插入新条目
5. **sentinel 条目**: `targetId=0`，代表"只知道这里有事，不知道是谁"（如 AllyAlert 声援信号）

### 仇恨淘汰机制

当新目标加入且已满 5 个时，移除 WeightedValue 最低的条目。WeightedValue = `value × ThreatTypeConfig.GetWeight(type)`。

## 五、六种威胁类型与权重衰减

**文件**: `Assets/_Game/Config/ThreatType.cs`

### ThreatType 枚举

| 类型 | 含义 | 触发来源 |
|---|---|---|
| `Damage` | 直接伤害 | 武器命中、近战攻击 |
| `Visual` | 视觉发现 | AI 视野探测到敌人 |
| `Sound` | 听觉感知 | 枪声、脚步声、爆炸 |
| `AllyDamage` | 友军受袭 | 盟友被攻击时的连带仇恨 |
| `Territory` | 区域入侵 | 进入敌方领地范围 |
| `Aggression` | 主动挑衅 | 嘲讽技能、挑衅动作 |

### 权重配置（ThreatTypeConfig.Weights）

用于 `GetTopTarget()` 排序，乘以裸值得到 WeightedValue：

| 类型 | 权重 | 说明 |
|---|---|---|
| Damage | 1.5 | 最高优先 — 被打最痛 |
| Aggression | 1.3 | 挑衅次之 |
| AllyDamage | 1.0 | 为同伴报仇 |
| Visual | 0.8 | 看到了不一定直接威胁 |
| Territory | 0.6 | 领地意识 |
| Sound | 0.5 | 只听到声音最不紧迫 |

### 衰减速率（ThreatTypeConfig.DecayRates）

每秒衰减量：

| 类型 | 衰减/秒 | 语义 |
|---|---|---|
| AllyDamage | 0.3 | **最慢** — 为同伴深仇大恨 |
| Damage | 0.5 | 记仇久 |
| Aggression | 0.5 | 与伤害同级 |
| Visual | 2.0 | 离开视线就掉 |
| Territory | 3.0 | 较快遗忘 |
| Sound | 5.0 | **最快** — 声音转瞬即逝 |

### 距离加速衰减

在 `DecayAll()` 中，若目标 `lastKnownPos` 与实体距离：
- **> _perceptionRange × 2**（默认60m）→ 衰减速率 ×3（快速遗忘走远的目标）
- **> _perceptionRange**（默认30m）→ 衰减速率 ×1.5（中等距离加速遗忘）

## 六、ThreatEntry 数据结构

**文件**: `Assets/_Game/Systems/Threat/ThreatEntry.cs`

```csharp
public struct ThreatEntry
{
    public int sourceInstanceId;           // 谁持有仇恨
    public int? targetInstanceId;          // 恨谁（null=sentinel，如AllyAlert）
    public float value;                    // 当前仇恨裸值
    public float timestamp;                // 最近更新时间(Unity Time.time)
    public ThreatType type;                // 威胁类型
    public Vector3 lastKnownPos;           // 目标最后已知位置
    public FactionType? sourceFactionType; // 威胁来源阵营（用于Sound/AllyAlert来源死亡后批量清理）

    public float WeightedValue => value * ThreatTypeConfig.GetWeight(type);
}
```

- 是 **struct**，不产生 GC 分配
- `WeightedValue` 是只读计算属性，每次调用重新计算

## 七、FactionComponent（自动注册）

**文件**: `Assets/_Game/Systems/Threat/FactionComponent.cs`

每个实体 GameObject 上挂载的 MonoBehaviour，负责声明阵营并自动注册。

```csharp
public class FactionComponent : MonoBehaviour
{
    [SerializeField] FactionData _factionData;
    FactionType? _directFaction;

    public FactionType Faction { get; }  // 优先 _directFaction，其次 _factionData，最后 Neutral

    void OnEnable()
    {
        // 自动注册到 ThreatSystem + InstanceRegistry
        ThreatSystem.Instance.RegisterEntity(GetInstanceID(), Faction);
        InstanceRegistry.Register(GetInstanceID(), transform);
    }

    void OnDisable()
    {
        // 自动注销
        ThreatSystem.Instance.UnregisterEntity(GetInstanceID());
        InstanceRegistry.Unregister(GetInstanceID());
    }

    // 运行时切换阵营（先注销旧阵营再注册新阵营）
    public void SetFaction(FactionType faction)
}
```

支持两种阵营来源：
1. **ScriptableObject 引用** (`_factionData`) — Editor 拖拽配置
2. **直接枚举** (`_directFaction`) — 代码运行时设置，优先级更高

## 九、Neutral 临时敌对机制

在 `AddThreat()` 中，当非 Neutral 阵营与 Neutral 阵营互打时，**双方**自动标记 30 秒临时敌对：

```csharp
_tempHostileTimers[(targetId, sourceId)] = 30f;
_tempHostileTimers[(sourceId, targetId)] = 30f;
```

- `IsTempHostile()` 检查任意方向是否有未过期的临时敌对标记
- 临时敌对期间 → 友军伤害倍率**不生效**（满额仇恨）
- 每帧 `TickTempHostile()` 扣减所有计时器，归零自动移除
- 实体死亡时（`UnregisterEntity`）清理涉及该实体的所有临时敌对条目

## 十、分帧 GC (RunFrameGC)

避免一次性全量扫描所有实体造成帧卡顿。

```
每帧处理 GC_BATCH(5) 个实体:

  队列为空? ──Yes──→ 重建队列（当前 _table.Keys 快照）
      │
     No
      │
      ▼
  Dequeue 5 个 sourceId:
   ├─ source 实体已不存在 → 移除 _table[sourceId]
   ├─ target 实体已不存在 → 移除 _table[sourceId][targetId]
   └─ 正常 → 不处理

  200实体 ÷ 5/帧 = 40帧 ≈ 0.67s 完成一圈
```

- 用 `Queue<int>` 轮询，Dequeue 即消耗，keys 动态变化不影响后续迭代
- 重建时 `new Queue<int>(_table.Keys)` 是 O(n) 拷贝，200实体约 0.01ms，可忽略
- 新增实体不在当前队列中，下一轮覆盖

## 十一、EventBus 集成

### ThreatSystem 作为订阅者

在 `Awake()` 中订阅三个事件：

| 事件 | 处理方法 | 行为 |
|---|---|---|
| `ThreatReportEvent` | `OnThreatReport()` | 将伤害转换为仇恨（target 恨 source） |
| `EntityDeathEvent` | `OnEntityDeath()` | 标记到 `_pendingRemoval`，LateUpdate 批量注销 |
| `FactionRelationChangedEvent` | `OnRelationChanged()` | 变为 Ally→清除仇恨，变为 Neutral→衰减50% |

### 为什么死亡延迟到 LateUpdate

`EntityDeathEvent` 在 `Update` 中标记，`LateUpdate` 中执行 `UnregisterEntity`。这保证了同一帧内先到达的 `ThreatReportEvent`（伤害事件）在死亡清理前已被完整处理——避免"先清理再伤害"的时序错乱。

### 事件发布者一览

| 发布者 | 事件 | 触发时机 |
|---|---|---|
| `WeaponShooting` | `ThreatReportEvent` | 武器命中目标（枪/弓） |
| `PlayerCombat` | `ThreatReportEvent` | 玩家近战命中 |
| `ZombieStateMachine` | `ThreatReportEvent` | 僵尸受到伤害/发现目标 |
| `AIBotCombat` | `ThreatReportEvent` (x4处) | AI机器人战斗各阶段 |
| `AIAgent` | `ThreatReportEvent` | AI感知到威胁 |
| `DamageableZombie` | `EntityDeathEvent` | 僵尸死亡清理 |
| `SurvivalSystem` | `EntityDeathEvent` | 玩家/角色死亡 |
| `AIBot` | `EntityDeathEvent` | 机器人报废 |

### AllyAlertEvent（友军声援）

**文件**: `Assets/_Game/Core/GameEvents.cs`

```csharp
public readonly struct AllyAlertEvent
{
    public int SourceId { get; }
    public Vector3 Position { get; }
    public float Radius { get; }
    public FactionType Faction { get; }
}
```

- **发布**: `AIAgent` 进入 Combat 状态时发布，携带位置、半径、阵营
- **订阅**: 同阵营其他 `AIAgent` 订阅，将其转为 sentinel 威胁条目（targetInstanceId=null）加入自己的仇恨表
- AIAgent 处理 sentinel 条目时走向 `lastKnownPos` 观察，不追具体目标
- 实现"听到同伴呼救→前去支援"的群体行为

### 帧内执行顺序

```
Update 阶段:
  ① EventBus 同步派发 ThreatReportEvent → ThreatSystem.OnThreatReport (写仇恨表)
  ② EventBus 同步派发 EntityDeathEvent → ThreatSystem.OnEntityDeath (标记待清理)
  ③ ThreatSystem.Update → DecayAll + TickTempHostile + RunFrameGC
  ④ AIAgent.Update → PerceptionTick (读仇恨表) + TickStateMachine
  ⑤ AIAgent 可能发布 AllyAlertEvent → 同阵营 AIAgent 写入 sentinel 仇恨

LateUpdate:
  ⑥ ThreatSystem.LateUpdate → 批量清理 _pendingRemoval 中的死亡实体
```

**保护措施**:
- `ScriptExecutionOrder` 保证 ThreatSystem.Update 先于 AIAgent.Update
- `HasProcessedThisFrame()` 运行时断言：Editor 下 AIAgent 检查 ThreatSystem 是否已处理本帧

## 十二、线程安全与帧标记

```csharp
int _frameProcessed;

void Update()
{
    TickTempHostile(Time.deltaTime);
    DecayAll(Time.deltaTime);
    RunFrameGC();
    _frameProcessed = Time.frameCount;
}

public bool HasProcessedThisFrame() => _frameProcessed == Time.frameCount;
```

AIAgent 在 Editor 下检查此标记，脚本执行顺序出问题时立即报告。

## 十三、存档策略

- **FactionSystem 关系变更**: 通过 `GetFactionDeltas()` / `ApplyDeltas()` 持久化（只存与默认不同的 delta）
- **ThreatSystem 仇恨表**: **不存档** — 视为易失状态。加载后仇恨从零开始，通过环境噪音和 AI 行为自然重建。这一设计简化了存档结构，避免了引用完整性验证的复杂性。

## 十六、设计要点

1. **两层解耦**: FactionSystem 管"谁跟谁一伙"，ThreatSystem 管"此刻恨谁"，分工明确互不侵入
2. **友军误伤**: ×0.1 系数防止内部消耗；Neutral 临时敌对期间该规则不适用
3. **仇恨上限**: 每实体最多 5 目标，淘汰权重最低者，防止表无限膨胀
4. **Sentinel 条目**: targetId=null 表示"只知道出事了不知道是谁"，驱动 AI 走向位置观察而非盲目追击
5. **Neutral 特殊性**: 被攻击后 30s 双向临时敌对，模拟"中立者反击"且不永久改变阵营关系
6. **分帧 GC**: 每帧 5 实体的清理节奏，零卡顿
7. **不存档仇恨**: 仇恨表视为易失状态，重新加载后自然恢复
8. **距离感知**: 衰减速率基于目标距离加速，远离的目标快速遗忘
