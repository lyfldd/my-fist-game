## 一、问题分析

### 现状：仇恨逻辑散落各处

```
ZombieStateMachine    → 视觉锥+听觉 → 自选目标
AIBotCombat           → 距离优先 → 自选目标
PlayerCombat          → OverlapSphere → 没有"目标"概念
DamageableZombie      → 受伤闪红 → 不记录"谁打的我"
```

每个系统各自实现目标选择，没有统一的"谁在威胁谁"。未来加入黑恶势力/NPC/好人帮后，仇恨关系会爆炸式复杂化。

### 目标

像 `DecibelSystem` 一样，提供一个独立的、可插拔的仇恨系统：
- 任何实体可以**报告威胁**（我威胁了谁）
- 任何实体可以**查询目标**（谁威胁我最多）
- 阵营关系**静态可配**（不硬编码）
- 威胁值**动态衰减**（自然遗忘）

### 3.4 FactionSystem 单例（MonoBehaviour，支持运行时变更）

```csharp
public class FactionSystem : MonoBehaviour
{
    static FactionSystem _instance;
    public static FactionSystem Instance => _instance;
    
    // 关系矩阵（默认从 SO 加载，运行时可覆盖）
    Dictionary<(FactionType, FactionType), FactionRelation> _relations;
    
    public enum FactionRelation { Ally, Hostile, Neutral }
    
    void Awake()
    {
        _instance = this;
        LoadDefaults();  // 从 FactionData[] SO 加载默认关系
    }
    
    public bool IsHostile(FactionType a, FactionType b)
        => _relations.TryGetValue((a, b), out var r) && r == FactionRelation.Hostile;
    
    public bool IsAlly(FactionType a, FactionType b)
        => _relations.TryGetValue((a, b), out var r) && r == FactionRelation.Ally;
    
    public bool IsNeutral(FactionType a, FactionType b)
        => !IsHostile(a, b) && !IsAlly(a, b);
    
    // ═══ 运行时变更 ═══
    public void SetRelation(FactionType a, FactionType b, FactionRelation relation)
    {
        _relations[(a, b)] = relation;
        _relations[(b, a)] = relation;  // 双向
        
        // ═══ 发射事件通知所有系统 ═══
        EventBus.Emit(new FactionRelationChangedEvent 
        { 
            factionA = a, 
            factionB = b, 
            newRelation = relation 
        });
    }
    
    public void ResetToDefault()
    {
        LoadDefaults();  // 恢复 SO 默认
    }
}
```

支持场景：玩家惹怒军方 → `SetRelation(Player, Military, Hostile)`，NPC 叛变 → `SetRelation(Survivor, Bandit, Ally)`。

**阵营变更通知**：`SetRelation` 会发射 `FactionRelationChangedEvent`。ThreatSystem 订阅后自动刷新已有威胁条目：变盟友→清除仇恨，变中立→衰减加速，变敌对→保持。

## 五、第二层：ThreatSystem（动态仇恨表）

### 5.1 ThreatType 枚举

```csharp
public enum ThreatType
{
    Damage,        // 造成伤害 → 仇恨值 = 伤害量 × 阵营系数
    Visual,        // 视觉发现 → 仇恨值 = 50（恒定）
    Sound,         // 听到声音 → 仇恨值 = 分贝值
    AllyDamage,    // 友军被攻击 → 仇恨值 = 伤害量 × 0.5
    Territory,     // 区域入侵 → 仇恨值 = 30（恒定）
    Aggression     // 主动挑衅 → 仇恨值 = 100
}
```

### 5.2 威胁权重（不同威胁类型不等价）

GetTopTarget 按 `value × weight` 排序，不是裸 value：

| ThreatType | 权重系数 | 说明 |
|------|:--:|------|
| Damage | ×1.5 | 被打 > 被看到 |
| Aggression | ×1.3 | 主动挑衅 |
| AllyDamage | ×1.0 | |
| Visual | ×0.8 | 看到了不一定 = 直接威胁 |
| Sound | ×0.5 | 听到声音最不紧迫 |
| Territory | ×0.6 | |

### 5.3 ThreatEntry

```csharp
public struct ThreatEntry
{
    public int sourceInstanceId;       // 谁恨
    public int? targetInstanceId;      // 恨谁（null=只知道位置不知道谁，如AllyAlert）
    public float value;                // 当前仇恨值（裸值）
    public float timestamp;            // 最近威胁时间
    public ThreatType type;            // 威胁类型
    public Vector3 lastKnownPos;       // 目标最后已知位置
    public FactionType? sourceFactionType;  // 威胁来源阵营（Sound/AllyAlert时用于来源死亡后批量清理）
    
    public float WeightedValue => value * GetWeight(type);  // 排序用
}
```

`targetInstanceId` 为 null 时，AIAgent 的 Investigate 状态走向 `lastKnownPos` 观察，不追具体目标。

### 5.5 AddThreat 完整逻辑

```csharp
// Neutral 被攻击 → 临时标记攻击者为敌对
Dictionary<(int neutralId, int attackerId), float> _tempHostileTimers;  // 剩余秒数

public void AddThreat(int sourceId, int? targetId, float value, ThreatType type, Vector3? position = null)
{
    // 内部：null targetId 用 0 作为 sentinel（Unity InstanceID 始终 ≥1）
    int internalKey = targetId ?? 0;
    
    if (sourceId == internalKey && internalKey != 0) return;  // 不对自己产生威胁
    
    var srcFac = GetFaction(sourceId);
    
    // ═══ 阵营系数（仅当有具体目标时） ═══
    float multiplier = 1f;
    if (targetId.HasValue)
    {
        var tgtFac = GetFaction(targetId.Value);
        
        // Neutral 被打 → 双向临时敌对（标记双方），持续 30s
        if (tgtFac == FactionType.Neutral && srcFac != FactionType.Neutral)
        {
            _tempHostileTimers[(targetId.Value, sourceId)] = 30f;  // Neutral 恨攻击者
            _tempHostileTimers[(sourceId, targetId.Value)] = 30f;  // 攻击者也恨 Neutral（双向）
        }
        // 对称情况：攻击者是 Neutral，被攻击者不是 Neutral
        else if (srcFac == FactionType.Neutral && tgtFac != FactionType.Neutral)
        {
            _tempHostileTimers[(sourceId, targetId.Value)] = 30f;
            _tempHostileTimers[(targetId.Value, sourceId)] = 30f;
        }
        
        // 友军误伤 → ×0.1（但如果攻击者已被临时标记为敌对，不减免）
        if (FactionSystem.Instance.IsAlly(srcFac, tgtFac) && !IsTempHostile(targetId.Value, sourceId))
            multiplier = _friendlyFireMultiplier;
    }
    
    value *= multiplier;
    
    // ═══ 写入威胁表 ═══
    if (!_table.TryGetValue(sourceId, out var inner))
    {
        inner = new Dictionary<int, ThreatEntry>();
        _table[sourceId] = inner;
    }
    
    if (inner.TryGetValue(internalKey, out var existing))
    {
        existing.value += value;
        existing.value = Mathf.Min(existing.value, _maxThreatPerTarget);
        existing.timestamp = Time.time;
        existing.lastKnownPos = position ?? existing.lastKnownPos;
        inner[internalKey] = existing;
    }
    else
    {
        if (inner.Count >= _maxTrackedTargets)
        {
            int weakestId = inner.OrderBy(kv => kv.Value.WeightedValue).First().Key;
            inner.Remove(weakestId);
        }
        
        inner[internalKey] = new ThreatEntry
        {
            sourceInstanceId = sourceId,
            targetInstanceId = targetId,  // 保持原始 null
            value = Mathf.Min(value, _maxThreatPerTarget),
            timestamp = Time.time,
            type = type,
            lastKnownPos = position ?? Vector3.zero,
            sourceFactionType = targetId.HasValue ? GetFaction(targetId.Value) : null
        };
    }
}

// 检查 (entityA, entityB) 任意方向是否有临时敌对标记
bool IsTempHostile(int entityA, int entityB)
    => _tempHostileTimers.TryGetValue((entityA, entityB), out var t) && t > 0f
    || _tempHostileTimers.TryGetValue((entityB, entityA), out t) && t > 0f;

void TickTempHostile(float dt)
{
    // 每帧衰减临时敌对计时器
    foreach (var key in _tempHostileTimers.Keys.ToList())
    {
        _tempHostileTimers[key] -= dt;
        if (_tempHostileTimers[key] <= 0f)
            _tempHostileTimers.Remove(key);
    }
}
```

**双向临时敌对**：Neutral 阵营任何一方被打 → 双方互相标记为临时敌对（30s），期间不受友军误伤减免。超时双向恢复中立。

**预留**：友军误伤累积叛变机制（Phase 3 人类阵营完善时实现）。每个实体记录对同阵营的误伤累计值，超过阈值 → `FactionSystem.SetRelation(Ally, Hostile)` 触发叛变。

### 5.6 衰减规则

```csharp
// 每Tick衰减量（每秒）
static Dictionary<ThreatType, float> DecayRates = new() {
    { ThreatType.Damage,      0.5f },  // 每10秒-5（记仇久）
    { ThreatType.Visual,      2.0f },  // 每5秒-10（离开视线就掉）
    { ThreatType.Sound,       5.0f },  // 每3秒-15（很快忘记）
    { ThreatType.AllyDamage,  0.3f },  // 衰减最慢（为同伴报仇）
    { ThreatType.Territory,   3.0f },  // 较快衰减
    { ThreatType.Aggression,  0.5f }   // 同伤害级
};

void DecayAll(float deltaTime)
{
    var outerToRemove = new List<int>();
    
    foreach (var (srcId, inner) in _table)
    {
        var srcPos = InstanceRegistry.GetTransform(srcId)?.position ?? Vector3.zero;
        bool srcExists = InstanceRegistry.Exists(srcId);
        var innerToRemove = new List<int>();
        
        foreach (var (tgtId, entry) in inner)
        {
            float decayRate = DecayRates[entry.type];
            
            // ═══ 距离衰减加速 ═══
            if (srcExists && entry.lastKnownPos != Vector3.zero)
            {
                float dist = Vector3.Distance(srcPos, entry.lastKnownPos);
                // 目标在视野外（距离 > 感知范围 × 2），衰减 ×3
                if (dist > _perceptionRange * 2f)       // 默认30m × 2 = 60m
                    decayRate *= 3f;
                // 中等距离（感知范围~2倍之间），衰减 ×1.5
                else if (dist > _perceptionRange)
                    decayRate *= 1.5f;
            }
            
            entry.value -= decayRate * deltaTime;
            if (entry.value <= 0f)
                innerToRemove.Add(tgtId);
            else
                inner[tgtId] = entry;
        }
        
        foreach (var tgtId in innerToRemove)
            inner.Remove(tgtId);
        
        if (inner.Count == 0)
            outerToRemove.Add(srcId);
    }
    
    foreach (var srcId in outerToRemove)
        _table.Remove(srcId);
}
```

`_perceptionRange` 默认 30m，在 ThreatSystem 上暴露为可配置字段。

**距离加速**：威胁目标距离超过感知范围的 2 倍时，衰减速率 ×3（快速遗忘走远的目标）；1~2 倍感知范围时 ×1.5。
```

威胁值归零 → 从表中移除。

### 6.2 ThreatSystem 处理伤害事件

```csharp
void OnEnable()
{
    EventBus.On<ThreatReportEvent>(OnThreatReport);
    EventBus.On<EntityDeathEvent>(OnEntityDeath);
    EventBus.On<FactionRelationChangedEvent>(OnRelationChanged);  // 见 §七
}

void OnThreatReport(ThreatReportEvent e)
{
    // 被攻击者恨攻击者（值=伤害量，权重内部自动×1.5）
    AddThreat(e.targetInstanceId, e.sourceInstanceId, e.damageAmount, ThreatType.Damage, e.position);
    
    // 注：AllyDamage 广播不在这里做！
    // 改为由 AIAgent 基类在进入 Combat 时发 AllyAlertEvent，
    // 同阵营 AIAgent 自己订阅 → 加 AllyDamage 到自己的威胁表。
    // ThreatSystem 只管存取，不管广播。
}
```

### 6.5 威胁来源总览

| 来源 | 触发时机 | ThreatType | value | 接入点 |
|------|------|:--:|------|------|
| 造成伤害 | EventBus ThreatReportEvent | Damage | 伤害量 | ThreatSystem 直接订阅 |
| 视觉发现 | AIAgent 感知 Tick | Visual | 50 | AIAgent → AddThreat |
| 听到声音 | NoiseEvent | Sound | 分贝值 | AIAgent → AddThreat |
| 友军被攻击 | AllyAlertEvent | AllyDamage | 20 | AIAgent → AddThreat |
| 区域入侵 | Zone 触发器 | Territory | 30 | ZoneManager → AddThreat |
| 主动挑衅 | NPC/事件 | Aggression | 100 | 调用方 → AddThreat |

**关键变更**：BroadcastAllyThreat 不再在 ThreatSystem 内做。改为 `AllyAlertEvent`（AIAgent 进入 Combat 时发射），同阵营 AIAgent 自己订阅处理。ThreatSystem 保持纯粹——只管存取和衰减。

## 七、实体死亡清理

```csharp
// ThreatSystem 订阅死亡事件
HashSet<int> _pendingRemoval = new();

void OnEnable()
{
    EventBus.On<ThreatReportEvent>(OnThreatReport);
    EventBus.On<EntityDeathEvent>(OnEntityDeath);
    EventBus.On<FactionRelationChangedEvent>(OnRelationChanged);
}

void OnRelationChanged(FactionRelationChangedEvent e)
{
    // 遍历所有实体，如果涉及变动的阵营，重新评估威胁表
    foreach (var (srcId, inner) in _table)
    {
        var srcFac = GetFaction(srcId);
        var toRemove = new List<int>();
        
        foreach (var (tgtId, entry) in inner)
        {
            if (tgtId == 0) continue;  // sentinel：Sound/AllyAlert 的未知目标，跳过
            var tgtFac = GetFaction(tgtId);
            
            if (MatchesChangedFactions(e, srcFac, tgtFac))
            {
                if (e.newRelation == FactionSystem.FactionRelation.Ally)
                    toRemove.Add(tgtId);  // 变成盟友 → 删除仇恨
                else if (e.newRelation == FactionSystem.FactionRelation.Neutral)
                {
                    // 变中立 → 衰减加速（保留一部分记忆但快速淡忘）
                    entry.value *= 0.5f;
                    inner[tgtId] = entry;
                }
                // 变 Hostile → 保持不变（本来就有威胁）
            }
        }
        
        foreach (var id in toRemove) inner.Remove(id);
    }
}

bool MatchesChangedFactions(FactionRelationChangedEvent e, FactionType a, FactionType b)
    => (a == e.factionA && b == e.factionB) || (a == e.factionB && b == e.factionA);

void OnEntityDeath(EntityDeathEvent e)
{
    // 只标记，不立即清理——确保同一帧内的 ThreatReportEvent 先处理完
    _pendingRemoval.Add(e.deadEntityId);
}

void LateUpdate()
{
    foreach (var id in _pendingRemoval)
        UnregisterEntity(id);
    _pendingRemoval.Clear();
}

public void UnregisterEntity(int instanceId)
{
    // 清"它恨谁"
    _table.Remove(instanceId);
    
    // 清"谁恨它"
    foreach (var inner in _table.Values)
        inner.Remove(instanceId);
    
    // 清阵营 + 临时敌对
    _entityFactions.Remove(instanceId);
    _tempHostileTimers = _tempHostileTimers
        .Where(kv => kv.Key.neutralId != instanceId && kv.Key.attackerId != instanceId)
        .ToDictionary(kv => kv.Key, kv => kv.Value);
}
```

**顺序保证**：死亡事件标记 → LateUpdate 清理。同一帧内的 ThreatReportEvent 在 Update 中已全部处理完毕。

**分帧 GC**：用队列轮询替代定时全量扫描，每帧处理 5 个实体，不集中卡顿：

```
队列空了？ ──Yes──→ 重建队列(当前 _table.Keys)
    │
   No
    │
    ▼
本帧取出 5 个实体ID
    │
    ▼
逐个检查：
 ├─ 源实体不存在 → 删整条 _table[srcId]
 ├─ 目标实体不存在 → 删 _table[srcId][tgtId]
 └─ 正常 → 不处理

200实体 ÷ 5/帧 = 40帧 ≈ 0.67s 完成一圈
```

```csharp
Queue<int> _gcQueue;
const int GC_BATCH = 5;

void Update()
{
    TickTempHostile(Time.deltaTime);
    DecayAll(Time.deltaTime);
    RunFrameGC();
}

void RunFrameGC()
{
    // 队列空了 → 从当前 table 重建
    if (_gcQueue == null || _gcQueue.Count == 0)
    {
        _gcQueue = new Queue<int>(_table.Keys);
        if (_gcQueue.Count == 0) return;
    }
    
    for (int i = 0; i < GC_BATCH && _gcQueue.Count > 0; i++)
    {
        int srcId = _gcQueue.Dequeue();
        
        // ① 源实体已销毁 → 删整条
        if (!InstanceRegistry.Exists(srcId))
        {
            _table.Remove(srcId);
            continue;
        }
        
        // ② 源可能已被 Unregister 清理 → 跳过
        if (!_table.TryGetValue(srcId, out var inner))
            continue;
        
        // ③ 逐个检查目标实体
        foreach (var tgtId in inner.Keys.ToList())
        {
            if (tgtId != 0 && !InstanceRegistry.Exists(tgtId))
                inner.Remove(tgtId);
        }
    }
}
```

**为什么用队列**：Dequeue 取出即消耗，key 变化不影响后续操作。重建时 `new Queue<int>(_table.Keys)` 是一次 O(n) 拷贝（200 实体 ≈ 0.01ms），可以忽略。新增实体不在当前队列中，下一轮重建覆盖到。

## 八、帧内事件处理时序

EventBus 是同步触发的——发射事件时所有订阅者回调当场跑完。不存在事件和感知交错的问题。

```
┌──────────── 单帧事件处理时序（ThreatSystem + AIAgent）──────────────┐
│                                                                      │
│  ① NoiseEvent          → AIAgent 声感知 → 可能请求 Investigate        │
│  ② ThreatReportEvent   → ThreatSystem 写威胁表                        │
│  ③ AIAgent.Perception  → 读威胁表 + 视野检测 → 可能请求 Combat/Flee   │
│  ④ ThreatSystem.Decay  → 衰减所有威胁值                               │
│  ⑤ AIAgent.TickState   → 执行当前状态行为（走/打/逃）                  │
│  ⑥ ThreatSystem.LateUpdate → 清理死亡实体标记                         │
│                                                                      │
│  强依赖（违反会有问题）：                                              │
│  ├─ ②→③ 写后读：刚发生的伤害必须在感知 Tick 时可见                     │
│  └─ ③→⑤ 读后决策：状态决定时感知数据必须是本帧的                       │
│                                                                      │
│  弱依赖（违反延迟一帧，可接受）：                                      │
│  ├─ ①→② 无依赖                                                     │
│  └─ ④→③ 读后衰减（上一帧衰减后的数据即可）                            │
│                                                                      │
│  保护措施：ScriptExecutionOrder 保证 ThreatSystem.Update 先于 AIAgent │
│  + 运行时 HasProcessedThisFrame 断言                                   │
└──────────────────────────────────────────────────────────────────────┘
```

运行时防御：

```csharp
// ThreatSystem
int _frameProcessed;
public bool HasProcessedThisFrame() => _frameProcessed == Time.frameCount;

void Update()
{
    // ... DecayAll / RunFrameGC / TickTempHostile ...
    _frameProcessed = Time.frameCount;
}
```

```csharp
// AIAgent
void Update()
{
#if UNITY_EDITOR
    if (!ThreatSystem.Instance.HasProcessedThisFrame())
        Debug.LogWarning($"[AIAgent] ThreatSystem 尚未处理本帧，感知数据可能过期");
#endif
    
    PerceptionTick();
    TickStateMachine();
}
```

成本为零，脚本执行顺序出问题时一秒定位。

## 十、性能

### 10.1 规模估算

```
115实体 × 平均3目标 = 345条目
DecayAll: 345次减法/0.5s → 可忽略
GetTopTarget: 遍历3个条目找最大 → O(3)
```

### 10.2 GC 考量

`Dictionary<int, Dictionary<int, ThreatEntry>>`:
- ThreatEntry 是 **struct**，不产生 GC
- 内层 Dictionary 在实体存活期间稳定（3-5 个条目），不会频繁 new
- DecayAll 内 `toRemove` 是 `List<int>`，每 0.5s 分配一次 → 可接受。如果优化可改为 `List<int>` 缓存复用。
