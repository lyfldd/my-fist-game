# 研究中心分层体系 + 成本翻倍计划

> **日期**: 2026-05-30 · **状态**: 📋 设计定稿 · **依赖**: ChemicalResearchManager + ProductionDevice + RecipeData

---

## 一、核心设计

### 1.1 分层研究哲学

```
前期/中期 (智力1-6):  研究「学科大类」→ 解锁一批设备（类现有模式）
后期/终局 (智力7-10): 研究「单个配方」→ 解锁一个具体物品的制造权
```

| 层级 | 智力 | 粒度 | 数量 | 研究成本基准 |
|------|:--:|------|:--:|------|
| **前期** | 1-3 | 大类→2~4设备 | 6项 | 基础材料 ×3倍 |
| **中期** | 4-6 | 大类→2~4设备 | 7项 | 中级材料 ×3倍 |
| **后期** | 7-8 | **逐个配方** | ~15项 | 高级材料 ×3倍 |
| **终局** | 9-10 | **逐个配方** | ~12项 | 稀有材料 ×3倍 |

### 1.2 成本翻倍原则

**全部研究项目成本在当前基础上 ×3 起步**，终局项目 ×4~5。

理由：
- 研究 = 质变性的解锁，不是跑腿任务
- 后期/终局配方研究代表了科技树的深度
- 让玩家真正感受到"为了造 X，我必须先研究 Y"的取舍

## 二、数据结构扩展

### 2.1 ChemicalResearchProject 新增字段

```csharp
public struct ChemicalResearchProject
{
    public string researchId;
    public string displayName;
    public string description;
    public ItemRequirement[] cost;
    public string[] unlockedDeviceNames;      // 解锁设备（前期/中期用）
    public string[] unlockedRecipeIds;        // 解锁配方（后期/终局用）← NEW
    public int requiredIntellectLevel;
    public ResearchTier tier;                 // 阶段标记 ← NEW
}

public enum ResearchTier
{
    Early,      // 前期 智力1-3
    Mid,        // 中期 智力4-6
    Late,       // 后期 智力7-8
    Endgame     // 终局 智力9-10
}
```

### 2.2 ChemicalResearchManager 新增

```csharp
// 检查配方是否已解锁（后期/终局用）
public bool IsRecipeUnlocked(string recipeId);

// 所有研究完成时解锁
```

### 2.3 门控联动

| 位置 | 检查 |
|------|------|
| `ProductionDevice.TryProduce()` | `IsDeviceUnlocked()` + `IsRecipeUnlocked()` |
| `ProductionDeviceUI` 配方列表 | 未解锁配方灰显"需研究" |
| `CraftingUI` 手工配方 | 终局配方检查 `IsRecipeUnlocked()` |

## 六、与现有系统的兼容

- **不影响已研究存档**：`_completedIds` 列表只增不改
- **旧研究项目保留**：现有 12 项保留，增加 28 项新项目
- **配方 ID 对应**：需要给每个后期/终局配方分配唯一 `recipeId`
- **Fallback**：若配方不在任何研究项目中 → 默认解锁（兼容前期/中期）
