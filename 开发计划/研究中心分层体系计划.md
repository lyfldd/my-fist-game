## 一、核心设计（v2.0 科技树模式）

### 1.1 两级研究结构

```
智力 1-4:  大类研究 → 一次性解锁设备 + 基础配方
智力 5+:   大类研究 → 打开子列表 → 逐个研究子项（子弹/枪械/药品）
```

| 层级 | 智力 | 结构 | 数量 |
|------|:--:|------|:--:|
| **基础大类** | 1-4 | 研究1次 → 全解锁 | 5项 |
| **进阶大类** | 5+ | 先研究大类 → 逐个解锁子项 | 7大类 |
| **子研究项** | 5-10 | 每个具体配方/弹药/枪械 | ~32项 |

### 1.2 用户体验

```
研究中心面板:
┌──────────────────────────────────────┐
│ [前期] [中期] [后期] [终局]           │
├──────────────────────────────────────┤
│                                      │
│ ✓ 基础金属加工                        │  ← 已研究
│   → 冲压机、粉碎机、回收站             │
│                                      │
│ ✓ 精密金属制造                        │  ← 已研究大类
│   ├ ✓ 手枪配方                       │  ← 已研究子项
│   ├ ✓ 霰弹枪配方                     │
│   ├ → 步枪配方          [研究]       │  ← 可研究子项
│   ├ 🔒 猎枪配方       需先研究大类    │  ← 大类未完成
│   └ 🔒 重狙配方       需智8          │  ← 智力不够
│                                      │
└──────────────────────────────────────┘
```

## 二、数据结构

### 2.1 ChemicalResearchProject v2

```csharp
public struct ChemicalResearchProject
{
    public string researchId;
    public string displayName;
    public string description;
    public ItemRequirement[] cost;
    public string[] unlockedDeviceNames;       // 解锁设备
    public string[] unlockedRecipeIds;         // 解锁配方
    public int requiredIntellectLevel;
    public ResearchTier tier;
    
    // v2 科技树字段
    public string parentResearchId;            // 前置大类ID（子项才有）
    public bool isCategory;                    // true=大类(可展开子列表)
}
```

### 2.2 ChemicalResearchManager 新增

```csharp

// 检查大类是否已研究（子项的前置条件）
```

## 四、UI 交互流程

```
1. 玩家打开研究中心 → 看到所有"大类"项目
2. 研究完一个大类 → 大类左侧出现 [▶展开] 按钮
3. 点击展开 → 显示子研究列表
4. 子项可逐个研究 → 研究完解锁对应配方/设备
5. 未展开的大类 → 子项不可见
```
