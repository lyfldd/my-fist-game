## 1. 系统概览

电力系统提供"发电 → 输电 → 用电"完整闭环。玩家建造发电设备、用电终端、生产设备，通过电缆连接形成电网，为设备提供电力加成（高速、零废料）。

核心设计原则：

- **功率匹配**：总输出(W) >= 总负载(W) = 全部运转，不模拟电压/电流
- **两极分化**：前期烧煤兜底（低速+废品率），后期电力爆发（高速+零废品）
- **电网拓扑**：终端之间可串联形成 mesh 网格，全电网共享总功率池
- **燃料链**：煤→精炼煤炭、原油→汽油、铀矿→浓缩铀 三条主线

## 3. 三组件架构

### 3.1 PowerSource（发电端）

挂载在发电设备上，实现 `IInteractable`（E 键打开面板）。

```csharp
// _Game.Systems.Power.PowerSource : MonoBehaviour, IInteractable
public PowerSourceType sourceType;  // 电源类型枚举
public float maxOutput = 100f;      // 最大输出功率(W)
public bool requiresFuel;           // 是否需要燃料
public float fuelPerHour = 1f;      // 每小时燃料消耗量
public string fuelItemName;         // 燃料物品名称
public ItemData fuelItemData;       // 燃料物品引用（从背包消耗）
public bool requiresOpenAir;        // 必须露天（预留）
public bool requiresWater;          // 必须水边（预留）
public bool daytimeOnly;            // 仅白天运行
public float noiseRadius = 10f;     // 噪音半径(m)
public float maxDurability = 100f;  // 最大耐久
public float durability = 100f;     // 当前耐久（预留）
```

**关键属性：**

| 属性 | 类型 | 说明 |
|------|------|------|
| `IsActive` | bool | 所有条件满足时为 true |
| `CurrentOutput` | float | IsActive ? maxOutput : 0 |
| `FuelRemaining` | float | 剩余燃料（小时） |
| `MaxOutput` | float | 最大功率 |
| `DurabilityPercent` | float | durability / maxDurability |

**运行时逻辑 (Update)：**

1. 检查 `daytimeOnly` — `FindObjectOfType<Light>().intensity < 0.1f` 则停摆
2. 燃料消耗 — `consumeInterval = 3600 / fuelPerHour` 秒消耗 1 单位
3. 燃料耗尽时自动从背包补充（`Inventory.RemoveItemByName`）
4. `fuelItemData.fuelValue > 0 ? fuelItemData.fuelValue : 1f` 计算补充量
5. 燃料余量 <= 0 时停摆

```csharp
// IInteractable — E 键打开面板
```

### 3.2 PowerTerminal（用电终端）

电网中枢节点，挂载在终端建造物上，实现 `IInteractable`。

```csharp
// _Game.Systems.Power.PowerTerminal : MonoBehaviour, IInteractable
public float supplyRadius = 15f;                       // 供电半径(m)
public bool allowCoalBackup = true;                    // 无发电端时允许烧煤兜底
public List<PowerSource> connectedSources;             // 直接连接的发电端
public List<PowerTerminal> connectedTerminals;         // 串联的其它终端
```

**电网计算 (ScanAndPower，每秒一次)：**

```
CollectGrid(this, visited, ref totalPower)
  1. DFS 遍历 connectedTerminals 图
  2. 对每个节点，累加其 connectedSources 中活跃电源的 maxOutput
  3. 得到 totalPower = 全电网总发电量

FindConsumersInRadius(terminal)
  1. FindObjectsOfType<PowerConsumer>()
  2. 筛选 enabled && activeInHierarchy
  3. 距离 <= terminal.supplyRadius 则纳入

通电分配：
  totalLoad = sum(consumer.EffectivePowerDraw)
  gridOk = totalPower >= totalLoad
  foreach consumer:
    consumer.SetPowered(this, gridOk && consumerPower <= remainingPower)
```

**电网状态：**

| 条件 | 状态 | 颜色 |
|------|------|------|
| `GridPower <= 0` | 无电源 | 红 |
| `GridPower >= GridLoad` | 正常（余量 W） | 绿 |
| `GridPower < GridLoad` | 超载（缺口 W） | 红 |

**连接 API：**

- `ConnectSource(PowerSource)` / `DisconnectSource(PowerSource)` — 电源连/断
- `ConnectTerminal(PowerTerminal)` / `DisconnectTerminal(PowerTerminal)` — 终端串联/断开
- 双向互操作，标记 `_gridDirty`
- OnDisable 时自动从所有邻居的列表中移除自己

### 3.3 PowerConsumer（用电端）

挂载在生产设备上，实现双模（电力 / 烧煤）切换。

```csharp
// _Game.Systems.Power.PowerConsumer : MonoBehaviour
public float requiredPower = 30f;            // 需求功率(W)
public bool allowCoal = false;              // 无电时允许烧煤兜底
public float coalPower = 15f;               // 烧煤模式等效功率(W)
public float electricSpeedMultiplier = 2f;  // 通电后生产速度倍率
public bool zeroWasteOnElectric = true;     // 通电后零废料
public string displayName;                  // UI 显示名称
```

**状态属性：**

| 属性 | 说明 |
|------|------|
| `IsPowered` | 电网供电中 |
| `IsCoalMode` | 烧煤模式（兜底） |
| `IsRunning` | IsPowered \|\| IsCoalMode |
| `IsManuallyOff` | 玩家手动关闭 |
| `SpeedMultiplier` | 电力: electricSpeedMultiplier, 煤/停: 1x |
| `EffectivePowerDraw` | 手动关闭: 0, 否则: requiredPower |

**三态状态机 (SetPowered)：**

```
SetPowered(terminal, powered)
  powered=true  → _isPowered=true,  _isCoalMode=false   (电力模式，高速)
  powered=false:
    if allowCoal → _isPowered=false, _isCoalMode=true    (煤模式，低速)
    else         → _isPowered=false, _isCoalMode=false   (停摆)

SetManualOff(true) → 强制停摆，_isPowered=false, _isCoalMode=false
```

状态变化时调用 `ProductionDevice.NotifyPowerState(_isPowered, _isCoalMode)` 通知生产逻辑。

## 4. 电网拓扑 (Grid Topology)

### 4.1 连接模型

```
PowerSource ──[电缆 30m]──→ PowerTerminal
                              │
                              ├──[半径 15m/25m 自动]──→ PowerConsumer × N
                              │
                              └──[电缆 30m]──→ PowerTerminal (串联)
                                                  │
                                                  └──[半径]──→ PowerConsumer × N
```

**连接规则：**

| 发起端 | 目标 | 方式 |
|--------|------|------|
| PowerSource | PowerTerminal | 电缆 (CableLinker.StartLinkingFromSource) |
| PowerTerminal | PowerTerminal | 电缆 (CableLinker.StartLinking) |
| PowerTerminal | PowerSource | 电缆 |
| PowerTerminal | PowerConsumer | 自动（半径范围，无需接线） |

- 电源之间不能直连
- 电源不能直连消费设备
- 电缆最大距离 30m（`CableLinker.MAX_CABLE_DISTANCE`）
- 每次连接消耗背包 1 根 "电缆"（`CABLE_ITEM_NAME = "电缆"`）

### 4.2 功率池模型

全电网所有活跃电源的总功率由所有终端半径内的消费者共享。

**分配策略（先到先得）：**
DFS 遍历终端图 → 汇总总功率 → 遍历所有消费者 → 当前剩余功率 >= 消费者需求时通电。

```
电网功率池 = Σ(每个终端.connectedSources 中活跃电源.maxOutput)
总负载 = Σ(每个消费者.EffectivePowerDraw)
gridOk = 功率池 >= 总负载
```

## 6. 双模系统 (Dual-Mode: Coal vs Electric)

### 6.1 模式切换

```
              ┌─ 电网有电 ─→ 电力模式（高速，零废料）
设备运行 ────┤
              └─ 电网无电 ─┬─ allowCoal=true  ─→ 煤模式（低速，有废料）
                           └─ allowCoal=false ─→ 停摆
```

设计理念：**不是禁止烧煤，而是烧到你主动想接电。** 用代价引导，不强制。

### 6.2 生产设备差异

| 属性 | 电力模式 | 煤模式 | 停摆 |
|------|----------|--------|------|

| 废料率 | 0%（zeroWasteOnElectric） | 正常 | — |
| 功率消耗 | 从电网取 requiredPower W | 不从电网取电 | — |
| 燃料消耗 | 无 | 烧煤（ProductionDevice 内部逻辑） | — |

## 7. 燃料链 (Fuel Chain)

### 7.1 煤系燃料链

```
煤矿 (Raw Coal)
  ├─→ 直接燃烧 → 简易发电机 (100W, 1煤/h)
  ├─→ 直接燃烧 → 发电机(旧, 200W, 2煤/h)
  ├─→ 炭窑/熔炉 → 精炼煤炭 (Refined Coal) → 火力发电站 (300W, 3/h)
  └─→ 熔炉·干馏 → 煤焦油 (Coal Tar) → 化工原料 → 汽油中间体
```

### 7.2 油系燃料链

```
原油/页岩/塑料废料 → 蒸馏器 → 汽油 (Gasoline) → 简易发电机 (100W)
```

### 7.3 核系燃料链（终局）

```
铀矿石 (Uranium Ore) → 离心机 → 浓缩铀 (Enriched Uranium) → 核电站 (5000W, 0.1铀/h)
```

### 7.4 电缆制造链

```
铜矿/废金属 → 熔炉 → 铜锭 → 拉线机 → 电线 → 装配台 → 电缆 (Cable)
  └─→ 用于连接发电端和终端 (1 根/次, max 30m)
```

## 8. CableLinker（电缆连线系统）

### 8.1 交互流程

```
1. 打开面板 (PowerSourceUI 或 TerminalUI)
2. 点击 "⚡ 连接终端" / "⚡ 连接发电端/终端"
3. 进入连线模式 (CableLinker.StartLinking / StartLinkingFromSource)
4. 左键点击目标 → 距离检查(≤30m) → 消耗 1 根电缆 → 建立连接 → 回到面板
5. 右键 → 取消连线 → 回到面板
```

```csharp
// CableLinker 静态 API
CableLinker.StartLinking(PowerTerminal source)        // 从终端发起
CableLinker.StartLinkingFromSource(PowerSource source) // 从电源发起
CableLinker.IsLinking                                  // 是否正在连线
CableLinker.CountCables()                              // 背包电缆数量
```

### 8.2 连接方向

| 发起端 | 可连目标 | 不可连 |
|--------|----------|--------|
| PowerTerminal | PowerTerminal, PowerSource | — |
| PowerSource | PowerTerminal | PowerTerminal(?) |

### 8.3 关键常量

```csharp
const float MAX_CABLE_DISTANCE = 30f;
const string CABLE_ITEM_NAME = "电缆";
```

### 8.4 成功连接后

1. `terminal.ConnectSource(src)` 或 `terminal.ConnectTerminal(other)`
2. `PowerLineRenderer.Create(transformA, transformB)` 创建可视化电缆线
3. 回到面板 UI

## 11. BuildableData 电力字段

`BuildableData.cs` 中包含完整的电力配置字段，运行时由 `BuildModeController` 读取并挂载对应组件：

| 字段 | 类型 | 触发条件 | 挂载组件 |
|------|------|----------|----------|
| `powerSupplyRadius` | float | > 0 | PowerTerminal (向周围供电) |
| `powerOutput` | float | > 0 | PowerSource (发电) |
| `powerSourceType` | PowerSourceType | powerOutput > 0 | 电源类型 |
| `powerRequiresFuel` | bool | powerOutput > 0 | 是否需燃料 |
| `powerFuelPerHour` | float | powerOutput > 0 | 每小时燃料消耗 |
| `powerFuelItemName` | string | powerOutput > 0 | UI 显示用 |
| `powerFuelItemData` | ItemData | powerOutput > 0 | 背包消耗用 |
| `powerDaytimeOnly` | bool | powerOutput > 0 | 仅白天 |
| `powerRequiresOpenAir` | bool | powerOutput > 0 | 需露天 |
| `powerRequiresWater` | bool | powerOutput > 0 | 需水边 |
| `powerNoiseRadius` | float | powerOutput > 0 | 噪音(m) |
| `powerRequired` | float | > 0 | PowerConsumer (用电) |
| `powerAllowCoal` | bool | powerRequired > 0 | 无电可烧煤 |
| `powerCoalPower` | float | powerRequired > 0 | 煤模式功率 |
| `powerElectricSpeedMul` | float | powerRequired > 0 | 通电速度倍率 |

**BuildModeController 运行时挂载逻辑（3 处 for 循环）：**

```csharp
// 放置新建筑
if (activeBuildable.powerSupplyRadius > 0f)  // 挂载 PowerTerminal
if (activeBuildable.powerOutput > 0f)         // 挂载 PowerSource
if (activeBuildable.powerRequired > 0f)       // 挂载 PowerConsumer

// 加载已保存的建筑（同样 3 处判断）
if (data.powerSupplyRadius > 0f) ...
if (data.powerOutput > 0f) ...
if (data.powerRequired > 0f) ...
```

## 12. 编辑器工具

`Tools → 配置电力组件参数` (`SetupPowerComponents.cs`)：

**一键执行 6 步批量配置：**

| 步骤 | 方法 | 配置内容 |
|------|------|----------|
| 1 | `SetupTerminals()` | 用电终端(supplyRadius=15m), 高级终端(25m) |
| 2 | `SetupPowerSources()` | 6 个电源的 type/output/fuel/noise |
| 3 | `SetupPowerConsumers()` | 19 个设备的 powerRequired/allowCoal/coalPower/speedMul |
| 4 | `MigrateOldGenerator()` | 旧"发电机" → EnergyIndustry 200W 烧煤矿 |
| 5 | `CleanupNonProductionDevices()` | 清理广播塔/太阳能板的旧 ProductionDevice |
| 6 | `AssetDatabase.SaveAssets()` + `Refresh()` | 保存并刷新 |

**配置写入逻辑 (ConfigConsumer)：**
1. 查找匹配 `displayName` 的 BuildableData
2. 直接写 `.powerRequired` / `.powerAllowCoal` / `.powerCoalPower` / `.powerElectricSpeedMul`
3. 如果 prefab 存在，同时添加/配置 PowerConsumer 组件 (`zeroWasteOnElectric = true`)

## 13. 完整交互清单

| 操作 | 方式 | 代码路径 |
|------|------|----------|
| 查看电源 | 准星对准电源 + E 键 | `PowerSource.IInteractable.OnInteract → PowerSourceUI.Show` |
| 查看终端 | 准星对准终端 + E 键 | `PowerTerminal.IInteractable.OnInteract → TerminalUI.Show` |
| 从电源连终端 | 电源面板 → "⚡ 连接终端" → 左键点终端 | `CableLinker.StartLinkingFromSource → TryLink → LinkFromSource` |
| 从终端连电源/终端 | 终端面板 → "⚡ 连接" → 左键点目标 | `CableLinker.StartLinking → TryLink → LinkFromTerminal` |
| 取消连线 | 连线模式下右键 | `CableLinker.CancelLink → 返回面板` |
| 断开电源 | 终端面板 → 点发电端行右侧"断开" | `terminal.DisconnectSource` |
| 断开终端串联 | 终端面板 → 点终端行右侧"断开" | `terminal.DisconnectTerminal` |
| 开关设备 | 终端面板 → 设备行"开启"/"关闭"按钮 | `consumer.IsManuallyOff = true/false` |
| 添加燃料 | 电源面板 → "添加燃料"按钮 | `PowerSourceUI.ConsumeFuelFromInventory` |
| 燃料自动补充 | 运行时自动 | `PowerSource.Update` — 燃料耗尽时 `Inventory.RemoveItemByName` |
