> 迭代追踪 | 第1轮: ⬜ | 第2轮: ⬜ | 第3轮: ⬜ | 最后审查: 2026-06-22

## 一、功能概述

角色创建时选择职业，获得差异化初始条件：**属性/技能修正**、**初始装备与物品**、**特殊开局能力**（制作人自带 AI 机器人）。系统基于 `CharacterTemplate` (ScriptableObject) 定义 9 个职业模板，运行时由 `SurvivalXPSystem` 触发 `ProfessionApplier` 进行两步投放：先穿装备扩容容器，再放物品填入容器。

## 三、架构与连接图

```
MainMenuUI.NewGame(slot)
  └─→ SceneManager.LoadScene("MainScene")
       └─→ GameBootstrap.OnAfterSceneLoad()
            ├─→ AddIfMissing(player, "ProfessionApplier")     // 挂载组件
            ├─→ AddIfMissing(player, "SurvivalXPSystem")      // 挂载组件
            └─→ AddIfMissing(player, "PlayerCharacter")       // 加载 DefaultCharacter.asset

PlayerCharacter.Start()
  └─→ 确保 SurvivalXPSystem.Instance 存在
       └─→ SurvivalXPSystem.Awake()  → ServiceLocator.Register() + DontDestroyOnLoad

SurvivalXPSystem.Start()
  └─→ InitFromCharacterData()
       ├─→ 从 CharacterData 读基础属性 (全5) + 技能初值 (近战1, 其余0)
       ├─→ 读取 CharacterData.profession  → CharacterTemplate (SO)
       ├─→ ApplyProfession(template)
       │    ├─→ 遍历 attributeMods  → 叠加到 _attributes[]
       │    └─→ 遍历 skillBoosts   → 叠加到 _skills[]
       └─→ ProfessionApplier.ApplyStartingGear(template)
            ├─→ Step 1: EquipItem(startingEquipment)   // 扩容容器
            ├─→ Step 2: AddItem(startingItems)         // 填入物品
            └─→ Step 3: SpawnStartingAIBot()           // 仅 startWithAIBot=true
                 └─→ NavMesh.SamplePosition (玩家右前方2m)
                 └─→ BuildModeController.PlaceStructureDirect(AIBot, pos)
```

**关键顺序**: 装备必须先于物品，因为装备扩容容器（皮带→负重、背包→格子）。若顺序颠倒，物品可能因超重无法放置。

## 四、CharacterTemplate 结构

```csharp
// Config/CharacterTemplate.cs
[CreateAssetMenu(menuName = "Game/Character Template")]
public class CharacterTemplate : ScriptableObject
{
    public string templateName;                    // 职业名称
    [TextArea(2,4)] public string description;    // 职业描述

    [Header("属性/技能修正")]
    public List<AttributeMod> attributeMods;       // 力量/敏捷/体质/耐力修正
    public List<SkillBoost> skillBoosts;           // 10种技能加成

    [Header("初始装备（开局自动穿上，扩容容器）")]
    public ItemData[] startingEquipment;

    [Header("初始物品（装备扩容后放入背包）")]
    public ItemRequirement[] startingItems;

    [Header("特殊开局")]
    public bool startWithAIBot;                    // 是否开局放置AI机器人
    public BuildableData startingAIBotBuildable;   // AI机器人Buildable引用
}

[System.Serializable]
public class SkillBoost
{
    public SkillType skillType;
    public int bonus;
}
```

- `startingEquipment` → 调用 `Inventory.EquipItem()` 穿到装备槽，扩容容器
- `startingItems` → 调用 `Inventory.AddItem()` 放入已扩容的背包
- `startWithAIBot` + `startingAIBotBuildable` → 通过建造系统直接放置 AI 机器人 prefab

## 七、制作人 AI 机器人开局详解

### 触发条件

`CharacterTemplate` 上两个字段同时设置：

```csharp
t.startWithAIBot = true;
t.startingAIBotBuildable = AssetDatabase.LoadAssetAtPath<BuildableData>(
    "Assets/_Game/Config/BuildableData/Buildable_AIBot.asset");
```

### 放置流程 (ProfessionApplier.SpawnStartingAIBot)

1. 获取 `BuildModeController` 组件
2. 计算位置: `player.position + forward * 2f + right * 1f` (右前方 2m)
3. `NavMesh.SamplePosition(pos, out hit, 5f, NavMesh.AllAreas)` — 在 5m 半径内找最近 NavMesh 点
4. 如果第一步失败，退到 `player.position + right * 2f` 重试
5. 如果仍失败（玩家在地图边缘/非 NavMesh），输出 Error 跳过
6. `buildCtrl.PlaceStructureDirect(buildableData, hit.position)` — 直接 Instantiate，**跳过建造耗时和材料消耗**

### AI 机器人的能力

- 自主跟随玩家战斗（AI 战斗模块）
- 自动拾取战利品（200kg 负重）
- 电击防御攻击
- 限 1 台/人
- 消耗电池组驱动，可额外加入浓缩铀启动增强模式（电击伤害+50%、移速+30%）
- 双进度条 UI：电量 + 核燃料

### 存档恢复

`SaveLoadManager` 恢复存档时会重建 AI 机器人：
- 从存档数据读取机器人位置、旋转、GUID
- 通过 `BuildModeController.PlaceStructureDirect()` 重建
- 覆盖 `PersistentGUID` 为存档值保持身份一致性

## 八、运行时初始化流程 (完整链路)

```
1. 场景加载 (MainScene)

2. GameBootstrap.OnAfterSceneLoad()
   ├─ 确认场景名 != "MainMenu"
   ├─ 找到 Player (tag=Player)
   ├─ PlayerRegistry.Register(player)
   ├─ 挂载所有系统组件 (包括 ProfessionApplier + SurvivalXPSystem)
   └─ AutoResolveReferences(): 加载 DefaultCharacter.asset 到 PlayerCharacter.characterData

3. PlayerCharacter.Start()
   └─ 确保 SurvivalXPSystem.Instance 存在
        └─ new GameObject("SurvivalXPSystem").AddComponent<SurvivalXPSystem>()

4. SurvivalXPSystem.Awake()
   ├─ 单例注册 (Instance)
   ├─ ServiceLocator.Register(this)
   └─ DontDestroyOnLoad(gameObject)

5. SurvivalXPSystem.Start()
   ├─ InitFromCharacterData()
   │    ├─ 从 CharacterData 读取基准属性 (力量/敏捷/体质/耐力 = 5)
   │    ├─ 从 CharacterData 读取基准技能 (近战=1, 其余=0)
   │    ├─ 读取 CharacterData.profession → CharacterTemplate
   │    ├─ ApplyProfession(template)
   │    │    ├─ 遍历 attributeMods → _attributes[i].value += mod (夹在 min-max)
   │    │    └─ 遍历 skillBoosts → _skills[i].level += bonus
   │    └─ ProfessionApplier.ApplyStartingGear(template)
   │         ├─ Step 1: foreach(equipment) → inv.EquipItem(item)
   │         ├─ Step 2: foreach(startingItems) → inv.AddItem(itemData, count)
   │         └─ Step 3: if(startWithAIBot) → SpawnStartingAIBot()
   └─ 订阅 EventBus: SurvivalXpGained → AddXP()
```

### 两步投放为什么必须严格有序

| 步骤 | 操作 | 效果 |
|------|------|------|
| Step 1: EquipItem | 装备到装备槽 | 扩容容器（皮带→负重↑, 背包→格子↑, 胸挂→容器容量↑） |
| Step 2: AddItem | 放入背包 | 依赖已扩容的容器空间，超重则部分无法放入 |

如果 Steps 1 和 2 颠倒，初始物品可能在装备扩容前就因超重而丢失。

## 九、职业对游戏进程的影响

### 开局影响

| 维度 | 说明 |
|------|------|
| 属性基础值 | 职业修正直接叠加（如伐木工 STR=8 vs 医生 STR=5） |
| 技能等级 | 职业修正直接提升初始技能，解锁高阶能力（如技工汽车改造 Lv4） |
| 装备槽/负重 | 士兵穿 4 件 → 负重最大；消防员/流浪者无装备 → 负重低机动高 |
| 材料储备 | 工程师有木材+钉子 → 开局即可建造；医生有药品 → 开局即可处理伤势 |
| 战斗伙伴 | 制作人开局即带 AI 机器人辅助战斗和搜刮 |
| 浓缩铀 | 制作人特有，驱动核电站/AI 增强模式的核心材料 |

### 长期影响

- 职业 **仅影响初始值**，后续成长完全由 `SurvivalXPSystem` 管理
- 任何职业都可以通过 `AddXP()` → `ConvertXPToPoints()` → `SpendPoint()` 弥补弱势领域
- 职业差异的边际效应随游戏时间增加而递减，约 2-3 小时后趋于平缓

## 十、编辑器工具

### 一键生成全部 9 个职业模板

菜单: `Game Tools → Create Profession Templates`

脚本: `Assets/_Game/Editor/CreateProfessionTemplates.cs`

行为:
1. 确保 `Assets/_Game/Config/Character/Templates/` 目录存在
2. 依次调用 9 个 `CreateXxx()` 方法创建 .asset
3. 已存在的模板跳过（不覆盖手动修改）
4. 自动绑定 `DefaultCharacter.profession` → "流浪者"（如果未设置）
5. `AssetDatabase.SaveAssets()` + `Refresh()`

### 新增职业

1. 在 `CreateProfessionTemplates.cs` 中添加 `CreateXxx()` 方法（参考现有模板）
2. 在 `CreateAll()` 中将新方法加入 `templates` 列表
3. 运行菜单生成 .asset
4. 如需运行时选角 UI，需在 MainMenuUI/角色创建界面增加选择入口

### 修改现有职业

直接编辑 `Assets/_Game/Config/Character/Templates/{职业名}.asset` Inspector，或重新运行生成工具（已存在则跳过，不会覆盖）。

## 十一、关联系统矩阵

| 系统 | 连接点 | 说明 |
|------|--------|------|
| **SurvivalXPSystem** | `InitFromCharacterData()` | 读取 profession → ApplyProfession() 叠加修正 → 触发 ProfessionApplier |
| **ProfessionApplier** | `ApplyStartingGear()` | 三步投放：EquipItem → AddItem → SpawnAIBot |
| **Inventory** | `EquipItem()` / `AddItem()` | 接收装备和物品，扩容/填入 |
| **BuildModeController** | `PlaceStructureDirect()` | 直接放置 AI 机器人 prefab（免建造） |
| **BuildableData** | `Buildable_AIBot.asset` | 制作人开局引用的 AI 机器人 Buildable |
| **PlayerCharacter** | `characterData` 引用 | 持有 DefaultCharacter.asset，其 profession 字段指向当前职业模板 |
| **CharacterData** | `profession` 字段 | 指向当前激活的 CharacterTemplate |
| **SaveLoadManager** | 存档时保存 profession + 建筑 | 读档时恢复 AI 机器人位置/GUID；当前未保存职业名到存档 |
| **MainMenuUI** | 新游戏 → LoadScene("MainScene") | 触发整个 Bootstrap → 职业初始化链路 |
| **GameBootstrap** | `OnAfterSceneLoad()` | 确保 ProfessionApplier + SurvivalXPSystem + BuildModeController 存在于 Player |
| **CreateProfessionTemplates** (Editor) | 一键生成 9 个 .asset | 硬编码物品路径引用 `Config/Equipment/` `Weapons/` `Consumables/` `Materials/` |
