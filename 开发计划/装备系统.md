## 一、功能概述

13 个装备槽位（武器/衣物/护甲），Dictionary 数据驱动。装备物品自动扩容对应容器，卸下物品恢复默认尺寸。腰带与武器槽幽灵占位系统。护甲值/保暖值通过事件推送至生存系统。武器切换系统（Q 键循环）+ 武器模型持有系统（骨骼绑定 / Capsule 偏移两种模式）。

---

## 二、装备槽位 (EquipSlot)

```csharp
// Config/EquipSlot.cs
public enum EquipSlot
{
    None,
    // 可穿戴容器（装备后扩容对应容器）
    Tops, Pants, Belt, Vest, Backpack,
    // 纯护甲（无容器功能）
    Head, BodyArmor,
    // 武器槽
    RightHand, LeftHand, KnifeBelt, SidearmBelt
}
```

武器槽(RightHand/LeftHand/KnifeBelt/SidearmBelt)共享同一容器但各有独立装备位。武器槽无容器功能 — `IsWeaponSlot()` 返回 `true` 时不走容器扩容分支，而是走幽灵占位 + 武器事件分支。

## 三、核心方法 (Inventory.cs)

### 3.1 EquipItem(ItemData item, EquipSlot targetSlot)

```csharp
public bool EquipItem(ItemData item, EquipSlot targetSlot)
```

**流程**：
1. 拒绝 `equipSlot == None`
2. 武器 → 武器槽：以拖放目标槽位为准（允许右/左/刀/枪互换）；非武器忽略 targetSlot，走自身 equipSlot
3. 小刀/手枪槽：检查 `HasBeltSpaceFor`（需腰带2格幽灵空间）→ 不足返回 false + 发布 "belt_full"
4. 先 `UnequipSlot(slot)` 卸下旧装备
5. 武器分支：`PlaceBeltGhosts` → 发布 `WeaponEquippedEvent`
6. 非武器分支：找到对应容器 → `storageWidth/Height` 覆盖网格尺寸 → `TryMoveItemsToOtherContainers`（将旧装备容器的物品转移到其他容器，放不下则丢弃）
7. 发布 `EquipmentChangedEvent(总护甲, 总保暖)`

**重载** `EquipItem(ItemData item)` — 走 item 自身的 `equipSlot`。

### 3.2 UnequipSlot(EquipSlot slot)

```csharp
public ItemData UnequipSlot(EquipSlot slot)
```

**流程**：
1. 武器：`RemoveBeltGhosts` → 发布 `WeaponUnequippedEvent`
2. 非武器：丢弃容器内物品(`DropItem`, 排除ghost) → 恢复默认尺寸
3. **级联**：卸下 Belt → 同时卸下 KnifeBelt + SidearmBelt（物品以 `SpawnDropAtFeet` 掉落脚下）
4. 发布 `EquipmentChangedEvent`

### 3.3 装备扩容机制

```
装备小背包(storageWidth=3, height=2) → Belt容器 1×1 → 3×2
卸下背包 → Belt容器恢复 1×1 → 旧物品尝试移到其他容器 → 放不下就丢弃
```

`TryMoveItemsToOtherContainers` 遍历所有非当前容器按 `AddItem` 尝试，移走了就从源容器移除。

## 四、护甲/保暖值

```
GetTotalArmor()  = Σ equipped[slot].armorValue
GetTotalWarmth() = Σ equipped[slot].warmthValue

每装备变化 → EquipmentChangedEvent.Publish
  → SurvivalSystem.OnEquipmentChanged:
      _cachedArmor = totalArmor   → 伤害减免公式 100/(100+armor)
      _cachedWarmth = totalWarmth → 体温目标+0.75°C per warmth
```

## 五、腰带幽灵系统

### 5.1 机制

```
刀刃槽(KnifeBelt) 武器 → 腰带容器中放置 2个1×1 Ghost格
手枪槽(SidearmBelt) 武器 → 腰带容器中放置 2个1×1 Ghost格
```

每个 Ghost 格是 `PlacedItem.Ghost(gridX, gridY, weaponSlot)`，`count=0, itemData=null`，只在腰带容器中占位，不在背包UI渲染（或被排除）。

### 5.2 HasBeltSpaceFor(weaponSlot, neededCells)

```csharp
bool HasBeltSpaceFor(EquipSlot weaponSlot, int neededCells)
```

1. 获取腰带容器 `GetContainer(EquipSlot.Belt)`
2. 遍历腰带所有格子，**排除同槽旧Ghost格**（卸下旧武器会释放这些空间）
3. 统计实际空闲格数 >= neededCells 则返回 true

### 5.3 PlaceBeltGhosts / RemoveBeltGhosts

- `PlaceBeltGhosts(slot)` — 在腰带中用 `FindFreeCell` 找2个空位放置Ghost
- `RemoveBeltGhosts(slot)` — `belt.placedItems.RemoveAll(p => p.isGhost && p.ghostSourceSlot == slot)`

### 5.4 级联规则

```
卸下腰带 → 自动卸下 KnifeBelt + SidearmBelt → SpawnDropAtFeet(物品)
```

`SpawnDropAtFeet` 生成 `WorldItem` GameObject：位置 `transform.position + forward*1.2 + up*0.1`。

## 六、武器切换系统 (WeaponSwitcher)

### 6.1 概述

`Systems/Weapon/WeaponSwitcher.cs` — 管理活跃武器槽位的切换与热键。

### 6.2 核心状态

```csharp
EquipSlot _activeSlot;         // 当前激活的武器槽
EquipSlot[] _cycleOrder =      // 循环顺序（兼容任意数量武器槽）
    { RightHand, LeftHand, KnifeBelt, SidearmBelt };
```

### 6.3 关键逻辑

**Q 键循环**：`InputRouter.BindKey(KeyCode.Q, InputPriority.Gameplay, HandleCycleWeapon)`

**CycleNext()** — 从当前槽沿 `_cycleOrder` 向后找第一个有武器的槽；全空则切 None。

**自动切换**：
- `OnWeaponEquipped` — 装备武器时若当前空手或新武器是 RightHand，自动切到该槽
- `OnWeaponUnequipped` — 当前槽武器卸下时自动切换到下一个有武器的槽

**SwitchTo(slot)**：
- 防重入（`_activeSlot == slot` 直接返回）
- 根据 `weapon.isFirearm` 切换 `WeaponShooting.enabled` / `PlayerCombat.enabled`（互斥）
- 发布 `WeaponSlotChangedEvent(slot, weapon)`

**ForceSwitch(slot)**：先重置 `_activeSlot = None` 再切（确保同槽重复触发）。

### 6.4 公开属性

```csharp
public EquipSlot ActiveSlot => _activeSlot;
public ItemData ActiveWeapon   // 从 inventory.equipped[activeSlot] 查
```

## 七、武器持有系统 (WeaponHolder)

### 7.1 概述

`Systems/Weapon/WeaponHolder.cs` — 管理武器3D模型的实例化、显示/隐藏、朝向跟随。

### 7.2 手部定位

两种模式自动降级：
1. **骨骼模式**：通过 `FindHandBone(handBoneName)` 在子Transform中递归查找名为 "RightHand" 的骨骼节点
2. **Capsule模式**（骨骼未找到时）：创建一个 `RightHandPoint` 子GameObject，`localPosition = CAPSULE_HAND_OFFSET (0.15, 0.55, 0.15)`, `localEulerAngles = CAPSULE_HAND_EULER (0, 90, -90)`

Capsule 模式下每帧 LateUpdate 保持偏移（防止动画覆盖）。

### 7.3 模型管理

```csharp
Dictionary<EquipSlot, GameObject> _models;  // 每个武器槽一个模型实例
```

- `OnWeaponEquipped` → `SpawnModel(item, slot)` — 实例化 `item.worldPrefab` 到手部Transform下，保持 localPosition/rotation 归零
- `OnWeaponUnequipped` → 销毁对应模型
- `OnSlotChanged` → 遍历所有模型，激活当前槽的模型，隐藏其余

### 7.4 朝向跟随

LateUpdate 中对当前激活模型做水平面朝向 Slerp：

```csharp
var flatDir = new Vector3(AimDirection.x, 0, AimDirection.z);
var targetRot = Quaternion.LookRotation(flatDir, Vector3.up);
activeModel.transform.rotation = Quaternion.Slerp(
    activeModel.transform.rotation, targetRot,
    WEAPON_ROTATION_SMOOTH_SPEED(20f) * Time.deltaTime
);
```

`AimDirection` 由外部（输入系统）每帧设置。

## 八、事件系统

| 事件 | 发布者 | 订阅者 | 负载 |
|------|--------|--------|------|
| `WeaponEquippedEvent` | Inventory.EquipItem | WeaponHolder(生成模型), WeaponSwitcher(自动切换) | slot, item |
| `WeaponUnequippedEvent` | Inventory.UnequipSlot | WeaponHolder(销毁模型), WeaponSwitcher(自动切换) | slot |
| `WeaponSlotChangedEvent` | WeaponSwitcher.SwitchTo | WeaponHolder(显示/隐藏模型), UI | slot, weapon |
| `EquipmentChangedEvent` | Inventory.EquipItem / UnequipSlot | SurvivalSystem(缓存护甲/保暖) | totalArmor, totalWarmth |
| `InventoryChanged` | Inventory (各种操作) | UI 提示 | 操作类型, 物品名, 数量 |

## 九、存档系统集成

### 9.1 保存 (GetSaveData)

`Inventory.GetSaveData()` 返回 `InventorySaveData`：
- **7个容器**的完整快照：containerName, gridWidth/Height, 每个 PlacedItem 的 instanceId/itemName/count/gridX/Y/rotated/isGhost/ghostSourceSlot
- **equipped字典**：非武器槽的装备物品通过 instanceId 定位（武器槽不在此存，武器自身在容器中已有数据）
- **ammoReserves**：弹药储备字典（由 SaveLoadManager 补充武器弹夹数据）

### 9.2 恢复 (RestoreFromSave)

`Inventory.RestoreFromSave(inv, itemCatalog)`：
1. 清空所有容器
2. 按存档逐容器重建 placedItems（非ghost通过 itemCatalog.Find 还原 ItemData；ghost通过 ghostSourceSlot 字符串解析还原）
3. 按 instanceId 定位 loadedItems，直接设置 equipped 字典（不调 EquipItem 避免副作用）
4. 更新 `_nextItemInstanceId` 计数器（max(savedIds) + 1）

### 9.3 武器弹药存档

```csharp
// 供存档系统采集/恢复武器弹夹子弹数
int GetWeaponAmmo(EquipSlot weaponSlot);   // → WeaponShooting.GetCurrentMag
void SetWeaponAmmo(EquipSlot weaponSlot, int ammo);  // → WeaponShooting.SetCurrentMag
```

只追踪当前活跃武器槽的弹夹数（WeaponShooting 内部只有一个 `_currentMag`）。
