# 2.3 装备系统 ✅ Phase 1 完成

> **状态**: 已实现 · **日期**: 2026-05-11 · **依赖**: L2 背包系统(InventoryContainer) + L1 ItemData

## 一、功能概述

13 个装备槽位（武器/衣物/护甲），Dictionary 数据驱动。装备物品自动扩容对应容器，卸下物品恢复默认尺寸。腰带与武器槽幽灵占位系统。护甲值/保暖值通过事件推送至生存系统。

---

## 二、装备槽位 (EquipSlot)

```csharp
// Config/EquipSlot.cs
public enum EquipSlot
{
    None,
    // 可穿戴容器（装备后扩容对应容器）
    Tops, Pants, Belt, Vest, Backpack,
    // 纯护甲（无容器功能）
    Head, BodyArmor,
    // 武器槽
    RightHand, LeftHand, KnifeBelt, SidearmBelt
}
```

武器槽(RightHand/LeftHand/KnifeBelt/SidearmBelt)共享同一容器但各有独立装备位。

## 三、核心方法 (Inventory.cs)

### EquipItem(ItemData item, EquipSlot targetSlot)
1. 拒绝 `equipSlot == None`
2. 武器：检查 `HasBeltSpaceFor`（腰带武器需2格幽灵占位）→ `PlaceBeltGhosts` → 发布 WeaponEquippedEvent
3. 非武器：找到对应容器 → `storageWidth/Height` 覆盖网格尺寸 → `TryMoveItemsToOtherContainers`（旧物品防溢出）
4. 发布 `EquipmentChangedEvent(总护甲, 总保暖)`

### UnequipSlot(EquipSlot slot)
1. 武器：`RemoveBeltGhosts` → 发布 WeaponUnequippedEvent
2. 非武器：丢弃容器内物品(`DropItem`) → 恢复默认尺寸
3. 级联：卸下Belt → 同时卸下 KnifeBelt + SidearmBelt（物品掉落脚下）
4. 发布 `EquipmentChangedEvent`

### 装备扩容
```
装备小背包(storageWidth=3, height=2) → Belt容器 1×1 → 3×2
卸下背包 → Belt容器恢复 1×1 → 旧物品尝试移到其他容器 → 放不下就丢弃
```

## 四、护甲/保暖值

```
GetTotalArmor()  = Σ equipped[slot].armorValue
GetTotalWarmth() = Σ equipped[slot].warmthValue

每装备变化 → EquipmentChangedEvent.Publish
  → SurvivalSystem.OnEquipmentChanged:
      _cachedArmor = totalArmor   → 伤害减免公式 100/(100+armor)
      _cachedWarmth = totalWarmth → 体温目标+0.75°C per warmth
```

## 五、腰带幽灵系统

```
刀刃槽(KnifeBelt) 武器 → 腰带容器中放置 2个1×1 Ghost格
手枪槽(SidearmBelt) 武器 → 腰带容器中放置 2个1×1 Ghost格

HasBeltSpaceFor(itemData):
  检查腰带空闲格 - 扣除同槽旧Ghost格占用的空间
  充足 → true / 不足 → 发布 belt_full 事件 → UI显示提示

卸下武器 → RemoveBeltGhosts (清理Ghost占位)
卸下腰带 → 级联卸下 KnifeBelt + SidearmBelt
```
