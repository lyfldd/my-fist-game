## 1. GameBootstrap — 运行时启动器

**路径**：`Assets/_Game/Core/GameBootstrap.cs`
**类型**：`static class GameBootstrap`

GameBootstrap 是整个游戏运行时入口。它通过 `[RuntimeInitializeOnLoadMethod]` 特性挂载到 Unity 初始化管线，不依赖编辑器脚本，场景丢失或电脑崩溃后也能自动恢复。

### 1.1 初始化序列

```
Unity Boot → BeforeSceneLoad → EnsureInputRouter()
                               → 创建/确保 Managers GameObject
                               → 挂载 InputRouter + 全局系统

Unity Boot → AfterSceneLoad  → 检测场景名，跳过 MainMenu
                               → 查找 Player (tag/name 三重兜底)
                               → PlayerRegistry.Register(player)
                               → 兜底添加 Rigidbody/CapsuleCollider/Animator
                               → AddIfMissing (50+ 组件按依赖顺序)
                               → 设置 FactionComponent → FactionType.Player
                               → 配置 CameraFollow
                               → AutoResolveReferences(player)
                               → EnsureInputRouter() (再次确认)
```

### 1.2 BeforeSceneLoad — 静态初始化

```csharp
[RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.BeforeSceneLoad)]
static void OnBeforeSceneLoad()
{
    EnsureInputRouter();
}
```

- 在第一个场景加载前执行。
- 目的：确保 InputRouter 单例 + 全局 Managers 对象在任何 Awake/Start 之前存在。

### 1.3 AfterSceneLoad — 延迟初始化

```csharp
[RuntimeInitializeOnLoadMethod(RuntimeInitializeLoadType.AfterSceneLoad)]
static void OnAfterSceneLoad() { ... }
```

完整初始化流程：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 场景名检测 | `MainMenu` 场景直接 return，不初始化玩家系统 |
| 2 | 查找玩家 | `FindWithTag("Player")` → `Find("player")` → `Find("Player")` 三重兜底 |
| 3 | 注册玩家 | `PlayerRegistry.Register(player)` 写入全局注册表 |
| 4 | 物理组件兜底 | Rigidbody (FreezeRotation) + CapsuleCollider (高2m，半径0.3m) |
| 5 | Animator 兜底 | 优先子对象已有，否则新建并加载 Avatar 和 Controller |
| 6 | 组件按序添加 | `AddIfMissing()` 调用 System.Type.GetType + AddComponent |
| 7 | 阵营设置 | `player.GetComponent<FactionComponent>().SetFaction(FactionType.Player)` |
| 8 | 摄像机配置 | 为主摄像机添加/获取 CameraFollow，设置 target |
| 9 | 自检 | 确认 PlayerCharacter 和 FactionComponent 存在 |

### 1.4 AddIfMissing — 反射组件添加

```csharp
static void AddIfMissing(GameObject go, string fullTypeName)
{
    var type = System.Type.GetType(fullTypeName + ", Assembly-CSharp");
    if (type == null) return;
    if (go.GetComponent(type) == null)
    {
        go.AddComponent(type);
        Debug.LogWarning($"[GameBootstrap] ⚠️ {type.Name} 未预置在 Player 上...");
    }
}
```

**组件添加顺序**（按功能域分组）：

| 类别 | 组件 |
|------|------|
| Player 核心 | PlayerController, Inventory, PlayerInteraction, PlayerCharacter, SurvivalSystem |
| 战斗 | PlayerCombat, WeaponShooting, WeaponAiming, WeaponHolder, SpreadVisualizer |
| HUD UI | SurvivalHUD, QuickItemBar, CrosshairUI, DecibelHUD, WeatherHUD, TopLeftHUD |
| 建造系统 | BuildMenuUI, GhostPreview, BuildModeController, BuildModeInputLock |
| 合成系统 | CraftingUI, ChemicalResearchManager, ChemicalResearchUI |
| 输入/角色 | MouseGroundProjector, StaminaSystem, SurvivalXPSystem |
| 其他 | DamageablePlayer, FactionComponent, WeaponSwitcher, ProfessionApplier, VehicleInputLock, InventoryTest |

### 1.5 EnsureInputRouter — 全局系统注册

```csharp
static void EnsureInputRouter()
{
    var go = GameObject.Find("Managers") ?? new GameObject("Managers");
    Object.DontDestroyOnLoad(go);
    // 依次添加: InputRouter, SoundEmitter, DecibelSystem, MuzzleFlashSystem,
    //           FactionSystem, ThreatSystem, SaveLoadManager, WorldItemManager,
    //           SaveTriggerSystem, PlacedStructureRegistry, ProductionDeviceUI
    if (go.GetComponent<T>() == null) go.AddComponent<T>();
}
```

- 创建或找到 `Managers` GameObject，标记 `DontDestroyOnLoad`。
- 挂载所有跨场景全局系统（输入、声音、阵营、威胁、存档、生产设备UI）。
- 在 BeforeSceneLoad 和 AfterSceneLoad 末尾各调用一次，确保存在。

### 1.6 AutoResolveReferences — ScriptableObject 自动引用

注入流程：

| 目标组件 | 字段 | 数据源 |
|----------|------|--------|
| SurvivalSystem | `survivalData` | `Resources.Load("SurvivalData_Default")` |
| SurvivalSystem | `playerCharacter` | `GetComponent<PlayerCharacter>()` |
| SurvivalSystem | `timeManager` | `ServiceLocator.Get<TimeManager>()` |
| PlayerCharacter | `characterData` | `Resources.Load("DefaultCharacter")` → Editor AssetDatabase 兜底 → `CreateInstance<CharacterData>()` 终极兜底 |
| BuildMenuUI | `catalog` | `Resources.Load("BuildableCatalog")` |
| ChemicalResearchManager | `_researchData` | `Resources.Load("ChemicalResearchData")` |
| FactionComponent | `_factionData` | Editor `FindAssets("t:FactionData")`，无 asset 返回 null（代码有自动创建逻辑） |

```csharp
static void TrySetField(object target, string fieldName, Func<Object> resolver)
{
    var field = target.GetType().GetField(fieldName,
        BindingFlags.Instance | BindingFlags.Public | BindingFlags.NonPublic);
    if (field == null) return;
    var current = field.GetValue(target);
    if (current != null && !(current is Object uo && uo == null)) return; // 已有值则跳过
    var value = resolver();
    if (value != null) field.SetValue(target, value);
}
```

- 使用反射（`System.Reflection`）读写任意可见性字段。
- 优先 `Resources.Load`，其次 `AssetDatabase.LoadAssetAtPath`（仅 Editor），最后 `CreateInstance` 终极兜底。

## 2. PlayerRegistry — 玩家全局注册表

**路径**：`Assets/_Game/Core/PlayerRegistry.cs`
**类型**：`static class PlayerRegistry`

唯一玩家引用的单一来源。只有 GameBootstrap 写入，所有其他系统只读。

### 2.1 API

| 成员 | 类型 | 说明 |
|------|------|------|
| `Transform` | `Transform` | 玩家 Transform（最常用），Lazy-init via `FindWithTag("Player")` |
| `GameObject` | `GameObject` | 玩家 GameObject |
| `Position` | `Vector3` | 玩家世界坐标快捷访问 |
| `Exists` | `bool` | 是否已注册 |
| `Get<T>()` | 泛型方法 | 获取玩家身上的组件，缓存友好 |
| `Register(GameObject)` | 方法 | 注册玩家（仅 GameBootstrap 调用） |
| `Clear()` | 方法 | 注销玩家 |

### 2.2 Lazy Init 机制

```csharp
public static Transform Transform
{
    get
    {
        if (_playerTransform == null)
        {
            var go = GameObject.FindWithTag("Player");
            if (go != null) { _playerTransform = go.transform; _playerObject = go; }
        }
        return _playerTransform;
    }
}
```

- 如果 GameBootstrap 未先注册，通过 `FindWithTag("Player")` 兜底查找并自动缓存。
- 注意：`FindWithTag` 是慢操作，正式流程应走 Register 路径。

### 2.3 使用模式

```csharp
// 获取 Transform（最常用）
var playerTransform = PlayerRegistry.Transform;

// 获取位置
Vector3 pos = PlayerRegistry.Position;

// 获取特定组件
var character = PlayerRegistry.Get<PlayerCharacter>();
var survival = PlayerRegistry.Get<SurvivalSystem>();
```

## 3. ServiceLocator — 统一服务定位器

**路径**：`Assets/_Game/Core/ServiceLocator.cs`
**类型**：`static class ServiceLocator`

替代 `FindObjectOfType` 的类型安全全局服务注册表。

### 3.1 API

| 方法 | 说明 |
|------|------|
| `Register<T>(T instance)` | 注册服务实例（通常 Awake 中调用）。同名已注册会用新的覆盖。 |
| `Get<T>()` | 获取服务。未注册时触发 `FindObjectOfType` 兜底并缓存。 |
| `Override<T>(T instance)` | 强制覆盖注册（运行时切换场景） |
| `GetAll<T>()` | 获取所有已注册实例（每次 `FindObjectsOfType`，不缓存） |
| `Unregister<T>()` | 注销服务 |
| `Clear()` | 清空所有注册（场景切换时） |

### 3.2 内部机制

- 内部存储：`Dictionary<Type, object>`
- Unity Object 生命周期检测：Get 时会检查缓存的 Unity Object 是否为 null（已被销毁），若是则自动移除并重新查找。
- Fallback 兜底：`FindObjectOfType` + Editor 警告（提示开发者在 Awake 加 Register）。

### 3.3 使用模式

```csharp
// 注册 — 在 Awake 中
void Awake() { ServiceLocator.Register(this); }

// 获取 — 任意位置
var inventory = ServiceLocator.Get<Inventory>();
var timeManager = ServiceLocator.Get<TimeManager>();
```

## 4. InputRouter — 优先级输入路由

**路径**：`Assets/_Game/Core/InputRouter.cs`
**类型**：`class InputRouter : MonoBehaviour`
**ScriptExecutionOrder**：-100（在所有系统之前运行）

### 4.1 架构

```
按键按下 → InputRouter.Update() → 按优先级排序的绑定列表 → 依次调用回调
                                                              ↓
                                              回调返回 true = 消费（阻断低优先级）
                                              回调返回 false = 放行
```

### 4.2 InputPriority 优先级定义

| 优先级 | 数值 | 用途 |
|--------|------|------|
| `Debug` | 200 | 调试工具面板（最先收到按键） |
| `UI` | 100 | UI 面板（背包/合成/容器/建造菜单） |
| `Action` | 50 | 交互动作（建造确认/取消、E键交互、上下车） |
| `Gameplay` | 0 | 游戏玩法（射击/近战/快捷栏/WASD） |

数值越大越先收到按键。

### 4.3 API

| 方法 | 说明 |
|------|------|
| `BindKey(KeyCode, InputPriority, Func<bool>, MonoBehaviour)` | 绑定按键。同 owner 同按键重复绑定会覆盖旧绑定。 |
| `BindMouse(int button, InputPriority, Func<bool>, MonoBehaviour)` | 绑定鼠标按键（0=左键, 1=右键, 2=中键）。 |
| `BindKeyHeld(...)` | 持续按住绑定（暂未实现，WASD/Shift 继续使用直接 Input 调用） |
| `UnbindAll(MonoBehaviour owner)` | 移除指定 owner 的所有键盘+鼠标绑定。应在 OnDisable/OnDestroy 调用。 |

### 4.4 内部实现细节

- **快照机制**：Update 中先将 `_keyMap` 和 `_mouseMap` 快照到临时列表再遍历，防止回调中 Bind/Unbind 导致 "collection modified" 异常。
- **同帧去重**：同一帧内同一按键/鼠标按钮只处理一次（`_lastKeyFrame` / `_lastMouseFrame` 检测）。
- **回调阻塞链**：按优先级降序排序，高优先级先执行；若任一回调返回 `true`，则 `break` 不再执行后续回调。
- **死绑定清理**：每帧 `CleanDeadBindings()` 会移除 owner 为 null 的绑定（MonoBehaviour 已被销毁）。

### 4.5 使用模式

```csharp
// 绑定 — OnEnable
void OnEnable()
{
    InputRouter.BindKey(KeyCode.E, InputPriority.Action, OnInteract, this);
    InputRouter.BindMouse(0, InputPriority.Gameplay, OnShoot, this);
}

// 回调 — 返回 true 消费，false 放行
bool OnInteract()
{
    if (TryInteract()) return true;   // 消费
    return false;                      // 放行给低优先级
}

// 解绑 — OnDisable
void OnDisable()
{
    InputRouter.UnbindAll(this);
}
```

## 5. EventBus — 全局类型安全事件总线

**路径**：`Assets/_Game/Core/EventBus.cs`
**类型**：`static class EventBus`

### 5.1 设计约束

- **泛型约束** `where T : struct`：所有事件必须是值类型（readonly struct），避免 GC 压力。
- **逐个 try-catch**：Publish 时每个订阅者独立 try-catch，防止一个异常中断整条链。
- **类型隔离**：按 `typeof(T)` 作为字典 key，不同类型事件互不干扰。

### 5.2 API

| 方法 | 说明 |
|------|------|
| `Subscribe<T>(Action<T>)` | 订阅事件 |
| `Unsubscribe<T>(Action<T>)` | 取消订阅事件 |
| `Publish<T>(T eventData)` | 发布事件。逐个调用所有订阅者，各自 try-catch。 |
| `Clear()` | 清空所有事件订阅（场景切换时调用） |
| `GetSubscriberCount<T>()` | 获取某事件订阅者数量（调试用） |
| `GetAllSubscriptions()` | 获取所有事件订阅完整摘要字符串（调试用） |

### 5.3 使用模式

```csharp
// 订阅 — OnEnable
void OnEnable()
{
    EventBus.Subscribe<PlayerDamaged>(OnPlayerDamaged);
    EventBus.Subscribe<InventoryChanged>(OnInventoryChanged);
}

// 处理
void OnPlayerDamaged(PlayerDamaged e)
{
    healthBar.SetValue(currentHealth - e.Damage);
}

// 取消订阅 — OnDisable
void OnDisable()
{
    EventBus.Unsubscribe<PlayerDamaged>(OnPlayerDamaged);
    EventBus.Unsubscribe<InventoryChanged>(OnInventoryChanged);
}

// 发布 — 任意位置
EventBus.Publish(new PlayerDamaged(10, "ZombieBite"));
```

## 附录 C：关键设计决策

1. **InputRouter 不处理持续输入**（WASD/Shift）：持续输入直接使用 `Input.GetKey()`，不移入路由系统。`BindKeyHeld` 方法签名存在但体为空。

2. **EventBus 泛型约束 `struct`**：强制所有事件为值类型，分配到栈上减少 GC 压力。

3. **GameBootstrap 编辑器兜底**：`#if UNITY_EDITOR` 块内使用 `AssetDatabase` 加载资源，确保运行时不依赖编辑器 API。

4. **ServiceLocator Fallback**：`FindObjectOfType` 兜底仅作为过渡期兼容，生产环境每个服务应在 Awake 中主动 `Register(this)`。

5. **PlayerRegistry Lazy-init**：`Transform` getter 中有 `FindWithTag("Player")` 兜底，确保场景重载后也能恢复引用。

6. **GameConstants 设计**：全项目唯一魔法数字来源。所有系统直接引用常量字段，避免散落硬编码。按功能域分组（网格、物理、属性、XP、天气等），注释全部中文。
