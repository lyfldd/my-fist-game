# 背包/物品栏系统 (Inventory System)

> 文档更新时间: 2026-06-22
> 状态: 已完成 (Phase 1-5 完整实现，含存档)

---

## 1. 系统概览

背包系统是生存游戏的核心模块，管理玩家携带的所有物品。采用**多容器分级网格**架构，模拟真实装备负载逻辑：不同装备位提供独立的存储空间（口袋/腰带/胸挂/背包），装备品质直接影响可用格子数，负重由玩家力量属性加成。

**核心设计理念**:
- **多容器管理**: 7 个独立容器（上衣口袋、裤子口袋、腰带、胸挂、背包、头盔、防弹衣），每个有独立的网格尺寸和物品列表
- **装备即容器**: 穿上装备后，容器自动扩展为该装备的 storageWidth × storageHeight；脱下时物品溢出则丢在地上
- **武器槽独立**: 主武器/副武器/小刀/手枪不走容器系统，直接挂载武器模型，腰带通过幽灵占位格消耗腰带空间
- **负重系统**: 总负重 = 所有容器物品重量之和，超过 EffectiveMaxWeight 进入超载（软限制），超过 overloadWeight 硬性拒绝添加
- **RectTransform 绝对坐标 UI**: 完全脱离 GridLayoutGroup，每个格子独立坐标定位，支持跨格物品渲染和滚动

## 3. 架构设计

### 3.1 数据流

```
ItemData (SO 配置)
   ↓
PlacedItem (运行时实例, struct)
   ↓ 放入
InventoryContainer (容器, 管理网格)
   ↓ 聚合
Inventory (MonoBehaviour, 管理所有容器 + 装备)
   ↓ BuildViewData()
InventoryViewData (数据快照, struct)
   ↓ 发布 InventoryViewChangedEvent
InventoryUI (只读渲染)
   ↓ 用户拖拽操作
DragDropManager → Inventory.AddItem/RemoveItem/EquipItem
```

### 3.2 事件总线

| 事件 | 发布者 | 订阅者 | 用途 |
|------|--------|--------|------|
| `InventoryChanged` | Inventory | InventoryUI / QuickItemBar / 浮动提示 | 物品增删移旋等操作通知 |
| `InventoryViewChangedEvent` | Inventory.PublishView() | InventoryUI | 推送完整数据快照驱动 UI 刷新 |
| `EquipmentChangedEvent` | Inventory.EquipItem/UnequipSlot | SurvivalSystem | 推送总护甲/总保暖值更新 |
| `WeaponEquippedEvent` | Inventory.EquipItem | WeaponHolder | 武器槽装备时生成3D模型 |
| `WeaponUnequippedEvent` | Inventory.UnequipSlot | WeaponHolder | 武器槽卸下时销毁3D模型 |

## 4. 核心类详解

### 4.1 Inventory (背包根组件)

**继承**: MonoBehaviour
**挂载**: Player GameObject

#### 字段
| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `maxWeight` | float | 30 (来自 GameConstants) | 基础负重上限 |
| `overloadWeight` | float | 50 | 硬性超载上限，超过无法添加 |
| `defaultTopsSize` ~ `defaultHeadSize` | Vector2Int | 1×1 | 各装备槽未装备时的默认格子尺寸 |
| `equipped` | Dictionary<EquipSlot, ItemData> | 空 | 当前装备字典，按槽位存储 |
| `containers` | List<InventoryContainer> | 7 个 | 容器列表（上衣/裤子/腰带/胸挂/背包/头盔/防弹衣） |
| `ActiveContainer` | InventoryContainer | null | UI 当前活跃容器（快显面板V键使用） |
| `_nextItemInstanceId` | int | 1 (自增) | PlacedItem 实例 ID 计数器（存档系统用） |

#### 装备槽位（EquipSlot 枚举）

```
None       — 非装备
Tops       — 上衣（决定上衣口袋大小）
Pants      — 裤子（决定裤子口袋大小）
Belt       — 腰带（决定快速栏大小；解锁小刀/手枪位）
Vest       — 胸挂（决定胸挂格大小）
Backpack   — 背包（决定主存储格大小）
Head       — 头盔（无容器，仅护甲+显示）
BodyArmor  — 防弹衣（单独护甲层，2×1占位容器）
RightHand  — 主武器（武器绑定位，始终可用）
LeftHand   — 副武器（武器绑定位，始终可用）
KnifeBelt  — 小刀位（需腰带解锁，占腰带2格幽灵位）
SidearmBelt — 手枪位（需腰带解锁，占腰带2格幽灵位）
```

**武器槽判定**: `IsWeaponSlot()` 返回 true 的槽位不走容器系统，而是发布 WeaponEquippedEvent 给 WeaponHolder 生成 3D 模型。KnifeBelt/SidearmBelt 额外在腰带容器中放置 2 个幽灵占位格，反映真实腰带容量占用。

#### 核心方法

**初始化**
- `Awake()`: 注册 ServiceLocator，初始化 7 个容器，每个容器关联一个 EquipSlot 和快捷键（T=上衣/P=裤子/V=胸挂/B=背包）
- `InitializeContainers()`: 清空并重建容器列表，应用已装备物品扩展网格，确保每个容器至少 1×1

**装备系统**
- `EquipItem(ItemData, EquipSlot)`: 穿上装备到指定槽位。武器类以 targetSlot 为准，非武器以 item.equipSlot 为准。装备后容器扩展为 item.storageWidth × storageHeight。若容器已有物品超出新尺寸则尝试迁移到其他容器，失败则 Warning。
- `EquipItem(ItemData)`: 穿到物品自身的 equipSlot
- `UnequipSlot(EquipSlot)`: 脱下装备。容器物品全部丢出（排除幽灵格），容器恢复默认尺寸。腰带拆除时级联卸下小刀和手枪。

**物品增删**
- `AddItem(ItemData, count)`: 按优先级（裤子→腰带→胸挂→背包）尝试放入。支持同类型堆叠。硬超载时拒绝添加。返回实际添加数量。
- `RemoveItem(ItemData, count)`: 遍历所有容器移除指定物品
- `RemoveItemByName(string, count)`: 按名称移除（供弹药/燃料系统使用）
- `CountItemByName(string)`: 按名称统计数量（供弹药/燃料系统使用）
- `DropItem(ItemData, count)`: RemoveItem 成功后在世界空间生成 WorldItem 预制体

**重量系统**
- `EffectiveMaxWeight`: maxWeight + 力量 × STRENGTH_CARRY_WEIGHT_BONUS (5/点)
- `CurrentWeight`: 所有容器物品重量之和
- `IsOverloaded`: CurrentWeight > EffectiveMaxWeight (软限制，仅UI提示)
- `IsHardOverloaded`: CurrentWeight >= overloadWeight (硬限制，拒绝添加)
- `OverloadRatio`: 0~1 超载程度，用于 UI 进度条颜色渐变

**数据快照**
- `BuildViewData()`: 构建只读 InventoryViewData，包含所有容器网格详情、装备名称、护甲/保暖/负重数据
- `PublishView()`: 发布 InventoryViewChangedEvent 给 UI 订阅者

**存档接口**
- `GetSaveData()`: 导出 InventorySaveData（7容器+装备字典+弹药储备）
- `RestoreFromSave(InventorySaveData, ItemCatalog)`: 从存档恢复容器内容、装备字典、instanceId计数器。武器弹药由 SaveLoadManager 调 WeaponShooting 恢复。

### 4.2 InventoryContainer (存储容器)

**类型**: [Serializable] class（非 MonoBehaviour）

#### 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `containerName` | string | 容器中文名（上衣口袋/裤子口袋/腰带/胸挂/背包/头盔/防弹衣） |
| `equipSlot` | EquipSlot | 对应装备槽 |
| `toggleKey` | KeyCode | 快捷键（None 表示无快捷键） |
| `gridWidth/Height` | int | 网格行列数（装备时动态变化） |
| `placedItems` | List\<PlacedItem\> | 已放置物品列表 |

#### 核心方法
- `IsSpaceFree(x, y, w, h)`: 矩形碰撞检测，判断指定区域无物品占用
- `IsSpaceFreeFor(x, y, w, h, excludeX, excludeY, ...)`: 同上但排除指定原位置物品（旋转时用）
- `FindSpace(w, h, out x, out y)`: 查找第一个可放下 w×h 物品的空位
- `AddItem(item, count, overloadWeight)`: 先尝试堆叠已有同类型，再找空位放新物品。受超载限制。
- `RemoveItem(item, count)`: 移除指定物品（优先减堆叠中的部分数量）
- `RemoveItemAt(gridX, gridY, count)`: 按精确坐标移除（拖拽时防同类型误扣）
- `GetItemAt(x, y)`: 获取指定坐标的物品
- `TotalCells` / `UsedCells` / `CurrentWeight`: 容量/占用/重量属性

### 4.3 PlacedItem (已放置物品结构体)

**类型**: [Serializable] struct

#### 字段
| 字段 | 类型 | 说明 |
|------|------|------|
| `instanceId` | int | 唯一实例 ID（存档系统用，Inventory 自增分配） |
| `itemData` | ItemData | 物品配置引用 |
| `count` | int | 堆叠数量 |
| `gridX/Y` | int | 左上角格子坐标 |
| `rotated` | bool | 是否已旋转 90°（CW） |
| `isGhost` | bool | 是否为腰带武器槽幽灵占位格 |
| `ghostSourceSlot` | EquipSlot | 幽灵格来源（KnifeBelt/SidearmBelt） |

#### 关键属性
- `TotalWeight`: itemData.weight × count（幽灵格为 0）
- `GridWidth/Height`: 考虑旋转后的实际占用（旋转时宽高互换；幽灵格固定 1×1）
- `OriginalWidth/Height`: 原始物品尺寸（不随旋转变化）

#### 静态工厂
- `Ghost(gridX, gridY, sourceSlot)`: 创建幽灵占位格，itemData=null, count=0

### 4.4 ItemData (物品配置)

**类型**: ScriptableObject (CreateAssetMenu: "Game/Item")

关键背包相关字段:
| 字段 | 说明 |
|------|------|
| `itemName` | 物品名称（唯一标识） |
| `icon` | UI 图标 |
| `gridWidth/Height` | 在背包中占用的格子尺寸 |
| `maxStack` | 堆叠上限 |
| `weight` | 单品重量（kg） |
| `equipSlot` | 可装备到哪个槽位（None=不可装备） |
| `storageWidth/Height` | 装备后展开容器的网格列数/行数 |
| `armorValue` | 护甲值（减伤） |
| `warmthValue` | 保暖值 |
| `category` | 物品类别（Weapon/Tool/Consumable/Resource 等） |

武器相关字段（category==Weapon 时生效）:
| 字段 | 说明 |
|------|------|
| `isFirearm` | true=枪械(远程)，false=近战 |
| `weaponDamage` | 单发伤害 |
| `fireRate` | 射击间隔(秒/发) |
| `magazineSize` | 弹匣容量 |
| `ammoItemName` | 消耗的弹药物品名 |
| `range` / `weaponRange` | 最大射程(米) |
| `baseSpread` / `maxSpread` / `spreadPerShot` / `spreadRecovery` | 散射系统参数 |
| `gunshotSoundType` / `muzzleFlashType` | 音效/枪口火焰类型 |
| `reloadTime` | 换弹耗时(秒) |

- ESC 关闭

### 5.2 DragDropManager — 拖拽系统

**类型**: MonoBehaviour 单例 (DontDestroyOnLoad)

**核心流程**:
1. **ItemDragHandler** (挂载在每个可拖拽格子上) 接收 IPointerDownHandler/IPointerUpHandler 事件
2. 转交 DragDropManager，判断是否为拖拽（移动距离 >= 8px 阈值）
3. 拖拽时创建半透明拖拽图标跟随鼠标
4. 松手时通过坐标检测找到目标 CellRegion（遍历已注册的 RectTransform 列表，从后往前）
5. 计算目标格子位置，执行 `TryMoveToContainer()`

**CellRegion 结构**: 记录每个格子的 RectTransform + 所属容器 + 网格坐标 + 是否武器槽标记，支持坐标→容器的反向查找。

**拖拽放置规则**:
- 目标为空闲格子 → 放入
- 目标为装备槽且物品可装备该槽 → 触发 EquipItem
- 目标为同容器不同位置 → 移动（原地旋转校验）
- 目标无效 → 回弹到原位
- 所有情况下都显示 Toast 提示（"放不下！" / "请在左边装备栏穿戴"）

**旋转系统** (T键):
- **拖拽中旋转**: toggle `_dragRotated` 标记，松手落点时以旋转状态校验放置。拖拽图标尺寸随旋转变化。
- **原地旋转**: 拖拽开始前按 T，在源容器内旋转。公式: `(rx, ry) → (ry, rx)`，保持点击点不动。
- **选中旋转**: 物品已选中时按 T，在容器内旋转。

**选中状态**:
- 点击物品 → 白色边框高亮（SelTop/SelBot/SelLeft/SelRight 四条子 RectTransform）
- 再次点击选中的物品 → 取消选中
- 点击空白处（LateUpdate 中拦截 MouseButtonDown） → 取消选中
- UI 重建后通过 `RefreshSelectionBorder()` 恢复选中框（遍历 CellRegion 查找同名 overlay）

**防崩溃保护**: Update 中检测鼠标已松开但状态未清理的异常情况，自动调用 EndDrag/ResetState。

### 5.3 数据快照架构

**目的**: 解耦 UI 层与 Inventory 系统，避免 UI 直接访问 Inventory 的内部状态。

```
InventoryViewData (struct)            ← BuildViewData() 构建
├── containers: List<ContainerViewData>
│   ├── containerName, equipSlot, gridWidth, gridHeight
│   └── items: List<ItemOnGrid>
│       ├── itemName, count, gridX, gridY, gridWidth, gridHeight
│       └── icon, itemData
├── equippedNames: Dictionary<EquipSlot, string>
├── totalArmor, totalWarmth
├── currentWeight, maxWeight
├── isHardOverloaded, overloadRatio
```

UI 订阅 `InventoryViewChangedEvent`，在 `OnViewChanged()` 中只更新内容区（不重建标签栏），实现增量刷新。`_currentTab` 保留当前选中的标签。

## 6. 存档系统

### 6.1 数据结构

```
InventorySaveData
├── containers: List<ContainerSaveData>       // 7 个容器
│   ├── containerName, equipSlotName
│   ├── gridWidth, gridHeight
│   └── slots: List<SlotSaveData>
│       ├── instanceId (PlacedItem 唯一 ID)
│       ├── itemName (null = 空格)
│       ├── count, gridX, gridY
│       ├── rotated, isGhost
│       └── ghostSourceSlot
├── equippedItems: Dictionary<string, EquippedItemSaveData>
│   └── equipSlotName → itemInstanceId (用 instanceId 定位装备物品)
└── ammoReserves: Dictionary<string, int>
    └── EquipSlot名 → 弹夹子弹数
```

所有存档类均实现 `ICloneable`，支持深拷贝。

### 6.2 关键设计决策

- **instanceId 定位**: 物品在容器中的实例 ID，由 Inventory 自增分配（`_nextItemInstanceId`），存档时写入，恢复时用 ID 查找装备映射
- **ItemCatalog 查找**: 存档只存 `itemName`（字符串），恢复时通过 `ItemCatalog`（ScriptableObject 索引）查找对应的 `ItemData` 引用
- **武器弹药独立存储**: `ammoReserves` 字典存储每个武器槽的弹夹子弹数，由 `SaveLoadManager` 调 `WeaponShooting.GetCurrentMag/SetCurrentMag()` 采集和恢复
- **委托外部补充**: `Inventory.GetSaveData()` 只导出基础数据，武器弹药、WorldItem 等由 `SaveLoadManager` 在外部补充
- **浅恢复策略**: `RestoreFromSave()` 直接设置 `equipped` 字典而不调用 `EquipItem()`，避免触发副作用（事件/模型生成），由 `SaveLoadManager` 后续统一处理武器模型恢复

### 6.3 恢复流程

1. `ItemCatalog.Build()` — 建立 itemName → ItemData 映射表
2. 清空所有容器 → 恢复容器内容和网格尺寸 → 收集最大 instanceId 更新计数器
3. 清空非武器 equipped → 按 instanceId 查找装备物品引用并设置字典
4. 武器弹药由 SaveLoadManager 后续调 SetWeaponAmmo() 恢复
5. 武器装备模型由 SaveLoadManager 后续处理

## 7. 世界物品交互

### WorldItem

挂载在动态生成的 GameObject 上，提供地面物品表现:
- **Billboard Quad**: 使用 `itemData.icon.texture` 作为贴图，`LateUpdate` 中始终面向摄像机
- **Box Collider** (isTrigger=true): 交互触发器
- **GroundItemInteract**: 附加的交互组件，处理玩家靠近拾取

### InventoryTest (开发测试)

快捷键:
- **I**: 添加测试物品到背包
- **O**: 丢弃背包中第一个物品（生成 WorldItem）
- **F**: 拾取最近 2.5m 内的地面物品（Physics.OverlapSphere 检测）
