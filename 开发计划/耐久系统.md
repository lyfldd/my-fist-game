## 一、系统概览

耐久系统是一个**横切系统**，嵌入武器、护甲、工具三个子系统，在运行时追踪每个物品实例的磨损，影响其性能，提供修理机制。

```

                    │           ItemData                │
                    │  hasDurability / maxDurability    │
                    │  durabilityMaterial (预留接口)     │
                    └───────────────┬───────────────────┘
                                    │
            ┌───────────────────────┼───────────────────────┐
            ▼                       ▼                       ▼
    ┌──────────────┐       ┌──────────────┐       ┌──────────────┐
```

## 二、核心数据结构

### 2.1 PlacedItem 扩展（唯一运行时数据源）

> **设计原则**：不建独立耐久字典。PlacedItem 是 itemDurability + repairCount 的唯一存储位置。DurabilitySystem 只读写 PlacedItem，不维护自己的状态。

```csharp
// PlacedItem.cs — 新增字段
[Serializable]
public struct PlacedItem
{
    // ... 现有字段 ...
    public float itemDurability;    // 当前耐久（命名区分于 PowerSource.maxDurability）
    public int repairCount;         // 已修理次数（影响下次修理扣除比例）
}
```

- `itemDurability == 0` 且 `itemData.hasDurability == true` → 视为满耐久
- 存档通过 `SlotSaveData` DTO 序列化（不是直接序列化 PlacedItem），见 **前置 A2**
- DurabilitySystem 通过 Inventory 提供的接口读写 PlacedItem，自己无字典

### 2.2 ItemData 扩展（模板层）

### 2.3 ItemMaterial 枚举（见前置B · 材质标签）

### 2.4 GameConstants 新增常量

```csharp
// 耐久基础值
public const float DURABILITY_DEFAULT_MAX = 100f;
public const float DURABILITY_WEAPON_MAX = 200f;   // 武器耐久高于一般物品
public const float DURABILITY_TOOL_MAX = 150f;

// 耐久消耗系数
public const float DURABILITY_ARMOR_PER_DAMAGE = 0.05f;    // 每点伤害扣 0.05 耐久
public const float DURABILITY_WEAPON_PER_SHOT = 1f;        // 开一枪扣 1 耐久

// 耐久性能阈值

public const float DURABILITY_BROKEN_THRESHOLD = 0f;       // ≤0=损坏

// 武器耐久性能衰减
public const float DURABILITY_WEAPON_SPREAD_MAX = 5f;      // 0%耐久额外散射（度）

// 修理
public const float DURABILITY_REPAIR_BASE_DEDUCTION = 0.05f; // 首次修扣5%最大耐久
public const float DURABILITY_REPAIR_CAP_DEDUCTION = 0.50f;  // 最多扣50%最大耐久（指数递增上限）
```

## 三、各子系统接入设计

### 3.1 武器系统 — 射击消耗 + 精准度衰减

**入口**：`WeaponShooting.cs` — `TryFire()` 方法

每次射击：
1. 扣耐久：`DurabilitySystem.ConsumeWeapon(instanceId, 1f)`
2. 查耐久比：`float ratio = DurabilitySystem.GetRatio(instanceId)`
3. 若 `ratio < 0.3`：散射扩大

```csharp
// WeaponShooting.cs — TryFire 中新增
void TryFire(ItemData weapon)
{
    // ... 现有弹药/换弹检查 ...

    // 射击扣耐久
    int weaponId = GetActiveWeaponInstanceId();
    DurabilitySystem.Instance?.ConsumeDurability(weaponId, 
        GameConstants.DURABILITY_WEAPON_PER_SHOT * GetMaterialFactor(weapon));

    // 低耐久精准度惩罚
    float durRatio = DurabilitySystem.Instance?.GetRatio(weaponId) ?? 1f;
    float durabilityPenalty = durRatio < GameConstants.DURABILITY_WORN_THRESHOLD
        ? (1f - durRatio) / GameConstants.DURABILITY_WORN_THRESHOLD 
            * GameConstants.DURABILITY_WEAPON_SPREAD_MAX
        : 0f;

    float effectiveSpread = _currentSpread * (1f - gunLevel * GUN_SKILL_SPREAD_REDUCTION) 
        + durabilityPenalty;  // ← 新增：耐久惩罚直接加到散射角

    // ... 现有射线/伤害逻辑 ...
}
```

**效果**：0% 耐久时额外 +5° 散射（锥形窗口显著变大），30% 以上耐久无影响。卡壳机制（15%概率空仓）留到 Phase 2。

### 3.2 护甲系统 — 受击消耗 + 减伤衰减

**入口**：`DamageablePlayer.cs` / `SurvivalSystem.cs` — 伤害处理

每次受击：
1. 查找当前装备的所有护甲 → 扣耐久
2. 护甲减伤公式中用有效护甲值替代原始值

```csharp
// SurvivalSystem.cs — 改造 TakeDamage 和 GetEffectiveArmor

public void TakeDamage(float rawDamage, string reason)
{
    // 护甲耐久消耗（每个装备的护甲按比例分担）
    float totalArmor = _cachedArmor;
    if (totalArmor > 0)
    {
        float durabilityLoss = rawDamage * GameConstants.DURABILITY_ARMOR_PER_DAMAGE;
        DurabilitySystem.Instance?.DistributeArmorWear(equippedArmorIds, durabilityLoss);
    }

    // 减伤（用有效护甲值）
    float effectiveArmor = GetEffectiveArmor();
    float multiplier = 100f / (100f + effectiveArmor);
    float actualDamage = rawDamage * multiplier;
    ModifyHealth(-actualDamage, reason);
}

float GetEffectiveArmor()
{
    float total = 0f;
    foreach (var kv in equipped)
    {
        if (kv.Value == null || !kv.Value.hasDurability) 
        { 
            total += kv.Value?.armorValue ?? 0; 
            continue; 
        }
        int id = GetInstanceIdForEquippedSlot(kv.Key);
        float ratio = DurabilitySystem.Instance?.GetRatio(id) ?? 1f;
        if (ratio <= 0f) continue;  // 损坏护甲=0减伤
        // 护甲效果 = 基础护甲值 × 耐久比
        total += kv.Value.armorValue * ratio;
    }
    return total;
}
```

**效果**：0% 耐久护甲完全失去减伤，30% 耐久=仅 30% 护甲值生效。

### 3.3 工具系统（预留架构）

## 四、耐久消耗公式

### 4.1 通用公式

```
实际耐久消耗 = 基础消耗 × 材质系数 × 品质系数
```

| 参数 | 来源 | 说明 |
|------|------|------|
| 基础消耗 | 系统定值 | 武器=1/发, 护甲=伤害×0.05, 工具=1/次 |
| 材质系数 | `ItemData.durabilityMaterial` | Wood=1.5, Steel=0.5, Titanium=0.3... |
| 品质系数 | `ItemData.quality` | Scavenged=1.2（旧货易损耗）, Handmade=1.0, Professional=0.8（最耐用） |

品质系数解释：搜刮品虽然是捡来的旧货，耐久消耗最快；专业制造品工艺精良，最耐磨损。

### 4.2 各场景速查

| 场景 | 触发点 | 消耗公式 | 文件 |
|------|--------|----------|------|
| 开枪 | WeaponShooting.TryFire | `1 × materialFactor × qualityFactor` | WeaponShooting.cs |
| 近战 | PlayerCombat.DoAttack | `1 × materialFactor × qualityFactor` | PlayerCombat.cs |
| 被击 | SurvivalSystem.TakeDamage | `damage × 0.05 × materialFactor × qualityFactor`（分担到所有装备护甲） | SurvivalSystem.cs |
| 建造 | BuildModeController | `1 × materialFactor` | BuildModeController.cs |
| 合成 | CraftingSystem | `1 × materialFactor` | CraftingSystem.cs |

## 五、损坏行为

### 5.1 损坏判定

`currentDurability <= 0` → 物品损坏。

### 5.2 按品质分级处理

| 品质 | 损坏行为 | 可否修理 | 残留物 |
|------|----------|:--:|------|
| **Handmade**（自制） | 直接销毁 | ❌ | 生成"损坏的XX残骸"（不可用，可拆解为材料） |
| **Professional**（专业制造） | 标记为"损坏"，不可使用 | ✅ | 保留物品，装备后无效果，可拿去维修台修 |
| **Scavenged**（搜刮品） | 直接销毁 | ❌ | 同 Handmade，搜刮品是捡来的旧货 |

### 5.3 损坏时的连锁反应

- **武器损坏** → `ItemBrokenEvent` → WeaponHolder 卸下武器 → 如果是 Handmade，生成残骸到背包/地面
- **护甲损坏** → `ItemBrokenEvent` → SurvivalSystem 重算护甲 → EquipmentChangedEvent 更新缓存
- **工具损坏** → `ItemBrokenEvent` → UI Toast "XX 已损坏"

## 六、修理系统

### 6.1 维修台设计

维修台是一种**建造物**（BuildableData, category=Workstation），分 3 级：

| 维修台 | 建造条件 | 可修物品品质 | 需要技能 |
|--------|----------|:--:|------|
| **基础修理台** (Tier 1) | 简易工作台 建造 | Handmade | 工匠制作 Lv.2 |
| **精密修理台** (Tier 2) | 高级工作台 建造 | Handmade + Professional | 工匠制作 Lv.5 |
| **工业维修站** (Tier 3) | 机械加工台 建造 | 全部（含 Scavenged） | 工匠制作 Lv.8 |

> **设计逻辑**：自制品（Handmade）坏了不值得修，就直接销毁。只有工业品（Professional）才值得修。T3 工业维修站可以修搜刮品（代表你有能力翻新军用级旧货）。

### 6.2 修理公式（指数递增扣除）

```
第 N 次修理后最大耐久 = 修理前最大耐久 × (1 - deduction)
其中 deduction = min(0.05 × 2^(N-1), 0.50)
```

| 修理次数 | 扣除比例 | 100 耐久→ | 200 耐久→ |
|:--:|:--:|:--:|:--:|
| 第 1 次 | 5% | 95 | 190 |
| 第 2 次 | 10% | 85.5 | 171 |
| 第 3 次 | 20% | 68.4 | 136.8 |
| 第 4 次 | 40% | 41.0 | 82.1 |
| 第 5 次 | 50%（封顶）| 20.5 | 41.0 |
| 第 6 次 | 50% | 10.3 | 20.5 |
| ... | 50% | → 0 | → 0 |

**数学特性**：
- 前 3 次修理比较划算（仍保留 68% 以上）
- 第 4 次开始急剧衰减
- 第 8 次修理后，100 耐久物品仅剩 ~0.4 耐久，实际上不再值得修
- **`maxRepairs = 8`** 作为硬上限，防止无限修理

### 6.3 修理消耗

```
修理材料消耗 = 同品质/同类型物品的制造材料 × repairCostRatio
其中 repairCostRatio = min(0.3 + 0.1 × RepairCount, 0.8)
```

即：第一次修需 40% 制造材料，第二次 50%，以此类推上限 80%。越修越贵。

### 6.4 修理流程（交互）

```
1. 玩家靠近维修台 → E 键交互
2. 打开 RepairUI 面板
   ├─ 左侧：背包中有耐久的物品列表（显示耐久条+修理次数）
   ├─ 右侧：选中物品的修理预览（需材料+修理后耐久+扣除提示）
   └─ 底部：修理按钮（材料不足灰显）
3. 点击修理 → 扣材料 → 扣最大耐久(指数公式) → 恢复当前耐久到新最大值
4. 发布 ItemRepairedEvent → 更新 UI / 护甲缓存
```

```
  ⑤ GameEvents +3 事件

Phase 2 — 核心运行时 (半天)
  ⑥ DurabilitySystem 单例 (ConsumeDurability / GetRatio / DistributeArmorWear)
  ⑦ Inventory 改造 (GetInstanceId / 装备耐久读写)
  ⑧ 物品创建时初始化 currentDurability = maxDurability

Phase 3 — 子系统接入 (1天)
  ⑨ WeaponShooting 接入 (消耗 + 精准度惩罚)
  ⑩ SurvivalSystem 接入 (护甲消耗 + 有效护甲)
  ⑪ PlayerCombat 接入 (近战消耗)
  ⑫ 损坏处理 (ItemBrokenEvent → 销毁/标记)

Phase 4 — 修理系统 (1天)
  ⑬ 3 级维修台 BuildableData
  ⑭ RepairUI 面板
  ⑮ 修理公式 + 材料消耗
  ⑯ 修理交互 (E键 → RepairUI)

Phase 5 — 存档 + UI (半天)
  ⑰ InventorySaveData 集成
  ⑱ 背包格耐久条
  ⑲ HUD 武器耐久指示器
```

**总计**: ~3.5 天。每 Phase 独立可测，不互相阻塞。

## 十一、边界情况

### 11.1 无耐久物品
`hasDurability == false` → DurabilitySystem 直接跳过，`GetRatio` 返回 1.0。

### 11.2 物品在容器间转移
PlacedItem 携带 `currentDurability`，跨容器转移自然保留。无需特殊处理。

### 11.3 丢到地面
WorldItem 从 PlacedItem 读取 `currentDurability`，存到 `WorldItemSaveData`。拾取时恢复。

### 11.4 堆叠物品
`maxStack > 1` 的物品（如弹药）不应有耐久（`hasDurability == false`）。编辑器校验：`OnValidate` 中若 `maxStack > 1 && hasDurability` → Warning。

### 11.5 刚加载场景后
DurabilitySystem 在 `Awake` 初始化空字典。Inventory 在 `RestoreFromSave` 中重建 PlacedItem 后，DurabilitySystem 从 PlacedItem 读取初始值。
