## 三、核心设计

### 3.1 两态容器（无 Empty 状态）

```
Unopened  ──搜索(进度条+掉落生成)──→  Opened  ──区块冷却到期──→  Unopened
                                              ──区块卸载──→  InventoryContainer释放(Record保留)
```

- **Unopened**: 第一次交互 → 搜索进度条 → LootTable随机生成掉落 → 标记Opened
- **Opened**: 再次交互 → 瞬间打开容器窗口(无进度条) → 物品交互(捡/拖) → 关闭后保持Opened
- **废弃了 Empty 状态**: 容器不再自动清空，永远可交互。即使物品拿完也不改变状态。

### 3.2 容器懒加载

- 场景中所有容器初始化时只创建 `ContainerRecord`(纯数据,<100字节)
- `InventoryContainer`(网格+物品数据)仅在第一次 `GetOrCreate()` 时才创建
- 未搜过的容器零内存开销（只有轻量 Record）
- Chunk 卸载时释放 `InventoryContainer`，保留 Record

### 3.3 LootTable 不放回加权随机

```
1. 过滤有效条目(itemData有效+maxCount>0)
2. 决定掉落类型数: Random(minTypes, maxTypes)
3. 构建索引池+总权重
4. 每次随机: 按权重累计→命中→生成count→移除该索引→减少总权重
5. 返回 List<(ItemData item, int count)>
```

每次搜索不会出现同种物品两次（不放回），保证掉落多样性。

### 3.4 区块级容器刷新

```
Chunk 加载 → RefreshHub.OnChunkLoad(chunkId, day)
  → ContainerRefreshHandler:
      遍历该Chunk所有ContainerRecord:
        isOpened=false → 跳过(懒加载)
        refreshEnabled=false → 跳过(玩家自建存储)
        已过冷却 → container=null + isOpened=false(重置)
        未过冷却 → 保持Opened(玩家物品保留)
```

## 四、地面物品

**WorldItem 结构**:
- `itemData` + `count` + `isPlayerDropped` 标记
- Quad 子物体作为广告牌(Unlit/Transparent + itemData.icon)
- LateUpdate 持续朝向摄像机
- BoxCollider 触发器 + GroundItemInteract(IInteractable)

**创建来源**:
1. 玩家丢弃: `Inventory.DropItem(item, count)` — 前方1.2m
2. 装备溢出: 卸下装备背包满时 `SpawnDropAtFeet()`
3. 装备槽双击: 概要视图双击 → `SpawnEquipDrop()`

**拾取**:
```
E键 → GroundItemInteract.OnInteract()
  → Inventory.AddItem(item, count)
    成功: count归零 → Destroy(gameObject)
    失败: 保留剩余物品在地面(背包满)
```
