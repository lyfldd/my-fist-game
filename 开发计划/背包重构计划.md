> **日期**: 2026-06-24 · **状态**: 设计定稿 v2.0 · 🔍 第1轮审查: ✅ (23 项全部定案)

## 核心修正（第 1 轮审查后）

基于 9 个严重问题 + 10 个中等问题 + 4 个轻微问题，在 v1.0 基础上做如下结构性调整：

1. **砍 ContainerLayout** — 每个容器独立 `GridInventory2D`，`Inventory` 持有 `Dictionary<EquipSlot, GridInventory2D>`
2. **GridInventory2D 内置堆叠+负重** — `TryAdd(item, count, maxWeight)` 含堆叠→找空格→重量检查
3. **GridInventory2D 内置幽灵格管理** — `PlaceGhost`/`RemoveGhost`/`CountFreeForGhost`
4. **Inventory 新增公开 API** — `RemoveByInstanceId(int)` 供 WeaponAttachmentHandler 调用
5. **instanceId 归属 Inventory** — GridInventory2D 只管理 slotId，instanceId 由外部传入
6. **GridSlot 替代 PlacedItem** — 含 CloneWithPosition / WithPosition 静态工具方法
7. **ContainerView 替代 InventoryViewData** — 合并为统一快照，事件驱动（非每帧）
8. **SlotData 补全** — 加 isGhost / ghostSourceSlot / repairCount
9. **TotalWeight 缓存** — Place/Remove 时增量更新
10. **ContainerRegistry 存档补耐久** — SlotSaveData 加 itemDurability/repairCount
11. **估时调整为 9-10 天** — Phase 2 拆为 2a+2b

## 一、新数据层 — GridInventory2D

```csharp
[Serializable]
public class GridInventory2D
{
    // === 存储 ===
    int _width, _height;
    int[] _grid;        // [y*w + x] = slotId, 0=空
    int _nextSlotId;
    Dictionary<int, GridSlot> _slots;  // slotId → 数据, O(1) 移除
    float _totalWeight;                // 缓存，Place/Remove 时增量

    // === 属性 ===
    public int Width => _width;
    public int Height => _height;
    public int UsedSlots => _slots.Count;
    public int TotalSlots => _width * _height;
    public float TotalWeight => _totalWeight;

    // === 空间查询 O(w×h) ===
    public bool CanPlace(int x, int y, int w, int h, int excludeSlotId = 0);
    public bool FindSpace(int w, int h, out int ox, out int oy);

    // === 堆叠+放置 ===
    public int TryAdd(ItemData item, int count, float maxWeight);
        // ① 遍历 _slots 找同 itemData 且 count<maxStack → 堆叠
        // ② 仍有剩余数量 → FindSpace → Place 新堆
        // ③ maxWeight 检查：超载截断。maxWeight 由 Inventory 传全局 overloadWeight
    public int Place(ItemData item, int count, int x, int y, bool rotated,
                     int instanceId, float durability, int repairCount);
        // 返回 slotId。跨容器移动时先 Place 目标成功再 Remove 源（防丢失）

    // === 移除 O(1) ===
    public void Remove(int slotId);
    public void RemoveByInstanceId(int instanceId); // 遍历 _slots 找 instanceId
    public GridSlot? GetSlot(int slotId);
    public GridSlot? GetSlotAt(int x, int y);       // O(1)

    // === 移动 ===
		public void Move(int slotId, int toX, int toY);

		// === 尺寸调整 ===
		public List<GridSlot> Resize(int newW, int newH);

    // === 幽灵格 ===
    public int PlaceGhost(int x, int y, EquipSlot source);
    public void RemoveGhosts(EquipSlot source);
    public int CountFreeForGhost(EquipSlot source, int needed);
        // = 空闲格子数 + 同 source 幽灵格数

    // === 遍历 ===
    public IEnumerable<GridSlot> AllSlots();
    public IEnumerable<GridSlot> SlotsFrom(EquipSlot source);
}
```

**GridSlot (替代 PlacedItem):**

```csharp
[Serializable]
public struct GridSlot
{
    public int slotId;         // GridInventory2D 内唯一
    public ItemData itemData;
    public int count;
    public int x, y, w, h;
    public bool rotated;
    public int instanceId;     // 外部分配（Inventory 或 AIBotInventory）
    public float itemDurability;
    public int repairCount;
    public bool isGhost;
    public EquipSlot ghostSourceSlot;

    public static GridSlot CloneWithPosition(GridSlot src, int newX, int newY, bool newRotated);
}
```

    public readonly bool isGhost;          // ← 补

}

### 🔴 严重（v2.0 新发现问题）

**S7. GridInventory2D API 缺少 Resize 方法**

v2.0 的 GridInventory2D API 列表里**没有 Resize**。但装备系统强依赖它：

- `Inventory.EquipItem:230-231` → `container.gridWidth = item.storageWidth; container.gridHeight = item.storageHeight;`
- `Inventory.UnequipSlot:407-408` → 恢复默认尺寸
- v2.0 三章 API 写了 `EquipItem → _grids[s].Resize`，但一章 GridInventory2D 定义里没有声明这个方法

**需要**：GridInventory2D API 补 `Resize(int newWidth, int newHeight)`，并定义缩小时的行为（见 M11）。

---

**S8. CountFreeForGhost 逻辑写反**

v2.0 注释写的是：
```
CountFreeForGhost(EquipSlot source, int needed)
    = 空闲格子数 − 同 sourceSlot 幽灵格
```

但原始 `HasBeltSpaceFor:258-284` 的实际逻辑是：
1. 遍历所有格子
2. 对每个格子，检查是否被占——**排除 `isGhost && ghostSourceSlot == weaponSlot` 的幽灵格**（即同槽幽灵格视为"不占用"）
3. 数空闲格数
4. 返回 `freeCount >= neededCells`

也就是说，同 sourceSlot 的幽灵格占的格子**算作空闲**。公式应该是：

```
可用格数 = 完全空闲格数 + 同 sourceSlot 幽灵格占的格数
```

v2.0 写的是 **减**，方向反了——会导致腰带上有同槽幽灵格时，空间检查永远失败，武器装不上。

**修正**：`CountFreeForGhost = 空闲格数 + 同 sourceSlot 幽灵格数`，或直接用原始逻辑"逐格检查时排除同 sourceSlot 幽灵格"。

### 🟡 中等（v2.0 新发现问题）

**M11. Resize 缩小时物品溢出策略未定义**

GridInventory2D.Resize 改了尺寸，但如果新尺寸比旧的小，超出新边界的物品怎么办？

当前代码有两个不同策略：
- `EquipItem:233-238` → 先改尺寸，再 `TryMoveItemsToOtherContainers`（移到其他容器）
- `UnequipSlot:386-389` → 先 `DropItem`（丢地上），再改尺寸

v2.0 没说 Resize 的行为。是直接裁掉超界物品？返回溢出列表让调用方处理？还是拒绝缩小？

**需要**：Resize 返回 `List<GridSlot>` 溢出物品（超出新边界的 slot），调用方（Inventory）决定移到其他容器还是丢地上。

---

**M12. 跨容器拖拽原子性**

DragDropManager 跨容器拖拽流程：
```
sourceGrid.Remove(slotId)   ← 物品已从源移除
targetGrid.Place(...)        ← 如果失败（空间不够/重量超限）→ 物品丢失
```

v2.0 没提事务性保证。应该 **先 Place 到目标，成功后再从源 Remove**，或提供 `TryMove(slotId, sourceGrid, targetGrid, x, y)` 原子操作。

---

**M13. TryAdd 的 maxWeight 参数来源未说明**

`GridInventory2D.TryAdd(item, count, maxWeight)` — maxWeight 谁传入？

当前代码 `Inventory.AddItem:690` 调用 `c.AddItem(item, remaining, overloadWeight)` 传的是全局 `overloadWeight`。

v2.0 需要明确：Inventory 调用 `_grids[slot].TryAdd(item, count, overloadWeight)` 时传全局 `overloadWeight`。GridInventory2D 自己不存储 maxWeight（它是 Inventory 层的概念）。

---

**M14. 存档版本识别机制缺失**

v2.0 说"旧→新转换：SlotSaveData.gridX/Y → GridSlot.x/y"，但没说 SaveLoadManager 怎么判断是旧存档还是新存档。

是看有没有 `width/height` 字段？还是存档版本号？当前存档系统有没有版本字段？

**需要**：明确判断逻辑（如 `GridSaveData == null && SlotSaveData != null` → 旧格式，或加 `version` 字段）。

---

**M15. DropItem 耐久丢失 bug 未提及**

之前审查 `背包系统.md` 时发现：`Inventory.DropItem:807` 只设 itemData + count，不设 itemDurability / repairCount → 武器砍到 50% 耐久丢地上再拾取变满耐久。

重构会动到 DropItem 相关代码，应顺带修复。v2.0 没提这个已有 bug。

### 🟢 轻微（v2.0 新发现问题）

**L8. UnequipSlot 行为描述不准确**

v2.0 三章 API 写 `UnequipSlot → 溢出→其他容器`。但实际代码 `UnequipSlot:386-389` 是 `DropItem`（丢地上），不是移到其他容器。只有 `EquipItem:235` 换装时才是 `TryMoveItemsToOtherContainers`。

两个路径行为不同，文档应区分：
- **EquipItem 换装**：溢出物品 → TryMoveItemsToOtherContainers
- **UnequipSlot 卸装**：溢出物品 → DropItem（丢地上）

---

**L9. GridSlot 是 struct 但 _slots 是 Dictionary<int, GridSlot>**

struct 在 Dictionary 中是值拷贝。修改某个字段必须：取出 → 修改 → 放回。

```csharp
// 错误写法（不生效）：
_slots[slotId].itemDurability = 50f;

// 正确写法：
var slot = _slots[slotId];
slot.itemDurability = 50f;
_slots[slotId] = slot;
```

当前代码 `InventoryContainer.MoveItem:91` 就有这个模式（`placedItems[i] = p`）。GridInventory2D 内部所有修改 slot 字段的地方都要注意。建议在 GridSlot 文档里标注，或改用 class。
