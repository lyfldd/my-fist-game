## 一、功能概述

统一交互接口 `IInteractable`，所有可交互对象（容器/建筑/车辆/地面物品/工作台/生产设备/电力设备）实现此接口。`PlayerInteraction` 通过 `Physics.OverlapSphere` 检测最近的可交互对象，E 键通过 `InputRouter` 路由触发交互。交互 Toast 反馈已连接，进度条由各实现类自行管理。

**核心文件 (3个)**：
| 文件 | 说明 |
|------|------|
| `Systems/Interaction/IInteractable.cs` | 可交互物体统一接口 |
| `Systems/Interaction/PlayerInteraction.cs` | 玩家交互检测 + E键分发 |
| `Systems/Interaction/TestInteractable.cs` | 调试用可交互物体 |

## 二、IInteractable 接口

```csharp
// Systems/Interaction/IInteractable.cs
namespace _Game.Systems.Interaction
{
    /// <summary>
    /// 可交互物体接口
    /// 任何实现此接口的 MonoBehaviour 都可以被 PlayerInteraction 检测并交互
    /// </summary>
    public interface IInteractable
    {
        /// <summary> 交互提示文本（如"搜索柜子"、"打开门"）</summary>
        string InteractionPrompt { get; }

        /// <summary> 交互耗时（0=瞬间完成，>0=需要按住 E 读条）</summary>
        float InteractionTime { get; }

        /// <summary> 当前是否可交互（用于条件判断）</summary>
        bool IsInteractable { get; }

        /// <summary>
        /// 执行交互
        /// </summary>
        /// <param name="interactor">发起交互的 GameObject（通常是玩家）</param>
        void OnInteract(GameObject interactor);
    }
}
```

## 三、PlayerInteraction 检测与分发

### 3.1 核心代码

```csharp
// Systems/Interaction/PlayerInteraction.cs
namespace _Game.Systems.Interaction
{
    public class PlayerInteraction : MonoBehaviour
    {
        public float detectRadius = GameConstants.INTERACT_DETECT_RADIUS;  // 2.5m

        private IInteractable _currentTarget;

        /// <summary>
        /// 建造模式开关。激活时跳过 E 键交互检测。
        /// BuildModeController 通过 EventBus 的 BuildModeEntered/Exited 事件控制此字段。
        /// </summary>
        [System.NonSerialized] public bool isBuildModeActive;

        private void Awake()
        {
            EventBus.Subscribe<BuildModeEnteredEvent>(OnBuildModeEntered);
            EventBus.Subscribe<BuildModeExitedEvent>(OnBuildModeExited);
        }

        void OnEnable()  { InputRouter.BindKey(KeyCode.E, InputPriority.Action, HandleInteract, this); }
        void OnDisable() { InputRouter.UnbindAll(this); }

        private void OnDestroy()
        {
            EventBus.Unsubscribe<BuildModeEnteredEvent>(OnBuildModeEntered);
            EventBus.Unsubscribe<BuildModeExitedEvent>(OnBuildModeExited);
        }

        private void OnBuildModeEntered(BuildModeEnteredEvent evt) => isBuildModeActive = true;
        private void OnBuildModeExited(BuildModeExitedEvent evt) => isBuildModeActive = false;

        void Update()
        {
            if (isBuildModeActive) return;
            DetectTarget();
        }

        bool HandleInteract()
        {
            if (isBuildModeActive || _currentTarget == null) return false;

            _currentTarget.OnInteract(gameObject);
            EventBus.Publish(new InteractionEvent(_currentTarget.InteractionPrompt, true));
            return true;  // 返回 true 消费按键，阻断低优先级处理器
        }

        void DetectTarget()
        {
            _currentTarget = null;

            Collider[] colliders = Physics.OverlapSphere(transform.position, detectRadius, -1);
            float closestDist = float.MaxValue;

            foreach (var col in colliders)
            {
                var interactables = col.GetComponents<IInteractable>();
                if (interactables == null || interactables.Length == 0)
                    continue;

                // 同一GameObject上可能有多个IInteractable（如 PlacedStructure + WorkstationInteract）
                // 优先选择非PlacedStructure的（工作站合成 > 拆除）
                IInteractable best = interactables[0];
                for (int i = 1; i < interactables.Length; i++)
                {
                    if (best is PlacedStructure && !(interactables[i] is PlacedStructure))
                        best = interactables[i];
                }

                if (!best.IsInteractable)
                    continue;

                float dist = Vector3.Distance(transform.position, col.transform.position);
                if (dist < closestDist)
                {
                    closestDist = dist;
                    _currentTarget = best;
                }
            }
        }

        void OnDrawGizmosSelected()
        {
            Gizmos.color = Color.yellow;
            Gizmos.DrawWireSphere(transform.position, detectRadius);
        }
    }
}
```

### 3.2 检测流程图

```
Update() 每帧
  │
  ├─ isBuildModeActive? ─── YES ──→ return (跳过)
  │
  └─ NO
       │
       └─ DetectTarget()
            │
            ├─ Physics.OverlapSphere(玩家位置, 2.5m, 所有层)
            │
            ├─ 遍历每个 Collider
            │    ├─ GetComponents<IInteractable>()
            │    ├─ 同一 GameObject 多个实现 → 优先非 PlacedStructure
            │    └─ 检查 IsInteractable → 计算距离 → 取最近
            │
            └─ _currentTarget = 最近的合法目标

E 键按下（经由 InputRouter，优先级 InputPriority.Action=50）
  │
  └─ HandleInteract()
       ├─ isBuildModeActive? → return false (放行)
       ├─ _currentTarget == null? → return false (放行)
       │
       └─ _currentTarget.OnInteract(gameObject)
            └─ EventBus.Publish(InteractionEvent)
                 └─ return true (消费)
```

### 3.3 InputRouter 集成

E 键交互通过 `InputRouter` 统一路由，优先级为 `InputPriority.Action = 50`，位于 UI 层(100) 和 Gameplay 层(0) 之间。

```csharp
// 绑定 (OnEnable)
InputRouter.BindKey(KeyCode.E, InputPriority.Action, HandleInteract, this);

// 解绑 (OnDisable)
InputRouter.UnbindAll(this);
```

`HandleInteract` 返回 `true` 表示消费按键，阻断低优先级的 Gameplay 层处理器；返回 `false` 放行。这使得建造模式或无目标时 E 键可以穿透到其他系统。

### 4.2 InteractionToast

```csharp
// UI/InteractionToast.cs
namespace _Game.UI
{
    /// <summary>
    /// 交互反馈 Toast — 监听 InteractionEvent → 显示 2s 后消失
    /// </summary>
    public class InteractionToast : MonoBehaviour
    {
        public Text toastText;

        void Start()
        {
            if (!UIModeConfig.UseUGUI) { enabled = false; return; }
            if (toastText != null)
                toastText.gameObject.SetActive(false);

            EventBus.Subscribe<InteractionEvent>(OnInteraction);
        }

        void OnDestroy()
        {
            EventBus.Unsubscribe<InteractionEvent>(OnInteraction);
        }

        void OnInteraction(InteractionEvent evt)
        {
            if (toastText == null) return;

            toastText.gameObject.SetActive(true);
            toastText.text = $"{evt.ObjectName}";
            CancelInvoke(nameof(Hide));
            Invoke(nameof(Hide), 2f);
        }

        void Hide()
        {
            if (toastText != null)
                toastText.gameObject.SetActive(false);
        }
    }
}
```

## 五、进度条实现

### 5.1 设计说明

交互进度条由 **各实现类自行管理**，而非由 `PlayerInteraction` 统一驱动。原因是不同交互类型的进度表现差异较大：

- **WorldContainer**：世界空间 Canvas + Slider，搜索时显示在容器上方
- **PlacedStructure**：纯协程等待，无 UI（拆除时不显示进度）
- **瞬间交互**（GroundItemInteract、VehicleInteraction、WorkstationInteract 等）：不需要进度条

### 5.2 WorldContainer 的世界空间进度条

```csharp
// WorldContainer.BuildProgressUI() — 在 Awake 中构建
private void BuildProgressUI()
{
    // 1. 创建 Canvas (WorldSpace, 不可交互)
    var canvasGo = new GameObject("SearchProgressCanvas",
        typeof(Canvas), typeof(CanvasScaler), typeof(GraphicRaycaster));
    canvasGo.transform.SetParent(transform, false);

    _progressCanvas = canvasGo.GetComponent<Canvas>();
    _progressCanvas.renderMode = RenderMode.WorldSpace;
    canvasGo.GetComponent<GraphicRaycaster>().enabled = false;

    var canvasRt = canvasGo.GetComponent<RectTransform>();
    canvasRt.sizeDelta = new Vector2(2f, 0.3f);
    canvasRt.localPosition = new Vector3(0, 1.2f, 0);  // 容器上方1.2m

    // 2. 创建 Slider (不可交互, 0~1)
    var sliderGo = new GameObject("Slider", typeof(Slider));
    sliderGo.transform.SetParent(canvasGo.transform, false);
    _progressSlider = sliderGo.GetComponent<Slider>();
    _progressSlider.interactable = false;
    _progressSlider.minValue = 0f;
    _progressSlider.maxValue = 1f;

    // 3. 背景 (深色半透明)
    var bgGo = new GameObject("Background", typeof(Image));
    bgGo.GetComponent<Image>().color = new Color(0.1f, 0.1f, 0.1f, 0.75f);

    // 4. 填充 (橙色)
    var fillGo = new GameObject("Fill", typeof(Image));
    fillGo.GetComponent<Image>().color = new Color(0.95f, 0.7f, 0.1f, 1f);

    // 默认隐藏
    _progressCanvas.gameObject.SetActive(false);
}
```

搜索协程中驱动进度条：

```csharp
// WorldContainer.SearchRoutine()
private IEnumerator SearchRoutine()
{
    _isSearching = true;
    float searchTime = profile != null ? profile.searchTime : 1f;

    // 显示进度条
    if (_progressCanvas != null)
    {
        _progressCanvas.gameObject.SetActive(true);
        if (_progressSlider != null) _progressSlider.value = 0f;
    }

    float elapsed = 0f;
    while (elapsed < searchTime)
    {
        elapsed += Time.deltaTime;
        if (_progressSlider != null)
            _progressSlider.value = elapsed / searchTime;
        yield return null;
    }

    // 隐藏进度条
    if (_progressCanvas != null)
        _progressCanvas.gameObject.SetActive(false);

    // ... 生成掉落 + 打开容器窗口
}
```

## 七、同一 GameObject 多 IInteractable 的处理

当建造物挂载 `PlacedStructure`（拆除）和 `WorkstationInteract`（合成）两个组件时，两者都实现了 `IInteractable`。`PlayerInteraction.DetectTarget()` 中做了特殊处理：

```csharp
// 优先选择非PlacedStructure的（工作站合成 > 拆除）
IInteractable best = interactables[0];
for (int i = 1; i < interactables.Length; i++)
{
    if (best is PlacedStructure && !(interactables[i] is PlacedStructure))
        best = interactables[i];
}
```

这意味着玩家靠近工作站时按 E 会优先打开合成 UI，而非拆除。如果要拆除，可以通过建造模式的拆除工具或额外按键完成。

建造模式激活时：
- `Update()` 跳过 `DetectTarget()`
- `HandleInteract()` 返回 `false`（放行 E 键给建造系统）
