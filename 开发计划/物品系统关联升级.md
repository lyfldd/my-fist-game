### 🔴 严重（遗漏系统 / 方案根本性错误）

**S1. 遗漏 AIBotUI — 7 处硬编码物品名不在升级清单中**

`AIBotUI.cs:730-778` `AddFuelFromPlayerInventory` / `RepairFromPlayerInventory`：
- `itemName == "电池组"`（电池能源）
- `itemName == "浓缩铀"`（核能）
- `itemName == "高级零件"`（高级修理）
- `itemName == "铁锭"`（基础修理）
- `itemName == "聚变核心(小)"` / `"聚变核心(大)"`（聚变核心识别）

升级清单只列了 AIBotCombat（§2.3），完全没提 AIBotUI。这 7 处硬编码不消除，"新建电池/铀/零件物品填字段即用"承诺就失效。

**需要**：ItemData 加 `energyValue` / `repairValue` / `isFusionCore` 字段（或用 usageTags），AIBotUI 改读字段而非名字。

---

**S2. 遗漏 AIBot.cs — 聚变核心识别硬编码**

`AIBot.cs:24-25`：
```csharp
public bool IsSmall => itemName == "聚变核心(小)";
public bool IsLarge => itemName == "聚变核心(大)";
```

升级清单未提。后果：新建一个"中型聚变核心"→ IsSmall/IsLarge 都返回 false → AIBot 装载逻辑全断。

**需要**：ItemData 加 `fusionCoreSize` 枚举（None/Small/Large），AIBot 改读此字段。

---

**S3. §2.1 VehicleController 方案错误 — "取第一个 fuelValue > 0"会有多燃料冲突**

文档方案：`遍历 containers 找 fuelValue > 0 的物品 → RemoveItem → 取第一个`。

极端情况：
- 背包同时有汽油（fuelValue=100）和煤油（fuelValue=80）→ 取到汽油（OK）
- 背包只有煤油 → 取煤油，但车可能只接受汽油
- 背包同时有"酒精"（fuelValue=50）和"汽油"（fuelValue=100）→ 按代码顺序可能取到酒精，效率低

车辆系统需要明确"接受哪些燃料"，不能任意取。

**正确方案**：
- VehicleData 加 `ItemData[] acceptedFuels` 字段（接受哪些燃料物品）
- 或加 `minFuelValue` 阈值 + 按燃料值排序取最高
- ConsumeFuel 遍历背包找匹配 acceptedFuels 的物品，按 fuelValue 转换成油量（不是固定 +20）

---

**S4. §2.2 ElectronicDevice 方案不完整 — 遗漏 ReplaceBattery + 多设备冲突未解决**

文档只说改 `GetDurabilityRatio`，遗漏两点：

1. **ReplaceBattery:158 硬编码 "电池组"**：
```csharp
if (pi.itemData != null && pi.itemData.itemName == "电池组" && pi.count > 0)
```
也要改为 `electronicDeviceType == Battery` 匹配。

2. **多设备冲突未解决**：原方案"`遍历背包找 electronicDeviceType == this.deviceType`"——背包有 2 个手电筒（一个满耐久一个快坏），还是只读第一个。

**正确方案**：ElectronicDevice 组件绑定 instanceId，`GetDurabilityRatio` 从 `Inventory.FindPlacedItem(instanceId).itemDurability` 直接读，不遍历。

---

**S5. §2.3 AIBotCombat 方案不完整 — 只删 const 字符串不够**

文档说删 4 个 const + GetWeaponItemName + GetRightArmFromItemName。但实际还有：

```csharp
public float pistolDamage = 20f;
public float rifleDamage = 30f;
public float shotgunDamage = 25f;
public float emRifleDamage = 40f;
public float pistolRange = 12f;
public float rifleRange = 18f;
// ... 还有 cooldown/range 共 12+ 个字段
```

这些 Inspector 字段和 ItemData.weaponDamage/weaponRange/fireRate **完全重复**。即使建了 AIBotWeaponMapping，伤害/射程/冷却还是从 Inspector 字段读，没从 ItemData 读。

**正确方案**：AIBotWeaponMapping.Entry 不只含 weaponItem/ammoItem，AIBotCombat 改读 `mapping.GetWeapon(weapon).weaponItem.weaponDamage` 等 ItemData 字段，删除 12+ 个 Inspector 字段。

### 🟡 中等（方案不完整 / 关联系统遗漏）

**M1. §2.4 SurvivalSystem 方案 — healAmount 与 itemEffects.RestoreHealth 会重复回血**

文档说加 `if (healAmount > 0) ModifyHealth(...)`。但现有代码已通过 `itemEffects` 中的 `RestoreHealth` 效果回血。

后果：
- 物品有 `healAmount=50` 且 itemEffects 含 `RestoreHealth(30)` → 使用后回 80 血
- 物品有 `healAmount=50` 但无 itemEffects → 按新方案回 50（OK，但原意可能不希望）
- 文档没说 healAmount 和 itemEffects 的优先级

**需要**：明确 healAmount 是独立效果还是 RestoreHealth 的替代，或合并规则。

---

**M2. §2.4 SurvivalSystem — foodType 触发识别与药品冷却逻辑冲突**

文档说"`foodType != null` 也进入处理（即使 itemEffects 为空）"。但现有代码在处理内部用 `foodType == "Medicine"` 判断药品冷却和医疗加成：

```csharp
if (item.foodType == "Medicine" && _medCooldowns.TryGetValue(...)) return;
if (!string.IsNullOrEmpty(item.foodType) && item.foodType != "Medicine") { ...同类递减 }
if (item.foodType == "Medicine") { ...医疗技能加成 }
```

如果物品 `foodType = "Meat"` 但没 itemEffects → 进入处理 → 走"同类递减"分支但无效果应用 → 浪费一次递减。

**需要**：foodType 拆成两个职责：`isFood`（是否触发食物处理）+ `foodCategory`（食物类型），或重写识别逻辑。

---

**M3. §2.6 Inventory 方案 — 遗漏 DropItem 耐久丢失 bug**

升级清单只加 storageWidth/Height 兜底。但 `Inventory.DropItem:816-822` 有已知 bug：
```csharp
foreach (var pi in c.placedItems)
    if (pi.itemData == item) { dur = pi.itemDurability; rp = pi.repairCount; break; }
```
找到第一个匹配就 break，如果背包有 2 个同名物品（一个满耐久一个快坏），可能取错那个的耐久。

物品系统升级时应顺带修复：按 instanceId 查而非按 ItemData 引用。

---

**M4. §2.6 Inventory 方案 — 遗漏 WeaponAttachmentHandler 直接操作 placedItems**

`WeaponAttachmentHandler.cs:54-66` 直接 `c.placedItems.RemoveAt(i)` 删附件。物品系统升级时，应让 Inventory 提供 `RemoveByInstanceId(int)` API，附件系统改用此 API，消除对 placedItems 的直接访问。

这不是物品识别问题，但是"系统接入物品系统"的硬伤——附件物品通过 instanceId 操作，但 Inventory 没有对应公开 API。

---

**M5. §2.7 ItemGraphBuilder 引用完整性检验 — 遗漏多种 SO 引用**

文档列了 4 种检验：RecipeData / 武器弹药 / BuildableData / ProductionDeviceData。实际还应检验：

1. **ProductionDeviceData.fuelItem**：`_data.fuelItem == null` 时 ConsumeFuel 直接返回 false，编辑期应警告
2. **ContainerLootProfile** 中物品引用（搜刮表）
3. **AIBotWeaponMapping.entries**（升级后新增的 SO）
4. **VehicleData.fuelItemName** 字符串指向的物品是否存在（VehicleController 升级前）
5. **ElectronicDevice** 硬编码物品名（升级前）—— 升级后此条作废

---

**M6. 遗漏：WeaponShooting 用 ammoItemName 字符串**

`WeaponShooting.cs:184,286,304` 用 `weapon.ammoItemName`（字符串）通过 `CountItemByName` / `RemoveItemByName` 查扣弹药。

这是半硬编码——武器 ItemData 上的 ammoItemName 字符串必须和弹药物品的 itemName 完全一致，改名就断。

**应升级为**：ItemData 加 `ItemData ammoItemData`（直接引用）替代 `string ammoItemName`，WeaponShooting 改用 `RemoveItem(weapon.ammoItemData, count)`。

---

**M7. 遗漏：PlayerInteraction 修理逻辑直接遍历 placedItems**

`PlayerInteraction.cs:84-91` 直接 `foreach (var pi in c.placedItems)` 收集材料。背包重构后会断，物品系统升级也应顺带改用 `Inventory.GetItemCount(item)` API。

**M8. 遗漏：AIBotInventory 按 itemName 查物品**

`AIBotInventory.cs:132,144` `slot.itemData.itemName == itemName` 按 name 查。新建武器/弹药改名字就识别不到。应改用 ItemData 引用或 usageTags。

## 二、详细方案

### 2.1 ItemData 新增字段

```csharp
// 5 个新字段
public float energyValue;              // 能量值（电池/铀/核心）→ AIBotUI 读
public float repairValue;              // 修理值（高级零件/铁锭）→ AIBotUI 读
public FusionCoreSize fusionCoreSize;  // 聚变核心大小（None/Small/Large）→ AIBot 读
public ItemData[] acceptedFuels;       // 接受的燃料物品（VehicleData 用，替代 fuelItemName）
public ItemData ammoItemData;          // 弹药直接引用（WeaponShooting 用，替代 ammoItemName 字符串）

enum FusionCoreSize { None = 0, Small, Large }
```

### 2.2 VehicleController

```
旧: fuelItemName="SyntheticGasoline" → RemoveItemByName

新: VehicleData.acceptedFuels = [汽油, 柴油, 煤油]
    遍历背包找 acceptedFuels 中的物品 → 取 fuelValue 最高的 → RemoveItem
    油量转换: fuelAmount = item.fuelValue × fuelConversionRate
    无匹配燃料 → 不可驾驶
```

### 2.3 ElectronicDevice

```
旧: GetDurabilityRatio() → foreach containers 找 itemName=="手电筒"
    ReplaceBattery() → 找 itemName=="电池组"

新: 加 _boundInstanceId 字段（装备时记录）
    GetDurabilityRatio() → Inventory.FindPlacedItem(_boundInstanceId).itemDurability（O(1)，不遍历）
    ReplaceBattery() → 找 electronicDeviceType==Battery 且 count>0
    多设备: 通过 instanceId 精确区分
```

### 2.4 AIBotCombat + AIBotUI + AIBot.cs

```
新建: AIBotWeaponMapping.asset
  Entry { RightArmWeapon weapon; LeftArmWeapon leftWeapon;
          ItemData weaponItem; ItemData ammoItem;
          ItemData leftWeaponItem; }

改 AIBotCombat:
  删: 4 个 const AMMO_XXX + 12 个 Inspector 字段(伤害/射程/冷却)
  改: FireRightArm → 读 mapping.weaponItem.weaponDamage/weaponRange/fireRate
  改: ConsumeAmmo → 读 mapping.ammoItem (ItemData 引用)

改 AIBotUI:
  删: 7 处 itemName=="电池组"/"浓缩铀"...
  改: energize → 遍历背包找 energyValue>0 的物品
  改: repair → 找 repairValue>0 的物品
  改: fusion core → 读 fusionCoreSize 枚举

改 AIBot.cs:
  删: IsSmall/IsLarge 硬编码
  改: 读 ItemData.fusionCoreSize
```

### 2.5 SurvivalSystem

```
foodType 拆为两个用途:
  isConsumable: 是否可食用（触发 OnItemUsed 处理）
  foodCategory: 食物类型（Meat/Vegetable/Drink/Medicine，同类递减 + 医疗加成用）

healAmount:
  独立于 itemEffects 处理 —— 先处理 itemEffects，再处理 healAmount
  不重复回血：如果 itemEffects 中已有 RestoreHealth，healAmount 作为额外加成

新识别条件:
  isConsumable || itemEffects.Count>0 || healAmount>0
```

### 2.6 Inventory 三项修复

```
1. storageWidth/Height 兜底: if (<=0) → 从 Inventory.defaultSizes 字典查默认值

2. DropItem 耐久 bug: 改按 instanceId 精确查找，不用第一个 ItemData 匹配

3. 新增 RemoveByInstanceId(int) API（供 WeaponAttachmentHandler / ElectronicDevice 等调用）
```

### 2.7 WeaponShooting 弹药引用

```
ItemData 加 ammoItemData 字段（直接引用弹药 ItemData）

改 WeaponShooting:
  旧: CountItemByName(weapon.ammoItemName) / RemoveItemByName(weapon.ammoItemName)
  新: RemoveItem(weapon.ammoItemData, count)

保留 ammoItemName 作为向后兼容（ammoItemData 为空时 fallback 到字符串查找）
```

### 2.8 ItemGraphBuilder 引用完整性

```
Build() 末尾自动扫描:
  1. RecipeData.resultItem + materials[i].itemData
  2. ItemData(isFirearm).ammoItemName + .ammoItemData（新字段）
  3. BuildableData.materials[i].itemData
  4. ProductionDeviceData.recipes(input/output) + .fuelItem
  5. ContainerLootProfile 中物品引用
  6. AIBotWeaponMapping.entries（升级后新增的 SO）
  7. VehicleData.acceptedFuels（升级后新增的数组）

输出: "[ItemGraph] ✅ 159 ✅ ❌ 3 断裂: ..."
```

### 2.9 PlayerInteraction + AIBotInventory

```
PlayerInteraction: 直接用 Inventory.GetItemCount(item) + RemoveItem
AIBotInventory: 改用 ItemData 引用匹配，不用 itemName 字符串
```
