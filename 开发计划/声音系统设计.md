## 一、设计目标

所有能发出声音的动作/人物/僵尸/动物都通过统一入口 `DecibelSystem.Emit()` 发声。僵尸监听非同类+非环境的声音，通过 Chunk 空间分区优化性能。持续声音（引擎/发电机）作为锚点或移动源吸附僵尸。

## 二、两级标签

```csharp
[Flags] enum SoundSource   // 谁发出的
{
    None = 0, Player = 1<<0, Zombie = 1<<1, Animal = 1<<2,
    Vehicle = 1<<3, Environment = 1<<4
}

[Flags] enum SoundTag      // 什么类型
{
    None = 0, Footstep = 1<<0, Combat = 1<<1, Gunshot = 1<<2,
    Building = 1<<3, Voice = 1<<4, Mechanical = 1<<5, Impact = 1<<6
}
```

```csharp
public readonly struct NoiseEvent
{
    public Vector3 Position { get; }        // 声音原点（不是物体位置）
    public float Radius { get; }
    public SoundSource Source { get; }
    public SoundTag Tag { get; }
    public GameObject SourceObject { get; } // 发声物体引用（可选）
    public bool IsContinuous { get; }       // 是否持续声音
}
```

## 三、DecibelSystem 完整处理流程

```
Emit(position, radius, source, tag, sourceObject, isContinuous)
  │
  ├─ 节流 1: 同源冷却
  │   同一 sourceObject + 同一 Tag → N秒内重复的跳过
  │
  ├─ 节流 2: 同位置合并
  │   同一位置附近已发过更大声音且在冷却期内 → 跳过
  │
  ├─ Modifier 管道
  │   注册的 ISoundModifier 依次处理（天气衰减/封闭空间衰减等）
  │
  ├─ 空间分区
  │   计算受影响 Chunk → 收集 RuntimeChunk.zombieInstances → 精准距离判定
  │
  └─ 通知僵尸: zombie.OnSoundHeard(noise)
       └─ 节流 3: 僵尸端冷却 + 大声音切换
```

### 持续声音接口

```csharp
// 固定持续: 发电机、警报器
StartContinuous(key, position, radius, source, tag);  // 每 N 秒自动 Emit
StopContinuous(key);

// 移动持续: 车辆引擎、奔跑玩家
UpdateContinuousPosition(key, newPosition);  // 更新锚点位置，下次 Emit 用新位置
```

## 四、节流机制

| 节流 | 位置 | 逻辑 |
|------|------|------|
| 同源冷却 | DecibelSystem | 同对象+同Tag，N秒内只生效一次 |
| 同位置合并 | DecibelSystem | 近处已有更大声音在冷却期 → 忽略小声音 |
| 僵尸冷却 | ZombieStateMachine | 听到声音后 2s 内忽略更小声音；更大的声音才切目标 |

## 五、僵尸声音处理

```
OnSoundHeard(noise):
  ├─ 忽略 Source = Zombie | Environment
  ├─ 对比当前"正在追踪的声音"
  │     新声音.Radius > 旧声音.Radius → 换目标
  │     否则 → 忽略（被更大的声音持续吸引中）
  ├─ 设置 lastHeardPosition = noise.Position（原点）
  └─ 切 Chase，目的地 = 声音原点
```

### 调查行为（僵尸 AI Phase2 实现）

```
Chase 到声音原点:
  有可攻击目标 → Attack
  没有目标 → 附近搜索一圈 → 放弃 → 回到 Idle/Wander
// 持续声音的物体不会被"到达"，僵尸会在原点附近徘徊
```

## 六、持续声音

| 类型 | 特点 | 例子 | 行为 |
|------|------|------|------|
| 固定持续 | 位置不变 | 发电机、警报器 | 静止锚点，持续吸附区域内僵尸 |
| 移动持续 | 位置随物体变化 | 车辆引擎、奔跑者 | 每N秒更新原点，僵尸跟着移动 |

只有更大的声音才能把僵尸从持续声源上拉走。

---

## 七、声音穿透（二期实现）

```
封闭空间衰减:
  发声方在封闭空间 → Radius × (1 - 封闭度 × maxAttenuation)
  接收方（僵尸）在封闭空间 → 所有传入声音 × (1 - 封闭度 × maxAttenuation)

封闭度: 0.0 (完全开放) ~ 1.0 (完全密封)
衰减随封闭度线性增加，最大衰减时内外几乎听不见
```

实现为 `ISoundModifier` 的一个实例，不改 DecibelSystem。

## 十三、三层架构

```
┌──────────────────────────────────────────────────────────┐
│                                                          │
│  玩家开枪      玩家跑步      车辆引擎      僵尸呻吟       │
│     │            │            │            │             │
│     ▼            ▼            ▼            ▼             │
│  ┌─────────────────────────────────────────────┐        │
│  │           SoundEmitter（声源层）              │        │
│  │    参数集中 · 单一切入 · 未来可接音频播放     │        │
│  └────────────────────┬────────────────────────┘        │
│                       │  Emit() / StartContinuous()      │
│                       ▼                                  │
│  ┌─────────────────────────────────────────────┐        │
│  │          DecibelSystem（声音处理层）          │        │
│  │  节流1+2 → Modifier管道 → 空间分区 → 分发     │        │
│  └────────┬──────────────────┬─────────────────┘        │
│           │                  │                           │
│           ▼                  ▼                           │
│  ┌──────────────┐   ┌──────────────────┐                │
│  │  僵尸AI消费   │   │  其他监听者消费    │                │
│  │  过滤→追击   │   │  ISoundListener   │                │
│  └──────────────┘   └──────────────────┘                │
│                                                          │
│  未来扩展（不动现有代码）:                                  │
│    AudioManager   ← 声源层加播放给玩家听                   │
│    封闭空间Modifier ← 室内声音衰减                         │
│    天气Modifier    ← 雨天声音衰减                          │
└──────────────────────────────────────────────────────────┘
```

| 层 | 职责 | 知不知道上层/下层 |
|------|------|:--:|
| **SoundEmitter** | 定义什么动作配什么声音参数，零监听 | 调用 DecibelSystem |
| **DecibelSystem** | 节流 + 处理 + 分发 | 不知道消费者是谁 |
| **僵尸/动物/NPC** | 听到声音后自己的行为逻辑 | 不知道声音从哪来 |

### SoundEmitter 设计

```csharp
public static class SoundEmitter
{
    // 战斗
    public static void EmitGunshot(Vector3 pos, bool isRifle = false);
    public static void EmitMeleeHit(Vector3 pos);
    // 移动（持续）
    public static void SetFootstep(string key, Vector3 pos, bool running);
    public static void StopFootstep(string key);
    // 建造
    public static void EmitBuildPlace(Vector3 pos);
    public static void EmitBuildDeconstruct(Vector3 pos);
    // 车辆
    public static void StartVehicleEngine(string key, Vector3 pos);
    public static void UpdateVehicleEngine(string key, Vector3 pos);
    public static void StopVehicleEngine(string key);
}
```

参数全在 SoundEmitter 内部，各系统调一个方法即可，不知道 DecibelSystem 存在。

#### 经验教训

- **字典遍历安全**：Unity Mono 运行时的 `Dictionary` 在 `foreach` 时即使只通过索引器更新已有 key 也会抛 `InvalidOperationException`。必须先收集再改字典。
- **噪声等级校准**：直接 `(1 - dist/radius)` 会导致玩家自己走路=100%噪声（因为距离为0）。必须加入声音类型的绝对响度：`(radius/MAX) × (1 - dist/radius)` 才能正确区分走路(4%)和枪声(63%~100%)。
- **OnGUI 散落问题**：多个系统各自画 OnGUI 导致位置重叠。提前设计统一的调试面板（Y 切换）比事后清理更高效。
- **声源层价值**：SoundEmitter 静态类让 WeaponShooting/PlayerCombat 等系统零感知 DecibelSystem 存在，未来加 AudioManager 播放声音给玩家也只需改 SoundEmitter 一处。
