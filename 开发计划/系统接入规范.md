**S2. §1.1 "电子设备"识别条件错误——ElectronicDevice 不读 ItemData.electronicDeviceType**

文档说："`electronicDeviceType != Flashlight`（非默认）→ ElectronicDevice 读取 deviceType 应用运行时行为"。

实际代码 `ElectronicDevice.cs`：
- 它是 **MonoBehaviour 组件**，`deviceType` 是组件自己的 public 字段，**不从 ItemData 读取**
- `GetDurabilityRatio():175-183` 用 `itemName == "手电筒"` / `"打火机"` / `"指南针"` / `"手表"` **硬编码字符串**查找背包
- `ReplaceBattery():158` 硬编码 `itemName == "电池组"`

后果：
- 新建一个 `electronicDeviceType = Flashlight` 的物品，只要 `itemName != "手电筒"`，**永远不会被 ElectronicDevice 识别**
- 改手电筒的 itemName → 功能立刻失效
- ItemData.electronicDeviceType 字段实际**完全没用**（ElectronicDevice 根本不查它）

文档 §1.1 的触发条件 "`electronicDeviceType != Flashlight`" 是错的。

**S3. §1.1 "食物/药品"识别条件错误——healAmount 字段完全没用，foodType 单独不触发**

文档说触发条件："`itemEffects.Count > 0` 或 `foodType != null` 或 `healAmount > 0`"。

实际 `SurvivalSystem.OnItemUsed:263`：
```csharp
if (evt.ItemData == null || evt.ItemData.itemEffects == null || evt.ItemData.itemEffects.Count == 0)
    return;
```
- **只检查 itemEffects.Count**，foodType 和 healAmount 都不会触发识别
- `healAmount` 字段在 SurvivalSystem 全文 **0 次使用**（`Grep healAmount` 无匹配）
- `foodType` 只在已进入处理后用作系数判断，不是触发条件

后果（极端情况）：
- 物品 A：`foodType = "Meat"` 但 `itemEffects` 为空 → **吃了没效果，不会被识别为食物**
- 物品 B：`healAmount = 50` 但 `itemEffects` 为空 → **完全无效，healAmount 是死字段**
- 物品 C：`itemEffects` 有条目但 `foodType` 为空 → 能用，但药品冷却/医疗技能加成逻辑全部跳过（因为都 `if (foodType == "Medicine")`）

**S6. §1.1 "武器附件"触发条件 "或" 逻辑错误——会把武器误识别为附件**

文档说："`attachmentSlot != None` 或 `hostSlots.Length > 0` → WeaponAttachmentHandler 挂载/拆卸"。

实际字段语义（`ItemData.cs:47-48`）：
- `attachmentSlot`：**此物品是**哪种附件（消音器/瞄具等）→ 附件物品用
- `hostSlots`：**此物品接受**哪些附件槽位 → 宿主武器/装备用

两个是不同概念。文档用"或"会导致：一把 AK47 设了 `hostSlots = [Muzzle, Scope, Underbarrel]`（它是宿主），按文档条件 → 被识别为附件 → WeaponAttachmentHandler 尝试把它挂载到别的东西上 → 错误。

实际 `MountAttachment:38` 只检查 `attachmentItem.attachmentSlot == AttachmentSlot.None` 就拒绝，**不看 hostSlots**——这是对的。文档的条件写错了。
