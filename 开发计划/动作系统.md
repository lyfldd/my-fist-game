## 〇、定位说明

**动作系统 ≠ 战斗系统**。战斗系统负责"打多少伤害/打没打中"，动作系统负责"看起来在打"。

| | 战斗系统（已有） | 动作系统（新增） |
|---|---|---|
| 职责 | 伤害公式、命中判定、冷却、散射、弹药 | 动画播放、动画事件、动作状态机、武器可见性 |
| 输入 | 伤害数值、范围参数 | 动画 clip、Animator 参数 |
| 输出 | TakeDamage / WeaponFiredEvent | AnimationEvent → 桥接到战斗系统 |
| 关系 | 逻辑层 | **表现层** |

**核心原则**：动作系统是战斗系统的"皮"，不改战斗逻辑，只改战斗的**时机**和**表现**。旧代码中 `PlayerCombat.TryMeleeAttack()` 的伤害计算和 `WeaponShooting` 的弹道结算全部保留，只是把它们从"鼠标点击瞬间执行"改为"动画播到命中帧时执行"。

## 一、动画来源：MeshyAI

### 1.1 导入管线

```
MeshyAI 生成动画 FBX
  → 下载到本地
  → 拖入 Unity Assets/_Game/Animations/[角色名]/
  → Rig 标签页：选 Humanoid → Apply
  → Animation 标签页：确认 clip 名称/循环/帧率
  → 拖入 Animator Controller 使用
```

### 1.2 Humanoid 重定向

Unity Mecanim 的 Humanoid Rig 自动将骨骼映射到标准人体结构，使得：

- MeshyAI 生成的任何 Humanoid 角色的动画 → 可以复用给另一个 Humanoid 角色
- 玩家动画、僵尸动画、AI 机器人动画可以共享同一套动作库
- 换模型不需要换动画

**关键操作**：导入后先设 Rig 为 Humanoid，Unity 会自动做骨骼映射。映射歪了去 `Configure Avatar` 手动纠正。

### 1.3 需要的动画清单

| 分类 | 动画名 | 优先级 | 说明 |
|------|--------|:----:|------|
| 移动 | Idle | P0 | 站立呼吸 |

## 二、Animator 状态机

### 2.1 总体结构

```
PlayerAnimController.controller
│
├── Base Layer（全身，权重 1.0）
│   ├── [Locomotion Blend Tree]  参数: Speed(float), IsRunning(bool)
│   │   ├── Idle       (Speed=0)
│   │   ├── Walk       (Speed=0~1)
│   │   └── Run        (Speed=1~2)
│   │
│   ├── [Death]                  参数: Dead(trigger)
│   └── Any State → Death
│
├── Upper Body Layer（腰部以上，权重 1.0，Avatar Mask 遮罩下半身）
│   ├── [Empty]                  默认空状态（不覆盖 Base Layer 上半身）
│   ├── [Melee Attack]           参数: MeleeAttack(trigger)
│   ├── [Ranged Idle]            参数: IsAiming(bool)
│   ├── [Ranged Fire]            参数: Fire(trigger)
│   ├── [Reload]                 参数: Reload(trigger)
│   └── [Hit Reaction]           参数: Hit(trigger)
│
└── IK Layer（可选，权重 0）
    └── 武器握持/瞄准 IK
```

### 2.2 Blend Tree 详解

```
Locomotion (2D Simple Directional)
  SpeedX: [-1, 1]  左右
  SpeedY: [-1, 2]  前后（后退慢、前进快）

  或者更简单的 1D Blend：
  Speed: [0, 2]
    0.0 → Idle
    0.5 → Walk
    1.0 → Run
    1.5 → Sprint
```

1D Blend 更简单、参数更少，适合 45° 俯视视角（移动方向由角色旋转 + 根运动处理，动画只负责前进方向的速度表现）。

### 2.3 过渡设计

| 从 | 到 | 条件 | 过渡时间(s) | 可打断 |
|---|---|---|---|---|
| Any | Death | Dead trigger | 0.1 | 是 |
| Empty | MeleeAttack | MeleeAttack trigger | 0.05 | 否 |
| MeleeAttack | Empty | 动画播完 (exit time 0.95) | 0.1 | 否 |
| Empty | RangedIdle | IsAiming=true | 0.15 | 否 |
| RangedIdle | Empty | IsAiming=false | 0.2 | 否 |
| RangedIdle | RangedFire | Fire trigger | 0.03 | 否 |
| RangedFire | RangedIdle | 动画播完 | 0.05 | 否 |
| Empty/RangedIdle | Reload | Reload trigger | 0.1 | 否 |
| Reload | Empty/RangedIdle | 动画播完 | 0.1 | 否 |

**Has Exit Time = false** 用于战斗响应类过渡（受击、死亡），确保立即切换不等待。

### 2.4 分层的价值

**Upper Body Layer（上半身遮罩）**：

```
Avatar Mask 配置:
  └── Humanoid
      ├── Hips          ← 不遮（腿部跟 Base Layer）
      ├── Spine         ← 遮
      ├── Chest         ← 遮
      ├── Right Arm     ← 遮
      ├── Left Arm      ← 遮
      └── Head          ← 遮
```

这意味着 **走路时能同时开枪**：腿部动画来自 Base Layer 的 Locomotion，上半身动画来自 Upper Body Layer 的 RangedFire。

## 三、动画事件驱动战斗

### 3.1 核心机制

现在的流程 vs 新流程：

```
【旧】鼠标 → 立即 Physics.OverlapSphere → 立即 TakeDamage
         ↑ 0 延迟，角色纹丝不动

【新】鼠标 → 设 Animator trigger → 播攻击动画
          → 命中帧 AnimationEvent → ActionEventHandler.OnAttackHit()
          → Physics.OverlapSphere → TakeDamage
         ↑ 延迟 = 动画到命中帧的时间（0.2~0.6s）
```

### 3.2 Animation Event 放置

以近战挥砍为例（假设动画全长 1.0s）：

```
MeleeAttack_01.anim 时间轴:
  0.0s  起手（刀抬起）
  0.35s 命中帧 ← AnimationEvent("OnAttackHit")
  0.7s  收尾（刀落下）
  1.0s  结束
```

命中帧在 MeshyAI 生成的动画中需要手动标记。在 Unity Animation 窗口中选择动画 clip → Events 标签 → 在对应帧添加 Event，Function 填 `OnAttackHit`。

### 3.3 ActionEventHandler 桥接

```csharp
// 挂在角色 Animator 所在 GameObject 上
// 动画事件到达时，桥接到战斗/武器系统
public class ActionEventHandler : MonoBehaviour
{
    // 近战命中帧回调
    public void OnAttackHit()
    {
        PlayerCombat.Instance?.ExecuteMeleeHit();
    }

    // 远程开枪帧回调
    public void OnFire()
    {
        WeaponShooting.Instance?.ExecuteFire();
    }

    // 换弹完成帧回调
    public void OnReloadComplete()
    {
        WeaponShooting.Instance?.ExecuteReloadComplete();
    }

    // 动作开始（体力扣除）
    public void OnActionStart(string actionType)
    {
        // 体力/耐久扣除放在这里，不是点击时
    }
}
```

### 3.4 动作锁定

动画播放期间，禁止某些操作：

```
攻击动画中:
  ✗ 不能再次攻击（防止动画叠加）
  ✗ 不能换武器
  ✓ 可以受击（Hit 打断攻击）
  ✓ 可以死亡（Death 打断一切）

换弹动画中:
  ✗ 不能攻击
  ✗ 不能换武器
  ✓ 可以受击
  ✓ 可以死亡
```

通过 `ActionStateHandler.IsLocked` 属性控制。

## 四、与现有系统对接

### 4.1 PlayerCombat 改造

```
旧: OnEnable 绑定鼠标 → TryMeleeAttack → 即时冷却+体力+伤害
新: OnEnable 绑定鼠标 → ActionController.RequestMeleeAttack()
      → 检查冷却/体力/动画锁定
      → 设 Animator trigger → 播动画
      → 动画事件 OnAttackHit 回调
      → PlayerCombat.ExecuteMeleeHit()  ← 保留原有伤害逻辑
```

**改什么**：
- `TryMeleeAttack()` → 拆成 `RequestMeleeAttack()`（设 trigger） + `ExecuteMeleeHit()`（原伤害逻辑）
- 输入绑定移到 ActionController

### 4.2 WeaponShooting 改造

```
旧: Update 中检测 isFire 状态 → 自动开火
新: ActionController.Update → 检测开火条件
      → 设 Animator trigger "Fire"
      → 动画事件 OnFire 回调
      → WeaponShooting.ExecuteFire() ← 保留原有弹道+散射逻辑
```

**改什么**：
- `Update()` 中的开火检测移到 ActionController
- `ExecuteFire()` 只做 Raycast + 伤害 + 弹药 + 事件发布
- 半自动/全自动逻辑保持在 ActionController

### 4.3 WeaponHolder 配合

动画播放期间，武器模型需要跟随手部运动：

- **骨骼模式**（现有）：武器挂右手骨骼 `WeaponHolder.SetBoneMode(Animator.GetBone(HumanBodyBones.RightHand))`，动画驱动手部骨骼 → 武器自动跟随
- **胶囊模式**（兜底）：没有骨骼动画时，武器挂胶囊体偏移位置

Humanoid 动画自动驱动骨骼运动，所以只需要确保 WeaponHolder 用的是骨骼模式即可。**现有代码基本不需要改。**

### 4.4 ItemData 扩展

```csharp
// ItemData 新增字段
public string meleeAttackAnimName;     // 近战攻击动画名（如 "MeleeAttack_01"）
public string fireAnimName;            // 开火动画名（如 "RifleFire"）
public string reloadAnimName;          // 换弹动画名（如 "RifleReload"）
public string equipAnimName;           // 掏武器动画名
```

不同武器可以用不同动画（匕首快刺 vs 大剑重劈），通过 Animator Override Controller 实现。

### 4.5 体力系统

体力消耗从"点击时扣除"改为"动作开始时扣除"：

```
旧: 鼠标点击 → 扣体力 → 伤害
新: 鼠标点击 → 检查体力 → 设 trigger
     → 动画事件 OnActionStart → 扣体力
```

**理由**：如果体力扣在动画前，玩家在动画播完前被打断 → 体力已经扣了但伤害没打出去。改为动画开始帧扣除更合理。

## 五、武器动画变体

### 5.1 Animator Override Controller

不同武器可能需要不同动画风格。不是新建完整 Controller，而是用 **Override Controller** 覆盖特定 clip：

```
PlayerAnimController.controller          ← 基 Controller（使用默认动画）
  │
  ├── PlayerAnim_Melee_Knife.override     ← 匕首变体（覆盖 MeleeAttack → 匕首刺击）
  ├── PlayerAnim_Melee_Sword.override     ← 长剑变体（覆盖 MeleeAttack → 剑挥砍）
  ├── PlayerAnim_Ranged_Pistol.override   ← 手枪变体（覆盖 Ranged 系列 → 手枪动画）
  └── PlayerAnim_Ranged_Rifle.override    ← 步枪变体（覆盖 Ranged 系列 → 步枪动画）
```

切换武器时，`WeaponSwitcher` 同时切换 Animator 的 runtimeAnimatorController 为对应的 Override Controller。

### 5.2 武器类型 → 动画映射

| 武器类别 | isFirearm | Override Controller | 攻击动画 |
|----------|:---:|---------------------|----------|
| 空手 | — | 默认 | — (无武器不做攻击动画) |
| 匕首/短刀 | 0 | Melee_Knife | 快速刺击 |
| 刀/剑/棍 | 0 | Melee_Sword | 单手挥砍 |
| 大剑/斧 | 0 | Melee_Heavy | 双手重击 |
| 手枪 | 1 | Ranged_Pistol | 手枪射击 |
| 步枪/霰弹 | 1 | Ranged_Rifle | 步枪射击 |

僵尸也需要动画，但目前模型是胶囊体+方块。如果没有骨骼模型，可以先不做僵尸动画（保留纯代码行为）。

如果 MeshyAI 生成了僵尸模型+动画：

- 僵尸也用 Humanoid Rig（人形）
- Animator Controller 更简单：Idle → Walk → Chase → Attack → Death
- `ZombieStateMachine.DoAttack()` 同样通过 AnimationEvent 驱动
- 不需要分层（僵尸不需要边跑边开枪）

## 九、风险与注意事项

| 风险 | 应对 |
|------|------|
| MeshyAI 生成的动画与 Unity Humanoid Rig 不兼容 | 导入后先测试一个动画，确认骨骼映射正确后再批量导入 |
| AnimationEvent 字符串回调依赖 GameObject 同名方法 | 方法名必须精确匹配，public void，且挂在 Animator 所在 GO 或其子物体 |
| Override Controller 覆盖 clip 时名字不匹配 | 基 Controller 中 clip 名与 Override 中 clip 名必须一致，用 Unity 的 Create Override 自动对齐 |
| 动画锁定期间被攻击打断导致状态错乱 | Has Exit Time=false + Any State 过渡 + 打断优先级表 |
| 根运动导致角色位移偏移 | 所有攻击动画关 Root Motion（Bake Into Pose），移动动画可选开启 |
