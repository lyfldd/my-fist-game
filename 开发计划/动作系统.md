## 二、Animator Controller：Player.controller

**路径**: `Assets/_Game/Config/Models/Characters/Player/Player.controller`

### 2.1 整体结构

```
Player.controller
└── Base Layer（权重 1.0）
    └── [Locomotion]  ← 唯一状态，运动为 1D BlendTree
        参数: Speed (Float, 默认 0)
```

- **单状态、零过渡**：Base Layer 只有一个名为 "Locomotion" 的状态，没有 Any State 过渡，没有子状态机。
- 默认状态就是 `Locomotion`，进入场景后 Animator 立即激活此状态。
- `writeDefaultValues = true`：退出状态时还原参数默认值。

### 2.2 Speed 参数

| 属性 | 值 |
|------|-----|
| 名称 | `Speed` |
| 类型 | Float (m_Type=1) |
| 默认值 | 0.0 |
| 来源 | PlayerController.Update() 中 `_animator.SetFloat("Speed", _smoothedSpeedRatio)` |

### 2.3 1D BlendTree：Locomotion

```
BlendTree "Locomotion" (Simple1D)
  参数: Speed
  范围: [0, 2]
  手动阈值（非自动）:
    threshold=0.0  → Standing Idle   (站立)
    threshold=1.0  → Walking         (行走)
    threshold=1.5  → Slow Run        (慢跑)
    threshold=2.0  → Fast Run        (快跑/冲刺)
```

**BlendType**: `Simple1D`（一维线性混合）。当 Speed 参数在阈值之间时，Unity 会自动对两个相邻动画做加权混合。

**为什么阈值不是均匀分布**：
- Idle (0) 到 Walk (1)：间隔 1.0，对应从静止到满速走路的整个加速区间。
- Walk (1) 到 SlowRun (1.5)：间隔 0.5，跑步启动区间。
- SlowRun (1.5) 到 FastRun (2)：间隔 0.5，冲刺区间。
- 不设 0.5 阈值（如慢走）：因为 1D 线性混合已经可以在 Idle 和 Walk 之间平滑过渡出任何中间速度的姿势，不需要额外的中间剪辑。

### 2.4 2D 升级（可选，未启用）

`UpgradeAnimator2D.cs` 提供了 `Tools/动画/升级到 2D 混合树` 菜单，可将控制器升级为 2D Simple Directional BlendTree：

```
参数: MoveX (Float, 左右 -1~1), MoveY (Float, 前后 -1~2)
BlendTree "Locomotion2D" (SimpleDirectional2D)
  (0, 0) → Standing Idle
  (0, 1) → Walking      (0, -1) → Walking (后退镜像)
  (0, 1.5) → Slow Run   (0, -1.5) → Slow Run (后退镜像)
  (0, 2) → Fast Run     (0, -2) → Fast Run (后退镜像)
  (1, 0) → Walking (右)  (-1, 0) → Walking (左)
  (2, 0) → Fast Run (右) (-2, 0) → Fast Run (左)
```

**当前状态**：项目使用的是 1D 版本，因为 45° 俯视视角下角色旋转由代码直接控制，动画只负责腿的速度表现。2D 版本预留给未来需要方向性动画（如横向走位）的场景。`Tools/动画/还原到 1D 混合树` 可随时回退。

## 四、Humanoid Rig

### 4.1 模型与 Avatar

| 项目 | 值 |
|------|-----|
| 骨骼来源 | Mixamo（标准人形骨架） |
| 主模型 FBX | `Player_Rigged.fbx` |
| Rig 类型 | Humanoid |
| Avatar 定义 | Create From This Model（从模型自身生成） |
| 动画 FBX 的 Avatar | Copy From Other（复制 Player_Rigged 的 Avatar） |

**为什么使用 Humanoid**：
1. Mixamo 自带完整的 Humanoid 骨骼映射，导入即可用，无需手动配置。
2. Humanoid 重定向允许动画和模型分离 —— 同一个动画可以应用于不同的人形角色（玩家/僵尸/NPC）。
3. 未来从 MeshyAI 或其他来源导入新动画时，只需设为 Humanoid 并引用同一个 Avatar，即可复用到当前模型。

### 4.2 Avatar 获取

`GameBootstrap` 和 `SetupPlayerCharacter` 都从 `Player_Rigged.fbx` 的 sub-assets 中查找 Avatar：

```csharp
// GameBootstrap.cs 行 64-67
var modelAssets = AssetDatabase.LoadAllAssetsAtPath(
    "Assets/_Game/Config/Models/Characters/Player/Player_Rigged.fbx");
foreach (var a in modelAssets)
{ if (a is Avatar av) { avatar = av; break; } }
```

## 五、PlayerController：动画驱动

**路径**: `Assets/_Game/Systems/Player/PlayerController.cs`

`PlayerController` 是整个移动 + 动画系统的中枢，每帧在 `Update()` 中执行。

### 5.1 Animator 引用获取

```csharp
// 行 41
_animator = GetComponentInChildren<Animator>();
```

使用 `GetComponentInChildren` 而非 `GetComponent`，因为 Animator 组件在模型子对象上（Player_Rigged 预制体自带），而非 Player 父对象上。

### 5.2 Speed 参数计算

```csharp
// 行 82-113
// 1. 基础速度
float currentSpeed = moveSpeed;                                    // 5 m/s (GameConstants.PLAYER_MOVE_SPEED)

// 2. 跑步加成
if (wantsRun && !_stamina.IsExhausted)
    currentSpeed *= runSpeedMultiplier;                            // × 1.6

// 3. 体力耗尽惩罚
if (_stamina.IsExhausted)
    currentSpeed *= GameConstants.STAMINA_EXHAUSTED_SPEED_MULT;   // × 0.6

// 4. 背包超重惩罚
if (_inventory.IsOverloaded)
    currentSpeed *= (1f - _inventory.OverloadRatio * maxOverloadSlow);  // 最多 × 0.5

// 5. 角色修正（技能/状态效果）
currentSpeed *= _playerCharacter.GetMoveSpeedModifier();

// 6. 右键瞄准减速
if (_weaponAiming.IsAimingDownSights)
    currentSpeed *= GameConstants.AIM_MOVE_SPEED_MULTIPLIER;      // × 0.4

// 7. 归一化为 Animator Speed 参数
float targetRatio = isMoving ? currentSpeed / moveSpeed : 0f;
```

**归一化公式**：`targetRatio = currentSpeed / moveSpeed`，其中 `moveSpeed = 5`。所以：
- 静止 → `targetRatio = 0` → Animator Speed = 0 → Idle
- 满速行走 → `targetRatio = 1.0` → Animator Speed = 1.0 → Walking
- 满速奔跑 → `targetRatio = 1.6` → Animator Speed = 1.6 → SlowRun(1.5) 到 FastRun(2) 之间
- 瞄准行走 → `targetRatio = 0.4` → Animator Speed = 0.4 → Idle 和 Walk 之间混合

### 5.3 平滑处理（加速度/减速度分离）

```csharp
// 行 19-20
public float animAcceleration = 3f;   // 加速时 Speed 上升速率
public float animDeceleration = 5f;   // 减速/松手时 Speed 下降速率（更快，跟手）

// 行 107-110
float smoothSpeed = targetRatio > _smoothedSpeedRatio ? animAcceleration : animDeceleration;
_smoothedSpeedRatio = Mathf.MoveTowards(
    _smoothedSpeedRatio, targetRatio, smoothSpeed * Time.deltaTime);

// 行 112-113
_animator.SetFloat("Speed", _smoothedSpeedRatio);
```

**设计理由**：
- 加速慢（3/s）：起步动画不会突变，有自然的加速过渡感。
- 减速快（5/s）：松手后角色更快回到 Idle，杜绝滑步感（松键后腿还在跑）。
- 使用 `Mathf.MoveTowards` 而非 `Mathf.Lerp`：保证线性逼近，不受帧率波动影响。

### 5.4 旋转：代码驱动 vs 根运动

角色旋转由 `PlayerController` 直接在 C# 中设置 `transform.rotation`，**不使用 Animator 的 Root Motion**：

```csharp
// 行 55-70
if (isMoving)
{
    // 移动时朝移动方向转（旋转速度 × 2，更灵敏）
    Quaternion targetRotation = Quaternion.LookRotation(moveInput, Vector3.up);
    transform.rotation = Quaternion.RotateTowards(
        transform.rotation, targetRotation, rotationSpeed * 2f * Time.deltaTime);
}
else if (_projector.HasValidTarget)
{
    // 静止时朝鼠标（瞄准用，正常旋转速度）
    Vector3 toTarget = _projector.GroundPoint - transform.position;
    toTarget.y = 0f;
    Quaternion targetRotation = Quaternion.LookRotation(toTarget.normalized, Vector3.up);
    transform.rotation = Quaternion.RotateTowards(
        transform.rotation, targetRotation, rotationSpeed * Time.deltaTime);
}
```

**为什么不用根运动**：
- 45° 俯视视角下角色需要面向移动方向，代码转向更精确。
- 根运动会与 Rigidbody 的速度产生冲突（双重力）。
- Mizamo 动画的根骨骼偏移量不标准，烘焙到 pose 后更容易控制。

### 5.5 移动速度常量汇总

| 常量 | 值 | 来源 |
|------|:--:|------|
| `PLAYER_MOVE_SPEED` | 5 m/s | GameConstants |
| `runSpeedMultiplier` | 1.6 | PlayerController 字段 |
| `STAMINA_EXHAUSTED_SPEED_MULT` | 0.6 | GameConstants |
| `PLAYER_MAX_OVERLOAD_SLOW` | 0.5 (即最多降速 50%) | GameConstants |
| `AIM_MOVE_SPEED_MULTIPLIER` | 0.4 | GameConstants |
| `PLAYER_ROTATION_SMOOTH_SPEED` | 540°/s | GameConstants |

### 5.6 体力状态连动

```csharp
// 行 75-79
bool actuallyRunning = wantsRun && !_stamina.IsExhausted;
_stamina.SetRunning(actuallyRunning);
_stamina.SetWalking(isMoving && !actuallyRunning);
```

体力系统 (`StaminaSystem`) 接收跑步/行走状态，用于体力消耗计算和 UI 显示。`IsExhausted`（体力耗尽）会阻断跑步并通过速度倍率降低 Animator Speed。

### 5.7 脚步声

```csharp
// 行 120-124
if (isMoving)
    SoundEmitter.SetFootstep(FootstepKey, transform.position, wantsRun);
else
    SoundEmitter.StopFootstep(FootstepKey);
```

脚步声由 `SoundEmitter`（挂载在 Managers 对象上）处理，根据 `wantsRun` 标志选择脚步声类型。不是通过 AnimationEvent，而是直接在 Update 中轮询。

## 六、运行时引导（GameBootstrap）

**路径**: `Assets/_Game/Core/GameBootstrap.cs`

`GameBootstrap.OnAfterSceneLoad()` 在场景加载后自动执行，确保 Player 的 Animator 组件存在且配置正确：

```csharp
// 行 54-71
// 检查 Animator 是否存在
if (player.GetComponentInChildren<Animator>() == null)
{
    var anim = player.AddComponent<Animator>();
    // 加载 Controller
    RuntimeAnimatorController ctrl = Resources.Load<RuntimeAnimatorController>("Player");
#if UNITY_EDITOR
    if (ctrl == null)
        ctrl = AssetDatabase.LoadAssetAtPath<RuntimeAnimatorController>(
            "Assets/_Game/Config/Models/Characters/Player/Player.controller");
#endif
    // 加载 Avatar
    Avatar avatar = null;
#if UNITY_EDITOR
    var modelAssets = AssetDatabase.LoadAllAssetsAtPath(
        "Assets/_Game/Config/Models/Characters/Player/Player_Rigged.fbx");
    foreach (var a in modelAssets)
    { if (a is Avatar av) { avatar = av; break; } }
#endif
    if (ctrl != null) anim.runtimeAnimatorController = ctrl;
    if (avatar != null) anim.avatar = avatar;
}
```

**兜底机制**：如果场景中的 Player 因丢失预制体连接或手动误删导致 Animator 缺失，`GameBootstrap` 会在运行时自动补救。构建后的 standalone build 中使用 `Resources.Load("Player")` 加载 Controller（前提是 Controller 放在 Resources 目录下或有 Editor 路径的额外处理）。

## 七、编辑器工具

### 7.1 SetupPlayerCharacter（完整创建管线）

**路径**: `Assets/_Game/Editor/SetupPlayerCharacter.cs`

**菜单**: `Tools/Characters/Setup Player (Full)`

完整的一键流程（5 步），从头创建角色+动画系统：

**配套菜单**: `Tools/Characters/Apply Player Components` — 仅挂组件（Rigidbody/Collider/Animator/所有游戏脚本），不做动画配置。

## 八、当前限制与后续规划

### 8.1 已实现

- 1D BlendTree 移动动画（Idle / Walk / SlowRun / FastRun）
- Speed 参数的平滑驱动（加速度/减速度分离）
- 体力/背包超重/瞄准减速全部联动 Animator Speed
- 运行时 Animator 兜底（GameBootstrap）
- 完整的编辑器工具链（创建/重建/升级/预配置）
- 脚步声类型切换（走/跑）联动声音系统

### 8.2 未实现（仍由代码直接驱动）

- **近战攻击动画**：`PlayerCombat.TryMeleeAttack()` 仍然是 "点击 → 立即伤害"，没有通过 Animator trigger → AnimationEvent 延迟执行。
- **远程射击动画**：`WeaponShooting` 在 Update 中检测开火状态并立即射击，没有 Animator 的 Fire 动画。
- **Upper Body Layer（上半身分离）**：没有 Avatar Mask，没有 Ranged Idle/Aim/Fire/Reload 状态，没有边走边打的叠加动画。
- **受击/死亡动画**：伤害和死亡瞬间执行，没有 HitReaction / Death 动画播放。
- **Animator Override Controller**：不同武器（刀/剑/枪）没有不同的动画变体。
- **AnimationEvent 驱动的体力扣除**：体力仍是点击时扣除，而非动画开始时扣除。
- **僵尸动画**：僵尸模型是胶囊体+方块，没有骨骼，没有动画。

### 8.3 下一阶段

参见 `SetupPlayerCharacter.cs` 和 `UpgradeAnimator2D.cs` 的设计模式：它们已经预留了扩展性（2D BlendTree 的 MoveX/MoveZ 参数可以直接用于横移动画，`SetupAllAnimations()` 可以一口气导入 MeshyAI 生成的所有动画）。战斗动画接入的核心工作是：

1. 导入战斗动画（MeshyAI 或 Mixamo）
2. 新建 Upper Body Layer + Avatar Mask
3. 在 PlayerController 中新增 trigger 参数（MeleeAttack/Fire/Reload/Hit/Death）
4. 改造 PlayerCombat/WeaponShooting 的伤害执行时机（从即时改为 AnimationEvent 驱动）

## 九、与现有系统的对接关系

| 系统 | 对接方式 |
|------|----------|
| **PlayerController** | 主力驱动，Update 中 `SetFloat("Speed")` |
| **StaminaSystem** | 通过 `SetRunning/SetWalking` 影响 `wantsRun` → 最终影响 Speed 参数 |
| **Inventory** | `IsOverloaded / OverloadRatio` → `currentSpeed *= (1 - ratio * maxOverloadSlow)` |
| **PlayerCharacter** | `GetMoveSpeedModifier()` → `currentSpeed *= modifier` |
| **WeaponAiming** | `IsAimingDownSights` → `currentSpeed *= 0.4` |
| **SoundEmitter** | `SetFootstep/StopFootstep` 根据 `wantsRun` 切换走/跑脚步声 |
| **MouseGroundProjector** | 静止时提供鼠标地面点，用于角色瞄准朝向 |
| **GameBootstrap** | 运行时确保 Animator 组件存在且配置完整 |
