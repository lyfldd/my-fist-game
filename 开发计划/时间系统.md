# 时间系统 (Time System)

> **文件**: `Assets/_Game/Systems/Time/TimeManager.cs`  
> **类型**: `TimeManager` (MonoBehaviour, ServiceLocator 单例)  
> **命名空间**: `_Game.Systems.Time`
> **状态**: Phase 1 完成

---

## 概述

TimeManager 负责游戏时间的流逝、日夜循环渲染、天数追踪，并通过 EventBus 发布 `TimeOfDayChanged` 和 `DayChanged` 事件。所有时间参数在 Inspector 中可调，支持运行时 Inspector 跳转调试。

---

## 核心参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `timeScale` | `float` | `72f` | 现实1秒 = 游戏72秒（1.2分钟），即1游戏天 ≈ 20现实分钟 |
| `startHour` | `float` | `8f` | 初始游戏时间（早上8:00） |
| `startDay` | `float` | `1f` | 初始天数（第1天） |

---

## 内部时间追踪

| 字段 | 公式/含义 |
|------|-----------|
| `_previousHour24` | 累计游戏小时数（单调递增，永不折回）。每帧 += `timeScale / 3600 * deltaTime` |
| `_totalGameDays` | 总游戏天数（含小数）：`startDay + _previousHour24 / 24f` |
| `debugCurrentHour` | 当前天内小时（0~24）：`_previousHour24 % 24f`，Inspector 暴露可运行时修改 |
| `_lastHourlyXP` | 已发放生存经验的最大整数小时数，防止重复发放 |

### 跨天检测逻辑（`Update()`）

```
每帧:
  deltaGameHours = timeScale / 3600 * deltaTime
  newHour24 = _previousHour24 + deltaGameHours

  oldDay = floor(_previousHour24 / 24)
  newDay = floor(newHour24 / 24)

  if newDay > oldDay:
      为 oldDay+1 到 newDay 的每一天发布 DayChanged(dayNumber)
      并发放每天生存经验 200 XP

  _previousHour24 = newHour24
  _totalGameDays = startDay + newHour24 / 24
  debugCurrentHour = newHour24 % 24
```

> 关键设计：使用单调递增的 `_previousHour24` 而非模24值，确保跨多天（如调试跳转48小时）时不会遗漏 DayChanged 事件。

## 公共 API

| 成员 | 类型 | 说明 |
|------|------|------|
| `CurrentHour` | `float` | 当前天的小时（0~24，含小数） |
| `CurrentMinute` | `float` | 当前分钟：`(CurrentHour - floor(CurrentHour)) * 60` |
| `CurrentDay` | `float` | 总流逝天数（含小数），供 ChunkManager 容器刷新冷却判定 |
| `GetPeriodName()` | `string` | 返回当前时段中文名 |
| `IsNight()` | `bool` | 当前是否夜晚（21:00 ~ 5:00） |

## 日夜循环光照

TimeManager 直接操控场景中的 Directional Light（太阳 + 月亮）。

### 太阳光 (`sunLight`)

- **旋转**: 绕X轴，角度 = `(h / 24) * 360 - 90`°，即 6:00 日出（0°仰角）、12:00 正午（+90°仰角）、18:00 日落（0°仰角）
- **强度**: 0.3（日出/日落）→ 1.2（正午），使用 `sin(仰角)` 平滑过渡
- **颜色**: 从 `dawnColor` (橙红) 平滑插值到 `dayColor` (暖白)，由 `dawnFactor` 控制过渡带
- **阴影强度**: 0.3 → 1.0
- **禁用**: 当 `sunIntensity < 0.1` 时完全关闭太阳光

### 月亮光 (`moonLight`)

- **方向**: 太阳角 + 180°（与太阳对面）
- **强度**: 0.05 ~ 0.35
- **颜色**: 固定 `moonLightColor`（偏蓝暗色 `(0.1, 0.12, 0.2)`）
- **阴影强度**: 0 ~ 0.3
- **禁用**: 当 `moonIntensity < 0.05` 时关闭

### 环境光

```csharp
float ambient = max(sunIntensity * 0.8, moonIntensity * 0.3);
RenderSettings.ambientIntensity = Lerp(0.08, 1.0, ambient);
```

| 时段 | 太阳强度 | 月亮强度 | 环境光强度 |
|------|----------|----------|-----------|
| 正午 | 1.0 | 0 | 1.0 |
| 日出/日落 | 0.3 | 0 | ~0.3 |
| 月夜 | 0 | 0.35 | ~0.18 |
| 深夜（无月） | 0 | 0 | 0.08 |

### Inspector 颜色配置

| 属性 | 默认颜色 | 说明 |
|------|----------|------|
| `dayColor` | `(1, 0.95, 0.85)` | 暖白正午 |
| `sunsetColor` | `(1, 0.5, 0.2)` | 橙红日落 |
| `nightColor` | `(0.05, 0.05, 0.1)` | 深夜蓝黑 |
| `dawnColor` | `(0.8, 0.6, 0.4)` | 橙红日出 |
| `moonLightColor` | `(0.1, 0.12, 0.2)` | 月光蓝 |

## 事件系统

### `TimeOfDayChanged` — 每帧发布

```csharp
// 定义于 GameEvents.cs
public readonly struct TimeOfDayChanged
{
    public float CurrentHour { get; }  // 当前天小时 (0~24)
}
```

订阅者: `TopLeftHUD`（更新时间显示）、`WeatherManager`（读取小时判定天气轮换）、`SurvivalSystem`.

### `DayChanged` — 跨天时发布（每天一次）

```csharp
// 定义于 GameEvents.cs
public readonly struct DayChanged
{
    public int Day { get; }  // 新天数编号
}
```

订阅者: `TopLeftHUD`（更新天数显示）、`ChunkManager`（容器刷新冷却）、存档系统.

## 生存经验发放

| 触发时机 | 经验值 | 常量 | 额外说明 |
|----------|--------|------|----------|
| 每存活1天（跨天时） | 200 XP | `XP_PER_DAY` | — |
| 每存活1小时（整点） | 5 XP | `XP_PER_HOUR` | 夜间 ×2 = 10 XP |
| 夜间倍率 | ×2 | `XP_NIGHT_MULTIPLIER` | `IsNight()` 判定（21:00~5:00） |

---

## 双套"夜晚"定义

| 系统 | 判定方式 | 夜晚时段 | 用途 |
|------|----------|----------|------|
| `TimeManager.IsNight()` | 内部方法 | 21:00~5:00 | 时段名称、XP夜间倍率 |
| ChunkManager / ZombieSpawner | 独立读取 `CurrentHour` | 18:00~6:00 | 僵尸生成倍率（更宽，提前天黑就加刷） |

## 调试功能

- `debugCurrentHour` 在 Inspector 中可直接修改，`OnValidate()` 会在运行时同步累计小时和天数，并自动发布 `DayChanged` 事件
- 可设置超出24的小时值（如 `26f` → 第2天凌晨2:00），系统正确分割天数和小时内部分

---

## 存档接口

存档数据字段：`TimeWeatherSaveData.totalGameDays`（float）

```csharp
// 保存
float days = timeManager.GetTotalGameDays();

// 恢复
timeManager.SetTotalGameDays(days);
```

> 恢复后 `_previousHour24` 由 `totalGameDays` 反算得出，无需单独保存。

## 依赖关系

```
TimeManager (ServiceLocator 注册)
├── EventBus         ← 发布 TimeOfDayChanged / DayChanged / SurvivalXpGained
├── Light sunLight   ← Directional Light，日夜太阳
├── Light moonLight  ← Directional Light，月光
└── RenderSettings   ← ambientIntensity 环境光

被依赖方:
├── WeatherManager   ← 读取 CurrentHour 判定天气轮换间隔
├── TopLeftHUD       ← 订阅事件，显示"第N天 HH:MM 时段"
├── SurvivalSystem   ← 订阅 TimeOfDayChanged，读取 timeScale 计算Tick间隔
├── ChunkManager     ← 读取 CurrentDay 判定容器刷新冷却
├── ZombieSpawner    ← 读取 CurrentHour 判定夜间生成倍率
└── SaveLoadManager  ← 存档/读档 totalGameDays
```
