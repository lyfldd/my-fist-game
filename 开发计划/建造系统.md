# 建造系统 (Building System)

> 文档生成日期: 2026-06-22
> 对应代码路径: `Assets/_Game/Systems/Building/` + `Assets/_Game/Config/BuildableData.cs` + `Assets/_Game/Config/BuildableCatalog.cs`

## 1. 系统概览

建造系统允许玩家在游戏世界中放置 46 种可建造物，涵盖墙壁、地板、家具、路障、工作台、工业设备、发电/供电设备等。系统通过状态机驱动完整建造流程：虚影预览 → 材料/技能/工具验证 → 建造读条 → 实例化并挂载相应运行时组件。

**核心文件清单:**

| 文件 | 类型 | 职责 |
|------|------|------|
| `BuildableData.cs` | ScriptableObject | 可建造物数据定义（材料、占地、血量、电力等） |
| `BuildableCatalog.cs` | ScriptableObject | 46个建造物的集中目录 |
| `BuildModeController.cs` | MonoBehaviour | 建造状态机 + 输入处理 + 材料扣除 + 实例化 |
| `GhostPreview.cs` | MonoBehaviour | 半透明虚影预览 + 网格吸附 + 碰撞检测 |
| `BuildMenuUI.cs` | MonoBehaviour | UGUI 底部面板 + 10标签页 + 材料提示 |
| `PlacedStructure.cs` | MonoBehaviour | 已放置建筑运行时组件（血量/拆除/摧毁） |
| `WorkstationInteract.cs` | MonoBehaviour | 工作站交互（E键→打开合成面板） |

## 2. BuildableData — 建造物数据定义

`BuildableData` 是一个 ScriptableObject，在编辑器中通过 `右键 → Create → Game/Buildable` 创建。它与 `ItemData`（背包物品）**独立分离**：材料需求通过 `ItemRequirement[]` 引用 `ItemData`。

### 2.1 字段全览

#### 基本信息
| 字段 | 类型 | 说明 |
|------|------|------|
| `displayName` | string | 显示名称（UI 按钮、提示文本） |
| `description` | string (TextArea) | 详细描述 |
| `icon` | Sprite | 图标 |
| `category` | BuildableCategory | 分类枚举（决定标签页归属） |

#### 模型
| 字段 | 类型 | 说明 |
|------|------|------|
| `previewPrefab` | GameObject | 半透明预览模型（材质需支持透明度） |
| `builtPrefab` | GameObject | 放置后的实体模型（带碰撞体） |

#### 占地与吸附
| 字段 | 类型 | 说明 |
|------|------|------|
| `placementSize` | Vector3 | OverlapBox 碰撞检测的尺寸，决定能否放置 |
| `snapSize` | float | 0=自由放置，>0=吸附到该网格步长 |

#### 建造需求
| 字段 | 类型 | 说明 |
|------|------|------|
| `buildDuration` | float | 建造耗时（秒） |
| `requiredTool` | ItemData | 需要的工具（null=不需要），工具需装备在 RightHand |
| `materials` | ItemRequirement[] | 材料清单（ItemData + count） |
| `skillRequirements` | SkillRequirement[] | 技能等级要求（空=无要求），支持多技能组合 |
| `hungerDrainPerSec` | float | 建造中每秒饥饿消耗 |
| `thirstDrainPerSec` | float | 建造中每秒口渴消耗 |
| `staminaDrainPerSec` | float | 建造中每秒体力消耗 |

#### 结构属性
| 字段 | 类型 | 说明 |
|------|------|------|
| `maxHealth` | float | 血量（0=无敌/不可破坏） |
| `blocksNavMesh` | bool | 是否阻挡 AI 路径（Phase 3） |
| `deconstructReturnRate` | float (0~1) | 拆除时返还材料的比例，默认 0.5 |

#### 电力（供电端/终端）
| 字段 | 类型 | 说明 |
|------|------|------|
| `powerSupplyRadius` | float | >0=用电终端，建造预览显示黄色供电圈 |
| `powerOutput` | float | >0=发电端，发电功率(W) |
| `powerSourceType` | PowerSourceType | 电源类型（仅 powerOutput>0 有效） |
| `powerRequiresFuel` | bool | 是否需要燃料 |
| `powerFuelPerHour` | float | 每小时燃料消耗量 |
| `powerFuelItemName` | string | 燃料物品名称（UI显示） |
| `powerFuelItemData` | ItemData | 燃料物品引用 |
| `powerDaytimeOnly` | bool | 是否仅白天发电 |
| `powerRequiresOpenAir` | bool | 是否必须露天 |
| `powerRequiresWater` | bool | 是否必须水边 |
| `powerNoiseRadius` | float | 发电机噪音半径(米) |

#### 设备用电
| 字段 | 类型 | 说明 |
|------|------|------|
| `powerRequired` | float | 需求功率(W) |
| `powerAllowCoal` | bool | 无电时允许烧煤兜底 |
| `powerCoalPower` | float | 烧煤功率 |
| `powerElectricSpeedMul` | float | 通电后生产速度倍率，默认 2x |

#### 工作台/生产设备
| 字段 | 类型 | 说明 |
|------|------|------|
| `isWorkstation` | bool | 是否为工作台（可交互打开合成面板） |
| `workstationTier` | WorkstationTier | 工作台等级（仅 isWorkstation=true） |
| `productionDeviceRef` | ProductionDeviceData | 关联自运转生产设备数据（null=纯工作台） |

#### 工业联动
| 字段 | 类型 | 说明 |
|------|------|------|
| `industrialChain` | ChainType | 所属产业链 |
| `producedItems` | string[] | 产出物品名列表（UI 预览用） |

## 3. 建造放置流程 (Ghost → Confirm → Instantiate)

### 3.1 状态机

```
  按B
Inactive ──→ MenuOnly ──→ Preview ──→ Building ──→ Preview (可连续建造)
  ▲            │  ▲          │  ▲         │
  │  右键/Esc  │  │ 点击物品 │  │ 左键确认│
  └────────────┘  └──────────┘  └─────────┘
      按B
```

**四种状态:**

| 状态 | 说明 |
|------|------|
| `Inactive` | 建造模式未激活，所有建造 UI 隐藏 |
| `MenuOnly` | 菜单可见，但未选中物品（无虚影） |
| `Preview` | 半透明虚影跟随鼠标，可调整位置，等待确认或取消 |
| `Building` | 建造读条中，显示进度 |

### 3.2 按键控制

| 按键 | 功能 |
|------|------|
| **B** | 进入/退出建造模式 |
| **左键**（菜单外） | 确认放置（在 Preview 状态下） |
| **右键 / Esc** | Preview→取消选中（回到 MenuOnly）；MenuOnly→退出建造 |
| **滚轮** | 翻页浏览建造物（ScrollRect 自动处理，鼠标在哪个区域就滚动哪个） |
| **数字键 1~7** | 快捷选择当前页的第1~7个建造物 |

### 3.3 完整放置流程

```
1. 玩家按 B 键
   → EnterBuildMode() → 状态变为 MenuOnly → 显示底部建造面板

2. 玩家点击标签页（墙壁/地板/家具/...）切换分类
   → BuildMenuUI 重建物品按钮列表

3. 玩家点击物品按钮 或 数字键选择
   → SwitchBuildable(data) → 状态变为 Preview
   → GhostPreview 生成半透明预览体

4. 预览体每帧跟随 MouseGroundProjector 的地面投影
   → 网格吸附（snapSize）
   → OverlapBox 碰撞检测
   → 颜色反馈：绿色(可放置) / 红色(阻挡)

5. 玩家左键点击菜单外区域
   → TryConfirmBuild() → TryStartBuilding()
   → 验证链：activeBuildable != null → 位置有效 → 材料充足 → 工具装备 → 技能等级 → AI限1台

6. 验证通过 → 状态变为 Building → 启动 BuildRoutine() 协程
   → 每帧扣体力(StaminaSystem.Consume)
   → 每帧扣饥饿/口渴(EventBus 发布 BuildProgressTickEvent)
   → Progress 0→1

7. 建造完成 → CompleteBuilding()
   → 扣除材料（DeductMaterials）
   → 实例化 builtPrefab 于 GhostPreview.PreviewPosition
   → 挂载运行时组件（PlacedStructure / WorkstationInteract / ProductionDevice / PowerSource / PowerConsumer / PowerTerminal）
   → 发布 StructurePlacedEvent + SurvivalXpGained
   → 状态回到 Preview（可连续建造同一物品）
```

### 3.4 DevTools 自由建造模式

当 DevTools 面板中启用"自由建造"时：
- 跳过材料检查与扣除
- 跳过工具检查
- 跳过技能等级检查

当启用"瞬间建造"时：
- `buildDuration` 设为 0，跳过读条

### 3.5 直接放置 API

`BuildModeController.PlaceStructureDirect(BuildableData, Vector3)` 跳过所有验证和读条，直接实例化。用于职业开局等特殊流程（如开局赠送篝火）。

## 4. GhostPreview — 虚影预览系统

### 4.1 生命周期

- 订阅 `BuildModeEnteredEvent` → `SpawnPreview()` 创建半透明预览体
- 订阅 `BuildModeExitedEvent` → `DestroyPreview()` 销毁
- 有 `previewPrefab` 则实例化之，否则创建默认 Cube 并移除碰撞体
- 遍历所有 Renderer，克隆材质并设为透明（Standard Shader → Transparent mode, renderQueue=3000）

### 4.2 每帧更新

每帧执行：
1. 读取 `MouseGroundProjector.GroundPoint`（地面投影的世界坐标）
2. 射线未打到地面 → 隐藏预览体
3. 网格吸附：`SnapToGrid(worldPos, snapSize)`，仅吸附 XZ 轴，Y 由地面决定
4. `OverlapBox` 碰撞检测（blockLayers 减去 ignoreLayers），无阻挡=可放置
5. 更新颜色：绿色(validColor) / 红色(invalidColor)
6. 供电半径预览：画出黄色圆形 Gizmo

### 4.3 关键属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `PreviewPosition` | Vector3 | 当前预览世界坐标（已吸附网格） |
| `IsValidPlacement` | bool | 当前位置是否可以放置 |
| `CurrentBuildable` | BuildableData | 当前建造数据 |

## 5. 材料/工具/技能验证

### 5.1 材料检查与扣除

`HasRequiredMaterials()`:
- 遍历 `activeBuildable.materials` 中的每一项 `ItemRequirement`
- 调用 `Inventory.HasItem(itemData, count)` 逐一检查
- 若材料为空数组 → 直接通过

`DeductMaterials()`:
- 建造完成时调用
- `Inventory.RemoveItem(itemData, count)` 逐一扣除
- DevTools 自由建造模式跳过

### 5.2 工具检查

`IsRequiredToolEquipped()`:
- 检查 `requiredTool` 是否为 null
- 检查 `Inventory.equipped[EquipSlot.RightHand]` 是否为该工具
- DevTools 自由建造模式跳过

### 5.3 技能等级检查

- 检查 `activeBuildable.skillRequirements` 是否非空
- 遍历每个 `SkillRequirement`，从 `PlayerCharacter.GetSkillLevel(req.skill)` 获取等级
- 任一技能不满足 → 阻止建造
- DevTools 自由建造模式跳过

### 5.4 特殊限制

**AI 机器人限 1 台/玩家：**
- 条件：`category == ElectronicsIndustry && displayName.Contains("AI")`
- 检查 `ServiceLocator.Get<AIBot>()` 是否存在且未死亡
- 已存在 → 阻止建造

## 6. 建造后实例化与组件挂载

建造完成后，`CompleteBuilding()` 按以下顺序处理：

### 6.1 非 AI 建筑（一般建造物）

| 步骤 | 组件 | 触发条件 |
|------|------|----------|
| 3.6 | `PlacedStructure` | 始终挂载，记录 `buildableData` |
| 3.7 | `WorkstationInteract` | `isWorkstation == true`，设置 `workstationTier` |
| 3.8 | `ProductionDevice` | `productionDeviceRef != null`，调用 `.Init(ref)` |
| 3.9 | `PowerTerminal` | `powerSupplyRadius > 0`，设置 `supplyRadius` |
| 3.10 | `PowerSource` | `powerOutput > 0`，设置全部 10 个电力字段 |
| 3.11 | `PowerConsumer` | `powerRequired > 0`，设置功率/烧煤/速度倍率 |

### 6.2 AI 机器人特殊处理

当 `category == ElectronicsIndustry && displayName.Contains("AI")` 时：
- 挂载 `AIBot`（AI 核心逻辑）
- 挂载 `AIBotCombat`（战斗，设置弹药回调连接到 AIBotInventory）
- 挂载 `AIBotInventory`（独立背包）
- 挂载 `AIBotBuildable`（记录建造数据）
- 添加 `Rigidbody`（isKinematic = true，NavMeshAgent 需要）

### 6.3 事件发布

- `StructurePlacedEvent(buildableData, placePos, structure)` — 通知其他系统
- `SurvivalXpGained(XP_PLACE_BUILDING, "build")` — 建造经验值

## 7. BuildableCatalog — 建造物目录

`BuildableCatalog` 是一个 ScriptableObject，在编辑器中通过 `右键 → Create → Game/Buildable Catalog` 创建。它包含一个 `buildables[]` 数组，**共 46 种建造物**，按 10 个分类组织。

### 7.1 公开方法

| 方法 | 返回 | 说明 |
|------|------|------|
| `GetBuildable(int index)` | BuildableData | 按索引获取，越界返回 null |
| `FindByName(string name)` | BuildableData | 按显示名称查找 |
| `GetByCategory(BuildableCategory)` | BuildableData[] | 按分类筛选所有匹配建造物 |
| `GetUsedCategories()` | List\<BuildableCategory\> | 获取有内容的分类列表（排序后） |
| `Count` | int | 建造物总数 |

### 7.2 46 个建造物分布

建造物作为 ScriptableObject 资产存放在 `Resources/BuildableCatalog` 中，BuildMenuUI 在 Awake 时加载：`Resources.Load<BuildableCatalog>("BuildableCatalog")`。

## 9. PlacedStructure — 已放置建筑运行时组件

### 9.1 功能概述

`PlacedStructure` 挂载在每一个建造后的建筑实例上，实现 `IInteractable` 接口。

### 9.2 IInteractable 实现

| 成员 | 值 | 说明 |
|------|-----|------|
| `InteractionPrompt` | `"拆除 {displayName}"` | 准星悬浮时显示的提示文本 |
| `InteractionTime` | `buildDuration * 0.5` | 拆除比建造快一半时间 |
| `IsInteractable` | `!_isDestroyed && activeInHierarchy` | 未摧毁且激活时可交互 |
| `OnInteract()` | 启动 DeconstructRoutine | 玩家按 E 键触发拆除读条 |

### 9.3 血量系统

```csharp
TakeDamage(float damage)
    → _currentHealth -= damage
    → 如果 _currentHealth <= 0 → DestroyStructure()
```

- `maxHealth == 0` → 无敌建筑，不受伤害
- `HealthPercent` → 0~1 比例，用于 Gizmo 血条可视化
- 血条颜色：绿(>50%) / 黄(>25%) / 红(<=25%)

### 9.4 拆除与材料返还

```
玩家按E → OnInteract() → DeconstructRoutine(Inventory)
    → 读条 duration = InteractionTime
    → 完成后：RefundMaterials(Inventory) → 发布 StructureDeconstructedEvent → DestroyStructure()
```

`RefundMaterials`:
- 遍历 `buildableData.materials` 中每一项
- 返还数量 = `max(1, RoundToInt(count * deconstructReturnRate))`
- 默认返还率 50%

### 9.5 注册与存档

- `Awake()` 时自动添加 `PersistentGUID`（存档系统用）
- `OnEnable()` 时注册到 `PlacedStructureRegistry`
- `OnDisable()` 时从注册表注销

## 10. WorkstationInteract — 工作站交互

### 10.1 功能

挂载在工作站建筑上的 `IInteractable` 组件。玩家按 E 键交互时：

1. **特殊处理：化学研究中心 (Chemistry)**
   - 发布 `ResearchStationOpenedEvent(workstationTier)` → 打开独立研究面板
   - **不**走合成 UI

2. **一般工作台**
   - 设置 `CraftingSystem.ActiveStation = workstationTier`
   - 发布 `WorkstationOpenedEvent(workstationTier)` → CraftingUI 订阅后打开合成面板

### 10.2 工作台等级与提示

| WorkstationTier | 交互提示 | 说明 |
|-----------------|---------|------|
| Hands | 徒手制作 | 默认，无需建造 |
| Campfire | 使用篝火 | 篝火（开局基础工作台） |

## 11. 事件总线集成

建造系统通过 EventBus 与其它系统通信：

| 事件 | 发布时间 | 订阅者 |
|------|---------|--------|
| `BuildModeEnteredEvent(BuildableData?)` | 进入建造模式 / 切换建造物 | GhostPreview, BuildMenuUI |
| `BuildModeExitedEvent()` | 退出建造模式 | GhostPreview, BuildMenuUI |
| `BuildProgressTickEvent(hunger, thirst)` | 建造读条每帧 | SurvivalSystem（饥饿/口渴消耗） |
| `StructurePlacedEvent(data, pos, go)` | 建造/直接放置完成 | 存档系统等 |
| `StructureDeconstructedEvent(data, pos, go)` | 建筑被拆除 | 存档系统等 |
| `SurvivalXpGained(XP, "build")` | 建造/放置完成 | 技能经验系统 |
| `WorkstationOpenedEvent(tier)` | 打开工作台 | CraftingUI |
| `ResearchStationOpenedEvent(tier)` | 打开研究中心 | ResearchUI |
| `WorkstationClosedEvent(tier)` | 关闭工作台（建造模式进入时） | CraftingUI |

## 12. 存档系统集成

- `PlacedStructure.Awake()` 自动附加 `PersistentGUID` 组件
- `PlacedStructure.OnEnable()` 注册到 `PlacedStructureRegistry`（单例）
- `PlacedStructure.OnDisable()` 注销
- 存档时：遍历注册表中的所有建筑，序列化其 GUID、BuildableData 引用、位置、血量
- 读档时：通过 GUID 恢复所有建筑（或重新实例化）
- 详见：`SaveLoadManager.cs` + `PersistentGUID.cs` + `PlacedStructureRegistry.cs`
