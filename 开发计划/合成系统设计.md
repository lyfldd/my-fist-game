## 一、核心设计范式

### 旧范式（已废弃）

```
工作台 = 配方目录
玩家 → 低级台点按钮(半成品) → 跑高级台 → 点按钮(成品) → 重复
问题：只有跑腿和点击，没有"工作感"，高级台臃肿混乱
```

### 新范式（v2.0）

```
工作台 = 生产设备制造厂
玩家 → 在工作台制造"生产设备" → 放置设备到世界 → 设备自己运转 → 玩家回来收成品
本质：建造+规划，不是重复点击
```

### 三级生产体系

| | 低级工作台 | 中级工作台 | 高级台+机械加工台+化学台 |
|---|---|---|---|
| **定位** | 手工自足 | 高效批量 | 工业自动化 |
| **设备类型** | 简易手工装置 | 优化机器 | 流水线模块 |
| **速度** | 慢（基准） | ×2~3 | 持续自动 |
| **材料效率** | 基准 | 节省20~30% | 节省40%+ |
| **操作** | 手动投料→等→取 | 批量投入→批量取 | 放原料→自动出→偶尔收 |
| **玩家做什么** | 亲自动手 | 半自动管理 | 建造产线→不管了 |

### 每级工作台都有完整的"生产线"

不跨级跑腿。每一级工作台都能做出**该级别的生产设备**，每个设备独立完成从原料到成品的闭环。

#### 示例：油脂/燃料链（三级递进，不跨级）

```
低级台 → 制造【简易油脂提炼器】
        ├─ 放置在地面上（建造系统联动）
        ├─ 投入：兽脂×2
        ├─ 产出：基础油脂×1（手动取出）
        └─ 闭环：不需要任何更高级工作台

中级台 → 制造【高效蒸馏装置】
        ├─ 放置在地面上
        ├─ 投入：基础油脂×3
        ├─ 产出：精炼燃油×1（批量产出，一炉出多个）
        └─ 闭环：低级油脂+中级装置=精炼燃油

机械加工台 → 制造【燃油提炼流水线】
        ├─ 多模块拼接：主控台×1 + 蒸馏罐×2 + 储油罐×1
        ├─ 管道连接，全自动运转
        ├─ 投入：兽脂/原油/种子（任意原料，智能识别）
        └─ 产出：储罐持续积累燃油，满时自动停机
```

#### 各系统三级设备总览

| 系统 | 低级设备(简易台) | 中级设备(中级台) | 工业流水线(高级/机械/化学台) |
|------|-----------------|-----------------|---------------------------|
| **油脂/燃料** | 简易油脂提炼器 | 高效蒸馏装置 | 燃油提炼流水线 |
| **化工** | 手工压药器 | 半自动化工台 | 化工自动化工厂 |
| **金属加工** | —（熔炉足够） | 手动冲压机 | 金属粉碎+冲压流水线 |
| **弹药** | — | 手工装弹台 | 弹药生产线 |
| **食品保存** | 晾肉架 | 熏肉架+腌菜缸 | 食品加工厂 |
| **纺织** | — | 简易织布机 | 纺织工坊 |
| **医疗** | 简易药剂台 | 半自动制药台 | —（化学台手工即可） |
| **电力** | — | 脚踏发电机 | 水车+风车+电网 |
| **水源** | 雨水收集桶 | 简易滤水塔 | 滤水+储水系统 |

### 玩家时间投入变化

```
前期(低级台):  手动操作多，设备需要投料/取料，但闭环自足
中期(中级台):  批量操作，一个设备一次处理一批，效率质变
后期(高级台+): 工业自动化，原料放进去，偶尔回来收成品，大部分时间自动运转
```

> **不变的原则：前期简单，后期丰富。** 低级台设备少但能闭环生存；中级台效率翻倍但操作简化；高级/化学/机械台才是真正的工业爆发，60%+ 的内容集中在后期。

## 二、设计原则

### 两个简化铁律

1. **同材质物品不设多层中间态**。木材就是木材，不存在"木材→木棍→木板→加厚木板"。数量表达难度，不新增物品类型。
2. **生存前期节俭，工业后期丰富**。第一阶段 30 分钟就该脱离徒手，中期建造基础，后期 60% 的内容量集中在高级工作台+工业设施。

### 中间态保留标准

一个新物品必须满足**至少两条**才独立存在：
- 多种成品都依赖它（如铁锭 → 工具/武器/零件）
- 它本身有独立用途（如绷带可以直用，也是急救包原料）
- 它在玩法上产生有意义决策（如燃料选择：木柴 vs 木炭 vs 精炼燃油）

三条都不满足 → 砍掉，用原材料数量表达。

## 三、核心底层规则

### 物品大类划分（全品类闭环）

| 大类 | 内容 | 来源 |
|------|------|------|
| 原材料 | 木头、石头、草药、矿石、兽皮、杂草、废水 | 野外采集/搜刮 |
| 半成品 | 铁锭、铜锭、布料、火药、通用零件 | 原材料加工 |
| 消耗品 | 食物、饮水、药品、投掷物、弹药、燃料 | 合成/搜刮 |
| 装备类 | 近战/远程武器、护甲、头盔、工具 | 合成/搜刮 |
| 建筑类 | 墙体、地板、门窗、陷阱、储物 | 建造系统 |
| 功能道具 | 图纸、技能书、钥匙、任务物品 | 搜刮/掉落 |
| 工作台 | 简易/中级/高级工作台、篝火、熔炉 | 合成后放置 |
| 工业设施 | 砂轮机、车床、炭窑、织布机、磨坊... | 合成后放置 |

### 物品三大核心属性

**品质** (`ItemQuality` 枚举，挂在 `ItemData`)：
```
Scavenged   — 地图刷新/搜刮品（属性最优，不可制造）
Handmade    — 自制（可制造，属性略低于搜刮品）
Professional — 高端自制（高级技能+稀有材料，属性接近搜刮品）
```

**耐久值**：装备/工具独有，归零报废，可维修。`ItemData` 加 `maxDurability`。

**用途标签** (`ItemUsageTag` 枚举，`ItemData` 加 `usageTags[]` 数组)：
```
烹饪 / 冶炼 / 建造 / 战斗 / 医疗 / 能源
```
用于 UI 分页 + 工作台权限匹配（熔炉只认"冶炼"标签材料）。

### 材料合并原则

同源物料大一统命名，删减细碎分类：
- 木材 = 树枝+原木+枯木（不设木棍/木板/加厚木板）
- 草本纤维 = 干草+植物纤维+芦苇+枯叶（不设绳索/粗布/棉布）
- 草药 = 普通+珍稀草药
- 石料 = 碎石+原石+砂石
- 生肉 = 所有野兽肉类
- 碎布料 = 破旧布料+布条
- 废金属 = 生锈金属+边角料
- 通用零件 = 老旧零件+小配件
- 空容器 = 玻璃瓶+塑料瓶

### 物品层级关联逻辑

核心规则：**高阶物品依赖低阶半成品，不跳级合成。同材质不拆分中间形态。**

| 链条 | 流程 | 最终成品 |
|------|------|----------|
| 金属链 | 铁矿 →(熔炉)→ 铁锭 → 铁质工具/武器/零件 | 武器/防具/工业设施 |
| 食物链 | 野果/生肉 →(篝火)→ 熟食 → 高级烹饪 | 长效食物/罐头 |
| 药品链 | 草药 → 研磨 → 基础药剂 → 高级药剂 | 急救包/抗毒血清 |
| 回收链 | 废旧装备 →(高级台拆解)→ 材料回流 | 再制新装备 |
| 弹药链 | 火药+弹壳(通用基底) → 换弹头 → | 手枪弹/步枪弹/霰弹 |

强制关联规则：
- 特殊道具只绑定专属产出途径（图纸解锁/僵尸掉落/宝箱搜刮）
- 半成品统一归类收纳，不拆分冷门素材
- 一物只留 1-2 种主流合成路线，不做同功能多材料合成

## 四、双门槛权限体系

### 门槛1：制作台权限锁

| 制作方式 | 可合成范围 | 禁止操作 |
|----------|-----------|----------|
| 徒手 | 基础工具、简易耗材、燃料加工 | 所有半成品、成品装备 |
| 简易工作台 | 木质建材、铁质工具、基础家具 | 金属制品、药品、枪械 |
| 中级工作台 | 铁质武器、皮甲、石砖建筑、储物、卫生 | 枪械、炸药、高阶药剂 |
| 高级工作台 | 全品类成品 + 拆解 + 维修 + 改装 | 无法熔炼矿石、无法生火做饭 |
| 熔炉 | 仅熔炼矿石、烧制原料锭 | 不做任何成品装备 |
| 篝火 | 仅烹饪、烧水、食材处理 | 无任何造物能力 |

### 门槛2：人物技能等级锁

- 建造拆解 → 解锁工作台升级 + 所有建筑配方
- 医疗生存 → 解锁各级药剂、急救物品合成
- 工匠制作 → 解锁工具升级、工业设施
- 枪械专精 → 解锁手枪→步枪→狙击全枪械 + 对应子弹
- 近战专精 → 解锁高级冷兵器
- 汽车改造 → 解锁车辆维修件/改装/燃料
- 智力 → 解锁稀有配方图纸（无线电/发电机/高级制药）

## 五、品质设计决策

品质挂在 `ItemData` 层（物品定义层），不同品质 = 不同 `.asset` 文件：
- `绷带_简易.asset` → quality=Handmade, 回血15, 消轻微出血
- `绷带_军用.asset` → quality=Scavenged, 回血40, 消重度出血+防感染

好处：图标不同、描述不同、属性固定、制作配方直接指定产出哪个，运行时不搞复杂。

## 十三、工业设施机制

工业设施不是"工作台"——是**放置在世界中的功能性建筑**，与建造系统联动：

| 设施 | 功能 | 动力加成 |
|------|------|---------|
| 砂轮机 | 维修武器不耗耐久材料 | 接动力 → 同时修两件 |
| 车床 | 批量加工木材（一次 N 个） | 接动力 → N×2 |
| 冲压机 | 铁锭→弹壳/零件（不徒手敲） | 接动力 → 自动模式 | **唯一量产通用零件来源** |

> **通用零件与高级零件的关系**：通用零件只能野外搜刮（后期靠冲压机量产），高级工作台将通用零件+铁锭加工为高级零件。所有高端物品（枪械/战术护甲/电力设备/车辆改装/工业设施/动力设施）消耗高级零件，不再直接消耗通用零件。链条：`搜刮通用零件 → 冲压机量产 → 高级工作台 → 高级零件 → 高端物品`。
| 磨坊 | 谷物→面粉 / 矿石→碎矿 | 接动力 → 效率×3 |
| 炭窑 | 木材→木炭（热值 3×） | — |
| 水泥桶 | 石料+石灰→水泥 | 接动力 → 自动搅拌 |
| 织布机 | 草本纤维→布料 | 接动力 → 产出×3 |
| 压油机 | 兽脂/种子→基础油脂 | 接动力 → 产出×2 |

动力来源：水车（水边持续）、风车（高地间歇）、脚踏发电机（人力稳定）。

### 21.1 背包系统对接

> **现状**: Inventory 有完整 API — `AddItem`/`RemoveItem`/`HasItem`/`GetItemCount`。合成系统直接调用即可，无需改造 Inventory 本身。

#### CraftingSystem 调用点

```csharp
// CraftingSystem.Craft(RecipeData recipe)
bool CanCraft(RecipeData recipe)
{
    foreach (var req in recipe.materials)
        if (!Inventory.Instance.HasItem(req.itemData, req.count))
            return false;
    return true;
}

bool Craft(RecipeData recipe)
{
    if (!CanCraft(recipe)) return false;

    // 扣材料
    foreach (var req in recipe.materials)
        Inventory.Instance.RemoveItem(req.itemData, req.count);

    // 出成品
    int added = Inventory.Instance.AddItem(recipe.resultItem, recipe.resultCount);
    if (added <= 0)
    {
        // 背包满了→回滚材料
        foreach (var req in recipe.materials)
            Inventory.Instance.AddItem(req.itemData, req.count);
        EventBus.Publish(new CraftingFailedEvent(recipe, "背包已满"));
        return false;
    }
    return true;
}
```

### 21.8 电力系统（从零新建）

> **现状**: 完全不存在。工业自动化需要动力：水车→水力、风车→风力、发电机+太阳能板→电力。

#### 设计原则

```
不铺电路，不模拟电压电流。
电力 = 动力输出(W) ≥ 设备需求(W) = 能运行
设备分为：独立自供（脚踏发电机） / 水边（水车） / 高地（风车） / 电网（发电机+电线）
```

#### 核心组件

```csharp
// Systems/Power/PowerSource.cs
public class PowerSource : MonoBehaviour
{
    public float maxOutput = 100f;     // 最大输出功率(W)
    public float currentOutput;        // 当前输出
    public bool isActive;              // 是否在发电
    public PowerSourceType sourceType; // Water/Wind/Combustion/Solar/Human
    public float fuelConsumptionRate;  // 燃料消耗率（燃烧型）
    public List<PowerConsumer> connectedConsumers;
}

// Systems/Power/PowerConsumer.cs
public class PowerConsumer : MonoBehaviour
{
    public float requiredPower = 50f;  // 需要功率(W)
    public PowerSource connectedSource;
    public bool isPowered => connectedSource != null 
        && connectedSource.currentOutput >= requiredPower;

    void OnPowered();    // 通电后行为（工业设备速度×2等）
    void OnUnpowered();  // 断电后行为（降级到手动模式）
}

// Systems/Power/PowerGrid.cs
public class PowerGrid : MonoBehaviour
{
    // 管理多个 PowerSource 与多个 PowerConsumer 的配对
    // 总输出 ≥ 总需求 → 全部运转
    // 总输出 < 总需求 → 低优先级设备断电
}
```
