## 一、架构总览

合成系统由两大子系统构成：

```
┌─────────────────────────────────────────────────────┐
│                    合成系统                          │
│                     CraftingSystem                   │
│  (单体 MonoBehaviour, 挂载于场景主游戏物体)          │
├────────────────────────┬────────────────────────────┤
│   A. 手工合成模式       │  B. 工业自动生产模式        │
│   CraftingSystem        │  ProductionDevice           │
│   + CraftingUI          │  + ProductionDeviceUI       │
│     (工作台交互)        │    (流水线 + 燃料/电力)     │
│                         │                            │
├────────────────────────┼────────────────────────────┤
│  C. 化学研究系统        │  D. 配方目录                │
│  ChemicalResearchManager│   RecipeCatalog             │
│  + ChemicalResearchUI   │  (ScriptableObject 总表)    │
│  (科技树解锁设备/配方)  │                            │
└────────────────────────┴────────────────────────────┘
```

**核心数据流**:
1. `RecipeCatalog` → 配方总表（ScriptableObject 引用所有 `RecipeData`）
2. `CraftingSystem` → 查询配方、校验技能/材料 → 扣材料 → 出成品 → 发 XP
3. `ProductionDevice` → 自运转：消耗输入槽原料 → 出产物到输出槽或下游设备
4. `ChemicalResearchManager` → 追踪已完成的研究 → 门控配方/设备解锁

## 二、工作台层级 (WorkstationTier)

定义于 `Assets/_Game/Config/WorkstationTier.cs`：

| 枚举值 | 中文名 | 层级 | 用途 |
|--------|--------|------|------|
| `Hands` (0) | 徒手制作 | 基础 | 最原始合成，消耗体力 10/次 |
| `Campfire` (1) | 篝火 | 基础 | 烹饪、基础制备，消耗体力 10/次 |
| `SimpleBench` (2) | 简易工作台 | 初级 | 木板/铁器/基础工具，消耗体力 5/次 |
| `Furnace` (3) | 熔炉 | 初级 | 冶炼金属、烧制材料，体力消耗 0 |
| `MediumBench` (4) | 中级工作台 | 中级 | 机械零件、武器配件、弹药，体力消耗 0 |
| `AdvancedBench` (5) | 高级工作台 | 高级 | 精密制造：电路板、高级零件、防具/武器 |
| `Chemistry` (6) | 研究中心 | 高级 | 化工配方、药品、工业设备配方解锁 |
| `Machining` (7) | 机械加工台 | 工业 | 重工业、工厂设备套件、车辆、动力 |
| `ElectronicsAssembly` (8) | 电子装配台 | 终局 | 芯片/电路/传感器/精密电子 |
| `ElementFurnace` (9) | 元素合成炉 | 终局 | 聚变核心、高级能源 |

**继承规则**: 高级工作台自动继承所有低级工作台的配方。`CraftingSystem.GetAllRecipesForStationAndBelow(tier)` 遍历 `0..tier` 汇总。

**UI 映射** (CraftingUI.GetStationName): 目前枚举到第 8 项 (Machining)，ElectronicsAssembly 和 ElementFurnace 在枚举中有定义但 UI 回退显示"工作台"。

## 三、核心合成系统 (CraftingSystem)

**文件**: `Assets/_Game/Systems/Crafting/CraftingSystem.cs`

### 3.1 单例注册

- `Awake()` 中：单例检查 → `ServiceLocator.Register(this)`
- 自动从 `Resources.Load<RecipeCatalog>("RecipeCatalog")` 加载配方总表
- 从同 GameObject 上获取 `Inventory`, `SurvivalXPSystem`, `StaminaSystem`（避免 ServiceLocator 时序问题）

### 3.2 配方查询 API

| 方法 | 说明 |
|------|------|
| `GetAvailableRecipes()` | 获取当前工作台可用配方（过滤技能+材料） |
| `GetAllRecipesForStation(tier)` | 获取指定工作台全部配方（不过滤） |
| `GetAllRecipesForCurrentStation()` | 获取当前工作台全部配方 |
| `GetAllRecipesForStationAndBelow(tier)` | 获取指定层级及以下全部配方（继承） |
| `GetAllRecipesForCurrentStationAndBelow()` | 当前层级及以下全部配方 |

### 3.3 合成执行流程

```
CanCraft(recipe)
  ├── recipe.requiredStation > ActiveStation → false (工作台不够高级)
  ├── CheckSkill(recipe) → false (技能不够)  
  ├── HasMaterials(recipe) → false (材料不足)
  └── true

Craft(recipe)
  ├── 扣材料: _inventory.RemoveItem(req.itemData, req.count) ×N
  ├── 扣体力: ConsumeStamina(recipe)
  ├── 加成品: _inventory.AddItem(resultItem, resultCount)
  │   └── 背包满 (added <= 0)
  │       ├── 回滚: 重新添加所有已扣除的材料
  │       ├── 事件: CraftingFailedEvent("背包已满")
  │       └── return false
  ├── 事件: CraftingCompletedEvent(名称, 产物, 数量, XP)
  ├── 发放 XP: _xpSystem.AddXP(recipe.xpReward)
  └── return true
```

### 3.4 辅助查询

| 方法 | 说明 |
|------|------|
| `CanCraft(recipe)` | 完整校验（工作台+技能+材料） |
| `GetCraftableCount(recipe)` | 根据背包材料可制作的最大次数 |
| `HasMaterialsFor(recipe)` | 仅校验材料（供 UI 区分显示） |

## 四、配方数据 (RecipeData)

**文件**: `Assets/_Game/Config/RecipeData.cs`

### 4.1 ScriptableObject 字段

每个配方是一个 `.asset` 文件，通过右键 `Create → Game/Recipe` 创建。

```csharp
[CreateAssetMenu(menuName = "Game/Recipe")]
public class RecipeData : ScriptableObject
{
    // -- 基础信息 --
    string recipeName;              // 配方名称
    string recipeId;                // 唯一ID (研究中心配方级解锁用, 留空=无需研究)
    Sprite icon;                    // 图标
    string description;             // 描述文本 (2-3行)
    RecipeCategory category;        // 配方分类 (14种)

    // -- 合成需求 --
    WorkstationTier requiredStation; // 需要的工作台等级
    bool isIndustrial;              // 是否为工业配方
    string productionDeviceName;    // 关联的生产设备名称
    SkillRequirement[] skillRequirements; // 技能需求
    ItemRequirement[] materials;    // 材料清单

    // -- 产出 --
    ItemData resultItem;            // 产物
    int resultCount = 1;           // 产出数量

    // -- 合成参数 --
    float craftTime = 2f;          // 合成耗时
    float xpReward = 15f;          // XP奖励

    // -- 拆解 --
    bool canDeconstruct;            // 是否可拆解
    float deconstructReturnRate = 0.5f; // 拆解回收率 (0~1)
}
```

### 4.3 技能需求 (SkillRequirement)

```csharp
[Serializable]

{
    public SkillType skill;  // 技能类型 (如 SkillType.智力)
```

### 4.4 配方 ID 与研究门控

- **两层门控**:
  1. **设备级**: `ChemicalResearchProject.unlockedDeviceNames` — 解锁整个设备，所有该设备配方可用
  2. **配方级**: `ChemicalResearchProject.unlockedRecipeIds` — 解锁指定配方（`RecipeData.recipeId`），用于后期/终局精细控制
- **默认解锁**: 不在任何研究项目中的设备和配方，视为默认已解锁
- **DevTools.FreeBuildEnabled**: 开启后所有门控跳过

### 6.3 交互流程

1. **打开**: 玩家与工作台交互 → `WorkstationOpenedEvent` → `OnStationOpened()`
   - 关闭其他 UI (背包、建造模式、容器窗口)
   - 设置 `CraftingSystem.ActiveStation` 为对应工作台层级
   - 刷新配方列表

2. **筛选**: 
   - **分类过滤**: 点击分类按钮切换 `_activeCategory`
   - **搜索过滤**: InputField 触发 `OnSearchChanged()`，模糊匹配配方名/产物名/材料名

3. **合成**:
   - 单次: `CraftOne()` → `DoCraft()` → `CraftingSystem.Craft()`
   - 批量: `CraftAll()` → 循环 `Craft()` 直到失败或次数用完

4. **关闭**: ESC → `HandleEsc()` → `Close()` → `WorkstationClosedEvent`
   - 或点关闭按钮

### 6.4 性能优化：脏刷新

```csharp
bool _needsRefresh = true;

void Update() {
    if (_needsRefresh || frameCount - _lastRefreshFrame > 30) {
        RefreshUGUI();
    }
}
```
- 操作触发 `MarkDirty()` 标记
- 30帧兜底刷新 (0.5秒)
- 配方数量变化才重建按钮列表，选中变化只更新高亮

## 七、生产设备系统 (ProductionDevice)

**文件**: `Assets/_Game/Systems/Crafting/ProductionDevice.cs`

### 7.1 架构

`ProductionDevice` 是实现 `IInteractable` 的 MonoBehaviour，挂载在放置后的设备 GameObject 上。

```
玩家背包 ──→ [输入槽 InventoryContainer] ──→ [生产逻辑] ──→ [输出槽 InventoryContainer]
                 4×3 网格                             4×2 网格
                                                          │
                                                    输出槽满时自动转发
                                                          │
                                                          ▼
                                               [下游设备 OutputDestination]
                                                   输入槽
```

### 7.2 关键属性

```csharp
// 运行时
ProductionDeviceData _data;               // 设备定义 (ScriptableObject)
InventoryContainer _inputSlot;             // 输入槽 (4×3)
InventoryContainer _outputSlot;            // 输出槽 (4×2)
ProductionDevice _outputDestination;       // 流水线下游目标
float _fuelRemaining;                      // 剩余燃料轮数
float _cycleTimer;                         // 生产周期计时器

// 电力模式
bool _isElectricPowered;                   // 电网供电
bool _isCoalPowered;                       // 烧煤模式
```

### 7.3 Update 自运转 (TryProduce)

```
Update() 每帧:
  cycleTimer += deltaTime
  通电加速: interval /= electricSpeedMultiplier
  
  if cycleTimer >= interval:
    TryProduce()
      1. 研究门控: 设备解锁? (ChemicalResearchManager.IsDeviceUnlocked)
      2. 电力门控: 有PowerConsumer需通电/烧煤才运转
      3. 燃料门控: 非通电模式消耗燃料
         - 燃料耗尽 → 尝试消耗输入槽燃料
         - 无燃料 → DeviceFuelDepletedEvent
      4. 遍历配方:
         - 配方级研究门控 (recipeId 解锁检查)
         - 多材料输入 (inputs[]) 优先级高于单材料 (input)
         - 材料齐全 → 扣材料 + PlaceOutput
         - 事件: ProductionCycleEvent
```

### 7.4 输出分发 (PlaceOutput)

```
PlaceOutput(item, count)
  ├── 有下游设备 (OutputDestination)?
  │   ├── 尝试转发到下游输入槽
  │   ├── 成功 → ProductionCycleEvent(下游设备)
  │   └── 失败 → DeviceOutputFullEvent(下游设备)
  ├── 自己的输出槽
  │   └── 失败 → DeviceOutputFullEvent(自身)
  └── 兜底: 玩家背包
```

### 7.5 配方来源 (GetActiveRecipes)

优先级：
1. `ProductionDeviceData.recipes` (内嵌配方，SO 中直接定义)
2. `ItemGraphManager.GetRecipesForDevice(deviceName)` (从 RecipeCatalog 查询工业配方)
3. 空数组

### 7.6 燃料系统

- `requiresFuel = true` 且指定 `fuelItem` 时生效
- 初始化: `_fuelRemaining = fuelPerCycle * 3f` (预充3轮)
- 每次 TryProduce 扣除 `fuelPerCycle`
- 耗尽时从输入槽消耗 1 个燃料物品，恢复 `fuelPerCycle * 10` 轮
- 通电模式跳过燃料消耗

### 7.7 电力系统 (PowerConsumer 集成)

```csharp
// 由 PowerConsumer 组件通知电力状态
NotifyPowerState(bool electricPowered, bool coalPowered)

// 状态判定优先级
IsRunning = electricPowered || coalPowered || (!requiresFuel && no PowerConsumer)
```

运行模式：
| 供电模式 | 速度 | 燃料消耗 |
|----------|------|----------|
| 电网供电 | `interval /= electricSpeedMultiplier` (加速) | 无 |
| 烧煤 | 正常速度 | 消耗煤 |
| 无电力 | 停摆 (有 PowerConsumer 时) | — |

### 7.8 IInteractable 接口

```csharp
string InteractionPrompt  => $"使用 {deviceName}"
float InteractionTime    => 0.3f
OnInteract(interactor)   => DeviceOpenedEvent(this)
```

玩家 E 键交互 → 打开 ProductionDeviceUI 面板。

```csharp
// 当前设备的产出能否被目标设备作为原料接收
bool CanLinkTo(ProductionDevice target)
{
    foreach (myRecipe in deviceData.recipes)
        foreach (theirRecipe in target.Data.recipes)

    return false;
}
```

```csharp
[Serializable]
public struct ChemicalResearchProject
{
    string researchId;                  // 唯一ID
    string displayName;                 // 显示名称
    string description;                 // 描述
    ItemRequirement[] cost;             // 研究费用 (材料消耗)
    string[] unlockedDeviceNames;       // 解锁的设备名称 (对应 ProductionDeviceData.deviceName)
    string[] unlockedRecipeIds;         // 解锁的配方ID (对应 RecipeData.recipeId)
    int requiredIntellectLevel;         // 智力等级需求

    // v2.0 科技树字段
    string parentResearchId;            // 前置研究ID (子项才有值)
```

### 10.3 研究解锁逻辑

```
IsDeviceUnlocked(deviceName):
  遍历所有研究项目:
    if 项目.unlockedDeviceNames 包含 deviceName:
      if 项目已完成 (IsResearched) → true
      else → false (锁定)
  未在任何项目中 → true (默认解锁)

IsRecipeUnlocked(recipeId):
  遍历所有研究项目:
    if 项目.unlockedRecipeIds 包含 recipeId:
      if 项目已完成 → true
      else → false (锁定)
  未在任何项目中 → true (默认解锁)
```

### 10.4 研究执行

```csharp
CanResearch(researchId, inventory):
  ├── 项目存在?
  ├── 已完成? → false
  ├── 智力需求: GetSkillLevel(智力) >= requiredIntellectLevel
  └── 材料充足? (所有 cost 物品)

TryResearch(researchId, inventory):
  if !CanResearch → false
  扣材料: inventory.RemoveItem ×N
  _completedIds.Add(researchId)
  → true
```

### 10.5 科技树结构 (v2.0)

- **大类 (isCategory=true)**: 作为父节点，可展开查看子项
- **子项 (parentResearchId != "")**: 需大类先完成后才能研究
- **独立项目**: parentResearchId 为空且 isCategory=false (兼容旧数据)
- `GetChildProjects(parentId)`: 获取指定大类的所有子项目

## 十五、双模式设计 (Craft vs Industrial)

### 15.1 手工合成 (Craft Mode)

- **触发**: 玩家与工作台交互 (E键)
- **工作流**: 选择配方 → 点击"制作" → 即时出成品
- **特点**:
  - 消耗体力
  - 受技能等级限制
  - 每次制作扣材料+出成品 (原子操作)
  - 背包满自动回滚材料
- **UI**: CraftingUI (UGUI 代码自建, sortingOrder=150)

### 15.2 工业生产 (Industrial Mode)

- **触发**: 放置 ProductionDevice 到世界 → 配置输入/输出/燃料/电力
- **工作流**: 持续自运转 (Update 循环)，自动消耗输入→产出
- **特点**:
  - 无需体力
  - 需燃料或电力
  - 支持流水线链接 (OutputDestination)
  - 连续批量生产
  - 研究门控 (配方和设备都需解锁)
- **配方来源**: 
  1. `ProductionDeviceData.recipes` (内嵌)
  2. `ItemGraphManager` → `RecipeCatalog.GetByDevice()` (动态查询)
- **UI**: ProductionDeviceUI (UGUI 代码自建, sortingOrder=155)

### 15.3 配方标记

`RecipeData` 有两个字段区分模式:
- `isIndustrial`: 标记为工业配方 (true=仅设备生产, false=手工合成)
- `productionDeviceName`: 关联的生产设备名称

## 附录 A: 关键设计决策

1. **代码创建 UI** (非 Prefab): 合成系统所有 UI 面板均通过代码动态创建 Canvas + 子元素。优点：无 Prefab 依赖错误；缺点：修改布局需要改代码。

2. **体力消耗阶梯**: 徒手/篝火=10, 简易工作台=5, 其他=0。模拟"使用工具减轻劳动"。

3. **配方 ID 门控**: 设计为两层（设备/配方），后期和终局使用配方级细粒度控制，前期/中期使用设备级大类解锁。

4. **流水线架构**: OutputDestination 支持设备间自动转发，形成生产链。存档系统通过 GUID 持久化链接关系。

5. **错误回滚**: Craft() 在背包满时回滚已扣材料，保证材料不丢失。
