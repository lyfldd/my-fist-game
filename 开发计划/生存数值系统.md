## 一、功能概述

生存数值系统是整个僵尸生存游戏的核心机制层。玩家角色拥有 4 项实时属性（健康/饥饿/口渴/体温），随时间持续衰减、受环境和装备影响、被物品使用恢复，最终驱动生存压力和策略决策。

系统通过 EventBus 与其他模块完全解耦：时间推进驱动 Tick，装备变化更新护甲/保暖缓存，物品使用触发效果应用，UI 层独立监听数值变化刷新 HUD。

## 二、四大属性

### 2.1 属性清单

| 属性 | 英文 | 最小值 | 最大值 | 初始值 | 常量 | Tick 行为 |
|------|------|--------|--------|--------|------|-----------|

## 三、Tick 衰减机制

### 3.1 Tick 间隔

- 常量: `SURVIVAL_TICK_INTERVAL = 60f`（游戏秒）
- 实时间隔: `60f / timeScale` 秒
- 在 `Update()` 中检查 `Time.time - lastTickTime >= TICK_INTERVAL / timeManager.timeScale`

### 3.2 每 Tick 执行顺序

```
TickSurvivalLogic()
  1. UpdateTemperature()       — 体温向目标值靠拢
  2. ApplyAttrDecay()           — 饥饿、口渴衰减
  3. ApplyNaturalRegen()        — 条件回血
  4. ApplyDangerDamage()        — 饥饿/口渴过低时扣血
  5. ApplyTemperatureDamage()   — 失温/过热扣血
  6. ApplyStateEffects()        — 出血/感染持续扣血
```

### 3.3 衰减倍率 (ApplyAttrDecay)

基础衰减由 `SurvivalData` ScriptableObject 配置:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `hungerDecayRate` | 0.2 | 每 Tick 饥饿基础衰减 |
| `thirstDecayRate` | 0.3 | 每 Tick 口渴基础衰减 |

倍率叠加规则（乘法）：

1. **回血加速消耗**: 当满足回血条件且健康不满时，饥饿/口渴衰减 × `regenAttrMultiplier`（默认 3x）
2. **交叉惩罚**:
   - 饥饿归零 → 口渴衰减 × 1.5
   - 口渴归零 → 饥饿衰减 × 1.5

### 3.4 自然回血条件 (ApplyNaturalRegen)

满足以下**全部**条件时自然回血（每 Tick `healthRegenRate`，默认 0.5）:

- 饥饿 > `regenAttrThreshold`（默认 80）
- 口渴 > `regenAttrThreshold`（默认 80）
- temperature ∈ [tempDangerMin, tempDangerMax]（默认 [30, 40]）
- 无 Bleeding 状态
- 无 Infected 状态

### 3.5 危险伤害 (ApplyDangerDamage)

| 触发条件 | Tick 伤害 | 原因标签 |
|----------|-----------|----------|
| 饥饿 < `hungerDangerThreshold` (20) | -0.4 | 饥饿 |
| 口渴 < `thirstDangerThreshold` (15) | -0.6 | 口渴 |
| 两者同时触发 | -1.0 | 饥饿+口渴 |

### 3.6 状态持续伤害 (ApplyStateEffects)

| 状态 | Tick 伤害 | 参数 |
|------|-----------|------|
| 出血 Bleeding | `bleedingHealthLoss` (0.5) | 持续扣血 |
| 感染 Infected | `infectedHealthLoss` (0.3) | 持续扣血 |

### 3.7 体温伤害 (ApplyTemperatureDamage)

| 状态 | Tick 伤害 | 参数 |
|------|-----------|------|
| 失温 Hypothermia | `hypothermiaHealthLoss` (0.2) | - |
| 过热 Overheated | `overheatHealthLoss` (0.15) | - |

## 四、体温系统

### 4.1 体温计算流程

```
targetTemp = currentEnvTemperature
  + (TemporaryWarmth 效果存在 ? +2°C : 0)
  + (装备保暖值 _cachedWarmth × 0.75)

temperature = MoveTowards(temperature, targetTemp, temperatureRegainRate)
```

### 4.2 环境温度来源

1. **时间系统驱动**: 订阅 `TimeOfDayChanged` 事件，按正弦曲线计算
   ```csharp
   currentEnvTemperature = sin(currentHour / 24 * 2π) * 10 + 15
   ```
   即: 正午峰值 ~25°C，午夜低谷 ~5°C

2. **外部 SetEnvironmentTemperature()**: 供天气系统、室内/室外判断调用

### 4.3 体温异常状态

| 状态 | 触发条件 | Tick 伤害 |
|------|----------|-----------|
| 失温 Hypothermia | temperature < `tempDangerMin` (30°C) | hypothermiaHealthLoss (0.2) |
| 过热 Overheated | temperature > `tempDangerMax` (40°C) | overheatHealthLoss (0.15) |

## 五、物品效果系统

### 5.1 效果类型枚举 (ItemEffectType)

| 枚举值 | 含义 |
|--------|------|
| `RestoreHealth` | 恢复健康值 |
| `RestoreHunger` | 恢复饥饿值 |
| `RestoreThirst` | 恢复口渴值 |
| `RestoreTemperature` | 恢复体温值 |
| `CureBleeding` | 治愈出血状态 |
| `CureInfected` | 治愈感染状态 |
| `FixFracture` | 修复骨折状态 |
| `TemporaryWarmth` | 临时保暖（持续 duration 秒，体温目标 +2°C） |

### 5.2 ItemEffect 数据结构

```csharp
[System.Serializable]
public class ItemEffect {
    public ItemEffectType effectType;   // 效果类型
    public float value;                 // 效果数值
    public float duration;              // 持续时间（TemporaryWarmth 专用）
    public bool isInstant;              // 是否即时生效
}
```

### 5.3 物品使用双重处理

**重要**: 物品使用存在两条独立订阅路径，由 SurvivalSystem 直接处理以保证药效/食物逻辑完整性：

#### 路径 A: SurvivalSystem.OnItemUsed()（主要路径）
- 直接订阅 `ItemUsedEvent`，独立处理食物递减、药品冷却
- **食物连续递减**: 5 分钟内食用同 `foodType` 的食物，效果 × 0.5
- **药品冷却**: 药品（`foodType == "Medicine"`）30 秒内不可重复使用同名药品

#### 路径 B: ItemUsageSystem.OnItemUsed()（辅助路径）
- 订阅 `ItemUsedEvent`，负责从背包扣除物品数量
- 调用 `Inventory.RemoveItem()` 后，遍历 `itemEffects` 调用 `SurvivalSystem.ApplyItemEffect()`
- 注意：SurvivalSystem 也订阅了 `ItemUsedEvent`（路径 A），因此可能存在效果重复应用的**潜在风险**（当前因 SurvivalSystem 在 ItemUsageSystem 之前注册 ServiceLocator，且两者均直接处理 itemEffects，实际运行中由 SurvivalSystem 路径 A 全权处理药效逻辑；如果出现重复回血，需在此处添加互斥标记）

### 5.4 ApplyItemEffect 逻辑

```csharp
public void ApplyItemEffect(ItemEffect effect, float multiplier = 1f)
switch (effect.effectType):
  RestoreHealth      → ModifyHealth(value * multiplier, "物品")
  RestoreHunger      → hunger = Min(100, hunger + value * multiplier)
  RestoreThirst      → thirst = Min(100, thirst + value * multiplier)
  RestoreTemperature → temperature = Clamp(value * multiplier, 0, 50)
  CureBleeding       → SetSurvivalState(Bleeding, false)
  CureInfected       → SetSurvivalState(Infected, false)
  FixFracture        → SetSurvivalState(Fracture, false)
  TemporaryWarmth    → temporaryEffects[TemporaryWarmth] = duration (刷新计时)
```

### 5.5 临时效果过期 (UpdateTemporaryEffects)

每帧遍历 `temporaryEffects` 字典，递减 `Time.deltaTime`，value <= 0 时自动移除。临时效果移除后，相应的体温加成（+2°C）自动消失。

## 七、HUD 显示 (SurvivalHUD.cs)

- 屏幕右上角，面板宽度 200px，每条 Bar 高 20px，间距 6px
- 使用 UGUI `Slider` + `Text` 构建，4 条垂直排列:

  - 口渴 (蓝色) - ThirstBar
  - 体温 (橙红色) - TempBar

### 7.2 刷新机制

### 7.3 UGUI 构建

- 使用统一 `UGUIBuilder.CreateCanvas()` / `CreateTextAnchored()` / `CreateProgressBar()` 工厂方法
- Canvas Sort Order = 100
- 锚点右上角 (anchor=1,1, pivot=1,1)
- 如果已存在 `SurvivalHUD_Canvas` 子对象则复用 FindBars()，避免重复创建

## 九、护甲与保暖缓存

`EquipmentChangedEvent` 由 Inventory 系统在装备变化时发布，SurvivalSystem 订阅后：

- `_cachedArmor` → 用于 `TakeDamage()` 护甲减伤: `actualDamage = rawDamage × 100 / (100 + armor)`
- `_cachedWarmth` → 用于体温计算: `targetTemp += warmth × 0.75`

不再在每个 Tick 内 Find Inventory，性能友好。

## 十、存档系统接口

### 导出 (GetSaveData)

返回 `PlayerSaveData`，包含：
- 4 属性数值: health, hunger, thirst, temperature
- activeStates: 当前活跃的生存状态字符串列表（如 `["Bleeding", "Hypothermia"]`）
- 食物连续递减状态: lastFoodType + lastEatGameTime（转换为游戏时间）
- 药品冷却: medCooldownsRemaining（实时秒数 → 游戏剩余秒数）

### 恢复 (RestoreFromSave)

从 `PlayerSaveData` 反序列化：
- 直接恢复 4 属性数值
- 清空所有状态后按 activeStates 重新激活
- 食物冷却时间还原（游戏秒数 → 实时时间）
- 药品冷却倒计时还原

## 十三、已知问题与注意事项

1. **物品效果双重处理风险**: `SurvivalSystem.OnItemUsed()` 和 `ItemUsageSystem.OnItemUsed()` 都订阅了 `ItemUsedEvent` 并分别遍历 `itemEffects`。SurvivalSystem 路径 A 包含食物递减/药品冷却逻辑，而 ItemUsageSystem 路径 B 直接调用 `ApplyItemEffect(eff)`（不带 multiplier），可能导致：
   - 同类食物重复恢复效果
   - 药品绕过冷却直接生效
   - **建议**: 在 `ItemUsageSystem.OnItemUsed()` 中增加互斥标记，仅当 SurvivalSystem 未处理时回退执行；或让 ItemUsageSystem 不再遍历 effects，仅负责扣物品。

2. **环境温度仅正弦驱动**: 当前仅由时间系统正弦公式计算昼夜温差，天气系统（雨/风暴）尚未接入 `SetEnvironmentTemperature()`，室内/室外温差也未实现。

3. **临时效果上限**: `temporaryEffects` 无数量上限，大量短时保暖物品理论上可叠加（但 TemporaryWarmth 效果值固定 +2°C，多次叠加仅刷新持续时间）。

4. **骨折仅在 SetSurvivalState 时调移速**: 如果存档恢复时骨折状态已激活但未调用 `playerCharacter.SetMoveSpeedModifier()`，则移速不会减半。查看 `RestoreFromSave` 中确实未调用移速修改。
