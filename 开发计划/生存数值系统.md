# 3.2 生存数值系统 ✅ Phase 1 完成

> **状态**: 已实现 · **日期**: 2026-05-11 · **依赖**: L2 时间系统(TimeManager/Tick) + L2 人物系统(PlayerCharacter) + L1 EventBus

---

## 一、功能概述

管理玩家四项核心生存数值（健康/饥饿/口渴/体温），以游戏时间刻度驱动衰减和自然恢复，支持出血/感染/骨折/失温/过热五种生存状态，物品使用效果通过事件总线接入，护甲减免战斗伤害。

## 三、四项核心数值

| 数值 | 默认最大 | Tick衰减 | 说明 |
|------|:------:|:------:|------|
| 健康值(Health) | 100 | — | 归零=死亡，可自然恢复 |
| 饥饿值(Hunger) | 100 | 0.2/tick | 低于20触发扣血 |
| 口渴值(Thirst) | 100 | 0.3/tick | 低于15触发扣血，消耗完加速另一项1.5x |
| 体温(Temp) | 36.5°(初始) | — | 向环境温度靠拢，<30失温/>40过热均扣血 |

---

## 四、Tick循环（每60游戏秒×timeScale）

```
SurvivalSystem.Update() → 检查tickTimer
  → TickSurvivalLogic():
    1. UpdateTemperature()     体温向(环境温度+保暖奖励)靠拢0.1/tick
    2. ApplyAttrDecay()        饥饿-0.2/tick  口渴-0.3/tick
    3. ApplyNaturalRegen()     条件: 饱食>80 且 口渴>80 且 体温正常 且 无状态
                               回血0.5/tick  但 饱食/口渴消耗×3
    4. ApplyDangerDamage()     饥饿<20 或 口渴<15 → 扣血
    5. ApplyTemperatureDamage() 体温<30(失温0.2/tick) 或 >40(过热0.15/tick)
    6. ApplyStateEffects()     出血-0.5/tick  感染-0.3/tick
```

## 五、五种生存状态

| 状态 | 效果 | 解除方式 |
|------|------|---------|
| 出血(Bleeding) | -0.5血/tick | 止血物品 |
| 感染(Infected) | -0.3血/tick | 抗生素/消毒 |
| 骨折(Fracture) | 移速×0.5 | 夹板 |
| 失温(Hypothermia) | -0.2血/tick | 升温后自动解除 |
| 过热(Overheated) | -0.15血/tick | 降温后自动解除 |

## 六、环境温度

```
TimeOfDayChanged事件 → SurvivalSystem.OnTimeChanged()
  envTemp = sin(hour/24×π×2) × 10 + 15
  范围约 5°C(深夜) ~ 25°C(正午)

目标体温 = envTemp + 装备保暖×0.75 + 临时保暖效果 + 2
```

---

## 七、物品效果系统

```
玩家右击物品 → QuickItemBar → UseItemCoroutine(进度条)
  → ItemUsedEvent.Publish
    → ItemUsageSystem: RemoveItem → SurvivalSystem.ApplyItemEffect()
      → 根据ItemEffectType:
          RestoreHealth/ Hunger/Thirst/Temperature: ModifyXxx(+value)
          CureBleeding/ Infected/ FixFracture: RemoveState()
          TemporaryWarmth: 持续时间buff
```

---

## 八、护甲减伤

```
DamageablePlayer.TakeDamage(damage)   // 来自战斗
  → SurvivalSystem.TakeDamage(rawDamage):
      actual = raw × 100 / (100 + _cachedArmor)
      → ModifyHealth(-actual)

总护甲 = Σ已装备物品的itemData.armorValue
总保暖 = Σ已装备物品的itemData.warmthValue
缓存更新: 订阅EquipmentChangedEvent
```
