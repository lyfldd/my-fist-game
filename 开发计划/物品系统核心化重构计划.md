## 一、现状诊断

### 1.1 核心问题

当前各系统与物品系统的连接方式不统一：

- **建造/合成系统**：✅ 正确读 ItemData + Inventory，是标杆
- **武器系统**：⚠ 读武器字段但不消耗弹药，magazineSize 死字段
- **生存系统**：⚠ 纯事件驱动，不主动查 Inventory，由 ItemUsageSystem 中转
- **电力/车辆**：🔴 燃料补给要么在 UI 层，要么完全缺失
- **声音/VFX**：🔴 武器→音色无映射，枪口火焰系统不存在
- **CableLinker**：🔴 用 AssetDatabase（仅编辑器API），生产构建崩溃

### 1.2 审计结果

18 个断链点，详见审计矩阵（本文末尾 §9）。

## 二、目标架构

```
                    ┌─────────────────────────────────┐
                    │           ItemData               │
                    │  (所有字段，单一起源)              │
                    │  + ammoItemName                   │
                    │  + gunshotSoundType               │
                    │  + muzzleFlashType                │
                    │  + reloadTime                     │
                    │  + foodType / fuelValue           │
                    │  + healAmount                     │
                    └───────────────┬───────────────────┘
                                    │ 被引用
                    ┌───────────────▼───────────────────┐
                    │           Inventory                │
                    │  (增删查改 + 装备 + 聚合)           │
                    │  Publish:                          │
                    │  - InventoryChanged                │
                    │  - EquipmentChanged                │
                    └───────────────┬───────────────────┘
                                    │ EventBus
        ┌───────┬───────┬───────┬───┴───┬───────┬───────┐
        ▼       ▼       ▼       ▼       ▼       ▼       ▼
    武器系统  生存系统  声音系统  VFX系统  电力系统  车辆系统  装备系统
    (读弹药) (读食物) (读音色) (读特效) (读燃料) (读燃料) (读属性)
```

**核心原则**：
- 系统来 ItemData/Inventory **读数据**
- 事件用来**通知变更**
- 每个系统入口自己判断 `if (不归我管) return;` — 不设中心路由器

### 5.2 WeaponShooting.cs — 加弹药消耗 + 自动换弹（改动最大）

```csharp
// 新字段
private int _currentMag;
private bool _isReloading;
private float _reloadTimer;

// 路由（已有，不改）
void Update() {
    var weapon = GetActiveWeapon();
    if (weapon == null || !weapon.isFirearm) return;  // ← 不是枪不管
}

// TryFire 改动
void TryFire(ItemData weapon) {
    if (_fireTimer > 0f || _isReloading) return;

    // === 新增：弹药检查 ===
    if (_currentMag <= 0 && !string.IsNullOrEmpty(weapon.ammoItemName)) {
        StartReload(weapon);  // 自动换弹
        return;
    }

    // === 原有射击逻辑 ===
    _currentSpread = Mathf.Min(_currentSpread + weapon.spreadPerShot, weapon.maxSpread);
    _fireTimer = weapon.fireRate;
    _currentMag--;  // ← 新增

    // 射线 → 伤害（不变）...

    // === 改动：事件携带完整数据 ===
    EventBus.Publish(new WeaponFiredEvent(
        weapon, weapon.ammoItemName, weapon.gunshotSoundType,
        weapon.muzzleFlashType, muzzle, spreadDir, hitSomething, target
    ));

    // === 删除：不再直接调 SoundEmitter ===
}

void StartReload(ItemData weapon) {
    int inInventory = _inventory.CountItemByName(weapon.ammoItemName);
    if (inInventory <= 0) {
        EventBus.Publish(new WeaponDryFireEvent(weapon, muzzle));
        return;
    }
    _isReloading = true;
    _reloadTimer = weapon.reloadTime;
    // 协程：计时结束 → 扣背包弹药 → 填弹匣 → 发 ItemReloadedEvent
}
```

路由：`if (!weapon.isFirearm) return;`

### 5.5 SurvivalSystem.cs — 加食物类型 + 药品冷却

```csharp
private float _lastEatTime;
private string _lastFoodType;
private Dictionary<string, float> _medCooldowns = new();

// 新增订阅（替代 ItemUsageSystem）
void OnItemUsed(ItemUsedEvent evt) {
    if (evt.ItemData.itemEffects == null || evt.ItemData.itemEffects.Length == 0) return; // 路由

    // 食物类型连续递减
    if (!string.IsNullOrEmpty(evt.ItemData.foodType) && evt.ItemData.foodType != "Medicine") {
        if (evt.ItemData.foodType == _lastFoodType && Time.time - _lastEatTime < 300f) {
            ApplyItemEffectsHalf(evt.ItemData.itemEffects);  // 5分钟内同类效果减半
            return;
        }
        _lastFoodType = evt.ItemData.foodType;
        _lastEatTime = Time.time;
    }

    // 药品冷却
    if (evt.ItemData.foodType == "Medicine") {
        if (_medCooldowns.TryGetValue(evt.ItemData.itemName, out var cdEnd) && Time.time < cdEnd)
            return;  // 冷却中
        _medCooldowns[evt.ItemData.itemName] = Time.time + 30f;
    }

    foreach (var eff in evt.ItemData.itemEffects)
        ApplyItemEffect(eff);
}
```

路由：`if (itemEffects == null || itemEffects.Length == 0) return;`
