## 2. 属性与技能定义

### 2.1 四属性 (AttributeType)

定义于 `Assets/_Game/Config/CharacterData.cs`

| 枚举值 | 简称 | 区间 | 效果 |
|--------|:----:|:----:|------|
| `力量` | STR | 1-10 | 近战伤害×0.05/点, 负重+5kg/点 |
| `敏捷` | AGI | 1-10 | 移速、闪避率、潜行效果 |
| `体质` | CON | 1-10 | 生命值上限、自然恢复速度 |
| `耐力` | END | 1-10 | 体力上限+10/点, 体力恢复+2/s/点 |

**升级公式** (SkillCostTable.GetAttributeCost):
```
cost = round(5 + currentLevel × 0.5)
Lv0→1=5, Lv1→2=6, ..., Lv9→10=10
```

**升级公式** (SkillCostTable.GetCost):
```
cost = round(baseCost + currentLevel × increment)
```
- 战斗/生存类低价 (10-15 base)，增量平缓 (0.6-1.7)
- 制造/智力类高价 (35-50 base)，增量陡峭 (1.9-3.3)
- 智力最贵：Lv9→10 需 80 技能点 = 8000 XP

|:------:|------|
| 0 | 篝火烹饪类 (Cooking) — 完全不需智力 |

| 5 | 碳纤维、钛合金、夜视仪、燃烧弹、电解槽套件 |
| 6 | 免疫增强剂、广播塔套件、离心机套件 |

### 4.4 设备运行时 (ProductionDevice)

定义于 `Assets/_Game/Systems/Crafting/ProductionDevice.cs`

**核心特性**:
- 实现 `IInteractable` — 按 E 键打开设备面板
- 双槽位: `_inputSlot` (4×3) + `_outputSlot` (4×2)
- 自运转循环: `_cycleTimer` 累加 → `TryProduce()` → 消耗材料 → 产出
- **电力集成**: `NotifyPowerState(electric, coal)` 由 PowerConsumer 通知
  - 通电时跳过燃料消耗 + 生产加速 (×electricSpeedMultiplier)
  - 无电时烧煤兜底 (`powerAllowCoal`)
- **研究门控**: `ChemicalResearchManager.IsDeviceUnlocked()` / `.IsRecipeUnlocked()` 决定是否允许生产
- **流水线链接**: `_outputDestination` — 产出物直接送入下一台设备的输入槽
- **存档完整**: `GetSaveData()` / `RestoreFromSave()` 保存燃料/计时器/输入输出物品/链接目标


## 7. 设计原则

1. **两极分化**: 生存前期技能成本低 (Lv0-4 10-20点)，工业后期指数增长 (Lv9-10 57-80点)。迫使玩家专精而非全修。
2. **双门槛分离**: "能不能造" (建造拆解) vs "懂不懂" (智力/医疗/枪械)。低级设施纯体力，高级设施必须双修。
3. **套件体系**: 工业设备的建造材料 = 套件×1 (来自合成台) + 结构材料。套件生产与建造消耗独立不重复。
4. **电力加速**: 通电设备跳过燃料消耗 + 速度翻倍 (×2)，鼓励玩家发展电力基础设施。
5. **研究门控**: 研究中心消耗材料解锁设备和配方，智力等级决定能研究什么。研究是先决条件，技能是执行条件。
6. **流水线链接**: 设备的 `_outputDestination` 允许串联成自动产线 (如 工业炉→冲压机→装配台)。
