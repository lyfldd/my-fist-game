## 前置 A2：存档 DTO 扩展

> **目标**: SlotSaveData 加耐久字段，SaveLoadManager 采/复耐久
> **依赖**: 无 · **估时**: 0.5 天

### 现状

存档用 `SlotSaveData` DTO，当前无 `itemDurability` / `repairCount` 字段。

### 实施

**SlotSaveData** — +2 字段：

```csharp
public float itemDurability;
public int repairCount;
```

**SaveLoadManager** — 采集/恢复各 +2 行：

```csharp
// 保存
slotSave.itemDurability = placed.itemDurability;
slotSave.repairCount = placed.repairCount;

// 读档
placed.itemDurability = slotSave.itemDurability;
placed.repairCount = slotSave.repairCount;
```

向后兼容：旧存档 JSON 反序列化 `itemDurability` 为 0。**0 值歧义处理**：在 SaveLoadManager 恢复时判断 —— 若 `itemDurability == 0 && repairCount == 0`（旧存档刚升级），初始化为 `maxDurability`；若 `repairCount > 0`（真损坏），保持 0。DurabilitySystem 内部用 -1 标记未初始化，首次使用时赋 `maxDurability`，之后 0=损坏。

> **目标**: 低耐久工具降低建造/合成效率
> **依赖**: 前置 A · **估时**: 0.25 天

### 公式

```
效率倍率 = 0.5 + 0.5 × (itemDurability / maxDurability)
```

| 耐久比 | 效率 | 
|:--:|:--:|
| 100% | 1.0× |
| 50% | 0.75× |
| 30% | 0.65× |

> **合成不接入**：`CraftingSystem.Craft()`（112-136行）是同步执行，无协程无耗时。工具效率不影响合成时间。未来合成改为协程后再接入。

```csharp
    DurabilitySystem.Instance?.ConsumeDurability(_currentWeaponInstanceId, 
        GameConstants.DURABILITY_WEAPON_PER_SHOT);
```


## 前置 G：生产设备磨损

> **目标**: 31 台工业设备每生产一个物品消耗自身耐久
> **依赖**: 耐久 v1.0 · **估时**: 1 天

### 现状

### 实施

```csharp
// ProductionDevice.cs — 新增
public float maxDurability = 1000f;
private float _currentDurability;

void CompleteProductionCycle()
{
    // ... 现有产出逻辑 ...
    _currentDurability -= 1f;
    if (_currentDurability <= 0)
    {
        enabled = false;
        EventBus.Publish(new DeviceBrokenEvent(deviceId));
    }
}
```
