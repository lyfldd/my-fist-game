    int cx = chunkId % _gridSize;

## 核心问题

世界生成管线算出了 districts/blocks/cityBuildings，但这份数据只存在 `WorldData` 里，**runtime 系统（ChunkManager、ZombieSpawner、ContainerRefresh）完全不消费它**。等于说区域数据"算完就丢了"。

## 目标

打通 **WorldData → Runtime 系统** 的传递链，让容器刷新和僵尸生成能按真实区域类型读取不同配置。

## 子任务拆解

### 4.4.1 ZoneConfig ScriptableObject

新建 `Assets/_Game/Config/ZoneConfig.cs`，一个 SO 定义每个区域类型的 runtime 行为：

```
ZoneConfig (SO)
├─ zoneType: ZoneType (CBD/Residential/Industrial/Green/Wilderness)
├─ displayName: "商业中心"/"住宅区"/"工业区"/"野外"
├─ spawnProfile: ZoneSpawnProfile 引用 (僵尸数量/类型权重)
├─ defaultLootTable: LootTable 引用 (容器默认掉落)
└─ lootMultiplier: float (1.0=标准, 商业区可设1.5, 野外0.5)
```

Runtime 时只有一个 `ZoneConfig[]` 数组（或 Dictionary<ZoneType, ZoneConfig>），挂在 ZoneManager 上。

**荒野/野外**：当前 ZoneType 枚举没有 Wilderness，但 Green（绿地/公园）不等于野外。新增 `Wilderness` 枚举值到 `ZoneType`，代表城市外的野外区域。Chunk 不在任何 district 内时 = Wilderness。

### 4.4.2 ZoneManager Runtime 单例

新建 `Assets/_Game/Systems/WorldGen/ZoneManager.cs`：

```
ZoneManager : MonoBehaviour (单例)
├─ zoneConfigs: ZoneConfig[] (Inspector 拖入)
├─ GetZoneType(chunkId): ZoneType
│   └─ 查询该 chunk 中心点落在哪个 district 的 circle 范围内
│       └─ 不在任何 district 内 → Wilderness
├─ GetZoneConfig(chunkId): ZoneConfig
│   └─ zoneType → 查 zoneConfigs 字典
└─ 数据来源: WorldGenerator 完成后从 WorldData.districts 拷贝
```

**查区域逻辑**：chunk 中心世界坐标 → 遍历所有 CityDistrict → 判断 `Vector2.Distance(chunkCenter, district.center) <= district.radius` → 命中则返回 district.zoneType。多个 district 重叠时取距离最近的。

### 4.4.3 RuntimeChunk 加 zoneType

`RuntimeChunk.cs` 新增字段：

```csharp
public ZoneType zoneType;  // 由 ZoneManager 在 chunk 首次加载时写入
```

ChunkManager 在 chunk 从 Preload 进入 Loaded 时，调用 `ZoneManager.Instance.AssignZone(chunk)`，写入 `chunk.zoneType`。

### 4.4.4 ZombieSpawner 接入真实区域

修改 `ZombieSpawner.GetProfile(int chunkId)`：

```
现状: hash 伪随机 → zoneProfiles[index]
改为: ZoneManager.Instance.GetZoneConfig(chunkId)?.spawnProfile
```

同时为四种 ZoneType + Wilderness 各创建一份 `ZoneSpawnProfile` SO：
- **CBD**：高密度（initialMin=4, maxPerChunk=12），快速僵尸权重高
- **Residential**：中密度（initialMin=2, maxPerChunk=8），普通僵尸为主
- **Industrial**：中高密度（initialMin=3, maxPerChunk=10），胖子权重高
- **Green**：低密度（initialMin=1, maxPerChunk=4）
- **Wilderness**：极低密度（initialMin=0, maxPerChunk=2），仅夜间出怪

### 4.4.5 ContainerRefresh 接入区域 Loot

修改 `RefreshHub` / `ContainerRegistry`，容器刷新时根据所在 chunk 的 zoneType 决定：
- 刷新用的 LootTable 优先用 ZoneConfig 的 defaultLootTable（而非 container 自身的 lootTable）
- lootMultiplier 影响掉落物品种类数（minLootTypes/maxLootTypes × multiplier）

### 4.4.6 BuildingTemplate SO（数据定义，不实例化）

BuildingTemplate (SO)

## 数据流

```
WorldGenerator (离线)
  └─ WorldData.districts ───────────────┐
                                         │
ZoneManager.Awake()                      │
  └─ 从 WorldData 拷贝 districts[]       │ ← 世界生成完成后
     └─ 构建 zoneType 查询表             │
                                        │
ChunkManager.OnChunkLoaded(chunkId)     │
  └─ ZoneManager.AssignZone(chunk) ─────┘ → chunk.zoneType 写入
     │
     ├─ ZombieSpawner.SpawnInitial(chunkId)
     │   └─ GetProfile(chunkId) → ZoneManager → ZoneConfig.spawnProfile
     │
     └─ RefreshHub.OnChunkLoad(chunkId)
         └─ ZoneManager.GetZoneConfig(chunkId) → LootTable + multiplier
```
