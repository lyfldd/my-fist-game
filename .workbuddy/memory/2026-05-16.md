## Bug 修复

### CS0234: `Time` 命名空间冲突
- **根因**: 项目存在 `namespace _Game.Systems.Time`（TimeManager.cs），与 `UnityEngine.Time` 冲突
- **修复**: VehicleController.cs:205 `Time.fixedDeltaTime` → `UnityEngine.Time.fixedDeltaTime`；CameraFollow.cs:55 `Time.deltaTime` → `UnityEngine.Time.deltaTime`

### Prefab Asset 直接修改报错
- **根因**: CreateVehicleData.cs 在 Prefab Asset 上直接调用 `SetParent` / `AddComponent`
- **修复**: 重构 `SetupPrefabAtPath()`，使用 `PrefabUtility.LoadPrefabContents()` → 修改 → `SaveAsPrefabAsset()` → `UnloadPrefabContents()` 流程

### NRE + 车轮坐标偏移
- **NRE 根因**: `OnDrawGizmosSelected()` 编辑器选中时触发，`_controller` 未初始化（Awake 仅运行时执行）
- **NRE 修复**: `GetDriverSeatOffset/GetExitOffset` 里 fallback 为 `GetComponent<VehicleController>()`
- **车轮偏移根因**: WheelCollider 位置硬编码 (-0.9,0.45,±1.4)，与模型实际包围盒不匹配
- **车轮偏移修复**: CreateVehicleData.cs 全面重写，从 Renderer 计算合并包围盒 → 自动推算车轮位置/半径、Collider、VehicleData 偏移量；半径 = carHeight × 0.28 (clamp 0.25~0.6)

### 车辆包围盒+轮子朝向修复 v3（2026-05-16 16:00）
- **包围盒膨大**: `renderer.bounds`（世界空间AABB）被子节点旋转膨大 → 改用 `MeshFilter.sharedMesh.bounds` 8角点逐级变换，精确紧致AABB
- **轮子方向错误**: 原代码硬编码 `Quaternion.identity`，假设车头=+Z；实际模型车头朝向未知 → 自动检测：水平长轴=车头(forwardAxis)，短轴=车右(rightAxis)；轮子旋转 = `Quaternion.FromToRotation(Vector3.forward, forwardAxis)`
- **轮子离车身太远**: 包围盒膨大导致偏移量过大 → 精确包围盒 + 沿检测轴向计算位置自动修正
- **用户手动参考值**: 左前轮(0.00724,-0.0036,-0.00354) 右后轮(-0.00632,0.00449,-0.00354) → 推断车头=-Y，左=+X，模型导入尺度极小(~0.014m宽)
- 新增 `EncapsulateTransformedBounds()` 辅助方法；BoundsInfo 新增 forwardAxis/rightAxis/upAxis/carLength/carWidth 字段

### 车辆包围盒+轮子修复 v4（2026-05-16 18:00）

- **缩放根物体**: `go.transform.localScale = Vector3.one * scaleFactor`，使模型达到真实世界尺寸
- **轮子参数补偿**: 所有 WheelCollider 的 localPosition/localRadius 除以 scaleFactor 补偿，保证世界空间正确
- **轮子旋转**: `FromToRotation` → `LookRotation(forwardAxis, Vector3.up)` 显式对齐
- **WheelCollider localScale**: `CreateWheelChild` 强制重置 `localScale = Vector3.one`，清理旧版残留
- **VehicleData 世界空间**: colliderCenter/colliderSize/driverSeatOffset/exitOffset 全部存世界空间值（× scaleFactor）
- 修改文件：`Assets/_Game/Editor/CreateVehicleData.cs`

### 车辆轮子位置/旋转修复 v6（2026-05-16 23:00）
- **用户突破性发现**: 模型导入时根节点 X=-90°（FBX 从 3D 工具 Y-forward/Z-up 导出，Unity 导入自动 X=-90 转为 Y-up）
- **问题根源**: 本地轴 (local Z=车头) 被 X=-90° 父旋转映射到世界 Y(朝上)；本地 LookRotation 的 Vector3.up 在旋转后 ≠ 世界 Y，WheelCollider 悬挂方向全错
- **手动验证**: X=0° 时位置对但旋转不对；X=-90° 时需 Y/X 轴一正一负
- **v6 修复**: 全部在世界空间计算 → Inverse 回本地
  1. 本地轴 → 世界轴: `worldForward = rootRot * rawBounds.forwardAxis`
  2. 世界空间轮子位置: `worldForward * wsOffset ± worldRight * wsSide + Vector3.up * wsBottomY`
  3. 转本地: `Quaternion.Inverse(rootRot) * (worldPos * invScale)`
  4. 世界空间轮子旋转: `LookRotation(worldForward, Vector3.up)` → 本地: `Inverse(rootRot) * worldRot`
- 修改文件：`Assets/_Game/Editor/CreateVehicleData.cs`
- 需在 Unity 运行 **Tools → Vehicles → Quick Setup OffRoad (Data + Prefab)** 验证
