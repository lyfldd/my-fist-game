## 编译错误修复 (2026-05-15 18:39)

BuildModeController.cs + PlacedStructure.cs 四个编译错误：

1. **CS0118** (BuildModeController:45,56): `Inventory` 命名空间与类名冲突
   - 原因：`using _Game.Systems.Inventory;` 同时存在 namespace `_Game.Systems.Inventory` 和 class `Inventory`
   - 修复：改为 `using Inventory = _Game.Systems.Inventory.Inventory;`（类型别名）
   
2. **CS1061** (BuildModeController:141): `ItemData` 无 `displayName` 字段
   - 修复：`requiredTool.displayName` → `requiredTool.itemName`

3. **CS0234** (BuildModeController:167 + PlacedStructure:124): `Time.deltaTime` 命名空间冲突
   - 原因：文件在 `_Game.Systems.Building` 命名空间，C# 将 `Time` 解析为 `_Game.Systems.Time` 命名空间（而非 `UnityEngine.Time`）
   - 修复：`Time.deltaTime` → `UnityEngine.Time.deltaTime`（两处）

**经验笔记**：在 `_Game.Systems.*` 子命名空间里写代码时，`Time`、`Inventory` 等与项目内命名空间同名的符号会被 C# 优先解析为项目命名空间。需要显式使用 `UnityEngine.Time` 或类型别名规避。

## B 键无反应修复 (2026-05-15 19:40)

**根因**：`EnterBuildMode()` 检查 `activeBuildable == null` 直接 return，但 `BuildMenuUI` 只在建造模式中显示——鸡生蛋死锁。

**修复**：
1. `EnterBuildMode()` — 移除 null 检查，允许无选中建造物进入 Preview 状态。GhostPreview.SpawnPreview() 已处理 null 情况（仅 log warning 不崩溃）
2. `TryStartBuilding()` — 顶部新增 `activeBuildable == null` 检查，防止未选建造物时点左键 NPE

**流程**：B → 进入建造模式（无选中）→ BuildMenuUI 显示 → 点菜单按钮/数字键 → SwitchBuildable 切换 → GhostPreview 生成预览
