## 2026-05-07 晚 — 步骤 1.3 EventBus 实现完成

### 创建的文件
- `Assets/_Game/Core/EventBus.cs` — 全局事件总线
  - `Dictionary<Type, Delegate>` 存储处理器
  - `Subscribe<T>`, `Unsubscribe<T>`, `Publish<T>`, `Clear()` 四个方法
  - 泛型约束 `where T : struct`
- `Assets/_Game/Core/GameEvents.cs` — 事件数据结构
  - `PlayerDied { string KillerName; int KillCount; }`
  - `PlayerDamaged { int Damage; string Source; }`
  - `ItemPickedUp { string ItemName; int Count; }`
  - `ZombieDied { float X; float Z; string LootGroup; }`
  - 所有 struct 为 `readonly`，构造函数初始化属性

### 设计要点
- 命名空间 `_Game.Core`，统一管理
- EventBus 是 `static class`，全局单例无需挂载 GameObject
- 所有事件数据为 `readonly struct`，值类型零 GC 压力
- `Clear()` 用于场景切换时重置订阅

### 增强改进
- Publish 改为 foreach 逐个调用 + 每个订阅者独立 try-catch，防止一个异常中断整条链
- 新增 `GetSubscriberCount<T>()`：查看某个事件的订阅者数量
- 新增 `GetAllSubscriptions()`：查看所有事件的订阅者完整列表（事件名、订阅者方法、实例/静态）
