# 🏙️ 程序化城市生成算法文档

> 2026-05-14 三层模块化架构

---

## 一、三层架构总览

```
一级模块（城市区块级 120~240m）← 决定城市骨架，现在只做这层
├── 商业区 / 住宅区 / 工业区 / 郊区 / 主干道 / 水域
│
├── 每个一级模块内部预留二级网格
│
▼
二级模块（建筑/地块级 40/56/72m）← 未来填充，90+个
├── 沿街商铺 / 独栋民宅 / 公寓 / 厂房 / 公园 / 停车场
│
├── 每栋建筑预留室内接口
│
▼
三级模块（室内级 3m起）← 未来填充
└── 房间 / 走廊 / 楼梯 / 家具
```

## 二、Pipeline（当前：只跑一级模块）

| Order | Stage | 说明 |
|-------|-------|------|
| 10 | SeedStage | 全局种子（System.Random） |
| 20 | **CityLayoutStage** | 定城市风格、繁华等级、模块数量配额 |
| 30 | **ModuleGridStage** | 初始化 20×20 基础网格（40m/格），构建尺寸感知候选集 |
| 35 | **ModuleAssignmentStage** | 多单元 WFC 约束求解 → 填入一级模块 |
| 80 | **MeshStage** | 多单元模块可视化（纯色平面 + 道路Cube + 标签） |

### 2.1 基础网格架构

- **基础单元**: 40m（所有模块尺寸的 GCD）
- **网格**: 20×20 = 400 格，覆盖 800m×800m
- **模块占格**: 1×1（道路/路口=40m）到 5×5（工业/郊区=200m）
- 每个格存储 `ModuleCandidate`（类型 + 宽 × 高），WFC 坍缩时选择具体尺寸

## 三、一级模块（现在只做这层）

### 3.1 模块清单（首批 ~12 个）

| 模块类型 | 尺寸 | 可视化颜色 | 说明 |
|---------|------|-----------|------|
| 商业区 A | 120×120m | 红色半透明平面 | 主干道贯穿，内部预留二级网格虚线 |
| 商业区 B | 160×160m | 红色半透明平面 | 大型商业中心 |
| 住宅区 A（密） | 120×120m | 蓝色半透明平面 | 高密度住宅 |
| 住宅区 B（疏） | 160×160m | 蓝色半透明平面 | 低密度+院子 |
| 工业区 | 200×200m | 灰色半透明平面 | 大厂房+堆场 |
| 郊区 | 200×200m | 浅绿色半透明平面 | 低密度+自然植被 |
| 主干道直段 | 40m宽×160m长 | 深灰色长条Cube | 连接各区的骨架 |
| T 字路口 | 40×40m | 深灰色 | 三岔路口 |
| 十字路口 | 40×40m | 深灰色 | 四岔路口 |
| 水域/河岸 | 按需 | 浅蓝色半透明平面 | 自然边界 |

### 3.2 模块接口

每个模块的四条边定义接口：

```csharp
struct ModulePort {
    int edgeIndex;           // 0=上,1=右,2=下,3=左
    float position;          // 边上位置 (0~1)
    float width;             // 路宽
    RoadType roadType;       // MainRoad / BranchRoad / None
}
```

拼接条件：相邻模块的接口必须**位置对齐**（误差 < 模块边长×5%）+ **路宽一致**（误差 < 0.5m）。

### 3.3 排列规则

按城市风格决定一级模块的排列规则：

| 风格 | 排列规则 |
|------|---------|
| 中心辐射型 | 商业区在中心，住宅区环状包围，工业区/郊区在外圈 |
| 沿路延伸型 | 道路模块贯穿，两侧交替住宅/商业，两端工业/郊区 |
| 沿河两岸型 | 水域模块贯穿，两岸排布商业/住宅 |
| 临森林边缘型 | 一侧为自然边界，另一侧扇形展开住宅/工业 |

> **注意：一级模块为矩形网格，弯曲感不由一级承担。** 所有道路弯曲、路口错位、转角朝向等视觉效果由二级模块实现。

## 七、三级约束体系

### 7.1 一级约束（区块级 120~240m）

WFC 求解城市功能区布局。

#### A. 邻接约束（WFC 核心规则）

```
商↔商 ✅  商↔住 ✅  商↔工 ❌  商↔郊 ❌  商↔路 ✅必须
住↔住 ✅  住↔商 ✅  住↔工 ❌  住↔郊 ✅(仅疏)  住↔路 ✅必须
工↔工 ✅  工↔住 ❌  工↔郊 ✅  工↔路 ✅必须  郊↔水 ✅
路↔任何 ✅（接口必须对齐）  水↔水 ✅（连续河流）  水↔商/住/郊 ✅（河岸聚落）
```

#### B. 城市风格 → 排列规则

| 风格 | 初始种子 | 传播方向 | 权重倾向 |
|------|---------|---------|---------|
| 中心辐射 | 网格中心 | 向外环形 | 商业中心→住宅中圈→工业外圈 |
| 沿路延伸 | 中轴线上 | 沿轴向两边 | 道路贯穿，两侧商住交替 |
| 沿河两岸 | 水域两侧 | 垂直河岸 | 紧邻水域商业/工业，后方住宅 |
| 临森林边缘 | 一侧边界 | 向另一侧扇形 | 禁建侧空，另一侧住宅/小工业 |

#### C. 繁华等级 → 模块数量配比

| 等级 | 商业 | 住宅 | 工业 | 郊区 | 道路宽 |
|------|------|------|------|------|-------|
| 贫瘠 | 少(1) | 多(4) | 极少(0~1) | 多(3) | 3~4m |
| 普通 | 中(2) | 中(3) | 中(1~2) | 中(2) | 4~6m |
| 繁华 | 多(3~4) | 多(3~4) | 多(2~3) | 少(1) | 6~10m |

每个模块类型设置最小/最大数量下限/上限，WFC 分配时约束。

### 7.2 二级约束（建筑级 40/56/72m）

WFC 求解建筑布局。

#### A. 类型继承（一级 → 二级候选集）

```
一级模块类型 → 二级模块候选集
──────────────────────────────
商业区 → 沿街商铺、转角建筑、内部公寓、广场、停车场
住宅区 → 独栋+院子、联排住宅、行列式公寓、街角公园
工业区 → 厂房、仓库、停车楼、堆场、废料地
郊区 → 独栋(稀疏)、荒地、小片林地、小型农场
水域/河岸 → 预留空位（二级不填充）
```

禁止跨类型填充。

#### B. 道路接口（二级网格起始约束）

```
1. 一级接口位置(x比例) + 方向 → 确定二级网格入口行/列
2. 沿接口方向延伸一条主干道 → 贯穿二级网格
3. 主干道两侧 → 强制放沿街建筑(面朝道路)
4. 道路必须连续，建筑不能阻挡
```

#### C. 繁华等级 → 内部密度

| 参数 | 贫瘠(×0.75) | 普通(×1.0) | 繁华(×1.25) |
|------|-------------|-----------|-------------|
| 建筑退线 | 1~2m | 3~4m | 5~6m |
| 建筑高度 | 1~2层 | 2~4层 | 4~8层 |
| 空地占比 | 40~50% | 20~30% | 10~20% |
| 建筑间距 | 8~12m | 6~10m | 4~6m |

#### D. 尺寸变体（对齐 8m 网格）

每种二级模块有 2~3 个合规变体，均为 8m 倍数：

```
沿街商铺: 8×12 / 8×16 / 8×20
独栋民宅: 8×8  / 10×8 / 12×10
公寓楼:   12×12 / 16×16 / 20×16
厂房:     16×20 / 24×32
```

### 7.3 三级约束（室内级 3m 起）

WFC 求解房间布局。

#### A. 房间模块类型

```
房间级(3×3m起, 模数1m): 卧室、客厅、厨房、卫生间、饭厅、走廊
特殊(6×6m起): 仓库大厅、车间、商店货架区、开放办公区
连接: 楼梯间、电梯井、阳台、门廊
```

按建筑类型限定候选：

```
民宅 → 卧室/客厅/厨房/卫生间/走廊
公寓 → 卧室/客厅/小厨房/卫生间/走廊(每户一套)
商铺 → 货架区/收银台/仓库/卫生间
厂房 → 车间大厅/办公室/仓库/卫生间
```

#### B. 邻接约束

```
卧室↔走廊 ✅(必须走走廊进)      卧室↔卧室 ❌(不直通)
客厅↔饭厅 ✅  厨房↔饭厅 ✅      卫生间↔走廊 ✅
所有房间↔外墙 ✅(必须有窗)       楼梯间↔每层走廊 ✅
门洞:宽0.9m(不能开在外墙上)     窗户:必须在外墙,宽1.2m
```

#### C. 家具约束

```
床 → 靠墙，床头不对门    桌 → 靠墙或居中
柜/书架 → 靠墙           沙发 → 面朝电视墙
不阻挡门洞/走廊          偏转 ±90°
```

家具作为房间 WFC 的后处理，不参与 WFC 求解。

| 优势 | 说明 |
|------|------|
| 现在就能跑 | 只需 ~12 个一级模块，纯色块，半天见城市全貌 |
| 逐级解耦 | 一级调通了，二级修改完全不影响一级 |
| 分批开发 | 先做市中心，再扩展郊区/工业区 |
| 视觉逐步升级 | 每一级先用方块验证逻辑，再替换模型 |
| 可扩展性 | 新模块类型只需加配置和接口规则 |
